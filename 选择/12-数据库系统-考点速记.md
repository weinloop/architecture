## 二、三级模式与两级独立性（★★★★★）

- **外模式**：用户视图，局部数据逻辑结构。**概念模式**：全体数据逻辑结构，DBA。**内模式**：存储结构、物理顺序、索引，系统程序员。
- **口诀**：外局改全内务——外（外模式）局（局部）改（概念）全（全体）内（内模式）务（物理）。
- **物理独立性**：应用程序与磁盘数据独立。**逻辑独立性**：应用程序与逻辑结构独立，逻辑结构变时应用可不变。

## 四、E-R 与逻辑设计（★★★★★）

### 4.1 E-R 三要素

- **实体**：矩形。**属性**：椭圆（单值）；双线椭圆多值；虚线椭圆派生）。**联系**：菱形，标注 1:1、1:n、m:n。

### 4.2 E-R 转关系模式

- **1:1**：独立关系模式（两实体主码+联系属性）或与任一端合并（另一端主码+联系属性放入）。
- **1:n**：独立关系模式（n 端主码）或与 **n 端合并**（1 端主码+联系属性放入 n 端）。
- **m:n**：联系**单独成表**，属性=两实体主码+联系属性，主码为两主码组合。

### 4.3 合并 E-R：合并→消除冲突（属性/命名/结构）→消除冗余。

## 五、函数依赖与范式（★★★★★）

- **完全函数依赖**：X→Y 且 X 的任一真子集不能决定 Y。**部分函数依赖**：X→Y 但 Y 不完全依赖于 X（存在真子集决定 Y）。**传递函数依赖**：X→Y，Y→Z，Y↛X，Z⊈Y。
- **候选键**：图示法——依赖作图，入度为 0 的属性集为起点，能遍历全图即为候选键；若不能则加入部分中间顶点再试。
- **无损分解**：ρ={R1,R2} 对 F 无损 ⇔ (R1∩R2)→(R1−R2) 或 (R1∩R2)→(R2−R1)。
- **范式口诀**：一范式无重复，二范式主键独，三范式无传递，BCNF 传递无，四范式多值除。

| 范式 | 关键 |
|------|------|
| **1NF** | 属性原子，不可再分 |
| **2NF** | 1NF + 无非主属性对候选码的**部分**函数依赖 |
| **3NF** | 1NF + 无非主属性对候选码的**传递**函数依赖 |
| **BCNF** | 任一非平凡依赖左侧均为**候选码** |
| **4NF** | BCNF + 无多值依赖（或多值依赖拆开） |

- **不规范问题**：**绒绣茶山**——冗余、修改异常、插入异常、删除异常。

## 六、事务与并发控制（★★★★★）

### 6.1 ACID

- **原子性**：全做或全不做。**一致性**：从一个一致状态到另一一致状态。**隔离性**：并发事务互不干扰，可串行化。**持久性**：提交后永久。

### 6.2 并发问题

| 问题 | 描述 |
|------|------|
| **丢失修改** | 两事务同读同改，后提交覆盖前提交 |
| **读脏数据** | 读到他事务未提交的修改，他事务回滚后数据无效 |
| **不可重复读** | 同一事务内两次读同一数据，中间被他事务更新，结果不同 |
| **幻读** | 同一事务内两次按条件读，中间他事务插入/删除，结果集不同 |

### 6.3 封锁

- **X 锁（排他）**：可读可写；其他事务不能加任何锁。**S 锁（共享）**：可读不可写；其他可加 S，不能加 X；全部 S 释放后才能加 X。
- **一级**：修改前加 X 直到结束；防丢失修改，不防脏读、不可重复读。**二级**：一级 + 读前加 S 读后即释；防丢失、脏读，不防不可重复读。**三级**：一级 + 读前加 S 直到结束；防丢失、脏读、不可重复读。**两段锁**：扩展阶段只能加锁、收缩阶段只能解锁；可串行化，可能死锁。
- **粒度**：粒度小并发高、开销大；粒度大并发低、开销小。

## 七、完整性、触发器、反规范化与索引（★★★★★）

- **实体完整性**：主键不为空。**参照完整性**：外键取被参照表主键值或空；级联/受限/置空等。**用户定义完整性**：业务规则。
- **触发器**：BEFORE/AFTER × INSERT/UPDATE/DELETE；INSERT 用 NEW，DELETE 用 OLD，UPDATE 用 NEW 与 OLD。
- **反规范化**：增加冗余以提升性能；带来冗余、修改/插入/删除异常。
- **索引准则**：常查询、少更新的属性建索引；索引过多影响增删改；小表可不建。

## 八、备份、OLAP 与分片（★★★☆☆）

- **物理备份**：拷贝数据文件、日志。**逻辑备份**：导出为 SQL 等逻辑对象。**完全**：全量。**增量**：自上次备份以来变化。**差异**：自上次完全备份以来变化。
- **OLTP**：日常事务、当前细节、面向应用。**OLAP**：分析决策、历史聚集、面向主题、多维。
- **钻取**：变粒度；**切片/切块**：部分维定值；**旋转**：行列互换。
- **水平分片**：元组划分，**并**恢复。**垂直分片**：属性划分，**连接**恢复；关键字各分片均需保留。

## 九、速记清单（考前扫一眼）

1. **三级模式**：外局改全内务；物理/逻辑独立性。
2. **E-R 转关系**：1:1 独立或合并一端；1:n 独立或并 n 端；m:n 单独成表。
3. **范式**：1NF 原子；2NF 无部分依赖；3NF 无传递；BCNF 左侧皆码；4NF 多值除。绒绣茶山。
4. **候选键**：图示法入度 0 起点遍历。
5. **无损**：(R1∩R2)→(R1−R2) 或 (R1∩R2)→(R2−R1)。
6. **并发**：丢失修改、脏读、不可重复读、幻读；X/S 锁；一二三级封锁协议；两段锁可串行化。
7. **触发器**：BEFORE/AFTER、INSERT/UPDATE/DELETE；NEW/OLD。
8. **分片**：水平并、垂直连。
