## 二、范围管理（★★★☆☆）

**五过程**（软考无“收集需求”）：

| 过程 | 描述 |
|------|------|
| 编制范围管理计划 | 描述如何定义、确认和控制范围 |
| 定义范围 | 详细描述产品与项目范围，编制范围说明书 |
| **创建工作分解结构 WBS** | 将项目工作分解为较小、易管理部分，自上而下至工作包，其总和即项目范围 |
| 确认范围 | 正式验收可交付成果 |
| 范围控制 | 监督范围状态，管理范围基准变更 |

**案例（航空运营智能管理平台）**：

- **范围管理计划**：约定范围说明书、WBS 模板与变更审批（如新增行李子域走 CCB）。
- **定义范围**：范围说明书对齐 **业务目标、七大功能表、联动流程、非功能指标**（并发、延迟、备份与合规）。
- **WBS**：第一层 **基础支撑 / 航信 / 旅客 / 票务 / 检修 / 数据服务 / 辅助管理**；下层拆到 **微服务/迭代版本** 等工作包；工作包之和覆盖项目范围。
- **确认范围**：里程碑验收（如 **购票闭环、压测通过、等保/合规材料**）。
- **范围控制**：防“智慧民航”口号下 **需求蔓延**（如无限堆 AI 功能），变更影响工期与成本。

## 三、进度管理（★★★★★）

### 3.1 过程

活动定义→活动排序→活动资源估算→活动历时估算→进度计划编制→进度控制。

**案例（航空运营智能管理平台）**：活动含 **基础平台就绪、核心票务联调、数据管道与实时分析、检修 AI、多终端、安全渗透与演练、生产切换**；排序上 **数据治理与基础支撑** 常前置，**票务闭环** 为关键里程碑；历时可用 **三点估算** 应对不确定模块（如 **对接民航局核验、PB 迁移**）。

### 3.2 网络图

- **AOV 网（单代号）**：顶点=活动，弧=先后关系；单代号以**节点**代表工作，箭线=逻辑关系。
- **AOE 网（双代号）**：顶点=事件，弧=活动；双代号以**箭线及两端节点编号**表示工作。求**最短/关键路径**时常用双代号。
- **PERT 图**：即网络图。

### 3.3 三点估算法

- **乐观时间 a**：一切顺利时完成时间。
- **最可能时间 m**：正常情况下完成时间。
- **悲观时间 b**：最不利情况下完成时间。
- **期望时间**：**(a + 4m + b) / 6**

**案例（航空运营智能管理平台）**：对 **“历史 10PB+ 数据入湖与 T+1 任务跑通”** 可给 **乐观 a**（环境顺利）、**最可能 m**、**悲观 b**（网络与schema大量返工），代入公式得 **期望工期**，用于排期缓冲。

### 3.4 关键路径与时差

- **关键路径**：从开始到结束的**最长路径**，决定项目最短工期。
- **总时差**：该活动可推迟而不影响总工期的最大时间。
- **自由时差**：该活动可推迟而不影响紧后活动最早开始的最大时间。真题常考总时差计算。

**案例（航空运营智能管理平台）**：**关键路径** 往往经过 **强依赖外部或长周期** 的活动链（如 **跨系统联调 + 压测达标 + 监管材料**），决定能否 **2025-08 上线**；非关键活动（如 **部分报表皮肤**）可有 **总时差**，春运前可挪用缓冲；**自由时差** 用于协调 **多团队并行** 时不改紧后 **联调窗口**。

## 四、成本管理（★★★☆☆）

| 过程 | 定义 | 要点 |
|------|------|------|
| 成本估算 | 对完成项目所需成本的估计与计划 | 自顶向下、自底向上、差别估算法 |
| 成本预算 | 将总成本分配到各工作包，建立成本基准 | 形成成本基准计划 |
| 成本控制 | 保证各项工作在预算内进行 | — |

**案例（航空运营智能管理平台）**：

- **估算**：**云资源（弹性扩容）、大数据与 GPU、商用中间件、安全测评、驻场集成**；可 **自顶向下** 按模块分摊或 **自底向上** 汇总工作包。
- **预算**：建立 **成本基准**（按阶段/环境：开发、测试、预生产、生产）；峰值前 **应急扩容预算**。
- **控制**：对比 **实际云账单 vs 预算**，春节峰值后 **缩容**；防 **非生产环境长期高开**。

## 五、配置管理（★★★☆☆）

| 内容 | 描述 |
|------|------|
| 编制配置管理计划 | 项目启动时制订，为开发工作基础 |
| 配置标识 | 确定配置项、命名与描述，是配置管理前提 |
| 变更管理与配置控制 | 对配置项变更加以控制，防止多次变更下失控 |
| 配置状态说明（配置状态报告） | 记录、报告配置信息，给出配置项当前状况 |
| 配置审核 | 验证配置项与配置标识一致，确保管理有效 |

**案例（航空运营智能管理平台）**：

- **配置计划**：规定 **镜像、Helm Chart、配置中心键、数据库脚本版本** 管理规则。
- **标识**：配置项含 **微服务版本、API 契约、数据模型、压测场景脚本**。
- **变更控制**：生产变更 **双人复核、灰度、回滚包**；防 **多次热修导致环境漂移**。
- **状态报告**：发布台账（谁、何时、何版本、何环境）。
- **审核**：发版前核对 **制品与文档一致**（Runbook、接口说明）。

## 六、风险管理（★★★☆☆）

**六过程**：风险管理计划编制→风险识别→风险定性分析→风险定量分析→风险应对计划编制→风险监控。

| 过程 | 要点 |
|------|------|
| **风险管理计划编制** | 如何安排与实施风险管理；角色与职责、预算、风险类别、概率与影响定义、汇报格式、跟踪；应急计划与应急储备 |
| **风险识别** | 系统化识别已知与可预测风险；反复进行；确定风险来源、条件、特征；收集资料→估计风险形势→识别潜在风险 |
| 风险定性分析 | 优先级排序；可能性与影响、风险类型 |
| 风险定量分析 | 灵敏度分析、期望货币价值、决策树、**蒙特卡罗**（多元函数→参数概率→随机数→模拟）等 |
| 风险应对 | **防范策略**（发生前：避免、转移、减轻）；**响应策略**（发生后执行）；正向风险：开拓、分享、提高 |
| 风险监控 | 跟踪已识别风险、残余风险，识别新风险，评价计划有效性 |

**案例（航空运营智能管理平台）**：

- **识别**：**春运/国庆峰值超容量**、**外部核验与支付接口不稳定**、**分布式事务错账**、**PB 查询超时**、**数据隐私与合规**、**人员流动** 等。
- **定性/定量**：对峰值风险用 **蒙特卡罗** 或压测样本推断 **TPS/延迟分布**；关键财务影响用 **EMV** 辅助是否多活投入。
- **应对**：**减轻**——弹性、缓存、降级、对账；**转移**——云 SLA、安全外包；**避免**——砍掉非核心上线项；**应急**——熔断、限流、公告；正向风险如政策补贴可 **开拓** 新功能试点。
- **监控**：节前 **红蓝演练**、复盘 **残余风险**（如单点外部依赖）。

## 七、速记清单（考前扫一眼）

1. **范围**：编制计划→定义范围→**WBS**→确认范围→范围控制。
2. **进度**：单代号=节点表工作；双代号=箭线+两节点；**期望时间=(a+4m+b)/6**；关键路径=最长路径；总时差/自由时差。
3. **成本**：估算→预算（成本基准）→控制。
4. **配置**：计划→标识→变更控制→状态说明→审核。
5. **风险**：计划编制→识别→定性→定量（蒙特卡罗）→应对（防范/响应）→监控。
