## 二、测试分类

### 2.1 按程序是否可见

| 类型 | 要点 |
|------|------|
| **黑盒** | 功能测试；不看内部，基于输入输出；等价类划分等；系统/确认测试常用 |
| **白盒** | 结构测试；了解内部逻辑，覆盖所有路径；逻辑/循环/基本路径覆盖；军工、航天等高可靠领域 |
| **灰盒** | 介于两者；多用于集成测试；既看输入输出也看内部表征 |

### 2.2 按是否执行程序

| 类型 | 要点 |
|------|------|
| **静态测试** | 不运行程序；分析语法、结构、接口等；代码检查、静态结构分析、质量度量；发现缺陷早，耗时长 |
| **动态测试** | 运行程序；比较结果与预期；构造用例、执行、分析输出 |

### 2.3 按测试对象

功能测试、性能测试、安全测试、兼容性测试、界面测试、易用性测试、稳定性测试等。

### 2.4 按测试阶段

| 阶段 | 对象 | 方法 | 要点 |
|------|------|------|------|
| **单元测试** | 最小可测单元 | 白盒为主、黑盒为辅 | 模块接口、局部数据；开发人员在组长监督下 |
| **集成测试** | 模块/子系统 | — | 模块间数据传输、高层设计；单元测试之后 |
| **系统测试** | 整个系统 | 黑盒 | 功能、性能、恢复、安全、压力等；集成测试之后 |

## 三、白盒覆盖标准（★★★★★）

**强度从弱到强**：语句 < 判定 ≈ 条件 < 判定条件 < 条件组合 < 路径。

**口诀**：**路条组判｜条条判语**——路（路径）条组（条件组合）判（判定条件）｜条（条件）条（条件）判（判定）语（语句）。

| 覆盖类型 | 描述 | 发现错误能力 |
|----------|------|--------------|
| 语句覆盖 | 每条语句至少执行一次 | 最弱 |
| 判定覆盖 | 每个判定的每个分支至少一次 | 2.5 |
| 条件覆盖 | 每个条件取到各种可能值，**不考虑组合** | 2.5 |
| 判定条件覆盖 | 同时满足判定覆盖与条件覆盖 | 4 |
| 条件组合覆盖 | 每个判定中各条件每一种组合至少一次 | 5 |
| 路径覆盖 | 所有可能路径都执行过 | 最强 |

## 四、功能测试与性能测试细分

### 4.1 功能测试

链接测试、表单测试、数据校验、Cookies 测试、数据库测试等；概念辨识即可。

### 4.2 性能测试辨析（★★★★★）

| 类型 | 要点 |
|------|------|
| **负载测试** | 在**已达一项或几项性能瓶颈**时，让系统继续运行一段时间，观察各项指标 |
| **压力测试** | **不断加压**（增加负载），直到某些指标达瓶颈（如 CPU 100%）；**过程性、渐进加压** |
| **强度测试** | 制造**极限状态**（超负荷或最差环境），看系统抵抗能力与性能下降幅度；侧重**极限状态**，非渐进过程 |
| **并发测试** | 负载+压力结合；逐渐增加负载至瓶颈或不可接受点，综合分析交易与资源指标 |
| **容量测试** | 预先分析某指标极限值（如最大并发用户数），在极限状态下测试是否故障或仍保持主要功能 |
| **可靠性测试** | 为满足可靠性要求的设计、分析、测试；系统测试阶段提高可靠性、发现对可靠性影响大的缺陷 |
| **配置测试** | 找系统资源最优分配，为调优提供依据；选典型配置测试 |
| **速度测试** | 网络连接速度、业务处理速度 |

**压力 vs 强度**：压力=慢慢加压、过程性；强度=制造极限状态（超负荷或环境）。

## 五、安全测试类型

数据加密、用户身份验证、日志文件、Session、备份与恢复、访问控制、安全漏洞（XSS、SQL 注入等）、TCP 端口、服务器端脚本、防火墙等；选择题概念辨识。

---

## 六、测试模型 W/H/X（★★★☆☆）

| 模型 | 特点 | 与开发关系 |
|------|------|------------|
| **W 模型** | 在 V 模型基础上发展；测试与开发**同步** | 同步 |
| **H 模型** | 测试活动**独立**，贯穿整个生命周期 | 独立 |
| **X 模型** | 针对**单独程序片段**编码与测试，**频繁交接**集成 | 交错 |

---

## 七、速记清单（考前扫一眼）

1. **黑盒**：功能、不看内部；**白盒**：结构、覆盖路径；**灰盒**：集成、两者结合。
2. **静态**：不运行、分析代码；**动态**：运行、比较结果。
3. **白盒覆盖**：路条组判｜条条判语（弱→强）；路径最强，语句最弱。
4. **负载**：已达瓶颈后继续运行；**压力**：不断加压至瓶颈；**强度**：极限状态（超负荷/环境）；**并发**：负载+压力结合。
5. **W/H/X**：W 同步、H 独立、X 交错。
