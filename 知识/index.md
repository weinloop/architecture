## 二、 专项突破：工程化能力深挖

### 1. 代码工程化：重构烂模块
- **目标**：新人接手代码，不用问你，直接能看懂、能扩展。
- **练习**：重构一个旧模块，实现统一异常、统一日志、统一返回、消除硬编码、可测试化。

### 2. 微服务分布式
- **目标**：能画出复杂系统的架构图，并解释每一步的设计理由。
- **练习**：设计“用户→订单→支付→库存”流转，解决数据一致性、超卖、超时等核心问题。

### 3. 可观测性：5 分钟定位
- **目标**：线上出问题，1 分钟定位方向，5 分钟找到原因。
- **重点**：掌握日志规范与链路追踪，建立标准的线上问题排查流程。

## 三、 3 个月强化计划（循序渐进）

### 第 1 个月：底层与中间件突击
- **重点**：JVM + 并发 + MySQL 索引 + Redis。
- **练习**：每天 1 小时练习线上问题模拟排查。

### 第 2 个月：架构与工程化落地
- **重点**：设计模式 + 微服务设计原则 + 代码规范重构。
- **练习**：按 DDD 或规范分层标准，重写一个核心业务逻辑。

### 第 3 个月：云原生与可观测性
- **重点**：Docker/K8s + 监控体系 + 自动化测试。
- **练习**：独立完成一个服务的镜像打包、K8s 部署及监控接入。
