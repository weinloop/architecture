## 摘要

2024年3月，我参与某航空公司运营智能管理平台建设，项目面向航空运营机构、近百个运营基地机场，覆盖数千万常旅客会员，年均服务旅客超3000万人次，提供航空信息管理、旅客全流程服务、票务交易、航空检修预警、数据智能分析等核心业务功能。项目中，我担任系统架构师，全面负责平台架构设计与核心技术落地。本文围绕**零信任与 API 安全架构**展开论述，通过**统一身份与 OIDC 令牌、RBAC+ABAC 最小权限与短效凭据降低越权与凭据滥用面**，基于**API 网关集中鉴权、限流与 mTLS 保障东西向通道一致策略**，结合**全链路审计、业务风控与边界 WAF 形成纵深防御与可追溯闭环**。系统于2025年8月正式上线，截至2026年5月已稳定运行10个月，各项功能及性能指标均达到预设标准，获得客户高度认可。

## 项目背景

某航空公司需管理覆盖全部航线网络的近百个运营基地与机场，服务数千万常旅客会员、年服务旅客超3000万人次，为其提供票务、值机、行程查询、航班变动通知、航空检修协同等全场景服务；随着 **OTA、商旅、联盟航司、机场地服**等合作方通过 **API** 接入增多，若仍采用「内网默认可信、边界一道防火墙」的传统模型，一旦出现**令牌泄露、接口横向移动或黄牛机器刷票**，将导致**旅客隐私、行程与支付数据**外泄或业务损失，与架构质量属性中的**安全性、合规性**要求严重冲突。**安全工程**强调在开放互联前提下持续验证身份与权限；**零信任**则以**永不信任、持续验证、最小权限**为纲，将控制面从网络边界前移到**每一次 API 调用**。在此背景下，该航空公司于2024年3月启动航空运营智能管理平台建设，将**零信任与 API 安全**列为与业务功能同优先级的架构能力。

我司中标后，我以系统架构师身份牵头**安全架构与 API 治理**：建设统一 **IdP** 与 **OAuth2/OIDC** 体系，服务间强制 **mTLS** 与**服务身份**；**API 网关**作为北向与东西向的统一策略执行点，串联**鉴权、限流、防重放与敏感字段治理**；配套**不可篡改审计、风控规则与 WAF**，并纳入**渗透测试与漏洞闭环**。为此，我们团队将**安全需求左移**到设计与代码评审，关键接口纳入**红队演练**场景。平台于2025年8月正式上线，通过多轮监管检查与第三方渗透复测，未发生**越权批量拉取行程**等严重事件，高危漏洞**日内级**闭环，获得安全与业务侧认可。

## 问题二回应与过渡

由于本项目**对外开放与合作接口多、调用主体涵盖人、机、批任务**，若缺乏**零信任与 API 安全**，将面临**凭据长期有效、权限过宽、入口策略分散**等系统性风险；**安全性**要求身份可鉴别、授权可证明、操作可审计。因此我们将**零信任与 API 安全架构**作为落地路径，其核心包括：第一，**身份与最小权限**，以短效令牌、细粒度策略与**服务间双向 TLS**落实「每次访问都验证」；第二，**API 网关策略**，在北向统一**鉴权、限流、防重放**并与 **OpenAPI** 契约对齐敏感字段；第三，**审计与纵深防御**，全链路日志支撑追责，**WAF+风控**覆盖边界与业务异常。

在本项目的实施中，我们通过**身份与最小权限、API 网关策略、审计与纵深防御**，完成了**零信任与 API 安全架构**的建设与落地，具体实践如下。

## 正文

### 一、身份与最小权限：解决越权访问与凭据滥用

OTA、商旅、联盟航司与机场地服通过API接入持续增多，若凭据长期有效、权限过宽或仍沿用「内网默认可信、边界一道防火墙」模型，易出现越权批量拉取行程、黄牛机器刷票与令牌泄露后的横向移动，旅客隐私、行程与支付数据面临实质性风险。我们落实零信任「永不信任、持续验证、最小权限」：人机与机机统一走OAuth2/OIDC，JWT短生命周期配合JWKS验签、刷新轮转，高危scope与敏感接口叠加ABAC属性策略（渠道、基地、数据分级）；服务间东西向强制mTLS与工作负载证书自动签发吊销，RBAC收敛到资源级操作。Spring Cloud Gateway作为统一策略执行点前置校验scope、租户与audience，与IdP联动步进式验证；与日均12万笔、5500TPS、P99≤800ms及99.993%可用性目标在安全评审会联签，不因安全策略牺牲核心链路容量签名。实施后越权与重放类事件可分钟级定位到主体、客户端与令牌版本，默认拒绝、显式允许策略落地；高危漏洞日内级闭环，与Prometheus鉴权失败率、ELK审计及Jira安全工单一致，监管抽检材料可追溯，重放与异常登录尝试较基线下降约38%，与错误预算及重大窗口冻结策略联动复核。

### 二、API 网关策略：解决入口缺乏统一鉴权与限流

多伙伴、多版本并存时，若各系统自行验签、限流与错误码处理，易出现策略漂移、重复建设与内部异常细节外泄。我们以北向流量统一经Spring Cloud Gateway集中执行：校验JWT声明、scope与audience，nonce与时钟窗口防重放，敏感接口叠加二次认证、mTLS客户端证书与IP白名单；多维限流与Sentinel熔断按渠道、应用与接口配额，OpenAPI标注PII字段并在响应侧做字段级过滤、掩码与最小必要返回。契约安全扫描、越权用例与敏感字段diff纳入GitLab MR门禁，与Sonar规则联动；traceId贯通网关、领域服务与审计日志，统一错误码对外脱敏对内可诊断。与WAF及业务风控引擎联动封禁异常指纹与高频试探，Kafka异步投递安全事件供SIEM关联分析。实施后渠道接入以审批scope与配额为主，安全左移可落地；联调周期缩短约22%，策略变更可审计可回放，与Nacos路由版本、Sentinel集群限流基线及金丝雀回滚一致，春运窗口与错误预算联动冻结高危规则变更，第三方渗透复测高危项清零周期缩短约40%，与API目录及开发者门户订阅审批闭环，与OpenAPI安全基线扫描版本对齐。

### 三、审计与纵深防御：解决攻击面不可见与事件不可追溯

监管与内审要求「谁、在何时、以何授权」访问了哪些敏感数据，且需抵御撞库、SQL注入、爬虫高频试探与供应链攻击面扩大。我们建设只增不改的审计链路与纵深防御：全量API访问写入防篡改日志管道，经ELK按traceId、主体与资源标识检索并拼接证据链；SIEM规则关联异常登录、批量拉取、异地并发与短时高频，与业务风控评分联动自动限流、验证码挑战与人工复核。边界WAF与网关策略双签生效，红队演练与渗透发现纳入缺陷Sprint与Capa闭环；敏感导出走数据网关审批、水印与留存期限，与Kafka审计主题及Prometheus异常流量指标互相印证。实施后争议场景可分钟级给出主体、时间、授权依据与配置版本，安全运营常态化；与99.993%可用性及On-call升级链一致，未发生越权批量拉取行程类重大事件，告警误报经抑制规则下降约25%，与季度安全架构周会及监管报送字段对齐，重大事故复盘材料与审计抽样合并归档，与IAM密钥轮换及证书计划在重大节假日前联合演练，与日均12万笔业务洪峰及5500TPS压测基线联合复核，与Sentinel异常拦截及Jira安全缺陷闭环一致，与等保测评留痕要求一致，全过程可审计。

## 总结

本平台响应智慧民航与数据安全合规要求，以**身份与最小权限、API 网关策略、审计与纵深防御**为核心，将零信任从理念落实为**可执行的策略与工程能力**，2025年8月上线后稳定运行10个月，超额达成预期目标。上线以来，系统日均处理票务交易超12万笔，核心业务响应时间≤800毫秒，运营效率提升35%，旅客投诉率下降40%，设备故障预警准确率92%，系统可用性达99.993%，峰值处理能力突破5500 TPS，成功应对节假日高并发与多渠道安全审查，获安全与业务一致认可。项目复盘仍存在不足：一是部分**遗留对接系统**仍依赖**共享密钥**，迁移 **mTLS** 需分阶段；二是**跨境合作方**时区与法务对日志留存要求不一；三是**AI 驱动的异常检测**尚未全量替代规则引擎。后续将推广 **SPIFFE** 风格**工作负载身份**、完善 **API 安全评分卡**与**自动化契约越权扫描**，持续收紧开放接口的攻击面。
