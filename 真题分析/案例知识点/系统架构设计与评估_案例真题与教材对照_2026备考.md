## 一、常考 · 直接背诵

### 1.1 效用树（Utility Tree）归类规则

- **结构**：根 → **质量属性**（常 **性能、可用性、安全性、可修改性**；ATAM 默认四类，题中可加 **易用性** 等）→ 分类（可选）→ **叶子 = 编号需求 (a)(b)…**  
- **判定口诀**：  
  - **时延 / 吞吐 / QPS / P99** → **性能**  
  - **切换 / 冗余 / RTO / RPO / 心跳 / 降级** → **可用性**（真题常把「故障后 N 秒内发现/切换」归 **可用性**）  
  - **攻防 / 审计 / 加密 / 权限 / 日志** → **安全性**  
  - **人天 / 人月改完、可配置、扩展模块、替换实现** → **可修改性**  
  - **界面友好、少步操作、帮助与无障碍** → **易用性**  
- **忌死记选项字母**；以 **题干树形与选项** 为准。

---

### 1.2 九大质量属性 · 各记一句对策

| 属性 | 背一句对策/关键词 |
|------|-------------------|
| **性能** | 优先级队列、加资源、**并发**、调度 |
| **可靠性** | 冗余、检测与恢复、**MTBF** 语境 |
| **可用性** | **心跳**、冗余/集群、**快速恢复**、**故障转移** |
| **安全性** | 认证授权、加密、防火墙、**审计** |
| **可修改性** | 模块化、**接口标准**、配置与版本 |
| **互操作性** | REST/SOAP、**中间件**、服务发现 |
| **可移植性** | 抽象层、减平台依赖 |
| **可测试性** | 模块化、**测试接口**、覆盖率 |
| **易用性** | 统一交互、反馈、帮助 |

**辨析**：**可用性** ≈ 正常运行时间比例、故障恢复速度；**可靠性** ≈ 错误下维持能力；案例题 **故障切换表述** 多归 **可用性**。

---

### 1.3 质量属性场景 · 六要素（必背公式）

**公式**：**刺激源** + **刺激** + **环境** + **制品** + **响应** + **响应度量**

**故事串**：**谁**（刺激源）在 **什么条件**（刺激）、系统 **何状态**（环境）、对 **谁**（制品）**怎么做**（响应），**如何量**（响应度量）。

**可用性标准表述片段（真题级）**（与《06》一致，按需整段默写）：  
刺激源：**系统内部、外部**；刺激：**疏忽、错误、崩溃、时间**；环境：**正常 / 降级**；制品：**处理器、信道、存储器、进程**；响应：**检测并记录、通知、隔离故障、恢复、降级或正常运行**；响应度量：**故障修复时间**。

**安全性响应** 常写：**认证、授权、审计、加密、监控**；度量：**检测成功、溯源、恢复**。

---

### 1.4 敏感点、权衡点、风险点、非风险点

| 概念 | 句式 |
|------|------|
| **敏感点** | 某构件特性 **「将影响」** 某一质量属性（单点） |
| **权衡点** | **影响多个质量属性**；「 **提升 A、降低 B** 」 |
| **风险点** | 「**若…，可能…，影响…**」 |
| **非风险点** | 成熟组件、**可接受**、经验证 |

---

### 1.5 ATAM 与效用树（简背）

- **ATAM**：在 SAAM 上强化 **性能、可用性、安全性、可修改性**；用 **效用树** **排序** 后评估 **折中**。  
- **效用树修剪**：叶为场景；标 **(重要度 H/M/L, 难易度 H/M/L)**，如 **(H,L)** = 重要且易实现 → 优先。  
- **ATAM 实践四阶段**：**演示** → **调查与分析**（含 **生成效用树**）→ **测试**（验证高优先级场景）→ **报告**（风险、非风险、敏感点、权衡点、改进建议）。  
- **SAAM**：**场景**挑刺；侧重 **可修改性**；过程：场景开发 → 架构描述 → 单场景评估 → **场景交互** → 总体。

---

### 1.6 三种架构评估方法

| 方法 | 一句 |
|------|------|
| **问卷/检查表** | 凭经验，**主观** |
| **基于场景（SAAM/ATAM）** | **系统化**，费工定义场景 |
| **基于度量** | **客观**，映射与度量成本高 |

### 1.7 软件架构风格 · 案例高频对比

**批处理 vs 管道-过滤器**

- **批处理**：前步完成 **整体** 传递，**无并行**（常「落盘/磁带感」）。  
- **管道-过滤器**：**输出即下步输入**，**可并行**，流式。

**进程通信 vs 事件驱动**

- **进程通信**：**显式** 点对点/RPC。  
- **事件驱动**：**隐式调用**、注册、**广播**。

**管道-过滤器 vs 仓库（Repository）**

| 维度 | 管道-过滤器 | 仓库 |
|------|-------------|------|
| 数据形态 | **线性阶段**、固定流水线 | **中央数据结构**，多构件 **围绕同一模型** |
| 适用 | 编译器、ETL、**固定步骤** | **IDE、共享模型**（如语法树）、多维读写 |
| 弱点 | 难 **交互式随意改流程** | 中心 **竞争与复杂度** |

**解释器 vs 面向对象（典型结论）**

- **解释器**：规则/语法 **外置**，**可修改性、灵活性** 强；多一层 **性能** 常略差。  
- **OO**：直观、**性能** 常较好；规则多变易 **类爆炸**。  
- 题干强调 **折扣/规则常改** → 常选 **解释器**（或规则系统，真题常与解释器同族）。

**解释器 vs 隐式调用（事件）**

- **解释器**：流程/规则 **集中可编排**、语义清晰。  
- **事件**：**解耦** 强，**全局流程可控性/可视性** 弱，需监控补齐。

---

### 1.8 Lambda / Kappa（必背差异）

| 项 | Lambda | Kappa |
|----|--------|-------|
| 路径 | **批处理层 + 速度层 + 服务层**（双路径） | **单一流处理链路** 统一实时与历史重算 |
| 心智 | **离线准确 + 实时近似** 合并对外 | **以流为唯一真相**，回放重算 |
| 运维 | 两套逻辑 | **单链路** 简化（团队要成熟） |

**填图记忆词**：**Flume** 采集；**Kafka** 缓冲；**HDFS** 冷数据；**HBase** 海量键值；**Spark** 批、**Spark Streaming/Flink** 流；**服务层** 合并视图对外 API。

---

### 1.9 微服务 vs 单体（案例常考）

| 单体 | 微服务 |
|------|--------|
| 部署简单、早期快 | **独立演进**、技术异构 |
| 可用性/扩展 **单点** | **弹性**、按域扩缩 |
| | 代价：**分布式事务、链路治理、测试复杂** |

---

### 1.10 Ping/Echo vs 心跳

| 模式 | 背法 |
|------|------|
| **Ping/Echo** | **主动探测** 关键实例是否存活 |
| **心跳** | 实例/设备 **周期上报**；检测 **静默故障** |

---

### 1.11 DDD 与弱网一致（2025-11 套路）

- **限界上下文**：划分 **业务边界**、**统一语言**、**独立演进**；跨界 **防腐层 / 领域事件**。  
- **弱网**：**本地队列**、**全局唯一订单号**、**幂等**、**对账 / 补偿**、**最终一致**。

---

### 1.12 AES 与长数据（2025-11 充电同构）

- **分组密码** 超长明文：**分块** + **填充（如 PKCS#7）** + **工作模式（CBC/GCM 等）** + **IV/Nonce**（依模式）。  
- 教材：**§4.4.2** 对称分组。

---

### 1.13 非功能四类（2018 题型）

- **操作**、**性能**、**安全**、**文化**（瘦/胖 C/S、组织过程等 —— **随题干** 归类）。

## 二、下次考试预测 · 直接背诵

**高概率**

1. **效用树**：给 (a)～(k) 句子 → **性能/可用/安全/可修改** 四分法 + 偶尔 **易用/可测试**。  
2. **敏感点 vs 权衡点**：各选 **一句题干** 套 **将影响 / 提升A降低B** 句式。  
3. **风格对比**：**管道-过滤器 vs 仓库**；**解释器 vs OO**；**大模型/DAG 与解释器** 填空。  
4. **Lambda 填图/判型**：题干 **批+实双路径 + 服务层合并** → **Lambda**；纯粹流重算演进提 **Kappa**。  
5. **六要素**：整题默写或 **可用性** 原文级；或迁移 **充电/AES** 子题。  
6. **微服务 + 质量**：单体/微服务 **优缺点** + **效用树** 叶子。

**中概率**

7. **Ping/Echo vs 心跳** + 场景句配对。  
8. **DDD、限界上下文** + **弱网幂等对账**（换行业场景）。  
9. **ATAM 四阶段** 或 **效用树 (H/M/L)** 优先级。  
10. **风险点** 论述：**需求未收敛、紧耦合遗留**。

**低概率 / 防偏**

11. **非功能四类** 再考 C/S 变体。  
12. **SAAM 过程** 名词序。  

**预测口诀**

**效用树看吞吐切换审计人天，敏感将影响权衡升降档，管道线仓库心解释外置 OO 硬，Lambda 批速服 Kappa 一流扛，六要素谁刺激环境制品响应量，微服拆分事务难单体单点扛，Ping 主动心跳报弱网幂等对账。**

## 六、综合类比 · 题型 ↔ 航空平台一句

## 七、考前一页纸

1. **效用树**：**时延吞吐→性能**；**切换冗余→可用**；**审计攻防→安全**；**人天可配→可修改**。  
2. **三句型**：敏感点 **将影响**；权衡点 **升A降B**；风险 **若…可能…**。  
3. **六要素**：刺激源、刺激、环境、制品、响应、响应度量 —— **故事串一遍**。  
4. **Lambda**：**批+速+服务** 三层；Kappa **单流**。  
5. **管道 vs 仓库**：**流水线** vs **中心模型**；解释器 **规则外置**。  
6. **不要死记效用树字母**；**随题干** 替换航空数字。  
7. **教材**：**第8章整章 + §7.3 + §19.5～19.6**。
