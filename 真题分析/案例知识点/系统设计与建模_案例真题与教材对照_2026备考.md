| **用例包含/扩展、类关系** | **包含**：值机 **必须先** 身份校验；**扩展**：延误补偿 **在延误成立条件下**；**组合**：机身-发动机；**聚合**：机队-飞机 |
| **状态图 vs 活动图、泳道** | **状态图**：航班 **计划/延误/登机/关闭**；**泳道活动图**：运控、地服、机务 **并行除冰/登机** |
| **Scrum、MVC、SSH** | **Sprint 状态图** 套 **航季需求迭代**；**MVC**：运控大屏 **视图**、放行规则 **控制器**、航班数据 **模型** |
| **SOA、ESB** | **2013 真题即航空**：**Ramp Control / Flight Management** 与 **ESB** 路由协议转换；平台中 **AODB、DCS、货运、气象** 经 **ESB** 集成 |
| **风险驱动安全、FTA、形式化** | **安全关键**：燃油加注、舱门、航材适航；**FTA**：顶事件「错误放行」分解；形式化用于 **规章可证明子集** |
| **区块链、溯源、智能合约** | **航材/时控件** 全链路溯源；**MRO 维修记录** 不可篡改；合约：**到寿更换、串件审批** 规则上链 |

**【平台案例总览 · 航空运营智能管理平台】**  
平台建模对象覆盖 **人（旅客/机组/地服）—机（航空器）—场（航站/机位/跑道）—流（计划/事件/货邮）**：结构化分析侧重 **数据从哪来、经哪些加工、落哪些库**；UML 侧重 **谁与系统交互、消息时序与状态**；集成题侧重 **服务划分与总线**；扩展题侧重 **安全证据链与不可篡改溯源**。下文各节 **【平台案例】** 均以此统一场景展开。

## 二、教材知识点对照（按主题）

### 1. 结构化分析与数据流图（§5.3.1）

| 真题考点 | 教程依据 |
|----------|----------|
| **SA** 常用手段：**数据流图、数据字典**、结构化语言、判定表/树 | **§5.3.1「1. 结构化分析」** |
| DFD 四要素：**数据流、加工、数据存储、外部实体（外部项）** | **§5.3.1「1）数据流图」** |
| **父图子图平衡**、加工须有输入输出；**黑洞、奇迹、灰洞** | 真题与 SA 规范；教程强调数据流与加工语义 |
| **数据字典**：对数据流、存储、数据项精确定义 | **结构化分析步骤**中出现「生成数据字典」 |
| **DFD vs 流程图**：数据流/并行 vs 控制流/单线时序 | 案例解析表述；教程以 DFD 功能建模为主 |
| **CRUD 矩阵**：过程与数据存储 **C/R/U/D** 一致 | 教材未单列，属 **数据-过程一致性** 实践 |

**【平台案例】**  
- **外部实体与边界**：如 **旅客**、**承运人/航司**、**机场运行部门**、**空管/流量管理接口**、**货运代理**、**机务 MRO**；数据穿越边界要有 **命名数据流**（如「订座记录」「放行许可」）。  
- **加工分解**：顶层可拆为 **计划与时刻**、**旅客与行李**、**配载与平衡**、**场面与除冰**、**结算与统计**；子图须与父图 **输入输出平衡**。  
- **数据字典**：对 **航班状态码、延误原因子码、行李牌号格式** 等做 **数据项/数据结构** 定义，支撑 **多系统同一口径**。  
- **黑洞/奇迹/灰洞**：如加工只有出无入（奇迹）、只有入无出（黑洞）——可举 **「统计报表」加工无数据源** 为错。  
- **DFD vs 流程图**：**离港** 强调 **数据并行到达**（旅客、行李、配载）用 DFD；若强调 **谁先签字后关门** 的控制顺序，更接近流程图（真题辨析）。  
- **CRUD 矩阵**：行是 **加工**（如值机、配载），列是 **存储**（旅客、航班、舱单），检查 **是否只有 R 无 U** 等遗漏。

### 2. UML 与面向对象分析（第2章）

| 真题考点 | 教程依据 |
|----------|----------|
| **用例图**：参与者、用例、边界 | **§2.x UML**（用例图说明） |
| **顺序图**：同步/异步/返回消息 | UML **序列图**（第2章软件工程/建模相关节） |
| **顺序图 vs 通信图（协作图）**：时间序 vs 结构连接 | 同章 |
| **状态图 vs 活动图**：事件驱动状态迁移 vs 活动流、并发 | 同章 |
| **状态图**、**活动图**（泳道） | 同章 |
| 序列图 **组合片段**：**alt、opt、loop、par、break** 等 | UML2 交互片段（以真题为准） |

**【平台案例】**  
- **用例图**：系统边界内写 **值机、行李托运、改签、不正常航班处置、电子放行** 等；**参与者** 与 **用例** 只连关联，不在用例图展开内部步骤。  
- **顺序图**：如 **旅客改签**：旅客 → App → **订座服务** → **运价规则** → **支付网关**（**同步调用**）；异步通知 **短信/推送**（**异步消息**）；**返回消息** 带结果码。  
- **顺序图 vs 通信图**：强调 **时间先后** 用顺序图；强调 **对象间链路与消息编号** 用通信图（协作图）——可举 **配载—离港—行李** 多对象协同。  
- **状态图 vs 活动图**：**航班生命周期**（计划/延误/登机/关闭）用 **状态图**；**除冰—登机—推出** 多部门 **泳道** 与 **分叉汇合** 用 **活动图**。  
- **组合片段**：**alt**「有座/无座」分支；**opt**「购买保险」可选段；**loop**「重试支付」；**par**「并行校验证件与行李额度」。

### 3. OMT / 三模型（2021 问题3）

- **对象模型**（静态结构）、**动态模型**（事件与状态）、**功能模型**（DFD/加工）。  
- 教材以 **结构化 + UML + 面向对象** 分章叙述，三模型完整定义可在软件工程参考书与真题解析中互证；答题按 **对象—动态—功能** 及「均可用于需求分析」表述。

**【平台案例】**  
- **对象模型**：**航班、航段、航空器、机位、旅客、行李、舱单** 等类及关联（可点到 **类图**）。  
- **动态模型**：**航班状态**、**登机口变更事件**、**除冰完成事件**（**状态图/事件表**）。  
- **功能模型**：**离港与配载** 的 **DFD/加工**（顶层数据流）。  
- 一句话：**静态谁和谁连、动态随事件怎么变、功能数据怎么流**——与平台 **运行指挥中心** 的需求分析对象一致。

### 4. 用例关系与类关系（2016）

| 概念 | 真题标准表述 |
|------|----------------|
| 用例关系 | **泛化、包含《include》、扩展《extend》** |
| **包含**：注册课程 **必须先** 登录 |  |
| **扩展**：补考 **在考试不及格条件下** 发生 |  |
| 类关系 | **关联、聚合、组合、依赖、泛化**；组合 **同生命周期** 更强 |

**【平台案例】**  
- **包含《include》**：**网上值机** **包含** **身份校验**（必须先做）。  
- **扩展《extend》**：**贵宾休息室准入** **扩展** **值机完成**（在 **两舱/金卡** 条件下）。  
- **泛化**：**旅客值机** 与 **团体值机** 泛化 **抽象用例「办理乘机手续」**（若题干如此设计）。  
- **类关系**：**聚合**「机队」—「飞机」（飞机可调入调出）；**组合**「飞机」—「机载设备套件」（换机时整体更换）；**关联**「航班」—「机长」。

### 5. SOA 与 ESB（2013）

| 考点 | 教程依据 |
|------|----------|
| **面向服务架构 SOA**、服务划分、集成 | **第13章** 等 **§13.x 面向服务架构** |
| **ESB**：协议转换、路由、格式转换、服务位置透明、监控等 | 扩展与 **§13** SOA 实践描述 |

**【平台案例】**  
- **服务划分**：按 **业务能力与边界** 划分，如 **航班计划服务、离港服务、配载服务、资源（机位）服务、气象服务**；**粗粒度** 可复用、**松耦合**。  
- **与 2013 真题同构**：题干已有 **Ramp Control（场面）** 与 **Flight Management（航班管理）**——可直接迁移到 **ESB 作为中间层**：上游 **AODB/ACDM**、下游 **离港、行李、显示、车辆调度**，**协议**（SOAP/REST、报文）、**字符集/时区** 不一，经 **ESB 转换与路由**。  
- **ESB 功能**：**服务目录与路由**、**协议/格式转换**、**日志与监控**、**限流熔断**（表述扣题即可）。

### 6. 敏捷 Scrum 与 MVC（2016-5）

| 考点 | 说明 |
|------|------|
| **Scrum**：Product Backlog、Sprint 计划、每日站会、评审/回顾 | **§5.1** 软件过程/敏捷可参照；细节以真题 |

| **MVC** + **Struts/Spring/Hibernate** 分层 | **分层架构** 与 J2EE 实践；教程 **MVC** 在多处提及 |

**【平台案例】**  
- **Scrum 状态图**：**Backlog → Sprint 计划 → 开发中 → 评审 → 回顾 → 下一 Sprint**，可套 **航季需求**（如 **新航站接入、新规章字段**）。  
- **MVC**：**运控工作台**——**View** 展示 **航班列表/Gantt**；**Controller** 处理 **改航、hold、除冰请求**；**Model** 封装 **航班聚合与业务规则**。**Struts/Spring** 作 Web 控制层，**Hibernate** 作 **ORM 持久化**（按真题空填）。

### 7. 区块链与溯源（2025-05）

| 考点 | 说明 |
|------|------|
| **六层**：数据、网络、共识、激励、合约、应用 | 真题背诵；教程 **第11章/新技术** 可能有区块链简述，以真题分层为准 |
| **智能合约**：参与方身份、执行规则、状态与结果 | 真题 |

**【平台案例】**  
- **六层**：可映射为 **应用**（航材 App/监管查询）→ **合约**（更换/串件审批规则）→ **共识**（联盟链多方确认）→ **网络**（航司/制造商/MRO 节点）→ **数据**（件号、批次、装机位置）。  
- **溯源三类角色**：**生产商/航材分销商、航空公司/MRO、监管/审计**（与真题「生产/流通/消费」类比）。  
- **智能合约三方面**：**参与方身份**（证书与组织）；**执行规则**（到寿、适航批准）；**链上状态与结果**（装机记录、更换哈希）。  
- **业务叙述**：**时控件、救生设备、发动机关键件** 需 **全生命周期可追溯**，与 **区块链溯源** 题干同构。

### 8. 安全分析、形式化与测试（2024-11 第5题）

| 考点 | 说明 |
|------|------|
| **风险驱动安全分析** 四步（识别、评估、安全需求、设计验证） | 教材未专章；按真题答案 |
| **FTA** 故障树填因 | 可靠性/安全扩展 |
| **形式化方法 vs 测试** | **§5.x** 软件工程综述中有形式化方法线索 |

**【平台案例】**  
- **风险驱动安全分析四步**：识别 **错误配载、错误放行、舱门未关** 等风险 → 评估严重度与概率 → 导出 **安全需求**（双人复核、互锁）→ **设计与验证**（测试用例、仿真）。  
- **FTA**：顶事件如 **「航班带缺陷放行」**，分解为 **工卡漏签、航材未装、软件版本错误** 等 **与门/或门**（按题干填空）。  
- **形式化 vs 测试**：**形式化** 用于 **小规模关键规约**（如 **间隔标准、冲突检测算法** 的可证明性质）；**测试** 用于 **系统级、穷尽不可行** 时的抽样验证——平台侧可写 **「规章子集建模」+「大量场景测试」** 互补。  
- **领域类比**：真题 **医疗泵** ↔ 平台 **燃油泵/液压关键执行器**（答题时 **扣题干设备**，勿硬套）。

## 三、高频默写清单

1. **DFD 四要素** + **平衡原则**（父-子、加工输入输出）。  
2. **黑洞 / 奇迹 / 灰洞** + 数据存储**不可**与外部实体直连。  
3. **用例三种关系** 与典型例句（登录⊃注册课程；考试—扩展→补考）。  
4. **类关系**：聚合 vs 组合（生命周期）。  
5. **顺序图三种消息**；**alt/opt/loop**。  
6. **状态图 vs 活动图** 一句本质差异。  
7. **ESB** 主要功能（路由、转换、位置透明、监控等）。  
8. **区块链六层**、**智能合约三方面**。

**【平台案例 · 与上表逐条对应】**  
1. **离港 DFD**：加工有入有出；父子图平衡。  
2. 灰洞例：**配载加工无重量来源**；存储不与外部实体直连。  
3. **包含**：值机 **必须先** 身份校验；**扩展**：延误补偿 **在延误成立条件下**。  
4. **机队—飞机** 聚合；**飞机—黑匣子** 组合（示例随题干）。  
5. 改签顺序图：**alt** 票规；**opt** 保险。  
6. **状态图** 航班状态机；**活动图** 泳道并行。  
7. **ESB**：**AODB/DCS/气象** 经总线 **路由+转换+监控**。  
8. **航材溯源** 六层 + **到寿/串件** 合约三要素。
