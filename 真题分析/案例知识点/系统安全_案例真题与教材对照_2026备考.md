## 一、常考 · 直接背诵

> 下列为 **2013-11-⑤ 得分逻辑** + 《07》中与案例 **同构** 的高频句，考场按序展开即可。

### 1.1 身份鉴别：口令 vs 公钥体系（PKI）

**口令认证 · 弱点**

- 实现简单；易受 **弱口令、字典攻击、嗅探、重放**；口令常与服务器 **共享**，难绑定「用户独有长期秘密」。

**PKI / 公钥认证 · 优势**

- **私钥不出网**；可配 **证书** 做强认证；私钥可用于 **签名** 或保护终端 **私有敏感数据**。  
- 题干若给 **百人级局域网**：公钥运算 **相对可接受**（写明「规模小、非全国并发」）。

**教材锚点**：§4.4.3 非对称；§4.5.2 **CA / 公钥证书**；§4.6.1 认证与访问控制。

**答题关键词**：**抗攻击**、**私钥不传输**、**私有数据保护**、**用户规模与性能可接受**。

---

### 1.2 大文件加密：流密码 + 数字信封

**选型**

- **数百兆级单文件**、吞吐优先 → 常答 **流密码（Stream Cipher）**：**连续处理**、适合 **大流量**（相对分组密码在案例中突出 **效率**）。  
- **数字信封**：用 **非对称** 保护 **随机对称会话密钥**；用 **对称密钥** 加密 **正文（bulk）** —— 兼顾 **密钥分发安全** 与 **对称高性能**（同 §4.5.3「公钥加密分配单钥」思想）。

**标准过程（背步骤）**

1. **加密**：生成随机对称密钥 → 用接收方 **公钥** 加密对称密钥，置 **文件头** → 用对称密钥对正文做 **流加密**。  
2. **解密**：用户用 **私钥** 解文件头得对称密钥 → 对称密钥解正文。

**教材锚点**：§4.5.3 会话密钥建立；§4.4.2 对称体系（分组在教材篇幅多，**流密码**按案例「长文件、吞吐」表述）。

---

### 1.3 库内敏感字段：加解密 API vs 透明加密（TDE）

| 方式 | 含义 | 优点 | 缺点 |
|------|------|------|------|
| **加解密 API** | 应用侧显式调用接口加解密 | **灵活**，字段级算法与密钥可定制 | **改代码**、安全职责 **分散**、维护成本高 |
| **TDE（透明加密）** | DBMS 按策略自动加解密，应用逻辑仍「明文视角」 | **应用少改或无改**、策略 **集中**、**安全/DBA** 统管密钥 | 灵活度略低于逐段手写 |

**案例结论句**：既要 **不同敏感级别不同密钥强度**，又要 **减轻开发与安全管理负担** → 倾向选 **TDE**。

**教材锚点**：§4.1.2 存储安全；第6章 库内字段保护（TDE 名词以案例与厂商为准）。

### 1.4 《07》中与案例同一张卷可用的「短句」（必背）

**安全三性**

- **可用性**：需要时可访问（备份、双机、限流）。  
- **完整性**：防未授权篡改 → **哈希、数字签名、事务**。  
- **机密性**：防泄露 → **加密、HTTPS、ACL、脱敏**。  
- **辨析**：完整性 = **防篡改**；机密性 = **防泄露**。

**对称 vs 非对称（口诀）**

- **对称**：**一钥**、**快**、**大数据**；AES、DES、3DES。  
- **非对称**：**密钥对**、**慢**、**小数据**；**密钥交换、签名、身份**；RSA、ECC、DH。

**非对称两场景**

- **数字签名**：**私钥签、公钥验**（本质是私钥对 **摘要** 加密，对方公钥解密比对）。  
- **保密通信 / 信封**：**公钥加密、私钥解密**；或 **公钥加密对称密钥**。

**数字签名流程（简背）**

- 发送：原文 → **Hash** → 摘要 → **私钥加密摘要** → 签名；发 **原文 + 签名**。  
- 验证：原文再 Hash 得摘要 A；签名用 **公钥解密** 得摘要 B；**A 与 B 一致** 则完整且来源可信。

**数字证书（简背）**

- **CA** 签发；绑定 **公钥与身份**；**CA 私钥** 签证书；验证方用 **CA 公钥**；**X.509**、**CRL**、**证书链**。

**Hash（杂凑）**

- **单向、抗碰撞、固定长度**；完整性校验与签名的输入摘要。

## 二、下次考试预测 · 直接背诵

> **依据**：案例卷 **仅 2013 一题**，与《07》**「再考概率大」**；近年上午/综合 **信息安全、等保、密码法、国密** 热度高，案例极可能以 **同构新场景**（医疗/政务/金融/仍可用航空素材）复现。

**高概率（模板可直接背）**

1. **选型论证**：**口令 vs 证书/PKI** 或 **OAuth 与证书** 混合表述 —— 仍按 **威胁（嗅探、重放、弱口令）→ 私钥不离网 → 规模** 展开。  
2. **大文件或批量数据**：**流密码 / 连续处理** + **数字信封**（随机对称密钥 + 公钥封装头 + 对称加密体）。  
3. **数据库落地**：**TDE vs API（应用层）vs 列加密/KMS** —— 题干若强调 **少改应用、策略集中** → **TDE**；强调 **算法与国密、与 HSM 深度耦合** → **API**。  
4. **数字签名 + 哈希**：支付/回调 **防篡改、防抵赖**；步骤背 **Hash → 私钥签摘要**。  
5. **HTTPS / TLS**：**握手协商对称会话密钥**，与 **非对称保护会话密钥** 同一叙述链。  

**中概率（与《07》叠答成加分段）**

6. **安全五服务**：**鉴别、访问控制、数据机密、数据完整性、抗否认** —— 各对应 **认证/签名、访问控制、加密、完整性机制、公证与审计**。  
7. **架构脆弱性**：**B/S、微服务、事件驱动** —— 攻击面、IPC、消息 **伪造/重放**；对策 **WAF、限流、TLS、mTLS、审计**。  
8. **国密答题句**：**SM2** 非对称、**SM3** 杂凑、**SM4** 对称；**TLS 国密套件**、合规场景区分。  

**低概率但防冷门**

9. **流密码 vs 分组密码** 细辨：**分组 + 模式/填充**；大文件仍优先案例式答 **流密码吞吐**。  
10. **公钥体系 vs 口令** 典型例题（见《07》四.5）整段论点迁移。  

**预测口诀（一整段）**

**口令弱嗅探重放私钥不出网百人可承受，大文件流密信封头包会话钥体对称扛，库内敏感要少改集中管选 TDE，签名先摘要私钥签公钥验证书信 CA，五服务五机制脆弱微服务 B/S 加分 WAF 与 TLS 国密 SM234。**

**场景钩子**：运控席位、签派放行 —— **强绑定、审计** 用证书；多人共用终端 —— 口令难绑个人长期秘密。

## 四、与《07-安全架构设计理论与实践·考点速记》交叉 · 直接背诵

**认证与加密（案例 + 选择通用）**

- **签名**：私钥签、公钥验；**加密**：公钥加密、私钥解密。  
- **TLS**：协商 **对称会话密钥**；素材 **全程 SSL/TLS**。  

**安全服务 ↔ 机制（简表，背服务名即可展开）**

| 安全服务 | 典型机制（记关键词） |
|----------|----------------------|
| **鉴别** | 认证、数字签名 |
| **访问控制** | 访问控制、路由控制 |
| **数据机密** | 加密、业务流填充 |
| **数据完整性** | 完整性机制 |
| **抗否认** | 公证、**审计日志** |

**架构脆弱性（案例「指出风险+对策」）**

| 架构 | 一句风险 |
|------|----------|
| **分层** | 层间依赖、单点瓶颈 |
| **C/S** | 客户端漏洞、网络暴露 |
| **B/S** | 恶意代码、脚本攻击面 |
| **事件驱动** | 控制弱、复杂交互、重放 |
| **微服务** | 面多、东向流量、mTLS/身份联邦复杂 |

**公钥体系 vs 口令（答题要点三句）**

1. **安全**：口令易猜易截；私钥 **不传**。  
2. **私有数据**：公钥体系下可用 **用户私钥** 护数更自然。  
3. **效率**：**小规模局域网** 公钥开销可接受。

**案例素材（航空，与《07》一致，可择句嵌入）**

- 三性：**99.99%**、对账验签、证件加密与 **脱敏**。  
- **RBAC、审计≥1 年、备份≥3 年、防 SQL 注入/XSS/DDoS、异地登录与爆破告警、民航局身份核验**。

## 七、考前一页纸

1. **论证套路**：**威胁** → **机制** → **规模/性能**（为何不用 A 用 B）。  
2. **信封必写**：随机对称钥 → **公钥封头** → **对称流加密正文**；解密 **私钥开头** 再解身。  
3. **TDE 必写**：**策略集中、应用少改**；API 对应 **灵活、改码、职责散**。  
4. **与《07》联动**：三性、对称/非对称、**签名四步**、证书 **CA**，五服务至少列 **名**。  
5. **提醒**：案例题可能 **不在第4章照抄**，但 **得分词** 仍是 **PKI、流密码、数字信封、TDE、签名、证书、威胁**。
