## 一、案例题一览（当前仓库可归纳）

| 考期 | 出处 | 核心考查 |
|------|------|----------|
| **2016-11 第5题** | `系统设计与建模.txt` | **Scrum** 流程：**Product Backlog / Sprint / Sprint Backlog / 每日站会 / 评审与回顾 / 燃尽图**；用 **UML 状态图** 描述敏捷生命周期；**MVC** 与 **Struts / Spring / Hibernate（SSH）** 分层填图 |

**说明**：案例中「项目组」「里程碑」等叙述在 **Web 应用设计、架构评估** 等题中常见，但 **一般不单独考** PMBOK 过程或挣值计算；**WBS、CPM、PERT、挣值、CCB、配置项状态** 等仍以 **上午题** 为主，见 **综合知识点** 文档。

**【考查点 · 与航空运营智能管理平台类比（备考「带场景」）】**  
| 考查点 | 平台中可写的素材 |
|--------|------------------|
| **案例 Scrum（2016）** | **Product Backlog**：航季 **新航站接入、规章字段、大屏指标** 等按商业价值排序；**Sprint**：2～4 周迭代交付 **可演示增量**（如离港联调通过）；**每日站会**：阻塞项如 **空管接口延迟、测试数据不全** |
| **进度 / WBS / 里程碑** | **一级**：需求冻结、核心运行模块、外部系统集成、试运行；**里程碑**：**AODB 联调完成、灾备演练通过、投产窗口** |
| **SCM / CCB / 配置项** | **配置项**：接口规范版本、离港接口包、数据库脚本；**变更**：**航季时刻表结构变更** 走 **CCB**，关联回归范围 |
| **风险** | **识别**：外部接口政策变更、关键人员离项；**应对**：原型验证、备份接口人 |
| **质量（McCall）** | **运行**：航班动态 **可用性**；**修改**：航季 **可维护性**；**转移**：移交运维 **文档与培训** |
| **需求追踪** | **正向**：**局方报表字段** → 数据库字段 → 接口 → 测试用例；**逆向**：线上缺陷 **回溯** 到需求条目 |

**【平台案例总览 · 航空运营智能管理平台】**  
平台建设类项目通常 **强依赖外部系统窗口**（离港、空管、机场 AODB）与 **固定航季/投产日**，项目管理重点落在 **范围基线与变更控制**、**里程碑驱动的集成计划**、以及 **敏捷迭代** 在「可演示增量」与「大集成里程碑」之间的衔接。下文 **【平台案例】** 均以此统一场景展开。

## 二、教材章节精确定位

### 1. §5.7 软件项目管理（进度 / 配置 / 质量 / 风险）

| 真题考点（上午/案例通用概念） | 教程位置 |
|------------------------------|----------|
| 管理对象：范围、风险、资源、任务、**里程碑**、成本、进度 | **§5.7.1** |
| 进度管理六活动：**活动定义、排序、资源估计、历时估计、制定进度计划、进度控制** | **§5.7.2** |
| **WBS**：项目→任务→工作→日常活动；**工作包**；计划/成本/风险等的基础 | **§5.7.2「1. 工作分解结构」** |
| **任务活动图**、前驱、里程碑、甘特图 | **§5.7.2「2. 任务活动图」** |
| **SCM**：标识、组织、控制修改；**版本控制**与**变更控制** | **§5.7.3** |
| 质量三维（**McCall**）：**产品运行 / 产品修改 / 产品转移**；SQA 要点 | **§5.7.4** |
| **风险管理**：辨识、评估、规划应对；Boehm / SEI 等体系简述 | **§5.7.5** |

**【平台案例】**  
- **范围与里程碑**：如 **M1 需求基线**、**M2 航班运行核心上线**、**M3 与离港/行李系统联调完成**、**M4 双活与演练**；里程碑常绑定 **机场投产窗口** 或 **航季切换日**。  
- **WBS**：可分解为 **业务域**（计划/运行/旅客）、**集成域**（ESB、接口）、**数据域**（主数据、历史迁移）、**非功能**（性能、安全、运维）；**工作包** 到人天可估算（如「某航站离港接口适配」）。  
- **进度六活动**：活动如 **接口联调、联调测试、性能压测、演练**；**历时** 受 **外部系统排期** 约束时常需 **缓冲**。  
- **SCM**：**版本** 管理 **接口 WSDL/JSON Schema、可执行包、库表脚本**；**变更控制** 与 **上线窗口** 绑定，避免「口头改接口」。  
- **质量三维**：**运行**—大屏与 App **SLA**；**修改**—航季需求 **变更成本可控**；**转移**—移交 **运行手册、监控告警、应急联系人**。  
- **风险**：例 **「空管流量接口测试环境不稳定」** → **缓解**：提前 **沙箱与桩**；**「关键 BA 离职」** → **减轻**：文档化与 **AB 角**。

### 2. §5.1.3 敏捷模型 — Scrum（与 2016 案例直接对应）

| 考点 | 教程表述要点 |
|------|----------------|
| Scrum **侧重项目管理**；迭代增量 | **§5.1.3「3. 主要敏捷方法简介 (3) Scrum」** |
| **产品 Backlog**：按商业价值排序的需求列表 | 同上 |
| **Sprint**：短迭代；从 Backlog 选高优先级需求组成 **Sprint backlog** | 同上 |
| 迭代结束递交**可交付增量**；全部 Sprint 完成后提交最终产品 | 同上 |

**与案例填图的关系**：真题还要求把 **制定 Product Backlog → Sprint 计划会议 → 每日站会 → 条件分支（仍有未完成故事 / 已全部交付）** 等画进 **状态图**，需在 **UML 状态图**（第2章）与上述 Scrum 语义之间切换。

**【平台案例】**  
- **Product Backlog**：条目示例——「**延误原因码与局方报表对齐**」「**某枢纽机场 A-CDM 事件订阅**」「**配载与离港数据一致性校验**」——按 **业务价值与依赖** 排序（运控优先于报表美化）。  
- **Sprint / Sprint Backlog**：单迭代目标如「**完成 T1 航站离港联调 + 演示**」；Sprint 内故事点受 **联调环境可用天数** 限制。  
- **每日站会**：同步 **昨日完成 / 今日计划 / 阻塞**；阻塞典型为 **接口证书未到位、测试航班数据未灌入、UAT 席位未排期**。  
- **评审 / 回顾**：评审向 **产品负责人/业务方** 演示 **可工作软件**（如改签闭环）；回顾改进 **Definition of Done、自动化回归范围**。  
- **燃尽图**：剩余故事点随 Sprint 递减；若曲线 **平台期**，常对应 **集成环境不稳定**（可写入案例分析）。  
- **状态图迁移**：与真题同构，仅将「故事」换成 **「航季需求条目」** 等 **领域词汇**。

### 3. §5.2.3 需求追踪（与变更、范围衔接）

- **正向跟踪**：需求 → 设计/代码/测试等后继制品。  
- **逆向跟踪**：从代码、文档等 **回溯** 到需求规格。  
- 上午题与 **范围/变更** 流程题常与此节 **概念对齐**（详见综合知识点文档）。

**【平台案例】**  
- **正向跟踪**：**运行规章** 要求 **舱单保留期限** → **数据库归档策略** → **备份任务** → **合规测试用例**。  
- **逆向跟踪**：生产问题「**旅客姓名与离港不一致**」→ 回溯 **接口映射需求** 与 **变更单** 是否引入回归遗漏。  
- **与变更/范围衔接**：**航季新增航站** 触发 **范围变更**，需更新 **WBS、接口清单、测试范围** 并走 **CCB**（与 §5.7.3 变更控制呼应）。

## 三、与《项目管理》选择题 PDF 的衔接

以下主题在 **案例卷** 出现频率低，但在 **《项目管理》PDF（上午汇编）** 中高度集中，建议与 **§5.7** 对照复习：

| 主题 | 教材锚点 | 备注 |
|------|----------|------|
| 产品配置、配置项、三状态、CCB | **§5.7.3** + 真题表述 | 选择题定义题多 |
| 范围：产品范围 vs 项目范围 | 以 **PMBOK 类** 题干为准；教程 **§5.7.1** 有「工作范围」叙述 | |
| **关键路径 / 总时差 / 三点估算 / 赶工** | **§5.7.2** | **计算题**需专项演练 |
| 成本预算与**成本基线**、变更流程 | **§5.7** 与组织过程常识 | 以真题步骤为准 |

完整条目见：**`综合知识点/项目管理_真题考点与教材对照_2026备考.md`**。

**【平台案例】**  
- **配置项 / 三状态 / CCB**：**配置项** 如 **V2.3 离港适配器**、**R2026 航季主数据脚本**；状态 **草稿/受控/发布**；**CCB** 审批 **投产日前一周** 的接口变更。  
- **范围双范围**：**产品范围**—平台交付的 **业务能力**（运行态势、改签、配载视图）；**项目范围**—**本期** 做哪些航站、哪些接口（避免范围蔓延成「全国一次全接」）。  
- **关键路径 / 赶工**：**关键路径** 常含 **外部联调**（不可压缩至零）；**赶工** 可加 **并行联调资源** 或 **延长每日窗口**（成本/风险上升）。  
- **成本基线与变更**：**预算** 含 **联调差旅、第三方接口费**；变更导致 **返工** 时对比 **成本基线** 与 **变更单**。

**【平台案例 · 与上表逐条对应】**
