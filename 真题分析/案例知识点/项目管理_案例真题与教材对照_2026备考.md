## 一、常考 · 直接背诵

### 1.1 进度管理（§5.7.2）

**六活动**（按序背）

1. **活动定义**  
2. **活动排序**  
3. **活动资源估计**  
4. **活动历时估计**  
5. **制定进度计划**  
6. **进度控制**  

**WBS 一句**

- 项目 → 任务 → 工作 → **工作包**；是 **计划、成本、风险** 等的基础。

**关联呈现**

- **任务活动图**、**前驱**、**里程碑**、**甘特图**。

---

### 1.2 配置管理 SCM（§5.7.3）

**核心**

- **标识、组织、控制修改**；**版本控制** + **变更控制**。

**案例答题句**

- **配置项**：接口规范、可执行包、**库表脚本** 等；变更走 **评审/Baseline/CCB**（措辞随题）。

---

### 1.3 质量 — McCall 三维（§5.7.4）

| 维度 | 记关键词 |
|------|----------|
| **产品运行** | 正确性、可靠性、效率、**可用性**、完整性 |
| **产品修改** | **可维护性**、灵活性、可测试性 |
| **产品转移** | **可移植性**、复用性、互操作性 |

---

### 1.4 风险管理（§5.7.5）

- **识别 → 评估 → 规划应对**（规避、减轻、转移、接受 —— 按题）。

---

### 1.5 需求追踪（§5.2.3）

- **正向**：需求 → 设计 / 代码 / **测试用例**。  
- **逆向**：缺陷 / 代码 → **回溯** 需求条目。  
- 与 **范围变更、回归范围** 一起答。

---

### 1.6 Scrum（§5.1.3）— 案例大分项

| 概念 | 背诵句 |
|------|--------|
| **Product Backlog** | 按 **商业价值排序** 的需求列表 |
| **Sprint** | 短迭代；从 Backlog 选条目 → **Sprint Backlog** |
| **交付** | 每 Sprint **可演示增量**；全部 Sprint 完成 → 最终产品 |
| **站会** | 昨日 / 今日 / **阻塞** |
| **评审** | 对干系人演示 **可工作软件** |
| **回顾** | 改进流程与工程实践 |
| **燃尽图** | 剩余工作量（故事点）随 Sprint **递减** |

**与状态图结合**（2016）：把 **制定 Backlog → Sprint 计划会 → 开发中 → 评审 → 回顾 → 下一 Sprint** 及 **条件分支（仍有未完成 / 已交付）** 画成 **UML 状态图** —— 迁移标签 **随题干**。

---

### 1.7 MVC + SSH 分层（2016 填图）

- **M**：**Model** — 数据与业务规则；**V**：**View**；**C**：**Controller**  
- **Struts**：Web **MVC / 控制**  
- **Spring**：容器、**业务与集成**  
- **Hibernate**：**ORM**、持久化

## 二、下次考试预测 · 直接背诵

**高概率**

1. **敏捷 Scrum** 名词默写 + **状态图填空**（或换 **Kanban** 列名，仍以 **迭代+可视化** 表述）。  
2. **MVC + 框架分层** 空位填 **Struts/Spring/Hibernate** 或同类（**随题**）。  
3. **WBS / 里程碑** 与 **外部系统联调窗口** 结合叙述（不考细算则写 **关键路径在外部**）。  
4. **需求追踪** 正向/逆向 + **变更影响测试范围**。  

**中概率**

5. **SCM**：配置项、**版本与变更**、与 **上线窗口** 绑定。  
6. **McCall** 三维 **各举一类** 非功能词。  
7. **风险**：接口政策、关键人员、环境不稳 → **缓解/减轻** 各一句。  

**低概率 / 防偏**

8. **纯挣值、双代号网络大题** —— 更在 **上午**；案例若出现，转 **综合知识点** 专项。  
9. **产品与项目范围** 双范围辨析。  

**预测口诀**

**六过程WBS里程碑，配置双控追正向逆向，McCall运改转风险识评对，Scrum backlog sprint站会燃尽状态迁，MVC三驼SSH填，上午算路挣值另卷看。**

## 四、与《01-需求分析·考点速记》交叉 · 直接背诵

> **何时用《01》**：案例 **Scrum 状态图**、**MVC** 与 **系统设计** 题共用 **UML**；**进度/WBS** 论述中若出现 **DFD**，用《01》规则 **防丢分**。

**DFD（与项目管理「外部依赖、数据落地」同卷时常混用）**

| 点 | 背一句 |
|----|--------|
| 四符号 | **流箭、加工圆、存直线、外部框** |
| 平衡 | 父子 **个数与方向一致** |
| 三洞 | **黑洞** 只进不出；**奇迹** 只出不进；**灰洞** 输入撑不起输出 |
| 硬规则 | **外部实体不经加工不能直连数据存储** |

**用例关系（箭头）**

- **<<include>>** → 指向 **被包含的抽象/公共用例**（**包含指抽象**）  
- **<<extend>>** → 指向 **被扩展的基本用例**（**扩展指基本**）  
- **泛化** → 指向 **父用例**

**顺序图 vs 通信图**

- **顺序图**：**时间先后**  
- **通信图（协作图）**：**结构链接**、消息编号  

**状态图符号**（Scrumer 填迁移线）

- **初态** 黑点；**终态** 双圈；**状态** 圆角矩形；**迁移** 箭头 + **事件/守卫**

**类关系强度（《01》口诀）**

- **饭（泛）食（实）组聚关羽（依）** —— 泛化=实现 > 组合 > 聚合 > 关联 > 依赖  

**与项目管理合写一句**

- **里程碑** 选 **可验证交付**（演示增量）；**DFD** 选 **数据从哪加工到哪存储** —— 同一项目 **计划层** 与 **分析层** 各一张「图」。

6. **追踪**：**正向 / 逆向** 各一句。  
7. **填图**：**MVC** + **Struts / Spring / Hibernate**。  
8. **《01》**：DFD **存储不经外部直连**；用例 **include/extend 箭头**；状态图 **事件在箭头上**。
