## 一、常考 · 直接背诵

> 下述为 **案例真题反复出现** 的表述，按句背即可考场展开。

**旁路缓存（Cache-Aside）**

- **读**：先读缓存 → **miss** 再读库 → **回填**缓存。  
- **写**：**先写数据库**，成功后再 **删除或失效** 缓存（忌简单「双写缓存+库」）。  
- **双写仍可能不一致**：一侧失败；**并发**下读写 **乱序**。  

**缓存三问题**

- **穿透**：无效 key 查库 → **布隆过滤器** 前置；合法 **规则校验**。  
- **雪崩**：大量 key 同时失效 → **TTL 随机化**、**多级缓存**、**互斥重建 / 队列限流**。  
- **空值缓存**：防反复穿透，但要防 **恶意随机 key 撑爆** → 布隆、短 TTL、容量与封禁。  

**反规范化（§6.3.4）**

- **手段**：**冗余列**、**派生列**、**表重组**、**表分割**（水平 / 垂直）。  
- **目的 / 代价**：提高 **读性能**；**冗余** 与 **更新异常**。  
- **冗余一致三法**：**应用程序同步**、**批量处理同步**、**触发器同步**。  

**分布式缓存**

- **普通哈希取模**：节点数变化 → **大量 key 迁移**。  
- **一致性哈希 + 虚拟节点**：扩缩容主要影响 **环上邻域**；虚拟节点 **摊平热点**。  

**Redis 一口清**

- **类型**：计数 **String(INCR)**；对象 **Hash**；时序列表 **List**；去重 **Set**；排行 **ZSet**。  
- **持久化**：**RDB** 快照恢复快、可能丢间隔内数据；**AOF** 更安全、文件大、重放慢。  
- **过期与淘汰**：**惰性 + 定期** 清不完「永不访问的过期键」→ **maxmemory-policy**（LRU/LFU/随机等）。  
- **与 MySQL 对齐**：应用双写、**Canal + Binlog**、**MQ**、**延迟双删**（按一致性与吞吐选）。  

**MemCache vs Redis（填表）**

- Memcache：**简单 K-V**、无持久化、集群方案弱。  
- Redis：**多数据结构**、**RDB/AOF**、**主从·哨兵·Cluster** 生态。  

**逻辑建模（2020 类）**

- **逻辑设计任务**：定数据模型；**E-R→关系**；**完整性约束**（实体 / 参照 / 用户自定义）；**用户视图**。  
- **超类/子类**：公共属性上提；**派生属性**：算不算、要不要落库（读频、审计、口径）。  

**数据架构一句话**

- 强一致事务 → **RDBMS**；海量文件/轨迹 → **HDFS** 等；宽列海量 → **HBase**；检索 → **ES**；文档 + 向量 → **MongoDB** 等；**内存库** 加速热点、**关系库** 落账。  

**异步准实时**

- 可先写库再 **异步刷新** 缓存，**秒级展示** 类业务常接受 **最终一致性**。

## 二、下次考试预测 · 直接背诵

> **预测依据**：近年已出现 **分层填图、向量、图库、Scrapy、异步 I/O**；缓存与反规范 **多年重复**；与综合知识 **分片、读写分离、主从延迟** 的 **交叉命题** 概率上升。

**高概率（写法可当模板背）**

1. **旁路 + 一致性强化**：**先库后删缓存**；并发下 **版本号 / 分布式锁 / 延迟双删**；与 **Canal+Binlog、MQ** 二选一或组合说清。  
2. **反规范化**：四手段 + 三同步；追问 **高频更新** 时答 **触发器或应用事务**。  
3. **Redis**：五类型 + RDB/AOF 对比 + 过期策略 + **maxmemory**；**ZSet 排行榜** 仍是送分场景。  
4. **哈希**：普通 vs **一致性哈希 + 虚拟节点**（扩缩容迁移量、负载均衡）。  
5. **穿透 / 雪崩 / 空值**：三件套口诀各一句 + **布隆**、**TTL 抖动**。  
6. **多模存储填图**：**接口 / 处理 / 存储**；**MySQL + HDFS + HBase + ES**；**MongoDB + 向量字段**（相似检索、RAG 类表述）。  
7. **图 + 采集**：**Neo4j 等** 多跳、**Cypher**；**Scrapy**：Engine → **Scheduler** → Downloader → Spider → **Item Pipeline**；**异步 I/O** 高并发网络。  

**中概率（与《11-数据库系统》速记叠答，易成加分点）**

8. **读写分离**：写主读从；素材 **票务高峰** 读多写少。  
9. **主从延迟**：刚写完读从可能旧值 → **半同步**；**中间件记 key 时间窗读主**；**写前 key 进短时缓存（如 500ms）读主**。  
10. **分片 vs 分区**：**分片跨实例扩规模**；**分区单实例内** 优查询与管理；订单/日志 **哈希 / 范围** 分片 + 按日期 **分区** 可一起写。  
11. **索引**：**常查少改、不宜过多**；热点查询字段建组合索引。  

**低概率但防冷门**

12. **逻辑建模**：E-R **超类/子类**、**派生属性** 仍可能单独小问。  
13. **文件系统 vs RDBMS**、**内存库 + 关系库** 混合（2015 逻辑在现代题中可改述为「对象存储 + 交易库」）。  

**预测口诀（一整段可背）**

**旁路先库后删防双写，布隆防穿抖动防崩，反规四招三法对齐 Redis 五型两持久，一致性哈希virtualnode，分层 MySQL 加 HDFS HBase ES，Mongo 向量图库 Scrapy 异步题。**

## 三、分主题精背（真题 + 教材）

### 3.1 逻辑设计与建模（2020-11-② 等）

| 考点 | 背诵句 |
|------|--------|
| 逻辑设计任务 | 定模型；**E-R→关系**；**完整性约束**；**用户视图**（§6.3、§6.3.2） |
| 超类/子类 | 公共属性上提，子类扩展；**泛化/特化** |
| 派生属性 | 可算可得；是否存储看 **读频、审计、口径** |

**场景钩子**：参与方超类；延误时长、费用合计是否落库。

### 3.2 范式、反规范化、冗余一致（2021-11-④）

**口诀**：**反规范为读，冗余要同步；三种手段，三种修法。**（手段四类、修法三类，勿混）

| 维度 | 内容 |
|------|------|
| 手段 | **冗余列**、**派生列**、**表重组**、**表分割（水平/垂直）** |
| 冗余一致 | **应用**、**批量**、**触发器** |
| 选用 | 高频更新 + 强一致 → **触发器 / 应用事务** |

**场景钩子**：航班宽表；航季分表；改机位应用事务或夜间批量对齐。

### 3.3 缓存架构 · 一致性与经典问题

见 **第一节「常考 · 直接背诵」**；此处只补 **教材线索**：穿透/雪崩等可关联教程 **绪论能力 §1.2** 表述，**答题以真题话术为准**。  

**场景钩子**：改航先落库再删缓存；布隆拦非法航班号；高峰 TTL 随机抖动。

## 四、与《11-数据库系统·考点速记》交叉 · 直接背诵（案例加分）

> 下列在 **案例/11-数据库系统-考点速记.md** 为 ★★★★★；案例卷若出现 **性能、扩展、读放大、主从**，可直接迁移术语。

**索引**

- **常查少改**、**不宜过多**、**小表可不建**；索引多拖累 **INSERT/UPDATE/DELETE**。  

**分片（概念）**

- **水平**：按行，**并** 还原；**垂直**：按列，**连接** 还原；**导出分片**、**混合**。  
- **算法**：**哈希/取模** 均匀；**范围** 时序；**列表** 枚举；**一致性哈希** 节点动态。  

**分区**

- **单实例内** 逻辑子集；**范围/列表/哈希/复合**；利于查询裁剪与生命周期。  

**分片 vs 分区 · 口诀**

- **分片跨节点扩规模，分区单机内提查询。**  

**读写分离**

- **一主多从**，写主读从；实现 **ShardingSphere、MyCat** 等。  

**主从复制（MySQL）**

- **binlog** → 从库 I/O → **relay log** → **SQL 线程** 应用。  

**主从延迟 · 三招**

| 方式 | 记一句 |
|------|--------|
| **半同步** | 等至少一从 ACK，主库 **写延迟↑、吞吐↓** |
| **中间件** | 记 **刚写过的 key**，时间窗内 **读走主库** |
| **缓存记 key** | 写前 key 进 cache 短 TTL，命中则 **读主** |

**选型口诀（与真题 Mongo/Redis/HBase 合并）**

- **缓存键值 Redis，海量列存 HBase/Cassandra，灵活文档 MongoDB，图 Neo4j，时序 InfluxDB/TDengine。**

**案例一句（航空）**

- **订单写多查少、航班动态读多写少**；**≥5000 TPS、≤1 秒**；**PB、单表 50 亿条+** → **索引 + 分片/分区 + 读写分离 + Redis**，与本文 **旁路缓存、反规范** 同案可写。

## 六、综合类比 · 真题考点 ↔ 航空平台一句

| 真题侧重 | 平台一句 |
|----------|----------|
| 准实时、一致性哈希、布隆 | 航班动态 <1s；扩缩容；**无效航班号** 拦截 |
| 反规范、ZSet、同步 | **宽表**；延误榜；双写 / Binlog / **延迟双删** |
| 逻辑模型、派生 | **参与方**；延误时长、费用合计 |
| Redis、RDB/AOF、淘汰 | **Hash** 快照；**List** 事件；航季 **TTL** |
| 旁路、穿透、雪崩 | **改航** 先库后删；高峰 **TTL 抖动** |

| 分层、ES、Mongo 向量 | **接口/处理/存储**；检索 + **相似案例** |
