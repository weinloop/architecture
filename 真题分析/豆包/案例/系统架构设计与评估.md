# 系统架构设计与评估（案例核心考点·极简总结）
本模块是**系统架构师案例题第一题必考内容**，无计算，纯**话术模板+固定选型+概念匹配**，核心围绕**质量属性、架构风格、架构评估、大数据架构、微服务、DDD**六大方向，精简背诵版如下：

## 一、核心重中之重：质量属性（效用树+场景匹配）
### 1. 六大核心质量属性（真题100%匹配）
| 属性 | 核心定义 | 典型场景（直接背） |
| ---- | -------- | ------------------ |
| 性能 | 响应时间、吞吐量、资源利用率 | 3秒内响应、CPU≤40%、任务1分钟内启动 |
| 可用性 | 故障恢复、不间断服务 | 宕机5分钟恢复、断电3秒切换备机、离线可用 |
| 安全性 | 数据加密、防攻击、权限控制 | AES加密、抵御黑客攻击、操作日志审计 |
| 可修改性 | 变更成本低、快速扩展 | 10天集成新API、3天功能上线、配置化改规则 |
| 易用性 | 用户体验、易学性 | 多语言、自适应界面、60秒上手操作 |
| 可测试性 | 易测试、仿真验证 | 远程测试接口、恶劣环境仿真 |

- 示例：集成新API=**刺激**；故障5分钟恢复=**响应度量**

## 二、架构风格选型（对比题固定结论）
所有场景**选型答案唯一**，直接背诵：
1. **解释器风格**：**规则灵活配置首选**
   适用：促销折扣规则、机器学习流程、模型训练平台；优势：动态改规则、无需改代码，性能略低。
2. **仓库风格**：**共享数据/IDE系统首选**
   适用：在线开发平台、代码仓库；优势：数据中心化、交互灵活，管道过滤器仅适合固定数据流。
3. **基于规则架构**：**动态分级/策略首选**
   适用：用户等级、折扣动态调整；优于面向对象，无需重启系统。
4. **瘦客户端C/S**：**多终端/易维护首选**
   优势：服务端集中部署，支持个性化定制、多语言、快速故障恢复。

## 三、架构评估三要素（必考定义+匹配）
1. **架构风险**：设计潜在隐患（需求未共识、模块紧耦合）；
2. **敏感点**：影响**单一**质量属性的关键设计；
3. **权衡点**：影响**多个**质量属性（经典：加密级别↑=安全↑性能↓）；
- 答题模板：直接套定义+题干对应描述即可。

## 四、大数据架构：Lambda vs Kappa
1. **Lambda架构（奥运场景首选）**
   三层结构：**批处理层（离线）+速度层（实时）+服务层（合并视图）**；适配实时+离线双分析需求。
2. **Kappa架构**：纯流处理单链路，架构简化，无批处理。
3. 核心技术栈固定匹配
   采集：Flume/Kafka；计算：Spark/Spark Streaming；存储：HDFS/HBase/ES

## 五、微服务架构（单体VS微服务）
1. **单体架构**：开发简单、部署一体；缺点：耦合高、扩展难、故障影响全系统；
2. **微服务架构**：按业务拆分、独立部署/扩展、故障隔离；缺点：运维复杂、通信开销大；
3. 配套能力：容器化、服务注册发现、统一配置中心。

## 六、高频补充考点
1. **可靠性检测**
   - Ping/Echo：主控主动探测，小规模适用，**资源消耗高**；
   - 心跳机制：节点主动上报，**大规模分布式首选，资源利用率高**。
2. **DDD领域驱动设计**
   - 核心：以业务领域为中心，统一语言建模；
   - 限界上下文：语义统一边界，**拆分微服务、降低复杂度、团队解耦**。
3. **弱网业务解决方案**
   本地缓存+本地事务+异步同步+全局唯一ID（幂等）+日终对账，保证**最终一致性**。
4. **AES超长数据加密**
   分块处理→末尾填充→分组模式（CBC/GCM）→逐块加密拼接。
