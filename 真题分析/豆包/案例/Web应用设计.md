# Web应用设计（架构师案例·极简应试总结）
本模块为**案例高频必考**，核心考**Web分层架构、通信协议、缓存优化、数据库架构、持久层框架、非功能需求、分布式锁/主从复制、边缘计算**，题型以**架构填空、概念对比、方案优劣分析**为主，直接套模板得分。

## 一、核心通信协议（必考对比）
### 1. MQTT协议（物联网/边缘计算专用）
- 定义：**发布/订阅模式**轻量级消息协议，基于TCP/IP，报文极小、低功耗；
- 核心特性：QoS三级消息质量、持久会话、遗嘱消息；
- 适用场景：**边缘设备-云端通信**（门禁、物联网设备指令下发/状态上报）；
- 选型结论：**Web交互用HTTP，设备通信用MQTT**。

### 2. TCP vs UDP（可靠通信必考）
| 特性 | TCP | UDP |
| ---- | ---- | ---- |
| 连接 | 面向连接（三次握手） | 无连接 |
| 可靠性 | 可靠、有序、重传、无丢包 | 不可靠、易丢包、无重传 |
| 速度 | 慢、开销大 | 快、开销小 |
| 选型 | 设备控制、状态反馈（智能家居） | 视频/语音直播 |

## 二、架构设计核心模型
### 1. MVC架构（JavaEE/Web通用）
- 三层职责：**View(视图)=JSP**（展示）、**Controller(控制器)=Servlet**（请求转发）、**Model(模型)=Service/JavaBean/DAO**（业务+数据）；
- EJB构件分类：**会话Bean**（业务逻辑）、**实体Bean**（持久化）、**消息驱动Bean**（异步消息）；
- 有状态/无状态：**有状态=身份认证、在线编辑**；**无状态=查询、统计、资源发布**。

### 2. 分布式Web分层架构（高并发标配）
**表现层→负载均衡→逻辑处理层→分布式通信→消息队列(Kafka)→缓存(Redis)→数据存储层**
- 核心组件：Nginx(负载均衡)、Kafka(消峰解耦)、Redis(热点缓存)、MySQL(持久化)。

### 3. 边缘计算 vs 传统云计算
- 优势：**低延迟**（本地处理）、**高安全**（原始数据不上传）、**高性能**（云端减负，边云协同）；
- 适用：智能门禁、工业物联网、实时识别场景。

## 三、非功能需求分类（送分题，必背）
1. **性能**：并发数、响应时间、实时性、吞吐量；
2. **安全性**：防SQL注入/XSS、数据加密、审计；
3. **可用性**：7×24运行、故障恢复时间(RTO)、无单点故障；
4. **易用性**：界面友好、学习成本低。

## 四、缓存优化（高并发核心，真题高频）
### 1. 缓存击穿解决方案
1. **互斥锁（物理过期）**：仅单线程查库重建缓存，其他线程等待；**强一致、性能差**；
2. **逻辑过期**：缓存存数据+过期时间，异步刷新，先返回旧数据；**高可用、最终一致**；
3. 选型：**电商读多写少用逻辑过期，金融强一致用互斥锁**。

### 2. Redis核心考点
- 主从复制：**主写从读**，读写分离、负载均衡、数据冗余；
- 持久化：**RDB（快照，快、丢数据）** + **AOF（日志，全、恢复慢）**；
- 分布式锁：数据库锁**性能差、易死锁**，Redis锁**高性能、需防死锁（加过期时间）**。

### 3. Memcached vs 数据库查询缓存
Memcached：**分布式、高扩展、缓存命中率高**，支持键值对/对象；数据库缓存：单库生效、表更新即清空，局限性大。

## 五、数据库架构优化
### 1. 主从复制+读写分离
- 好处：提升读并发、负载均衡、数据备份、故障容错；
- 适用：电商、社交系统（读多写少）。

### 2. 数据库分区
- 水平分区：按**用户ID/范围**拆分，适合社交/物流按用户查询场景；
- 垂直分区：按列拆分，适合大表字段分离。

### 3. SQL注入防御（必考简答）
1. **参数化查询/预编译**（根本方案）；2. 输入白名单过滤；3. 数据库最小权限。

## 六、持久层框架对比
1. **ORM框架（Hibernate）**：全自动ORM、**无SQL、跨数据库、适合复杂对象关系**；
2. **SQL映射（MyBatis）**：半自动、手写SQL、灵活调优、适合复杂查询；
3. 持久层技术分类：JDBC封装→SQL映射→ORM→EJB实体Bean。

## 七、企业集成架构（SOA/ESB）
1. SOA：将功能封装为**可复用服务**，松耦合、标准化接口；
2. ESB（企业服务总线）：SOA核心，**协议转换、消息路由、格式转换、解耦异构系统**；
3. 安全措施：HTTPS加密、数字签名、防火墙/入侵检测。

## 八、前端/接口设计
1. 响应式Web：**流式布局+媒体查询**，适配多终端（PC/手机/平板）；
2. RESTful接口：**HTTP方法(GET/POST/PUT/DELETE)表达操作**，无状态、JSON返回，前后端分离标配；
3. JWT令牌：**Header(算法)+Payload(用户信息)+Signature(防篡改)**，无状态认证。

## 九、应试秒杀结论
1. **协议选型**：人机交互HTTP，设备通信MQTT，可靠控制TCP；
2. **缓存原则**：热点数据Redis，击穿用逻辑过期，强一致用互斥锁；
3. **数据库**：读多写少做读写分离，大表水平分区，防注入用预编译；
4. **架构分层**：MVC解耦，高并发加负载均衡/消息队列/缓存；
5. **优劣分析**：永远从**一致性、可用性、性能、复杂度、扩展性**五个维度作答。
