# 系统架构设计师-软件测试 核心考点速记
本模块**必考2~3分**，纯概念+易混区分，无复杂计算，核心围绕**测试分类、阶段流程、覆盖强度、性能测试、自动化测试**五大方向，以下为极简背诵版。

## 一、测试核心原则（必考判断）
1. 测试**只能发现缺陷，无法证明软件无错/绝对正确**
2. 缺陷群集效应：**80%缺陷集中在20%模块**（二八定律）
3. 测试用例必备：**输入条件 + 预期输出**，缺一不可
4. 测试应尽早介入，贯穿全生命周期

## 二、测试方法分类（高频区分）
### 1. 静态测试 vs 动态测试
- **静态测试**：**不运行程序**；桌面检查、代码审查、代码走查
  静态分析：**数据流分析**（数据初始化/赋值异常）、控制流分析
- **动态测试**：运行程序；包含所有白盒/黑盒测试

### 2. 白盒/黑盒/灰盒测试
| 类型 | 核心特点 | 常用方法 | 覆盖强度排序 |
| ---- | -------- | -------- | ------------ |
| 白盒测试 | 看**内部逻辑结构** | 语句/判定/条件/路径覆盖 | **路径覆盖（最强）> 组合覆盖 > 判定/条件覆盖 > 条件/判定覆盖 > 语句覆盖（最弱）** |
| 黑盒测试 | 功能测试，**不看内部** | 等价类、边界值、**判定表（多条件组合动作）**、因果图 | - |
| 灰盒测试 | 兼顾**输入输出+内部逻辑** | - | - |

- 易错点：条件覆盖**不一定包含**判定覆盖，二者无绝对强弱；组合覆盖要求**所有条件组合至少执行一次**

## 三、测试阶段流程（顺序+依据 必考）
**标准顺序**：单元测试 → 集成测试（组装测试）→ 系统测试 → 确认/验收测试
1. **单元测试**
   测单个模块，依据**详细设计文档**；驱动模块（调用被测）、桩模块（模拟子模块）；**自顶向下无需额外写驱动模块**
2. **集成测试**
   测模块间接口，依据**概要设计文档**；组装策略：**一次性组装、增量式组装**
3. **系统测试**
   测整体系统，依据**需求规格说明书**
4. **确认测试（有效性测试）**
   验证符合用户需求；包含：**Alpha（开发环境内测）、Beta（用户环境公测）、验收测试**
5. **回归测试**
   代码修改后执行，核心目的：**验证变更无新缺陷、原有功能不受损**

## 四、性能测试分类（易混名词秒杀）
1. **强度测试**：资源受限下，测试系统**极限运行**能力
2. **容量测试**：测试系统**最大并发用户数**
3. **负载测试**：正常负载下，验证系统性能指标
4. **压力测试**：超负载运行，**定位性能瓶颈/崩溃点**
5. **基准测试**：建立系统性能基线，用于对比回归
6. 精确度排序：**真实程序 > 核心程序 > 小型基准测试 > 合成基准测试**

## 五、自动化测试
1. 脚本类型
   - **线性脚本**：直接录制手工测试生成
   - **数据驱动脚本**：测试数据存**独立文件**，与脚本分离
2. 适用场景：需求稳定、回归频繁；**不适合：需求频繁变更的系统**
3. 代码覆盖率：主要衡量**单元测试**的充分性

## 六、特殊测试与补充
1. **Web新型测试**：**A/B测试**（同时间、同质用户随机分组，对比多版本效果）
2. Web服务器性能评测：含基准/压力/可靠性测试，**不含UI测试**
3. 面向对象测试：多态需测**所有动态绑定**；子类继承方法**仍需测试**
4. 链接是**Web应用**核心特征

## 七、测试 vs 调试（终极区分）
1. **测试**：**发现缺陷**，有计划、流程固定、周期可预估
2. **调试**：**定位+修复缺陷**，无固定流程、周期不可预估

## 八、高频易错秒杀清单
1. 白盒测试**不含功能测试**；黑盒测试**不含路径覆盖**
2. 空行/注释**不计入**语句覆盖统计
3. 测试用例必须覆盖**合法+非法输入**
4. 单元测试依据详细设计，集成依据概要设计，系统依据需求文档
