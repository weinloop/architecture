# 软考系统架构设计师：真题核心必背知识点+考题题型预测
结合**2013-2025年系统架构师真题**（你提供的真题全覆盖）+ 官方教材，提炼**100%高频必背考点**，精准预测考题形式，聚焦**上午单选题（70分）** 核心考法，兼顾下午案例题高频方向。

# 一、核心必背知识点（9大模块，真题反复考查）
## 模块1：软件架构基础概念
1. 架构核心三要素：**结构、行为、属性**
2. 架构描述语言**ADL四要素**：**组件、接口、连接件、配置**（必考填空）
3. 消息传递架构风格：**异步传输、点到点/发布订阅、构件独立、连接件=消息传递**
4. 架构核心作用：**非功能质量保障、干系人沟通、结构建模**（**不负责表达功能性需求**）
5. 架构设计核心：**组件结构、属性、交互作用**，用**多视图**描述架构

## 模块2：基于架构的软件开发（ABSD/ABSDM）
1. **ABSD三大驱动**：**商业、质量、功能需求**
2. **ABSD三大基础**：功能分解、**架构风格实现质量需求**、软件模板设计
3. **ABSD方法特征**：**自顶向下、递归细化**，最终产出**构件/类**
4. **ABSDM六大过程（必考顺序）**：需求→设计→**文档化**→**复审**→**实现**→演化
5. 文档化输出：**架构规格说明、质量设计说明书**
6. 架构描述：**视角+视图**；需求描述：**用例（功能）+质量场景（质量）**
7. 体系结构演化6步（必考顺序）：需求归类→制定计划→增删改构件→**更新构件交互**→组装测试→技术评审

## 模块3：软件架构评估（最高频考点）
### 1. 三大评估方法核心
- **SAAM**：最早，关注**可修改性**，输入：问题描述+需求说明+架构描述
- **ATAM**：主流，**质量属性权衡**，核心工具**效用树**，关注4大属性：**性能、可用性、安全性、可修改性**
- **CBAM**：ATAM扩展，**成本效益/ROI** 决策
- 静态分析方法：**SASAM**

### 2. 核心概念（必考辨析）
- **敏感点**：单构件影响**单个**质量属性
- **权衡点**：影响**多个**质量属性（如加密级别=安全+性能）
- **风险点**：潜在缺陷；非风险点：无影响的设计
### 3. 效用树结构（必考层级）
**树根→质量属性→属性分类→质量属性场景（叶子）**
### 4. 质量场景六要素（必考填空）
**刺激源→刺激→环境→制品→响应→响应度量**
### 5. ATAM场景分类：用例场景、增长场景、**探索性场景（增长极端形式）**

## 模块4：4+1视图模型（必考匹配）
1. 核心思想：**关注点分离**
2. **4基础视图+1场景**：
   - 逻辑视图：功能需求、类图/对象模型
   - **进程视图**：并发、同步、运行时动态行为
   - 开发视图（实现视图）：代码静态结构、模块划分
   - 物理视图（部署视图）：软件→硬件映射、节点部署
   - +1：**场景视图（用例）**，串联验证所有视图
3. Hofmeister 4视图：**概念、模块、执行、代码**

## 模块5：特定领域软件架构（DSSA）
1. **三大核心阶段（必考）**：
   - 领域分析→产出**领域模型**
   - 领域设计→产出**DSSA架构**
   - 领域实现→开发复用资产
2. 域分类：**垂直域（单领域系统族）、水平域（跨领域公共功能）**
3. 人员角色：领域专家、**领域分析师**、领域设计者、领域实现者
4. 三层环境：领域开发→领域特定应用开发→应用执行

## 模块6：软件质量属性（场景题必考）
### 1. 分类
- **运行期**：性能、可用性、安全性、健壮性、互操作性
- **开发期**：可维护性、可扩展性、可测试性、可移植性、可重用性
### 2. 核心属性定义（真题场景匹配）
- 性能：响应时间、吞吐量、并发处理
- 可用性：故障恢复（MTTR/MTBF）、双机热备、心跳检测
- 安全性：加密、防攻击、追踪审计
- 可修改性：变更成本、模块解耦
- 可移植性：跨平台/环境运行
- 健壮性：异常/非法操作下正常运行
### 3. 经典战术
- 性能：资源调度、并发、增加资源
- 可用性：**主动冗余、心跳检测**
- 安全性：追踪审计、权限控制
- 可修改性：信息隐藏、接口分离

## 模块7：新技术架构（送分题）
1. 云计算三层：**SaaS（应用层）、PaaS（平台层）、IaaS（基础设施层）**
2. 微服务核心：**去中心化、独立部署、轻量级通信**；SOA：**集中式（ESB）**
3. 断路器三态：**关闭、打开、半打开**
4. RESTful：**资源为中心**，HTTP方法语义（GET查/POST增/PUT全更/PATCH部分更）
5. WebService三要素：
   - **WSDL**：服务接口描述（做什么/怎么访问/在哪）
   - **SOAP**：远程调用协议
   - **UDDI**：服务注册、发现、集成
6. 区块链分类：**公有链、私有链、联盟链**（无行业链）
7. 边缘计算：**低延迟、省带宽、高安全**（不降低功耗）
8. Lambda架构：**批处理层（全量）、加速层（增量实时）、服务层**

## 模块8：软件架构复用
1. 复用类型：**机会复用（临时发现）、系统复用（提前规划）**
2. 复用三阶段（必考顺序）：**构造/获取→管理→使用**可复用资产

## 模块9：SOA与微服务
1. ESB（企业服务总线）：SOA核心中间件，**解耦服务请求/提供者，消除直连**
2. BPEL：**服务编排**，组合多个Web服务
3. 微服务vs SOA：微服务**去中心化**，SOA**集中式ESB**

# 二、高频易混考点（真题错题重灾区）
1. **敏感点≠权衡点**：单属性=敏感点，多属性冲突=权衡点
2. **4+1视图职责**：并发=进程视图，代码结构=开发视图，硬件部署=物理视图
3. **ABSDM阶段**：文档化出规格说明，复审找缺陷，演化处理需求变更
4. **质量场景**：刺激=触发事件，响应=系统动作，环境=触发条件
5. **DSSA阶段产出**：分析=领域模型，设计=DSSA架构
6. **RESTful URI**：**多URI对应1资源**（多对一，非多对多）

# 三、考题题型&考点分布预测
## （一）上午单选题（70分，核心占比30%）
### 1. 必考题型1：**概念定义填空**
- 考法：ADL要素、ABSDM过程、质量场景六要素、4+1视图定义
- 预测考点：连接件、效用树结构、演化6步、DSSA三阶段

### 2. 必考题型2：**场景匹配题**
- 考法：给业务场景，匹配**质量属性/架构风格/评估概念**
- 预测考点：加密级别=权衡点、故障恢复=可用性、高并发响应=性能

### 3. 必考题型3：**技术辨析题**
- 考法：微服务vs SOA、SAAM vs ATAM、云计算三层、WebService三要素
- 预测考点：ESB作用、RESTful方法语义、断路器状态

### 4. 必考题型4：**顺序排序题**
- 考法：ABSDM过程、架构演化步骤、软件复用阶段
- 预测考点：演化6步、复用三阶段

## （二）下午案例题（75分，高频出题方向）
1. **架构评估案例**：区分敏感点/权衡点/风险点，ATAM流程、效用树绘制
2. **质量属性案例**：给系统场景，识别质量属性+设计对应战术
3. **4+1视图案例**：匹配视图与UML图、描述各视图职责
4. **微服务/SOA案例**：微服务架构设计、ESB作用、服务解耦方案

# 四、考前速记口诀
1. 架构三要素：**结构行为加属性**，ADL四件：**组接接配**
2. ABSD驱动：**商质功**，细化：**自顶向下到构件**
3. 评估三兄弟：**SAAM改、ATAM衡、CBAM算钱**
4. 权衡敏感记：**单敏多权**，加密性能安全是经典
5. 4+1视图：**逻功进并开静部硬**，场景串联全验证
6. DSSA三步：**分析建模、设计架构、实现复用**
7. 质量属性：**运性可用安，开维扩测移**
8. 云三层：**S应P平I基**，微服务去中心，SOA靠ESB
