这份资料是**系统架构设计师-信息安全**模块历年选择题真题解析（2013-2025年），无复杂计算，以**概念辨析、技术分类、攻防原理、标准等级**为核心题型，覆盖**加解密、认证签名、安全基础、抗攻击**四大核心模块，精简总结如下：

# 一、核心考点分模块速记
## 1. 信息加解密技术
1. 算法分类（必考）
    - 对称加密：DES（56位）、3DES（**112位主流**，三密钥168位）、AES；
    - 非对称加密：**RSA**（唯一代表，用于数字签名、密钥交换）。
2. 哈希算法：MD5输出**128位定长摘要**，相同输入必输出相同结果，核心作用**防数据篡改**。
3. 密钥管理：加密存储**必须严控密钥权限**，密文仍可被破解，非绝对安全。

## 2. 访问控制&数字签名&认证体系
1. 数字签名（高频）
    - 流程：生成消息摘要（**防篡改**）→ 发送方私钥加密摘要（**防抵赖**）；
2. 数字证书：验证**CA的签名**，核心目的**验证网站真伪**。
3. 两大认证体系
    - PKI：非对称加密，核心组件**CA认证中心**；
    - Kerberos：对称加密，核心组件**KDC密钥分发中心**，用**时间戳**防重放攻击。
4. 补充概念
    - 电子签名：≠手写签名扫描图，是加密认证手段，支持电子印章/代码实现；
    - TBAC访问控制：由**工作流、授权结构体、受托人集、许可集**组成。

## 3. 信息安全基础知识（核心送分）
1. 核心安全属性（必考）
    - 机密性：防信息泄露；完整性：防非法篡改；
    - 可用性：防拒绝服务；可控性：管控信息传播；抗抵赖：审计取证、防否认。
2. OSI五大安全服务：认证、访问控制、机密性、完整性、**抗抵赖**。
3. 标准与等级
    - 等保五级：最高为**访问验证保护级**；
    - SHARE78灾备最高级：**零数据丢失+自动故障切换**。
4. 安全体系&治理
    - 三大措施：技术+管理+**政策法律**；核心技术：加密、数据备份、灾难恢复；
    - 数据治理3目标：合规、管控风险、促进数据利用；
    - 安全审计四要素：控制目标、安全漏洞、控制措施、控制测试。
5. 安全威胁：篡改=破坏完整性、DoS=阻断可用性、**业务流分析=被动攻击**。

## 4. 抗攻击技术
1. 攻击分类
    - 被动攻击：**流量分析、监听**（仅窃取，不修改数据）；
    - 主动攻击：篡改、伪装、重放、拒绝服务（修改/破坏系统）。
2. 典型攻击原理
    - SYN Flood：利用TCP三次握手，制造大量半连接耗尽资源（DoS攻击）；
    - TCP/IP漏洞攻击：Land、Ping of Death、Teardrop；**SQL注入是应用层攻击，非TCP/IP漏洞**。
3. 防御要点：参数化查询防SQL注入，**ORM框架可显著降低注入风险**。
4. 冷门考点：区块链挖矿是PoW/Hash碰撞，**挖矿不防双花攻击**（最长链机制防双花）；文件共享会**降低系统安全性**。

# 二、高频易错点（避坑必记）
1. 3DES密钥：**112位（双密钥）** 为标准答案，168位仅理论值；
2. 签名作用：摘要防篡改、私钥加密防抵赖，二者不可混淆；
3. 组件对应：PKI=CA、Kerberos=KDC，术语严禁混用；
4. 攻击定性：流量分析是**被动攻击**，SQL注入是应用层攻击；
5. 电子签名：无图像属性，是具备法律效力的加密认证手段。

# 三、整体概括
本模块为**纯记忆型考点**，无计算题，核心考查**概念边界区分、技术分类、攻防原理**；**加密算法、安全四属性、数字签名、攻防分类、等级标准**为必考核心，易混点少，是系统架构考试的高频送分模块。
