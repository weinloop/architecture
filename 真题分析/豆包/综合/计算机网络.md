# 系统架构设计师-计算机网络 核心考点总结
本模块为**必考选择题（3-5分）**，考点集中在**网络架构设计、设备层级、协议端口、存储RAID、通信原理、IP子网计算**，纯记忆+简单计算，无复杂题型。

## 一、网络规划与架构设计
1. **SDN三层架构**：应用层、控制层、转发层（**无逻辑层**）
2. **三层网络模型**
    - 核心层：**高速转发**，不做过滤/策略路由
    - 汇聚层：路由汇总、协议转换、策略控制
    - 接入层：用户接入、MAC认证、信息采集
3. **设计原则**：层次化设计**从接入层开始**，禁止随意加额外连接；冗余备份路径**默认不转发流量**，仅主路径故障启用
4. **拓扑结构**：星型拓扑通信**最多2跳**；局域网是**封闭性网络**，覆盖小、拓扑多样
5. **逻辑设计内容**：拓扑图、IP方案、软硬件规划（**不含用户培训计划**）

## 二、网络设备与OSI层级（必考）
| 设备 | 层级 | 核心功能 |
| ---- | ---- | -------- |
| 集线器/中继器 | 物理层 | 信号放大，无寻址 |
| 二层交换机 | 数据链路层 | MAC地址转发，初始MAC表为空 |
| 路由器/三层交换机 | 网络层 | IP路由、寻址转发 |
| 防火墙/HTTPS | 应用层 | 安全控制、加密通信 |

- 易错点：**路由选择是网络层功能**，数据链路层不负责；TCP端口作用：**寻址应用层进程**

## 三、核心协议与端口（速记）
1. **基础协议**
    - HTTP(80)、HTTPS(443，加密)；SMTP(25，发邮件)、POP3(110)/IMAP(143，收邮件)
    - DNS配置文件：Linux **/etc/resolv.conf**；正向解析A记录，反向解析**PTR记录**
    - DHCP：客户端检测IP冲突发送**DhcpDecline**；可部署多台服务器做冗余
2. **安全协议**
    - 无线加密：WPA2用**AES**（最安全），WEP/WPA不安全
    - 邮件安全：**S/MIME/PGP**；MIME仅支持附件，无安全功能
    - SSL特性：保密、可靠、完整（**无不可抵赖性**）
    - Kerberos：基于**KDC**分发票据，无CA，时间戳防重放
3. **网络技术**
    - 5G核心：**网络切片**（逻辑独立虚拟网络）
    - Internet核心交换：**分组交换**
    - QoS模型：IntServ三服务（无突发式）；DiffServ用**ToS字段**标记DS码点

## 四、IP与子网（计算必考）
1. **IPv6**：128位地址；无广播，只有单播/组播/任播；链路本地地址**FE80::/10+MAC地址**
2. **子网计算**
    - 掩码255.255.255.248：主机位3位，可用主机数=**2³-2=6**
    - 管理距离：数值**越小越可靠**（直连0，静态1）
3. **DNS查询**：根域名服务器**禁用递归查询**（性能损耗大），默认迭代查询

## 五、网络存储（RAID5）
- 容量公式：**(硬盘数-1)×最小硬盘容量**
- 示例：3块80G → 160G；2块80G+1块40G → 80G

## 六、综合布线
结构化布线6子系统：**水平子系统连接干线子系统+工作区子系统**

## 七、通信原理（计算+概念）
1. **采样定理**：采样频率≥2倍信号最高频率；3kHz采样→最高信号**1.5kHz**
2. **信道容量**
    - 奈奎斯特（无噪声）：C=2B×log₂V
    - 香农（有噪声）：C=B×log₂(1+S/N) → **提升带宽+信噪比**可提速
3. **编码**：以太网用**曼彻斯特编码**（自带时钟同步）
4. **通信模式**：半双工**1个逻辑通道**（双向不同时）；全双工2个
5. **调制解调**：实现**模拟信道传输数字信号**

## 八、高频易错避坑
1. 交换机**源MAC学习**，无表项时洪泛转发，不丢弃
2. 服务器延迟核心：**队列延迟+磁盘IO延迟**；路由器延迟＞交换机
3. OSI安全服务：**会话层无安全服务**
4. 财务系统方案：虚拟化优势是**业务连续性**；**网闸物理隔离不符合校园网访问需求**
