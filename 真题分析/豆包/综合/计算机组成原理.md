# 系统架构设计师-计算机组成原理 核心考点总结
本模块为**必考选择题（2-4分）**，核心考点集中在**CPU参数、指令系统、存储体系、流水线计算、总线、嵌入式处理器、AI芯片、I/O控制**，以**概念记忆+简单公式计算**为主，无复杂题型。

## 一、基础硬件与核心概念
1. **AI芯片三大架构**：**GPU、FPGA、ASIC**（CPU/SOC/DSP均不属于）
2. **CPU频率计算**：**主频 = 外频 × 倍频**（例：200MHz×13=2.6GHz）
3. **指令系统 RISC vs CISC**
    - RISC（精简）：指令固定、寻址少、寄存器多、**硬布线逻辑**
    - CISC（复杂）：寻址丰富、指令功能强、微程序控制
4. **总线传输模式**
    - 半双工：双向**轮流**传输；全双工：双向**同时**传输
    - 串行总线：按位传输，依赖**校验码**纠错，波特率可动态调整
5. **I/O控制方式**
    - DMA：**主存与外设**直连，绕过CPU，批量数据传输
    - 通道：执行通道程序管理I/O，最大化CPU与I/O并行度

## 二、存储体系（高频）
1. **速度排序**：**寄存器组 > Cache > 内存 > Flash/磁盘**（寄存器最快）
2. **分级存储目的**：平衡**容量、价格、速度**三者矛盾
3. **ROM分类（只读存储器）**
    - PROM：**一次可编程**，烧录后不可修改
    - EPROM：紫外线擦除，多次编程
    - EEPROM：电擦除，多次可编程
    - 掩膜ROM：工厂固化，不可编程

## 三、流水线技术（必考计算）
1. **核心公式**
    - 流水线周期 = 各阶段**最长耗时**
    - 最大吞吐率 = 1 / 流水线周期
    - 流水线总时间 = 单条指令总时间 + (指令数-1)×流水线周期
    - 加速比 = 顺序执行总时间 / 流水线执行总时间
2. **经典例题**
5级流水线（最长阶段3Δt），10条指令：吞吐率=1/3Δt，加速比=5:2

## 四、嵌入式系统与处理器
1. **处理器架构**
    - DSP：**哈佛结构**（指令/数据存储分离，高速处理）
    - 通用CPU：冯诺依曼结构（共享存储）
2. **嵌入式处理器分类**
    - MPU：通用微处理器，非专用大运算场景
    - MCU：单片机，体积小、低成本、控制类
    - DSP：数字信号处理专用
    - SoC：片上系统，**集成完整软硬件**，≠单一处理器
3. **MMU内存管理单元**
    - 无MMU：Cortex-M3；适配系统：**UC/OS-II**
    - 有MMU：ARM9、PowerPC；适配系统：Linux、Windows CE
4. **芯片温度等级**
    - 民用级：0~70℃；**工业级：-40~85℃**；军用级：-55~125℃
5. **专用处理器**
    - NPU：神经网络处理器，**高速低耗、AI性能强**
    - GPU：峰值性能＞100 TFlops，并行计算核心

## 五、校验与编码
1. **CRC循环冗余校验**
    - 校验码位数 = 生成多项式最高幂次
    - 计算规则：信息码左移N位 → 模2除法（异或）→ 余数为校验码
2. 示例：G(x)=x⁵+x³+x+1 → 5位校验码

## 六、高频易错避坑
1. SoC是**完整系统集成**，不是单一处理器芯片
2. RISC**寻址方式少**，丰富寻址是CISC的特点
3. DMA不经过CPU，仅在**主存-外设**间传输
4. 嵌入式实时系统典型架构：**层次化+递归模式**
5. AI芯片四大特征：新型计算范式、训练+推断、大数据处理、可重构能力
