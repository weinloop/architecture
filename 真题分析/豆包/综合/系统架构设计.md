# 系统架构师-软件架构核心考点总结
这份资料聚焦**软件架构全章节高频真题考点**，覆盖架构基础、设计方法、评估、视图、领域架构、质量属性、新技术、复用、面向服务架构等核心模块，核心考点精简总结如下：

## 一、软件架构基础概念
1. **核心定义**：架构是系统**结构、行为、属性**的高级抽象，关注组件结构、属性与**交互作用**，通过**多视图**描述。
2. **ADL架构描述语言**：核心四要素=**组件+接口+连接件+配置**，连接件是组件交互核心。
3. **架构核心作用**：早期选型、干系人沟通、描述结构/交互；**不直接表达功能性需求**，核心满足**质量属性**。
4. **易错点**：架构设计**不负责需求捕获细化**；消息传递架构**构件完全独立**（高内聚低耦合）。

## 二、基于架构的软件开发（ABSD）
1. **核心驱动**：**商业+质量+功能需求**三大需求驱动。
2. **三大基础**：功能分解、**架构风格**实现质量/商业需求、软件模板设计结构。
3. **开发流程（6阶段）**：需求→设计→**文档化**→**复审**→**实现**→**演化**
    - 文档化输出：架构规格说明、质量设计说明书；
    - 复审核心：**用户代表+领域专家**判定架构合理性；
    - 演化6步骤：需求归类→定计划→增删改构件→**更新构件交互**→组装测试→技术评审。
4. **核心特征**：**自顶向下、递归细化**，最终产出**软件构件/类**；用**视角与视图**描述架构，**用例**描功能、**质量场景**描质量需求。

## 三、软件架构评估（高频核心）
### 1. 核心评估方法
- **SAAM**：最早的评估方法，关注**可修改性**，输入=问题描述+需求说明+架构描述；
- **ATAM**：SAAM扩展，**质量属性权衡**，核心工具**效用树**，聚焦4大属性：**性能、可用性、安全性、可修改性**；
- **CBAM**：ATAM经济扩展，按**ROI投资回报**选架构；
- **SASAM**：唯一**静态分析**方法。

### 2. 关键概念（必考区分）
- **敏感点**：单构件特性影响**单一**质量属性；
- **权衡点**：影响**多个**质量属性（如加密级别=安全+性能）；
- **风险点**：潜在无法满足需求的设计；**非风险点**：可接受的设计。

### 3. 质量属性场景六要素
**刺激源→刺激→环境→制品→响应→响应度量**；ATAM场景分三类：**用例场景、增长场景、探索性场景**（增长场景极端形式）。

### 4. 效用树结构
**树根→质量属性→属性分类→质量属性场景（叶子节点）**。

## 四、4+1视图模型
1. **核心思想**：**关注点分离**，5个视角完整描述架构；
2. **4基础视图**
    - 逻辑视图：功能需求、对象模型（类图）；
    - 进程视图：**并发、同步、运行时动态**；
    - 开发视图（实现视图）：代码静态组织、模块结构；
    - 物理视图（部署视图）：软件→硬件映射、部署拓扑；
3. **+1视图**：**场景视图（用例）**，串联验证所有视图；
4. 易混：Hofmeister 4视图=**概念+模块+执行+代码**。

## 五、特定领域软件架构（DSSA）
1. **三大阶段**
    - 领域分析：产出**领域模型**（核心目标）；
    - 领域设计：产出**DSSA架构**；
    - 领域实现：开发复用资产、落地架构。
2. **域分类**：**垂直域**（特定领域系统族）、**水平域**（跨领域公共功能）；
3. **核心角色**：领域专家、分析师、设计者、实现者；
4. **建立过程**：**并发、递归、反复**。

## 六、软件质量属性（必考场景匹配）
### 1. 分类
- **运行期**：性能、可用性、安全性、健壮性、互操作性、可伸缩性；
- **开发期**：可维护性、可扩展性、可测试性、可重用性、可移植性。

### 2. 核心属性定义&战术
| 质量属性 | 核心定义 | 典型战术 |
|----------|----------|----------|
| 性能 | 响应时间、吞吐量 | 资源调度、并发、增资源 |
| 可用性 | 故障恢复能力（MTTR/MTBF） | 心跳、主动冗余、故障转移 |
| 安全性 | 防攻击、数据保密 | 追踪审计、权限控制、攻击检测 |
| 可修改性 | 变更难易度 | 信息隐藏、接口分离 |
| 易用性 | 用户操作便捷性 | 学习曲线、操作效率 |
| 可移植性 | 跨环境运行 | 平台无关设计 |

### 3. 易错点
- 健壮性：异常/故障下正常运行；**稳定性非可靠性子特性**；
- 可伸缩性：用户/数据量增长下保持服务质量；
- 互操作性：系统间数据交换/服务调用难易度。

## 七、新技术架构（云原生/微服务/大数据等）
1. **云计算**：IaaS（基础设施）、PaaS（平台）、SaaS（应用）；虚拟化技术=KVM/Xen/Hyper-V，**LVS是负载均衡非虚拟化**。
2. **微服务**：独立部署、去中心化、轻量级通信；**断路器三态**：关闭→打开→半打开；**不可变基础设施**：实例部署后不修改，重建替代更新。
3. **WebService/SOA**
    - SOAP：通信协议，**信封与编码不同命名空间**；
    - WSDL：描述服务**做什么、怎么访问、在哪**；
    - UDDI：服务**描述、发现、集成**；
    - ESB：SOA核心，**解耦服务请求/提供者**，集中式管理；微服务=**去中心化**。
4. **大数据**：Lambda架构=批处理层（全量）+**加速层（增量实时）**+服务层。
5. **其他**：CDN/反向代理核心=**缓存**；区块链分类=公有链/私有链/联盟链（无行业链）；大语言模型核心=**Transformer**；边缘计算核心=低延迟、省带宽。

## 八、软件架构复用
1. **复用类型**：**机会复用**（开发中发现即复用）、**系统复用**（提前规划复用）；
2. **复用三阶段**：**构造/获取**可复用资产→**管理**资产→**使用**资产。

## 九、架构风格（场景选型）
1. **规则系统**：家用机器人、专家系统（条件规则自适应）；
2. **消息传递**：分布式/并发系统（高内聚低耦合、异步、点到点）；
3. **层次型**：物联网（感知层→网络层→应用层）；
4. **事件驱动**：GUI、异步交互系统。

## 十、高频真题易错结论
1. 架构设计**不满足功能需求**，核心满足**质量属性**；
2. 加密级别调整=**权衡点**，影响安全+性能；
3. 双机热备/心跳=**可用性**战术；资源调度=**性能**战术；
4. DSSA领域分析→**领域模型**，领域设计→**DSSA架构**；
5. 微服务去中心化，SOA（ESB）集中式；
6. 质量属性场景**响应**=刺激后的系统动作。
