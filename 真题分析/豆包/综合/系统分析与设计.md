# 系统分析与设计（架构师）核心考点速记
**分值8~12分**，全选择题，**结构化+面向对象+UML+设计模式**四大核心，无复杂计算，记定义+分类秒杀。

## 一、需求工程（必考）
1. **需求抽取（获取）**
迭代过程：**需求发现→分类组织→协商→文档化**（≠需求确认/规格说明）
2. **需求变更管理**
流程：问题分析→**变更分析&成本计算**→变更实现；**CCB审批**；变更**有利有弊**，必须受控
3. **需求陈述原则**
完整、准确、可行、必要；**需求分优先级，绝非同等重要**
4. **JRP联合需求计划**
核心：**获取需求**（≠分析验证）；成本高、高效；避专业术语、严格按议程

## 二、结构化分析（SA）
1. **核心工具DFD数据流图**
- 四要素：**加工、外部实体、数据流、数据存储**
- 关键规则：**不描述执行顺序**；父子图输入输出**完全一致**
- 加工逻辑细化：**判定表/判定树**
2. **模型对应**
功能模型=DFD；行为模型=**状态转换图**；数据模型=ER图
3. **业务建模工具**
**Petri网**：图形化+数学化，支持并发/同步，嵌入式数据流建模首选

## 三、结构化设计（SD）
### 1. 模块独立性（黄金考点）
- 内聚（高→低）：**功能>顺序>通信>过程>时间>逻辑>偶然**
  秒杀：**功能内聚最高**，单一完整功能；过程内聚=按顺序执行无数据依赖
- 耦合原则：**多用数据耦合，少用控制耦合，禁用内容耦合**
### 2. 结构度量
**扇出**：模块直接调用下层数量；**扇入**：被上层调用数量
### 3. 设计阶段&工具
- 概要设计：产出**数据结构+系统结构**；工具：模块结构图、层次图、HIPO图
- 详细设计：工具：流程图、N-S盒图、PAD、IPO图
- 三大控制结构：**顺序、分支、循环**
### 4. 软件设计四大活动
**数据设计、结构设计、人机界面设计、过程设计**

## 四、面向对象分析设计（OOA/OOD）
1. **类的三大职责**
- 边界类：**界面/外部接口隔离**
- 控制类：**协调用例逻辑**，核心调度
- 实体类：存储业务数据
2. **核心设计原则**
**依赖倒置**：抽象不依赖细节，面向接口编程；开闭、里氏替换、迪米特法则
3. **模型构成**
- OOA：顶层架构图+用例图+**领域概念模型**
- OOD：包图（架构）+交互图（用例实现）+类图+状态图+活动图
4. **其他**
DDD领域驱动设计：核心=**领域模型**；COM重用：**包含（转发）、聚集（暴露接口）**

## 五、UML建模（高频易混）
1. **图的适用场景**
- 需求分析：用例图、活动图、类图；**构件图（设计阶段）排除**
- 物理结构：**部署图**；流程建模：活动图；时序交互：顺序图
2. **用例图核心**
- 用例关系：**包含、扩展、泛化**；**聚合（类图专属）排除**
- 参与者关系：仅**泛化（继承）**
3. **顺序图**
复杂交互=**序列片段**（循环/选择）；消息：同步、异步、返回、创建、销毁
4. **活动图**
泳道=执行者；**分支=条件判断，分叉=并发**；初始节点=原子起始态
5. **进阶概念**
UML2.0基础结构=**元模型**；SysML新增=**需求图**

## 六、设计模式（必考分类+场景）
### 1. 三大分类
按目的：**创建型、结构型、行为型**；按范围：类模式、对象模式
### 2. 创建型（对象创建）
- 单例：唯一实例；抽象工厂：创建**一族相关对象**
- 建造者：**构建与表示分离**；原型：**克隆创建**，无需知道类细节
### 3. 结构型（组合/解耦）
- 桥接：**抽象与实现分离**，多维度变化
- 装饰：**动态加功能**，比静态继承更灵活
- 外观：统一接口简化子系统；代理：控制对象访问
### 4. 行为型（对象交互）
- 命令：封装请求，支持**撤销/排队**
- 中介者：封装多对象交互，解耦
### 5. 秒杀判错
桥接/装饰=结构型；原型=创建型；观察者=行为型

## 七、补充高频考点
1. 系统建议方案：**不含详细设计方案**
2. McCabe环形复杂度：**判定节点数+1**
3. 界面设计：**一致性优先**，禁止频繁修改交互模型
4. 性能设计流程：**计算→分析→设计→评估**
