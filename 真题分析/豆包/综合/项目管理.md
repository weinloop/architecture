这份资料是**系统架构设计师-项目管理**模块历年选择题真题解析（2013-2025年），聚焦**概念辨析、流程记忆、进度计算**三大题型，核心覆盖**配置管理、范围管理、进度管理、成本管理、变更管理**五大考点，精简总结如下：

# 一、核心考点分模块速记
## 1. 配置管理（最高频必考）
1. 核心定义
    - 产品配置：**文档+计算机程序+部件及数据**（完整集合）；
    - 配置项：软件生命周期阶段审批产物，**工作计划（管理文档）不属于产品配置项**。
2. 配置项状态：**草稿、正式发布、正在修改**（三大标准状态，评审/审批是流程非状态）。
3. 变更控制（CCB）
    - CCB可设多个小组，**可拒绝不合理变更**；
    - 标准流程：**变更申请→风险评估→变更批准→实施→验证→存档**。
4. 标准体系：**UML无配置管理**，ISO9000/PMBOK/CMMI均包含配置管理。
5. 软件质量（McCall模型）：用户视角三维度**产品运行、产品修改、产品转移**。

## 2. 范围管理
1. 范围定义**核心输入**：项目章程、范围管理计划、组织过程资产、批准的变更申请；
2. 易错概念：**项目范围≠产品范围**（产品是功能，项目是交付工作）；范围定义是**项目计划的基础**。

## 3. 文档与需求跟踪
1. 文档分类
    - 用户文档：安装手册、参考手册、版本说明；
    - 系统文档：**系统设计文档**（不属于用户文档）。
2. 需求跟踪
    - 正向：需求→设计/代码/测试；逆向：代码/文档→需求；
    - 易错点：**工件回溯需求=逆向跟踪**。
3. 团队分工：项目管理组输出计划/质量报告，**源代码由开发组产出**。

## 4. 进度管理（计算题核心）
1. 六大标准过程：活动定义→排序→资源估算→历时估算→制定计划→进度控制；
2. 核心工具：**WBS（工作分解结构）** 用于活动定义，甘特图/PERT用于进度展示；
3. 必考计算
    - 三点估算：工期=(乐观+4×最可能+悲观)÷6；
    - 关键路径：**最长路径=总工期**，总时差=活动可拖延天数（不影响总工期）；
    - 赶工优化：优先压缩**关键路径单位成本最低**的活动，平衡直接费+间接费。

## 5. 成本管理
- 核心区分：**成本估算**（算总成本）→**成本预算**（分配至工作包，建立成本基线）→成本控制；
- 唯一考点：**成本预算是建立成本基线的过程**。

## 6. 变更管理
1. 需求变更流程：**问题分析与变更描述→变更分析与成本计算→变更实现**；
2. 工具特性：核心支持**变更记录（日期/责任人）**，不自动判定变更、不自动更新文档。

# 二、高频易错点
1. 配置项状态无“评审/版本变更”，仅草稿、正式发布、正在修改；
2. 需求跟踪逆向≠正向，代码回溯需求是逆向跟踪；
3. 赶工仅压缩**关键路径**活动，非关键路径压缩无意义；
4. 成本基线由**成本预算**生成，非成本估算；
5. CCB可多组设立，并非唯一小组。

# 三、整体概括
本模块以**记忆性概念+流程排序+简单计算**为主，**配置管理、进度关键路径、变更流程**为三大必考核心，无复杂计算，重点区分概念边界、牢记标准流程与公式即可掌握。
