这份资料是**系统架构设计师考试-系统运行与维护模块**的历年选择题真题解析（2013-2024年），核心围绕**评价方法、性能指标、遗留系统、系统维护**四大考点展开，核心总结如下：

# 一、核心考点分模块总结
## 1. 评价方法
1. 软件复杂性：**环路数（圈复杂度）** 唯一直接反映代码结构复杂度，模块数、对象数均不直接体现。
2. 系统监视3种方式：系统命令（Linux ps/last）、日志文件、可视化监控工具（Windows Perfmon）；top/netstat为命令行工具，iptables是防火墙工具。
3. 性能评价程序准确度：**真实程序＞核心程序＞小型基准程序＞合成基准程序**（合成基准程序准确度最低）；核心程序是基准测试核心，TPC-C为**在线事务处理（OLTP）** 专用基准。
4. Web服务器核心指标：最大并发连接数、响应延迟、吞吐量；**丢包率属于网络指标，非服务器性能指标**。

## 2. 性能指标
1. Web服务器测试：**链接正确跳转属于功能测试，非性能指标**；性能测试方法含基准测试、压力测试、**可靠性测试**。
2. 数据库优化核心参数：CPU/内存、**查询语句性能**、进程/线程状态、日志文件大小。
3. 应用系统优化核心参数：可用性、响应时间、**并发用户数**、系统资源占用。
4. 硬件/DBMS指标：计算机核心为数据处理速率；数据库核心为**最大连接数**、索引数量、并发事务能力。

## 3. 遗留系统演化策略（高频必考）
按**技术水平×业务价值**四象限划分，固定匹配规则：
- 低技术+低价值：**淘汰**
- 低技术+高价值：**继承**
- 高技术+高价值：**改造**
- 高技术+低价值：**集成**（消除信息孤岛，最高频考点）

## 4. 系统维护
1. 四大维护类型（核心区分）
    - 改正性维护：修复系统bug/缺陷
    - 适应性维护：适配硬件/软件环境变化
    - 完善性维护：**新增功能、优化性能/效率**（最高频）
    - 预防性维护：提前优化，规避潜在风险
2. 工具与流程：**版本控制工具（Git/SVN）** 管理软件多版本；可行性分析**不包含现有系统问题分析**（属系统调查阶段）。

# 二、高频易错点
1. 指标归属：丢包率、时延抖动为**网络指标**，不归属Web/数据库性能；
2. 工具区分：Perfmon是可视化综合监控工具，top/netstat仅为命令行工具；
3. 场景匹配：新增需求=完善性维护、高技术低价值遗留系统=集成策略；
4. 测试分类：黑盒/白盒/功能测试均不属于Web服务器性能测试方法。

# 三、整体概括
本资料以真题为核心，聚焦系统运行与维护的**概念区分、指标判定、策略选择**三大题型，高频考点集中在性能评价程序、遗留系统四象限、软件维护类型，核心考查**场景匹配与概念边界区分**，无复杂计算，以记忆性考点为主。
