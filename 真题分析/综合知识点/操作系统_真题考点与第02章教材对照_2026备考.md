| 考点域 | 平台中可写的素材 |
|--------|------------------|
| **磁盘 / 缓冲 / 调度** | **QAR/舱音/日志** 顺序与随机读；**离港高峰** 磁盘 I/O；**双缓冲** 类比 **采集与落盘** 并行 |
| **内存 / 分页置换** | **运控席位工作站** 多应用并发；**机载任务** 内存受限与 **页面置换** 策略 |
| **嵌入式 / RTOS** | **机载显示管理计算机、FMS 相关任务、廊桥/行李控制、场面车辆终端**；要求 **硬实时**、**VxWorks/类 RTOS**；**看门狗** 防死锁挂起 |
| **前趋图 / PV** | **除冰完成 → 推出许可 → 起飞** 等 **前驱约束**；**机位、跑道** 等 **互斥资源** 用 **信号量** 建模 |
| **进程 / 死锁 / 银行家** | **多席位同时申请** 有限 **机位/跑道**；**安全序列** 类比 **不冲突的推出顺序** |
| **文件 / FCB** | **飞行计划、舱单、适航文件** **打开文件** 先读 **元数据/FCB**；**路径** 从 **根目录** |
| **OS 分类与特征** | **旅客查询** 偏 **分时交互**；**防撞/自动油门相关控制链** 偏 **硬实时**；**离港批处理报表** 偏 **批处理** |
| **并发与异步** | 宏观上 **多席运控同时操作**；微观 **单核分时**；**不确定性** 体现在 **事件到达顺序** |

**【平台案例总览 · 航空运营智能管理平台】**  
平台侧既有 **通用服务器/桌面 OS**（运控、离港、BI），也有 **嵌入式实时环境**（机载、场面设备）。备考时可用 **「同一业务链条上的硬实时 vs 交互式 vs 批处理」** 区分 **实时 / 分时 / 批处理**；用 **机位/跑道/除冰位** 理解 **互斥与死锁**；用 **日志与飞行数据归档** 联想 **磁盘调度与缓冲**。下文 **【平台案例】** 均以此统一场景展开。

## 二、与第2章原文可直接对应的高频点

以下内容在 **第2章 2.3.2** 中有明确表述，与真题高度一致，建议**以教材原文为准**背诵。

### 1. 操作系统的定位与组成

- 资源管理者：**处理机、存储、设备、文件** 管理；配置在硬件上的**第一层软件**。  
- **内核**提供进程/任务管理、存储管理、文件管理、设备管理等；对硬件**抽象**，通过 **系统调用 / API** 为应用服务。  
- *真题关联*：用户态通过系统调用使用特权指令 → **访管中断**；RTOS 内核与应用接口 → **API**（真题选项）。

**【平台案例】**  
- **资源管理**：**CPU**（多任务运控）、**内存**（离港终端）、**设备**（打印机/登机牌/扫描枪）、**文件**（计划与舱单）均由 OS 统一抽象。  
- **系统调用**：业务进程 **读库、写串口、发网络包** 经 **内核态** 完成——与 **访管/陷入** 题干一致。

### 2. 操作系统的特征（四性）

- **并发性**：单 CPU 上宏观并行、微观串行；多 CPU 可并行。  
- **共享性**：同时共享与互斥共享。  
- **虚拟性**：物理实体与逻辑对应物的映射。  
- **不确定性（异步性）**：进程推进次序与速度不可预知。  
- *真题关联*：单 CPU 多任务「交替运行」与并发叙述一致；多道程序提高 **CPU/内存/I/O 利用率**。

**【平台案例】**  
- **并发**：单席位 **CPU** 上 **Gantt 与告警** 多窗口「宏观同时、微观轮流」。  
- **共享**：**只读航班库** 可同时读；**写舱单** 需 **互斥** 或 **事务**。  
- **虚拟**：**虚拟内存** 让大查询「以为」内存足够；**虚拟机/容器** 隔离 **测试与生产**。  
- **异步**：**延误、除冰、登机** 事件到达 **不可预知**，进程推进次序 **不确定**。

### 3. 操作系统的分类（必背清单）

教材列举：**批处理**（单道/多道；多道特点：多道、宏观并行、微观串行）、**分时**（时间片、多路性/独立性/交互性/及时性）、**实时**（**在被控对象允许的时间内**做出响应；实时控制 vs 实时信息处理）、**网络操作系统**、**分布式操作系统**、**微型计算机操作系统**、**嵌入式操作系统**。  
- *真题关联*：分时系统 **时间片固定时，用户数增多 → 轮转周期变长**；实时系统典型产品（如 **VxWorks**）与通用 OS（iOS/Android）区分；多道批处理作业与 CPU-I/O 重叠画时间线。

**【平台案例】**  
- **批处理**：夜间 **T+1 报表、日志归档、批量对账**——吞吐优先、少交互。  
- **分时**：**订座员、值机员** **交互终端**——时间片轮转，用户增多则 **响应变慢**。  
- **实时（硬）**：**防撞、飞控相关任务链、关键场面联锁**——必须在 **时限内** 完成（与 **被控对象允许时间** 表述一致）。  
- **嵌入式**：**廊桥、行李转盘控制器、RFID 读头、机载模块**——**微型化、可裁剪、BSP/HAL**。

### 4. 嵌入式操作系统（第2章列出的特点与产品）

- 特点：**微型化、可定制、实时性、可靠性、易移植性**（**HAL、BSP** 等底层技术）。  
- 常见嵌入式实时系统：**VxWorks、μClinux、PalmOS、Windows CE、μC/OS-Ⅱ、eCos** 等（教材原表）。  
- *真题扩展*：鸿蒙 **微内核**（非宏内核）、分布式能力；RTOS **可裁剪**、**非「通用性」**；**抢占式**调度为主；**看门狗超时**中断；**JTAG** 调试；软件低功耗常考 **编译优化、软硬件协同、算法优化**。

**【平台案例】**  
- **机载/场面嵌入式**：**显示管理、数据集中器、部分机载维护终端** 常跑 **RTOS**；**抢占** 保证 **高优先级告警** 打断低优先级任务。  
- **BSP/HAL**：不同 **航电总线/传感器接口** 由 **板级支持包** 适配，上层应用 **可移植**。  
- **看门狗 / JTAG**：**场面控制器** 死循环时 **看门狗复位**；**机务维护** 用 **JTAG** 现场调试。  
- **低功耗**：**手持巡检终端、机上便携设备** 考 **算法与协同省电**（与真题选项一致）。

## 三、真题中高频、但第2章未细讲的模块（需专项复习）

下列内容在 `操作系统.pdf` 中反复出现，**教程第2章仅不涉及或仅一笔带过**，建议单独整理笔记：

| 主题 | 典型考点 |
|------|----------|
| 磁盘 | **移臂调度**（SSTF）、同一柱面内按扇区序；记录**块布局与旋转延迟**；**双缓冲**时间计算 |
| 内存 | **页号+页内偏移**、页表项有效位；**段式越界**；**段页式**定义；置换时 **访问位/修改位** |
| 进程 | **三态转换**（阻塞不能直接→运行）；**银行家算法**与安全序列；**死锁**环路与预防（**不破坏互斥**）；**n 进程每进程需 r 个同类资源**的最少资源数 \(n(r-1)+1\) |
| 同步 | **PV 与前趋图**；互斥信号量初值 **1**；信号量为负与等待队列含义 |
| 线程 | 线程 **私有栈与栈指针**，共享进程内打开文件等 |
| 文件 | **打开文件先读 FCB/元数据**；绝对路径自**根目录**起 |
| 杂项 | **契约式设计**（DbC）等与操作系统弱相关题目混排 |

*说明：架构师教程在后文**嵌入式**等章节会再出现信号量、任务通信等内容（如共享内存、消息队列），可与进程管理真题对照学习。*

**【平台案例】**  
- **磁盘 / SSTF / 缓冲**：**飞行品质数据、日志** 大块读写；**移臂调度** 可联想 **多任务抢磁盘** 时 **缩短寻道**；**双缓冲** 联想 **采集进程与写入进程** 并行。  
- **内存 / 页与置换**：**运控终端** 多应用切换触发 **缺页**；**访问位/修改位** 决定 **换出** 候选（与 LRU 类策略题一致）。  
- **银行家 / 资源公式**：**n 个推出任务各需跑道占用 r** 类题，可记 **「资源下界防死锁」** 与 **机位/跑道配额** 类比（**数学仍按公式**）。  
- **PV / 前趋图**：**登机门开闭、除冰完成、牵引车到位** 画 **前趋边**；**互斥信号量** 保护 **共享缓冲区**（如 **离港报文队列**）。  
- **进程三态**：**等待离港应答** 阻塞，不可 **直接** 变运行。  
- **文件 / 路径**：打开 **电子舱单** 先载入 **目录项/FCB**；**绝对路径** 从 **文件系统根**。

## 四、第2章与真题的「分工」小结

- **第2章能直接覆盖的得分点**：OS 定义与作用、内核与 API、**四大特征**、**七大类 OS** 及各自关键词、**嵌入式 OS 特点与典型产品**、实时性表述。  
- **真题卷中其余大部分题目**：属于传统 **操作系统原理** 考点，与第2章 **2.3.2 节「概述+分类」** 形成「**概念骨架 + 原理填充**」关系：第2章解决「是什么、分哪几类」，磁盘/内存/进程大题解决「怎么做题」。

**【平台案例】**  
- 做题时若抽象概念记不住，可用 **航空运营智能管理平台** 快速 **锚定**：**分时**→**柜台/订座交互**；**硬实时**→**机载/关键联锁**；**嵌入式**→**廊桥/行李/场面终端**；**死锁/PV**→**机位与推出依赖**。**计算题**（银行家、甘特图、缓冲时间）仍以 **公式与步骤** 为准，场景只辅助 **不替代推导**。

## 五、2026 年 5 月考查倾向预测（供参考）

1. **第2章可背内容**：分时/实时/嵌入式辨析、嵌入式特点（含 BSP/HAL）、并发与异步表述、常见 RTOS 名称。  
2. **原理类延续高频**：分页地址转换、**PV 与前趋图**、银行家与安全序列、进程状态与 **死锁预防**、**SSTF** 与缓冲。  
3. **新技术结合**：国产 OS 架构（微内核、分布式）、**M2M** 分层、低功耗与嵌入式调试（JTAG/看门狗）仍易出现。  
4. **做题注意**：部分年份存在 **ATAM/质量属性** 等题目混入不同 PDF 的情况；本 PDF 以 OS 与嵌入式为主，若见 **DbC、XML** 等更偏软件工程或数据表示的题，按该题所属知识域作答即可。  
5. **案例素材（与其他备考稿一致）**：**航空运营智能管理平台** 用于 **OS 分类、并发、嵌入式、资源互斥** 的概念题 **口语化**；**不与** 架构案例中的 **Redis/微服务** 混为一谈——本 PDF 仍以 **第2章 + OS 原理** 为纲。

## 六、复习建议（简要）

1. 先通读 **第2章 2.3.2**，建立分类与特征框架，确保「分时 vs 实时 vs 嵌入式」不混淆。  
2. 再用真题 PDF 按 **磁盘 / 内存 / 进程 / 嵌入式** 四类刷题，薄弱项回到操作系统通用教材或教程后续章节补概念。  
3. 嵌入式特色题（鸿蒙、混成系统、EDF/RMS/LLF）与 **2.3.2 嵌入式** 及大纲中嵌入式章节一起复习，性价比更高。  
4. 概念混淆时，用 **平台一句话**：**「旅客 App 是分时体验，机载告警是实时约束」**，再回教材定义核对。
