## 二、分模块常考知识点详解

### 1. 架构风格（最高频）

**教材依据：第7章 7.3（软件架构风格）**

- **定义（必背）**  
  - 架构风格是某领域中系统组织方式的**惯用模式**。  
  - 一个体系结构定义**一个词汇表和一组约束**；风格反映众多系统共有的**结构和语义**特征，并指导如何将模块组织成完整系统；强调对架构**设计**层面的重用。  
  - *真题多次以填空或选词形式考查「约束」「语义」「设计」。*

- **大类与代表风格（必会与场景挂钩）**  
  - **数据流**：**批处理**（整批数据、步序严格、前步结束再执行后步）vs **管道-过滤器**（流式、过滤器+管道；过滤器可流水线并行，不必等上游完全结束）。  
  - **调用/返回**：主程序/子程序、面向对象、**层次结构**、**C/S**（二层：胖客户/瘦服务器；三层：增加**应用服务器**，表示层/功能层/数据层；表示层为 UI，功能层为业务逻辑）。  
  - **以数据为中心**：**仓库**（中央数据结构说明状态 + **独立构件** 对数据操作）、**黑板**（共享黑板 + 知识源；语音识别、复杂推理）。  
  - **虚拟机**：**解释器**、**规则系统**（规则集、解释器、规则/数据选择器、工作内存等）。  
  - **独立构件**：进程通信、**事件系统/隐式调用**（发布订阅、事件触发）。  
  - **C2**：构件与连接件均有顶/底，**构件不得直连**，须通过连接件按顶底规则连接；可概括为「连接件绑定的并行构件网络」。  

- **典型场景（真题反复变体）**  
  - 国家政策/规则常变 → **规则系统**。  
  - 用户自定义地图、行为、脚本组合 → **解释器**（或虚拟机类）。  
  - 编译多阶段、IDE 与工具链共享符号表/语法树 → 传统编译流水线常考 **管道-过滤器** 或 **顺序批处理**；IDE 整体常考 **仓库/数据共享**；代码一改多工具响应 → **隐式调用/事件**。  
  - 语音识别分阶段、多知识源协作 → **黑板**。  
  - 新闻/兴趣推送 → **事件驱动**。  
  - 分层架构 **脆弱性**：底层错误可波及上层；层间通信带来**性能开销**（真题常考「底层故障 + 层间机制导致性能下降」类选项）。  

- **与教材一致性**  
  上述分类与术语与第7章 **7.3.2～7.3.6** 的划分及图示一致；**C2** 在教程第7章正文中若未展开，可结合辅导资料与 Kruchten/经典文献补充，但真题已多次单独命题，需掌握连接规则。

**【平台案例】**  
- **批处理 vs 管道-过滤器**：**日终 T+1 报表** 偏 **批处理**；**实时报文流水线** 偏 **管道-过滤器**（**不必等上游整批结束** 的叙述扣管道）。  
- **仓库 vs 黑板**：**统一航班/机位模型** 多子系统读写 → **仓库**；**除冰、气象、场面** 多源 **黑板** 协同推理 → **黑板**。  
- **规则/解释器**：**运价、限载** 常变 → **规则系统**；**用户自定义脚本/策略** → **解释器**。  
- **事件驱动**：**航班状态变更** → **订阅** 登机口/行李/配餐。  
- **层次脆弱性**：**底层消息中间件故障** 或 **层间 RPC 过多** → **上层时延与可用性** 连带受损（真题选项）。

### 2. 基于架构的软件开发（ABSD）

**教材依据：第7章 7.2**

- **ABSD 特征**：**体系结构驱动**；由**商业目标、质量目标与功能需求**的组合驱动；**自顶向下、递归细化**，直至产生**软件构件和类**。  
- **三个基础**：功能分解（模块内聚耦合）；**通过选取体系结构风格**满足质量与商业需求；**软件模板**的运用。  
- **ABSDM 六个子过程**（顺序题高频）：**体系结构需求 → 体系结构设计 → 体系结构文档化 → 体系结构复审 → 体系结构实现 → 体系结构演化**。（注意：真题中常设干扰项「体系结构测试」「体系结构管理」等，**非**标准六步。）  
- **需求捕获**：用例描述功能；**质量场景**描述变更、性能、可靠性、交互性等（含预期与非预期场景）。  
- **视角与视图**：从多个视角观察架构；逻辑视图等记录功能与接口（与 **4+1** 等模型衔接）。

**【平台案例】**  
- **驱动**：**「枢纽准点率 KPI」**（商业）+ **「动态 P95」**（质量）+ **「国际中转行李」**（功能）共同驱动 **服务划分与缓存策略**。  
- **六过程**：**航季** 从 **架构需求** → **设计** → **文档化/评审** → **实现** → **上线后演化**（**顺序题** 仍背 **标准六步**）。  
- **质量场景**：**「早高峰并发」** 非功能场景与 **用例** 并列输入（与 **7.2** 表述一致）。

### 3. ADL 与生命周期中的架构（概念题）

**教材依据：第7章 7.1.2**

- **ADL**：支持**构件、连接子、配置**（及接口等）的描述；连接子在 ADL 中地位突出。  
- **多视图**：4+1、Hofmesiter 四视图、Views and Beyond 等（选择题常考「关注点分离」）。  
- **构件组装失配**：构件对基础设施/控制/数据模型的假设与连接子对协议/数据模型的假设不一致等（**Garlan** 等提出的三类失配来源，可背概念）。  
- **中间件与连接子**：中间件可支持连接子实现并引出 **中间件导向的体系结构风格**（教材原句）。

**【平台案例】**  
- **ADL**：构件 **「离港适配器」**、连接子 **「MQ 通道」**、配置 **「双活拓扑」**（**概念题** 扣 **构件/连接子/配置**）。  
- **失配**：**旧系统假定同步 RPC**、**新总线假定异步消息** → **三类失配** 类选项（**Garlan** 口径）。  
- **多视图**：**开发视图** 看 **微服务仓库**；**物理视图** 看 **机房与分区**。

### 4. 特定领域软件体系结构（DSSA）

**教材依据：第7章 7.5**

- **必备特征**：明确问题域与解域、普遍性、对构件组织模型的抽象、领域可重用元素。  
- **垂直域**：同一系统族；**水平域**：多系统族共有功能片段。  
- **三阶段**：**领域分析**（领域模型）→ **领域设计**（DSSA）→ **领域实现**（可重用资产）。  
- **四种角色**：**领域专家、领域分析人员、领域设计人员、领域实现人员**（职责辨析题）。  
- **建立过程五阶段**（Tracz）：定义领域范围 → 定义领域特定元素 → 定义设计与实现约束 → 定义领域模型与体系结构 → 产生/搜集可重用产品单元；过程具有**并发、递归、迭代**特点。

**【平台案例】**  
- **垂直域**：**「全服务航司运行与订座」** 一条产品线的 **参考架构**。  
- **水平域**：**支付、短信、身份** 等 **跨行业可插拔** 片段。  
- **四角色**：**签派领域专家**、**领域分析/设计/实现** 分工（**名称题** 背原文）。  
- **三阶段**：**分析** 出 **航班/资源领域模型** → **设计 DSSA** → **实现** **构件库**。

### 5. 软件架构复用

**教材依据：第7章 7.4**

- **产品线**：围绕**核心资产库**（架构、构件、文档、测试资产等）。  
- **复用类型**：**机会复用** vs **系统复用**（规划驱动）。  
- **基本过程三阶段**：构造/获取可复用资产 → **管理**（构件库：分类与检索）→ **选择/定制/组装**。

**【平台案例】**  
- **产品线**：**多航司/多航站** 共用 **核心资产库**（**参考架构、接口包、测试集**）。  
- **机会复用**：**某项目** 临时 **复用** 一旧 **适配器**；**系统复用**：**公司级** 规划 **统一主数据与集成平台**。

### 6. 架构评估与质量属性（真题 PDF 中题量很大，教材主要在第8章）

**教材依据：第8章 8.1～8.2**（第7章末仅作过渡）

- **SAAM**：输入常考 **问题描述、需求声明、体系结构描述**；场景从**风险承担者**角度；场景描述要素 **刺激、环境、响应**。  
- **ATAM**：关注**质量属性权衡**；活动领域常考 **属性模型构造和分析**、架构视图与场景实现、折中等；**效用树**对质量属性进行刻画与排序；**敏感点**（影响某质量属性的构件特性）与 **权衡点**（影响多个质量属性且需折中）——加密级别、编码方式等是经典例子。  
- **方法辨析**：**SAAM** 偏最终架构评估与场景对比；**ATAM** 偏多质量属性折中；**CBAM** 偏成本与收益；**SASAM** 在辅导资料中称为**静态**架构分析（真题 2023）。  
- **ATAM 最初强调的四种质量属性**：不同年份真题选项表述略有差异（**性能、可用性、安全性、可修改性** 与 **性能、安全性、可修改性、可用性** 等价表述）。备考以**教材第8章 ATAM 小节**为准。  

*说明：此块不在第7章正文中展开，但属于「系统架构设计」同一选择题分类下的必考内容，需与第7章一并复习。*

**【平台案例】**  
- **SAAM**：对比 **「旧单体离港」** 与 **「服务化拆分」** 对 **场景**（如 **延误处置**）的响应。  
- **ATAM**：**效用树** 叶结点 **「动态 P95」「RTO」「审计」**；**权衡点**：**全链路加密** vs **席位响应**；**敏感点**：**会话保持策略** 影响 **可用与性能**。  
- **CBAM**：**投入容灾与双活** 的 **成本—收益**（若题干出现）。
