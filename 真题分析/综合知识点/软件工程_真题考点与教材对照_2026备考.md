## 二、高频考点清单（按主题）

### 1. 软件生命周期与过程

- **传统三阶段**：**软件定义、软件开发、软件运行与维护**（可行性研究属**软件定义**）。  
- **四大活动**（教材 **5.1.1**）：软件规格说明（Plan）、开发（Do）、确认（Check）、**演进**（Action）；真题亦常见「软件描述、软件开发、有效性验证、**软件演化**」表述。  
- **瀑布模型**：阶段间**因果/先后依赖**，前一阶段输出为后一阶段输入。  
- **原型模型**：**原型开发阶段 + 目标软件开发阶段**。  
- **螺旋模型**：在**快速原型**基础上扩展；每轮含**目标设定、风险分析、开发/验证、评审**（真题口径）。  
- **增量模型**：分增量交付，利于早期获得可运行价值、降低变更成本（与敏捷题干区分）。  
- **喷泉模型**：面向对象、阶段重叠与迭代（真题常考「描述 OO 过程」）。  
- **敏捷**：**适应型**、**以人为本**非以过程为本、迭代增量；**不适合**单纯「团队极其庞大」等表述。  
- **Scrum**：固定 **Sprint**、**产品负责人 / Scrum Master / 开发团队**、**Daily Stand-up**；Product Backlog 排序首要常强调**商业价值**。  
- **其他敏捷方法**：**开放式源码**利于**地域分布**协作；**FDD** 有**首席程序员与类程序员**分工。  
- **RUP**：**用例驱动**、**以体系结构为中心**、**迭代与增量**；四阶段 **初始、细化、构造、移交**（**细化**阶段确定**架构基线**与详细计划/资源）；9 个 **Discipline** 含业务建模、需求、分析设计、实现、测试、部署、**配置与变更管理**、**项目管理**、**环境**——**不含「成本」独立工作流**。  
- **RAD**：模块化程度高时结合**基于构件**快速开发（真题口径）。  
- **W 模型**：开发与测试**同步贯穿**各阶段（教程未用该名称时，以真题为准）。  
- **MDA**：**PIM** 在不绑定具体平台下详述系统；**CIM** 偏业务领域；**PSM** 含平台实现细节。  
- **工作流建模**：活动图、BPMN、Petri 网等；**用例图**描述参与者与用例关系，**不表示过程流转**（不可单独当工作流描述）。  
- **生命周期划分原则**：真题曾出现「各阶段**时间**尽可能**独立**」等选项，以**当年解析**为准。

**【平台案例】**  
- **瀑布**：**局方接口冻结** 后 **需求→设计→实现** 顺序推进（题干强调 **文档闸门**）。  
- **敏捷/Scrum**：**航季** 每 **Sprint** 交付 **可演示** 的 **改签/行李** 功能；**Backlog** 按 **收益与风险** 排序。  
- **RUP**：**智慧运控** 项目 **细化阶段** 确定 **服务划分与架构基线**；**9 工作流** 不含 **成本** 独立流（高频题）。  
- **MDA**：**PIM** 描述 **航班/资源领域服务**；**PSM** 绑定 **具体中间件/消息中间件**。  
- **用例图 vs 工作流**：**旅客值机** 用 **用例** 表达 **参与者—用例**；**登机口变更审批** 若考 **流程** 选 **活动图/BPMN**（非单独用例图）。

### 2. 逆向工程与再工程

- **抽象层次**：**实现级**（AST、语句级）→ **结构级**（模块依赖、调用关系）→ **功能级**（程序段功能及关系）→ **领域级**（业务语义）。  
- **再工程**：在逆向所得信息基础上**修改/重构**得到新版本。  
- **方法**：用户指导的搜索与变换多恢复**实现级+结构级**；**基于领域**的方法利于**功能级+领域级**（与另一题「用户指导」范围区分）。

**【平台案例】**  
- **实现级**：从 **旧离港代码** 抽 **AST**；**领域级**：抽象 **「航班状态机」** 业务语义。  
- **再工程**：将 **单体遗留** **重构** 为 **可复用服务** + **新接口**。

### 3. 软件开发环境与工具

- **SDE**：**环境信息库**集中存储开发相关信息并支持共享；**过程控制与消息服务器**支撑**过程集成与控制集成**。  
- **配置管理工具**：版本控制、变更管理、配置状态、访问与安全等（**版本控制工具**常被视为配置管理的一部分或子能力）。  
- **需求分析工具**：基于自然语言/图形 vs **形式化需求定义语言**。  
- **移植**：**移植方法**在**计划阶段**确定（真题口径）。

**【平台案例】**  
- **环境信息库**：集中 **接口 WSDL、版本、缺陷、测试报告**（多项目共享）。  
- **配置管理**：**R2026 航季主数据包**、**离港适配器 v2.x** 的 **版本/变更/状态**。

### 4. 基于构件的软件工程（CBSE）

- **构件特性**：独立**部署单元**、第三方组装、**无外部可见状态**；部署单元**不可随意拆分**；同一进程多拷贝等叙述需按真题解析甄别。  
- **接口**：已命名的一组操作集合。  
- **服务端模型**：**EJB + COM+** 偏**应用服务器**；**Servlet** 等偏 **Web 服务器**（真题分类）。  
- **构件分类**：**关键字**（树/有向无环图分解）、**刻面**（Facet）、**超文本**组织。  
- **组装技术**：**基于功能 / 基于数据 / 面向对象**；**「基于实现」** 非标准分类。  
- **组装层次**：**定制、集成、扩展**。  
- **组装方式**：层次、叠加、顺序等；**循环组装**非常见推荐方式。  
- **接口不兼容**：**操作不完备**（提供接口为请求接口子集）等。  
- **EJB**：**会话 Bean**（客户端交互）、**实体 Bean**（持久化）、**消息驱动 Bean**（异步/并发）。  
- **CORBA**：**Servant** 为对象真正实现。  
- **J2EE 五种构件**：**Applet、Servlet、JSP、EJB、Application Client**（勿与 JDBC/JNDI 等 API 混淆）。  
- **IDL**：**接口描述**为核心；**模块**映射为 Java **Package** / C++ **Namespace**。  
- **接口标准化**：重点在**消息模式、格式与协议**。

**【平台案例】**  
- **构件**：**独立部署** 的 **「离港适配构件」「运价规则构件」**；**接口** 为 **订座/退改签操作集合**（与 **5.6** 定义一致）。  
- **组装层次**：**定制** 航站参数 → **集成** 多构件 → **扩展** 业务插件。  
- **J2EE 五种构件**：**旅客 App** 常见 **Servlet/JSP**；**后台** **EJB**（真题分类题）。

### 5. 净室软件工程

- **理论基础**：**函数理论** + **抽样理论**。  
- **辨析**：强调正确性验证与统计测试；**「完全不做传统模块测试」** 类绝对表述多为**错误**（教材亦指出完全不测试不现实）。

**【平台案例】**  
- 仅作 **概念联想**：**燃油计算、配载关键** 模块若题干强调 **形式化验证+统计测试**，勿选 **「完全不做单元测试」** 绝对项（与 **5.5** 辨析一致）。

### 6. CMMI、CMM 与 DCMM

- **CMMI**：**5 个成熟度等级**；框架知识体系常考 **系统工程、软件工程、IPPD、供应商采购** 四类。  
- **体系文件**（真题）：**顶层方针 → 过程文件 → 规程文件 → 模板类**。  
- **DCMM**：8 个能力域含 **数据战略、治理、架构、标准、质量、安全、应用、生存周期** 等（以国标与真题为准）。  
- **成熟度叙述**：组织级流程标准化、制度化 → 常对应 **已定义级** 等（结合题干）。

**【平台案例】**  
- **CMMI**：**航司软件中心** 推 **过程标准化**、**度量与基线**（等级与题干对应）。  
- **体系文件塔**：**信息安全方针** → **研发过程** → **发布规程** → **模板**（真题四层）。  
- **DCMM**：**航班主数据、旅客数据** **治理与质量** 与 **8 域** 题干配对（以国标与真题为准）。

### 7. 软件重用

- **水平/横向**：标准函数库、通用算法、通用 UI 组件等。  
- **垂直**：领域专用资产。  
- **双生命周期**：**领域工程**（生产可复用资产）+ **应用工程**（组装应用）。  
- **重用元素**：需求/设计文档、代码、测试用例、领域知识等。

**【平台案例】**  
- **水平重用**：**通用日志、支付、短信** 组件。  
- **垂直重用**：**民航订座/运行** **领域模型与规则包**。  
- **双生命周期**：**领域工程** 产 **可复用资产**；**应用工程** 组装 **某航站/某航司** 项目。

### 8. 其他标准与模型（教材分散）

- **DO-178**：适航核心要素真题曾考 **目标、活动、数据**（Objectives / Activities / Life Cycle Data）类归纳。  
- **软件工程过程四方面**：真题曾出现 **计划、执行、检查、处理** 与 **5.1.1** 的 Plan-Do-Check-Act 叙述对照。

**【平台案例】**  
- **DO-178**：**机载/航电相关软件** 题干与 **适航目标/活动/数据** 归纳（**与运控地面平台项目** 区分，**按题选**）。
