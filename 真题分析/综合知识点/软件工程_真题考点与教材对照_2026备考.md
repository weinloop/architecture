## 一、常考 · 直接背诵

### 1.1 生命周期与过程活动（§5.1.1 + PDF）

**传统三阶段**：**定义 → 开发 → 运行维护**（可行性研究属 **定义**）。  

**四活动**（教材 PDCA 系）：规格说明（Plan）、开发（Do）、确认（Check）、**演进**（Action）。  
真题亦常见：**描述、开发、有效性验证、演化** —— **与同一套对应**。

**瀑布**：阶段 **因果依赖**；**不适合需求多变**。  
**原型**：**原型阶段 + 目标软件开发阶段**。  
**螺旋**：**快速原型 + 风险分析**；每轮 **目标—风险—开发/验证—评审**（随题干）。  
**增量**：**分块交付**、早获价值；与敏捷 **按题辨**。  
**喷泉**：**OO**、阶段 **重叠反复**。  
**敏捷**：**适应型、以人为本**；**不大**团队即唯一选型等 **绝对项慎选**。  
**RAD**：构件化成熟 + **快速**（随解析）。

---

### 1.2 MDA 与工作流（PDF）

- **CIM**：业务/领域目标；**PIM**：**平台无关** 细致规格；**PSM**：**含平台实现细节**。  
- **工作流**：**活动图、BPMN、Petri 网** 等；**用例图** 表 **参与者—用例**，**不** 单独表示 **过程流转**。

---

### 1.3 逆向工程与再工程（与《01》五一致 + 真题层次名）

- **层次（低→高）**：**实现级**（AST）→ **结构级**（调用/依赖）→ **功能级** → **领域级**。  
- **再工程**：逆向信息 + **新需求/重构** → 新版本。  
- **恢复方法**：**用户指导搜索** 多 **实现+结构**；**基于领域** 多 **功能+领域**（**按题**）。

---

### 1.4 软件开发环境与工具

- **环境信息库**：集中 **规格、版本、缺陷** 等共享信息。  
- **过程控制与消息服务器**：**过程集成 / 控制集成** 基础。  
- **配置管理工具**：版本、变更、状态、访问与安全；**版本控制** 常归 **SCM 能力子集**（随题）。  
- **需求工具**：自然语言/图形 **vs** **形式化需求语言**。  
- **移植**：**移植策略** 多在 **计划阶段** 定（真题口径）。

---

### 1.5 CBSE 真题扩展（§5.6 + PDF）

**构件特性（真题句）**：独立 **部署单元**、**第三方组装**、**无外部可见状态**（与《01》「可见性」一致口径）、部署单元 **不可随意拆分** 等 —— **绝对化句子对题**。  

**分类方法**：**关键字 / 刻面（Facet）/ 超文本**。  
**组装技术**：基于 **功能、数据、OO**；**非**标准项：**「基于实现」**。  
**组装层次**：**定制、集成、扩展**。  
**组装方式**：顺序、层次、叠加等；**循环组装** 多 **不推荐**。  
**接口不兼容**：参数/操作/ **操作不完备**（子集关系）。  

**EJB 三类**：**会话、实体、消息驱动**。  
**J2EE 五种构件**：**Applet、Servlet、JSP、EJB、Application Client**（勿与 **JDBC/JNDI** 等 API 混）。  
**CORBA**：**Servant** = 对象真正实现。  
**IDL**：**接口** 描述；**模块** → Java **Package** / C++ **Namespace**。  
**服务端归类**（真题）：**EJB、COM+** 偏 **应用服务器**；**Servlet** 偏 **Web 服务器**。  
**接口标准化**：**消息模式、格式、协议**。

**《01》萨默维尔六特性、构件五类、三种组装** → **详背《01》六章**，本文不重复大表。

---

### 1.6 净室（§5.5 + 《01》四）

- **理论**：**函数理论** + **抽样理论**（口诀 **玩一阵**）。  
- **辨析**：**不是**「完全不做传统测试」的银弹（**绝对错项** 多）。

---

### 1.7 RUP（§5.1.4 + 《01》八）

- **特征**：**用例驱动**、以 **体系结构为中心**、**迭代与增量**。  
- **四阶段**：初始、**细化**（**架构基线**、详细计划）、构造、移交。  
- **九核心工作流**：**建需分食测、部配变项环** —— **无独立「成本」工作流**（高频）。

---

### 1.8 敏捷与 Scrum（§5.1.3 + 《01》九）

- **宣言**：四 **胜过** —— **见《01》9.1**。  
- **适用**：**小、变**（规模可控、需求易变）；**高安全单一路径** 题干需 **与质量体系结合**。  
- **Scrum**：PO / SM / Team；Sprint；Backlog **商业价值** 排序；**开放式源码** 利于 **分布式** 协作；**FDD**：**首席程序员 / 类程序员**。  
- **工件与活动** —— **《01》9.4**。

---

### 1.9 CMMI / CMM / DCMM（§5.1.5 + PDF）

- **CMMI 5 级**：《01》口诀 **储罐定粮油**（与真题等级名 **逐字核对**）。  
- **四知识体系**：**系统工程、软件工程、IPPD、供应商采购** —— **细软开工**。  
- **体系文件塔**（真题）：**方针 → 过程 → 规程 → 模板**。  
- **DCMM**：**8 能力域**（战略、治理、架构、标准、质量、安全、应用、生存周期等 —— **以国标·题为准**）。  
- **CMM 五级**：《01》**初重定管优**。

---

### 1.10 软件重用（PDF + §5.6）

- **水平**：通用库、通用组件；**垂直**：**领域资产**。  
- **双生命周期**：**领域工程**（生产资产）+ **应用工程**（组装应用）。

---

### 1.11 其它标准（PDF）

- **DO-178**：适航 **目标 / 活动 / 生命周期数据** 归纳题 —— **机载/航电题干**；**勿** 与 **地面信息系统** 混答。  
- **V 模型测试依据**：《01》7.6 表；**系统测试依据** 有的解析写 **需求分析/合同** —— **以当年题为准**。

## 二、下次考试预测 · 直接背诵

**高概率**

1. **RUP**：九流 **无成本**；**细化** 阶段 **架构基线**。  
2. **Scrum**：角色、Backlog **商业价值**。  
3. **螺旋** = 原型 + **风险**；**喷泉** OO 重叠。  
4. **CBSE**：构件特性、组装层次、**J2EE 五种**、**EJB 三类**。  
5. **逆向** 四层；**再工程** 定义。  
6. **CMMI** 等级、四知识体系、**文件四层塔**。  
7. **增量 vs 迭代**（《01》7.7）。  

**中概率**

8. **MDA** CIM/PIM/PSM；**用例图≠工作流**。  
9. **净室** 理论与 **非**完全无单测。  
10. **重用** 双生命周期、横纵。  
11. **SDE** 信息库、过程服务器。  
12. **DCMM** 域名配对。  

**低概率**

13. **DO-178**（题干涉适航）。  
14. **RAD**、**生命周期划分原则** 非常规句 —— **跟解析**。  

**预测口诀**

**三阶段定义开发维过程四活动演螺旋原型风险喷泉OO叠增量块敏捷人小变瀑布冻结门RUP用例架构基九流无成本MDA业务无关PIM用例不当流逆向实现结构功领域CBSE部独状态五构件J2EE五EJB三CORBA Servant组装定集扩展非实现净室函数抽样玩一阵敏捷宣言胜过Scrum PO backlog CMMI储罐四体系方针过程规程模DCMM八域双生命周期领域加应用适航DO178机载单行记。**
