## 一、常考 · 直接背诵

### 1.1 测试方法（5.4.1 + 《05》二）

- **静态**：**不运行**；走查、评审、结构分析等。  
- **动态**：**运行**；黑盒（等价类、边界值、**因果图/判定表** 等）、白盒（逻辑/路径等）。  
- **黑盒**：**功能**，不看实现；需 **多条件→多动作** → 常选 **判定表**。  
- **白盒**：看 **内部结构**；**功能测试属黑盒，非白盒**。  
- **灰盒**：**介于二者**；集成常用。

---

### 1.2 白盒覆盖强弱（背《05》三 + 真题抠字）

**弱 → 强**：**语句 < 判定 ≈ 条件 < 判定-条件 < 条件组合 < 路径**。

**口诀**：《05》— **路条组判｜条条判语**。

| 覆盖 | 要点 |
|------|------|
| **语句** | 每条语句至少一次 —— **最弱** |
| **判定** | 每个判定 **两分支** 至少各一次 |
| **条件** | 各条件真假都出现，**组合未必全** |
| **判定-条件** | **判定覆盖 + 条件覆盖** |
| **条件组合** | 一判定内 **条件组合** 覆盖 |
| **路径** | **所有路径** —— **最强** |

**注意**：判定覆盖与条件覆盖 **互不必然包含**，**不可简单比强弱**（**按题表述**）。

---

### 1.3 测试阶段与依据（高频）

| 阶段 | 依据（V 模型经典） |
|------|---------------------|
| **单元** | **详细设计**；对象 **模块/函数** |
| **集成** | **概要设计**；**接口、协作** |
| **系统** | **SRS（需求规格说明书）** |
| **验收/确认** | **用户需求/合同** 等 |

**顺序**：**单元 → 集成 → 系统**（→ 验收类）。  
**回归**：变更后 **验证修改 + 不破坏原有**。

---

### 1.4 桩（Stub）与驱动（Driver）

- **桩**：模拟 **被测模块所调用的下层**（**替身下级**）。  
- **驱动**：模拟 **调用被测模块的上层**（**替身上级**）。  
- **自顶向下**：常需 **桩**；**一般不需额外驱动**（从顶层下）。  
- **自底向上**：常需 **驱动**；**一般不需桩**。

---

### 1.5 性能测试辨析（同《05》4.2，一句版）

| 类型 | 背关键词 |
|------|-----------|
| **负载** | **已达部分瓶颈** 后 **持续运行**，观指标 |
| **压力** | **渐进加压** 至 **瓶颈/不可接受点**（**过程性**） |
| **强度** | **极限状态**（超负荷、恶劣环境），**非**强调渐进过程 |
| **并发** | 负载+压力思路 **结合**，综合交易与资源 |
| **容量** | **预先给定极限指标**（如最大并发用户），在极限下是否仍满足 |
| **可靠性** | 为提高 **可靠性** 的试验与发现缺陷 |

**压力 vs 强度**：《05》— 压力**慢慢加压**；强度**直接造极端**。

---

### 1.6 自动化与 A/B

- **线性脚本**：录制回放。  
- **数据驱动**：数据 **外置**，脚本循环读（**DDT**）。  
- **不宜自动化**：**需求频繁变**、脚本维护成本极高等。  
- **A/B**：**同时**、用户 **随机** 分版本，比指标。

---

### 1.7 原则与辨析

- **测试目的**：**发现缺陷**；**不能证明无错**。**调试** = **定位修复**，周期难估。  
- **缺陷群集（二八）**：少数模块集中多数缺陷。  
- **基准程序真实度**：**真实程序 > 核心程序 > 小型基准 > 合成基准**。

---

### 1.8 Web、OO 与其它

- **Web 服务器性能评测**：常用基准/压力/可靠性等；**UI 测试** **不是** 服务器性能方法。  
- **OO 测试**：**多态** 需对 **可能绑定** 分别测；**测遍方法 ≠ 测透类**。

---

### 1.9 测试模型 W / H / X（《05》六）

| 模型 | 一句 |
|------|------|
| **W** | **V 的延伸**；开发与测试 **同步**（左移） |
| **H** | 测试活动 **独立**，**贯穿** 全生命周期 |
| **X** | **片段** 开发与测试 **频繁交替**、再集成 |

---

### 1.10 安全测试类型（《05》五）

加密、身份认证、日志、Session、备份恢复、访问控制、漏洞（XSS/SQL 注入）、端口、脚本、防火墙等 —— **概念归属题**。

## 二、下次考试预测 · 直接背诵

**高概率**

1. **黑/白/灰** 界定；**路径覆盖** 属白盒。  
2. **V 模型**：**系统测试 ↔ SRS**；**集成 ↔ 概要设计**；**单元 ↔ 详细设计**。  
3. **桩 / 驱动** 与 **自顶向下 / 自底向上**。  
4. **负载 vs 压力 vs 强度** 关键词。  
5. **回归**、**Alpha/Beta** 场景。  
6. **测试 vs 调试**。  
7. **白盒覆盖** 排序或「不可简单比较」陷阱。  

**中概率**

8. **静/动**；**判定表** 多条件。  
9. **自动化**：数据驱动、不宜自动化。  
10. **A/B**。  
11. **基准程序** 真实度排序。  
12. **Web 性能** 不含 UI。  

**低概率**

13. **OO 多态**、净室统计测试交集。  

**预测口诀**

**黑盒功能白盒路灰盒集成桩下驱上自顶要桩底上要驱动，静不运动要跑语句弱路径强判定条件可比看原句，单元详设集成概要系统需求V字对上号，负载瓶颈继续跑压力渐加压强度极端造回归改后防副作用，Alpha内Beta外自动化数据驱变太快别录播，W同步H独立X碎片真实基准胜合成调试修错测试找错证不全。**

## 七、考前一页纸

1. **黑/白/灰**；**静/动**。  
2. **覆盖**：路条组判｜条条判语；**路径最强**。  
3. **V**：单元—详设；集成—概要；系统—**SRS**。  
4. **桩** 代下、**驱动** 代上；顶向下要 **桩**。  
5. **负载** 瓶颈后持续；**压力** 渐加压；**强度** 极限环境。  
6. **回归**：改对 + 不破坏旧功能。  
7. **测试** 发现；**调试** 修；**不能证明无缺陷**。  
8. **基准**：真实 > 核心 > 小型 > 合成。  
9. **W 同步、H 独立、X 交错**。  
10. **详表与细分** → **`选择/05-软件测试-考点速记.md`**。
