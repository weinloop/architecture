## 二、高频考点清单（按主题）

### 1. 数据库设计与 E-R

- **六阶段**（教材 **6.3.1**）：需求分析 → 概念结构（**E-R**）→ **逻辑结构（关系模式）** → 物理结构 → 实施 → 运行维护。  
- **关系模式**主要在 **逻辑结构设计** 阶段形成；真题亦考「反规范化」多在 **逻辑设计** 阶段权衡。  
- **E-R 集成冲突**：**结构冲突**（同一对象不同抽象、属性个数/次序不同等）、**属性冲突**（域不一致）、**命名冲突**（同名异义/异名同义）；合并策略依题干。  
- **派生属性**：如由出生日期可算**年龄**，一般不存入关系，查询时计算。

**【平台案例】**  
- **六阶段**：从 **需求（规章与业务）** → **E-R** → **关系模式** → **索引分区** → **上线** → **航季运维**。  
- **E-R 冲突**：**同一航站** 在 **货运与客运** 子系统 **抽象层级不同**（**结构冲突** 类）。  
- **派生属性**：**延误时长** = 实际 − 计划，可 **不落库** 计算（与教材例一致）。

### 2. 三级模式与体系结构

- **外模式**：用户视图、局部逻辑结构。  
- **概念模式（模式）**：全局逻辑结构、完整性安全性等（真题表述与「用户视图」区分）。  
- **内模式**：物理存储、索引（**聚簇索引**改变内模式）。  

**【平台案例】**  
- **外模式**：**旅客** 只见 **本人行程**；**运控** 见 **全量动态**。  
- **概念模式**：**全局** **航班—航段—资源** 关系与约束。  
- **内模式**：**按航季分区**、**聚簇按航班号** 等物理设计。

### 3. 事务与并发

- **ACID**：原子性、一致性、隔离性、**持久性**；**并发性**是系统能力，**不是**事务四性之一。  
- **封锁**：**共享锁（S）** 一般**只读不写**；排他锁（X）可读写。  
- **两阶段提交（2PC）**：**表决（准备）阶段** + **执行（提交/回滚）阶段**；勿与 **两阶段锁（2PL）** 混淆。

**【平台案例】**  
- **ACID**：**改签** 要么 **全程成功** 要么 **回滚**（**原子性**）；**并发** 不是 **四性** 之一（高频判断）。  
- **S 锁**：**大量读动态** 时 **共享锁**；写 **机位分配** 用 **X 锁**。  
- **2PC**：**多库扣座** **准备—提交** 与 **2PL** **加锁—释放** 分开记。

### 4. 关系代数与 SQL

- **笛卡尔积**：\(m\times n\) 个元组。  
- **自然连接**：去重**同名属性列**，列数为两关系属性并集去重后的个数。  
- **恒等式**：\(R-(R-S) = R\cap S\)。  
- **查询优化**：**选择 \(\sigma\) 尽早下推**，先缩小关系再做连接，通常优于先大连接再筛选。  
- **HAVING**：与 **GROUP BY** 配合，对**分组后**结果过滤。  
- **自然连接转 SQL**：公共属性 **AND** 连接；条件忌滥用 **OR**。

**【平台案例】**  
- **笛卡尔积**：**航班表×旅客表** 若无连接条件则 **m×n**（题眼）。  
- **选择下推**：先 **σ 日期=今天** 再 **⋈**，减少中间结果（**优化题**）。  
- **HAVING**：按 **航线** **GROUP BY** 再筛 **COUNT>100**（**分组后过滤**）。

### 5. 函数依赖与范式

- **Armstrong**：自反、增广、传递；导出规则如 **分解**（\(X→Y,\ Z⊆Y \Rightarrow X→Z\)）、**合并**（\(X→Y,\ X→Z \Rightarrow X→YZ\)）。  
- **候选键**：属性闭包能否覆盖全部属性；注意**最小性**。  
- **2NF**：消除**非主属性对码的部分依赖**。  
- **3NF**：消除非主属性对码的**传递依赖**（在 2NF 基础上）。  
- **4NF**：消除**多值依赖**（在 BCNF 前提下，教材 **6.2.3**）。  
- **模式分解**：判断是否 **无损连接**、是否 **保持函数依赖**（交集属性能否决定差集等套路）。

**【平台案例】**  
- **候选键**：**（航班号，日期）** 能否 **决定** 航段与机型等（**闭包题**）。  
- **3NF**：**航站名** 依赖于 **航站代码** 而非 **主键** 时 **传递依赖** 需拆表（示意，以题干 FD 为准）。  
- **分解**：**订座** 与 **旅客档案** 拆分时的 **无损/保持依赖** 仍按 **算法步骤**。

### 6. 完整性

- **实体完整性**：主键。  
- **参照完整性**：外键参照主键。  
- **用户定义完整性**：如性别枚举、取值范围、CHECK 等。

**【平台案例】**  
- **实体**：**航班号+日期** **主键** 唯一。  
- **参照**：**行李.航班号** **外键** 指向 **航班**。  
- **用户定义**：**舱位** ∈ {F,C,Y}、**延误原因码** **CHECK**。

### 7. 安全与访问封装

- **存储过程**：封装更新逻辑，对外隐藏表结构（真题口径）。

**【平台案例】**  
- **存储过程**：封装 **「改舱+重算费用+写日志」**，App **只调过程名**（与真题 **封装** 表述一致）。

### 8. 分布式、数据仓库与嵌入式

- **透明性**：**位置透明**—不必知物理站点；**逻辑透明（局部映像透明）**—不必知局部 DBMS/模型；**分片透明**—不必知分片；**复制透明**—不必知副本。  
- **全局概念模式**：描述分布式库**整体逻辑结构**，用户像访问非分布式一样使用。  
- **数据仓库**：面向主题、集成、**相对稳定（非易失）**、反映历史变化。  
- **NDB**：客户端+协议+远程服务器；**非**典型「本地文件直接读写」模型。  
- **嵌入式 DB**：现代产品常**支持多线程并发**（辨析题）。

**【平台案例】**  
- **透明性**：用户 **不必知** **行李库在哪个城市**（**位置**）；**不必知** **分片键**（**分片**）。  
- **数据仓库**：**T+1 装载** **历史航班、收益**，**相对稳定**、面向 **主题**（**正常性、航线**）。  
- **NDB**：**旅客 App** 通过 **协议** 访问 **中心库**，**非** 本地文件当库。  
- **嵌入式**：**场面终端** 内嵌 **轻量库** **多线程** 写本地缓存。
