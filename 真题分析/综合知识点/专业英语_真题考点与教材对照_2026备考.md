## 二、高频术语速记（按场景）

### 1. 微服务与分布式

- **micro-service architecture**  
- **decompose / manageable / maintain**  
- **develop / deploy / scale / expand** **independently**

**【平台案例】**  
- **订座、行李、结算** 拆为 **micro-services**，团队 **develop** 与 **deploy** **independently**，整体仍要 **maintain** 一致 **SLA**（**micro-service architecture, maintain, independently**）。

### 2. C/S 与 Web

- **thin client** vs **fat client**  
- **database server** vs **application server** vs **web server**  
- **presentation / application logic / data** layers

**【平台案例】**  
- **航站 kiosk** 常为 **thin client**；本地 **离港** 终端可能是 **fat client**；**flight inventory** 在 **database server**，静态页面走 **web server**（**thin/fat, database server, web server**）。

### 3. 数据库与规范化

- **data redundancy**、**data integrity**（注意与冗余题区分）  
- **ERD (entity-relationship diagram)**  
- **normalization**、**first/second/third normal form (3NF)**  
- **referential integrity**（relational DB）

**【平台案例】**  
- **航班—旅客—行李** 画 **ERD**；**normalization** 到 **third normal form** 降低 **data redundancy**；**ticket** 与 **booking** 间 **referential integrity**（**ERD, normalization, 3NF, redundancy, referential integrity**）。

### 4. 架构设计任务

- **allocate** software to **hardware**  
- **data storage → data access logic → application logic → presentation logic**  
- **clients, servers, and network**（三大硬件逻辑划分）

**【平台案例】**  
- **requirements analysis** 产出模型后定 **application architecture**；逻辑链 **data storage → data access logic → application logic → presentation** 可对应 **舱单库 → DAO → 业务规则 → 值机 UI**；硬件 **clients, servers, and network** 覆盖 **航站、机房、专线**（**application architecture, requirements analysis, four logic layers**）。

### 5. 非功能需求英文标签

- **operational requirements**（运行环境）  
- **performance requirements**（响应时间、容量、可靠性等）  
- **security requirements**（防破坏、防数据丢失）

**【平台案例】**  
- **724 运行环境** → **operational**；**高峰值机响应** → **performance**；**旅客隐私与防篡改** → **security**（**nonfunctional requirements, operational, performance, security**）。

### 6. 架构结构三类（必背英文）

- **Module structures**（代码/数据单元）  
- **Component-and-connector structures**（运行时行为与交互）  
- **Allocation structures**（映射到 CPU、网络、团队等非软件环境）

**【平台案例】**  
- **值机模块** → **Module**；**ESB 上消息流** → **Component-and-connector**；**构件部署到双活中心** → **Allocation**（三类 **structures** 必辨）。

### 7. 质量属性场景（ATAM 等）

- **architect**  
- **quality attribute scenarios**  
- **stimulus / response / measure**（题干中可出现 **events** 与 performance 社区）

**【平台案例】**  
- 从 **architect** 视角写 **quality attribute scenarios**；**performance** 场景里社区常谈 **events** 与 **stimulus–response–measure**（**ATAM** 语境词）。

### 8. 其他

- **legacy database**、**network database**（指针）  
- **master / lookup / transaction** files  
- **software architecture reconstruction**：**information extraction**、**view fusion**

**【平台案例】**  
- **旧 AODB** → **legacy database**；**指针式** 老库 → **network database**；**国家码表** → **lookup file**；逆向遗留系统 **SAR**：**information extraction**、**view fusion**（**legacy, master/lookup, SAR**）。

## 三、备考提示与复习建议

**【平台案例】**  
- 与其他综合稿统一 **航空运营智能管理平台** 场景，**只帮助判断段落主题**；**完形答案** 仍依赖 **搭配与排除**。可与 **`综合知识点/系统架构设计_真题考点与教材对照_2026备考.md`**、**`综合知识点/数据库系统_真题考点与教材对照_2026备考.md`** **对照朗读** 中英文术语。


1. **题型**：全部为 **语境选词**，需兼顾 **语法（动宾搭配）** 与 **术语精确性**（如 thin/fat、database server vs web server）。  
2. **方法**：先扫 **5 个空** 的主题句（往往考同一概念链），再按 **固定搭配** 锁定 1～2 空，其余用排除法。  
3. **与综合知识联动**：复习 **第7章架构风格与 C/S**、**第6章范式与完整性**、**第8章质量属性与场景** 时，**顺带朗读英文小节标题与教材中已有英文缩写**（如 ERD、DFD），可显著降低陌生感。  
4. **易混拼写**：题目中偶见 **typo**（如 Serve-sider、dataaccess），以 **选项合法拼写** 为准。  
5. 一句话串平台：**Micro 独立 deploy，Thin 瘦 Fat 胖，四层逻辑记 storage 到头，Module CC Allocation 三结构，3NF 减 redundancy。**
