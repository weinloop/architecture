## 一、常考 · 直接背诵

### 1.1 微服务与分布式

- **micro-service architecture**；**decompose / manageable / maintain**  
- **develop / deploy / scale / expand** **independently**（并列常考）  

---

### 1.2 C/S 与 Web

- **client/server**；**thin client**（弱终端，仅 **UI**）vs **fat client**  
- **database server**（共享库、SQL）vs **application server** vs **web server**（HTTP/站点）  
- **presentation / application logic / data** 分层  

---

### 1.3 系统设计

- **application architecture**（技术总纲）；模型来自 **requirements analysis**  
- **physical data flow diagram**；**system users**；**system proposal**（已批准）  

---

### 1.4 数据库与规范化

- **data structure**；**data redundancy**  
- **entity-relationship diagram (ERD)**  
- **normalization**；**first/second/third normal form (3NF)**  
- **referential integrity**（主外键一致）  

---

### 1.5 文件与库类型

- **master files**（核心业务）；**look-up files**（码表）；**transaction files**  
- **legacy database**；**network database**（指针链）  

---

### 1.6 非功能与四层逻辑

- **nonfunctional requirements**；**operational / performance / security** requirements（辨析）  
- 逻辑链（常考顺序）：**data storage → data access logic → application logic → presentation logic**  
- 硬件三分：**clients, servers, and network**  

---

### 1.7 架构结构三类（必背英文）

- **Module structures**  
- **Component-and-connector structures**  
- **Allocation structures**  

---

### 1.8 SAR 与黑板、质量

- **software architecture reconstruction**：**information extraction**；**elements and relations**；**view fusion**；**visualization and interaction**；**pattern definition and recognition**  
- **Blackboard**：**decentralized**；**lack** 固定算法；**sharing information**；**partial** solution  
- **quality attribute scenarios**；**stimulus / response / measure**；performance 与 **events**（题干语境）  
- **MAS**：**uniform** knowledge；中心失效 **entire** system；**share**；**coordinating**（按题）

## 五、解题动作（简）

1. **先读 5 空** 是否 **同一概念链**（常是）。  
2. **锁定** 1～2 个 **固定搭配**（动宾、并列）。  
3. 其余 **排除** 语义/词性不符项。  
4. 题干偶见 **拼写噪声**，以 **选项合法拼写** 为准。
