## 二、高频考点清单

### 1. 需求工程阶段划分（开发侧）

教程将需求工程活动分为（**§5.2** 开篇）：  
**需求获取 → 需求分析 → 形成需求规格（文档化）→ 需求确认与验证 → 需求管理**。

- **需求获取、需求建模（分析）、需求评审（验证环节）** 属于 **需求开发** 过程，**不是**「需求管理」定义下的主要活动（与 **2021 年第18、19 题** 选项辨析一致）。

**【平台案例】**  
- **获取 / 分析 / 规格 / 确认**：**航季前** **业务与规章** 联合 **需求采集**；**「不正常航班」** 规则 **分析建模**；**SRS** 含 **值机、改签、配载**；**评审** 通过后方可 **基线**。

### 2. 需求管理的主要活动（管理侧）

与 **图5-5** 一致，应掌握四类：

1. **变更控制**：建议变更、影响分析、决策、交流、合并、需求稳定性测量等。  
2. **版本控制**：需求文档版本、单条需求版本。  
3. **需求跟踪**：与其他需求及系统元素之间的 **连接链**（**§5.2.3**）。  
4. **需求状态跟踪**：需求状态定义与生命周期状态跟踪。

**易错**：**文档管理** 很重要，但教程在「需求管理」图示中强调的是 **版本控制 + 可追溯**，广义 **文档管理** 常被归入 **配置管理/项目管理**，真题中「不属于需求管理主要活动」多选 **文档管理**（**2018 年第18 题**）。

**【平台案例】**  
- **变更控制**：**接口字段** 或 **航司规则** 变更 → **影响分析** → **CCB** 决策（与 **§5.2.2** 衔接）。  
- **版本控制**：**《旅客服务 SRS》v1.2**、单条需求 **REQ-CHK-015** 版本。  
- **需求跟踪 / 状态**：**矩阵** 链到 **舱单服务**、**测试用例**；状态 **已批准 / 已实现 / 已验证**。

### 3. 需求基线与需求约定

- **需求基线**：经 **正式评审批准** 的需求集合，作为后续设计、实现、测试的依据。  
- **需求约定**：基线在客户与开发者之间就 **功能需求与非功能需求** 达成的 **一致约定**，是 **需求开发** 与 **需求管理** 的 **桥梁**（**2020 年第16 题**）。

**【平台案例】**  
- **基线**：**SRS 评审签字** 后，**设计/开发/测试** 均以 **基线包** 为准。  
- **约定**：**航司** 与 **承建方** 对 **「可改签条件、审计日志保留期」** 等 **书面一致**（**桥梁** 关键词背题）。

### 4. 需求跟踪（§5.2.3）

- **定义**：编制 **每个需求与系统各元素之间联系** 的文档；元素包括 **其他需求、体系结构、设计部件、源代码、测试、帮助文件、文档** 等。  
- **目的**：维护 **需求—设计—编码—测试** 一致性，支撑 **变更影响分析**。  
- **实现**：常用 **需求跟踪矩阵**；常与 **配置管理工具** 结合。

**【平台案例】**  
- **跟踪链**：**「旅客身份校验」** 需求 → **架构** 中 **认证网关** → **设计** 中 **接口** → **代码** 模块 → **测试** **用例 TC-AUTH-01**（**元素** 与教材列举一致）。

### 5. 与「需求变更」衔接（扩展）

**§5.2.2** 变更过程三步：**问题分析与变更描述 → 变更分析与成本计算 → 变更实现**（与项目管理 PDF 中变更题一致，可合并复习）。

**【平台案例】**  
- **变更三步**：**「航季时刻表变更」** → **评估离港/库存接口** 影响与 **人天** → **发布实现**（与 **`综合知识点/项目管理_真题考点与教材对照_2026备考.md`** 变更题 **合并背**）。

## 三、备考提示

**【平台案例】**  
- **`需求工程.pdf`** 与 **`软件工程.pdf`** 中需求题 **交叉练**；**场景** 统一用 **航空运营智能管理平台** 便于 **记忆图5-5** 与 **跟踪一段话**。


1. **背图5-5**：需求管理四项活动名称及与「开发活动」的区分。  
2. **背清定义**：**需求跟踪** 一段话（可与配置管理中的需求追溯题对照）。  
3. **背基线两句**：**基线** + **约定** + **桥梁** 三个关键词。  
4. 本 PDF 题量小，建议做 **`软件工程.pdf`** 中需求、变更、追溯类题补全训练。  
5. 一句话串平台：**获取分析规格确认是开发，变更版本跟踪状态是管理；基线评审后约定，文档管理不算图5-5。**
