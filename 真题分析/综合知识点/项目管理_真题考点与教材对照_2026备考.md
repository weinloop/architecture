## 二、高频考点清单（按主题）

### 1. 配置管理与产品配置

- **产品配置**：生命周期各阶段各种形式、各种版本的 **文档、计算机程序、部件及数据** 的集合（教程 **5.7.3** 从「标识、控制修改」角度阐述 SCM；真题定义偏「集合」表述）。  
- **配置项**：上述集合中的元素；**工作计划** 属项目管理产物，**不是**产品组成部分工作成果（真题多次考）。  
- **配置项状态**（真题固定表述）：**草稿、正式发布、正在修改**。  
- **审批后的阶段产物**：常考 **软件配置项**（B）。  
- **变更控制委员会 CCB**：可按变更类型设多个或分组，**并非**只能一个小组（选「不正确」时常选此项）。  
- **标准与配置管理**：**UML** 为建模语言，**不包含**配置管理主题；**ISO 9000、PMBOK、CMMI** 等体系或指南中会涉及配置/变更与过程控制（以题为纲）。

**【平台案例】**  
- **产品配置**：**航班库脚本、离港适配器、测试报告、部署手册** 等 **受控集合**；**项目周计划** **不是** **配置项**（真题辨析）。  
- **CCB**：**多类变更**（**接口 / 数据 / 安全**）可 **分组** 评审，**非** 只能一个会（选「不正确」时常考）。

### 2. 软件质量三维（McCall）

- 从用户视角三组因素：**产品运行**（正确性、可靠性、效率等）、**产品修改**（可维护性、灵活性、可测试性等）、**产品转移**（可移植性、可重用性、互操作性等）。  
- 与教材 **5.7.4** 图5-8 及文字一致。

**【平台案例】**  
- **运行**：**动态查询正确、可用**；**修改**：**航季改规则** 成本 **可控**；**转移**：**移交运维** **文档与培训**。

### 3. 范围管理（概念题）

- **产品范围**：产品或服务应具备的功能与特性。  
- **项目范围**：为交付产品而必须完成的**全部工作**。  
- **范围定义** 是编制进度、成本、资源等**项目计划**的前提（真题判断「正确的是」常选此项）。  
- **定义范围过程的输入**：以各年真题选项为准（常见组合含项目章程、范围管理计划、组织过程资产、已批准变更等），与 PMBOK 版本差异处**背真题逻辑**即可。

**【平台案例】**  
- **产品范围**：**旅客能做什么**（**改签、行李、动态**）；**项目范围**：**本期做哪些接口、哪些航站**（**全部工作**）。  
- **范围定义**：**先定范围** 再 **排进度/算成本**（判断题常选）。

### 4. 文档与需求跟踪

- **用户文档**：安装手册、参考手册、版本说明等面向使用与运维的文档。  
- **系统设计** 等面向开发维护人员的属 **系统文档**（「不属于用户文档」选系统设计）。  
- **正向跟踪**：规格中的**每条需求**是否在后继工作成果中有对应（教材 **5.2.3**）。  
- **逆向跟踪**：设计、代码、用例等是否均能**回溯**到需求规格中的出处。  
- 选项将「代码回溯到需求」说成正向跟踪 → **错误**。

**【平台案例】**  
- **用户文档**：**值机终端操作手册**；**系统文档**：**AODB 接口规格**（「不属于用户文档」类题）。  
- **正向**：**每条需求** 是否映射到 **表字段/接口**；**逆向**：**缺陷单** 能否 **追溯到** **需求条目**。

### 5. 项目组织

- **源代码** 由开发组产出；项目管理与支撑组产出 **计划、跟踪、质量报告** 等，不产源代码。

**【平台案例】**  
- **PMO/PM** 产出 **里程碑、风险登记册**；**不交付** **离港源代码**（组织题）。

### 6. 进度管理（计算与概念）

- **六个过程**（与 **5.7.2** 一致）：活动定义、活动排序、活动资源估算、活动历时估算、制定进度计划、进度控制。  
- **活动定义** 常用工具：**WBS**（把工作包细化为活动）。  
- **关键路径**（CPM）：最长路径决定最短总工期；求「最少需要多少天」即关键路径长度。  
- **总时差**：非关键路径活动可拖延的天数而不影响总工期（需画网络或列表计算）。  
- **三点估算**（PERT）：期望 \(t = (t_o + 4t_m + t_p)/6\)，再结合关键路径与总时差。  
- **时间-成本权衡（赶工）**：压缩关键路径上**单位费用最低**的活动；出现多条关键路径需**同时**压缩；比较 **赶工增加的直接费用** 与 **缩短工期节省的间接费用**；直至再压缩不划算为止，得到**最低总费用**对应工期。

**【平台案例】**  
- **WBS**：**「单航站离港适配」** → **活动**（**安装、联调、UAT**）。  
- **关键路径**：**联调** 在 **外部系统窗口** 上，**拖延即拖全局**（**计算仍按图**）。  
- **赶工**：**加并行联调人力**、**延长每日窗口**（**直接费↑** vs **间接费↓**）。

### 7. 成本与变更

- **成本预算**：把成本估算分配到活动与工作包，形成 **成本基线**（**不是**单纯估算、跟踪或控制）。  
- **需求变更管理流程**（填空）：① **问题分析与变更描述** → ② **变更分析与成本计算** → ③ **变更实现**。  
- **变更控制工具**典型能力：**记录每次状态变更的日期与操作者**（可追溯）；**自动判定能否实施**等通常不合理。  
- **配置项变更顺序**：先 **申请** → **风险评估** → **批准** → **实施** → **验证** → **存档**。

**【平台案例】**  
- **成本基线**：**预算** 分解到 **「接口联调」「性能压测」** 工作包；变更导致 **返工** 对比 **基线**。  
- **变更三步**：**「航季字段变更」** → **评估与报价** → **开发实现**。  
- **工具**：变更单 **记录** **时间、操作人、状态**（可追溯）。
