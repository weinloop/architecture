## 二、高频考点清单（便于背诵）

### 1. 信息化规划方法（第12章原文口径）

- **CSF（关键成功因素法）**：识别关键成功因素 → 关键信息集合 → 常考确定系统开发的 **优先次序**；目标分解链：**目标 → CSF → 性能指标识别 → 数据字典**。  
- **SST（战略目标集转化法）**：把组织战略目标视为「**信息集合**」，转化为 MIS 战略目标。  
- **BSP（企业系统规划法）**：**自上而下**识别目标、企业过程与数据，**自下而上**设计信息系统。

**【平台案例】**  
- **CSF**：若题干问「先建什么系统」，可联想 **「航班运行与安全监管数据」** 往往是 **高优先级信息集合**（具体选项仍看 **目标→CSF→指标** 链）。  
- **SST**：**「国际化枢纽」** 战略目标 → 转化为 **信息系统的战略目标**（如 **全球航班态势可视**）。  
- **BSP**：先画 **企业过程**（**签派、离港、机务、结算**）再落 **数据类** 与 **子系统划分**。

### 2. 企业战略与数字化

- 数字化转型五阶段：**初始级、单元级、流程级、网络级、生态级**（顺序与用词以真题/当期标准为据）。  
- 信息化需求三层：**战略需求、运作需求、技术需求**。

**【平台案例】**  
- **五阶段**：从 **「本站离港单机」** 到 **「多航站一张网」** 再到 **「与伙伴系统互联」**，对应阶段名 **背真题用词**。  
- **三层次**：**战略**—联盟与枢纽定位；**运作**—**正常性、收益、服务**；**技术**—**云、接口、安全、数据平台**。

### 3. 电子政务

- 行为主体：**政府、企（事）业单位、居民**（真题亦用「公民」表述选项）。  
- 模式：**G2G、G2B、G2C、G2E**；企业面向政府为 **B2G**。  
- 干扰项：**Government to Customer** 非我国教材常用标准表述（多用 **Citizen**）。

**【平台案例】**  
- **B2G**：**航司向民航管理部门** **报送运行数据、申请航线时刻** 等。  
- **G2B**：**监管部门向企业** 下发 **规章、批复**（题干给场景再选）。  
- **G2C**：**公共航空服务信息** 类政务（按题归类，勿凭直觉硬套）。

### 4. ERP（第3章 3.7）

- **三流**：物流、资金流、**信息流**。  
- 计划链（需能辨析题干）：**经营计划** → **生产计划大纲（PP）** → **主生产计划（MPS）** → **MRP** → **CRP（能力需求计划）** 等；**库存管理**管物料进销存。  
- 企业信息化与战略：常以「战略数据模型」与战略规划衔接（见真题解析表述）。

**【平台案例】**  
- **三流**：**飞机/航材/餐食** 为 **物流**；**票款、燃油、起降** 为 **资金流**；**计划、动态、舱单** 为 **信息流**。  
- **计划层次**：题干若出现 **航季经营→航班计划→运力/资源平衡**，与 **PP→MPS→MRP→CRP** **类比理解**，**术语以教材为准**。

### 5. 集成与智能

- 集成平台：「跨库互操作、共享信息模型」偏 **信息集成**；「应用协作」偏 **应用集成**。  
- **EDI** 四模块：**通信、格式转换、联系、消息生成与处理**（以真题为准）。  
- **BI**：**数据仓库 + OLAP + 数据挖掘**；**OLAP 不算数据挖掘任务**（选题时注意）。  
- **企业集成自下而上**（真题给定顺序）：**网络集成 → 数据集成 → 会聚集成 → 服务集成 → 接口集成**。  
- **数据集成三种模式**：**数据联邦、数据复制、基于接口的数据集成**。  
- **事件驱动架构**：模块 **独立**、关系 **非耦合**。

**【平台案例】**  
- **信息集成**：各系统 **航班号、航站、时刻** **主数据一致**（**联邦/复制** 属 **数据集成** 手段）。  
- **应用集成**：**改签** 触发 **离港+行李+结算** 协同，偏 **应用/流程**。  
- **自下而上顺序**：先 **航站网络互通**，再 **主数据/接口**，再到 **服务编排**（与真题给定顺序 **严格一致**）。  
- **BI**：**数据仓库** 存 **历史航班与收益**；**OLAP** 多维分析 **航线维度**；**挖掘** 做 **延误关联、旅客细分**（**OLAP 不是挖掘任务** 类判断题常考）。

### 6. 系统工程（霍尔）

- 系统工程分析要素：结构、元素、**信息**、反馈；**最优规划**等目标。  
- 霍尔三维：**时间维、逻辑维、知识维**；时间维阶段中**研制**阶段涉及研制方案与生产计划（以教材 **2.8** 与真题为准）。

**【平台案例】**  
- **霍尔三维**：**智慧运控项目**——**时间维** 从 **规划到上线运维**；**逻辑维** **需求→方案→实施**；**知识维** **运行规章 + IT + 数据**。  
- **专家系统**（若题）：**排故规则库 + 推理机** 与 **机务知识库** 联想（仍以选项定义为准）。

## 五、平台案例一句话（速记）

- **规划三法**：CSF 找 **要害**、SST 转 **目标**、BSP 对齐 **过程与数据**（**航空**：运行与安全往往排在 **信息优先级** 前列——具体以题为准）。  
- **集成顺序**：**先修路（网络）再通粮（数据）再协同（服务/接口）**。  
- **ERP 三流**：**物、钱、信**（**航材餐食油料 / 票款结算 / 计划与动态**），计划链 **自上而下分解、自下而上平衡**。
