# 论智能运维（AIOps）技术在数据资产智能管理平台中的应用

## 1. 摘要
2024年3月，我参与了某国有航空集团数据资产智能管理平台建设。该项目面向集团总部、5大区域中心和23个分公司，服务数据治理委员会、数据管家、安全管理员及运维人员等多类角色，提供数据资产目录管理、数据质量管理、数据安全治理、血缘分析、数据共享和智能问答等核心功能。在项目中，我担任系统架构师，负责平台总体架构设计和关键技术落地。本文围绕智能运维（AIOps）在异构数据资产与大规模微服务场景中的应用展开论述，通过构建运维数据中台汇聚指标、日志与链路，实现时序异常检测与容量预测；基于日志聚类、知识图谱与智能告警收敛及分级自愈动作，大幅降低告警噪声与重复劳动；结合自动化编排与值班体系联动，显著缩短故障定位与平均恢复时间（MTTR）。系统于2025年8月正式上线，截至2026年5月已稳定运行10个月，各项功能和性能指标良好，获得客户高度认可。

## 2. 项目背景
某国有航空集团业务涵盖航空客运、旅客服务、机务维修、地面服务等领域，总部数据中心、5 大区域中心及 23 家分公司长期积累了大量分散在数据库、数据仓库中的多模态数据资产。依据民航局智慧民航数据管理相关政策标准，集团启动数据资产智能管理平台建设，亟需构建统一的数据资产目录、质量规则库、安全策略库、血缘图谱及智能知识库，实现资产盘点、共享申请、质量治理、安全管控、智能运营一体化管理。平台需适配集团统一治理、区域分级运营、分公司轻量接入架构，支撑 10 万 + 资产目录、5000 条质量规则、年 10 亿次以上数据服务调用，同时满足 5000 并发用户、简单查询 2 秒内响应、系统可用性 99.99% 以上等高并发、高一致性技术指标。
本人作为中标方系统架构师，负责平台总体技术路线设计。经分析，平台由数十个微服务组成，底层依赖 TiDB、图数据库、Kafka 及私有化大模型等复杂中间件。传统的基于固定阈值的监控系统面临严峻挑战：多源异构数据分散导致故障排查如同大海捞针；一旦某区域中心网络抖动，会瞬间触发“数据库连接超时”、“血缘解析失败”、“审批流异常”等数百条告警，造成严重的“告警轰炸”；此外，平台高度依赖外部业务系统的数据源接入，排障过度依赖运维老专家的个人经验，难以满足 7×24 小时极速恢复的 SLA 要求。我们需要将这种被动救火的局面转化为可解释、可编排、可度量的智能运维能力。
所以我们团队决定基于智能运维（AIOps）技术建设平台的稳定性与可观测性闭环。平台 2025 年 8 月顺利上线，稳定支撑总部 — 区域多级协同访问，高峰期运行平稳，全面达成建设目标。

## 3. 问题2回应 + 过渡
由于本项目面临底层组件异构、多级架构下告警噪声极高；故障排查链路长且过度依赖个人经验；月末全量资产盘点高峰期的故障恢复速度直接影响集团数据治理工作等挑战，所以我们选用智能运维（AIOps）技术作为平台性能治理和稳定性保障的重要支撑手段。其核心包括：第一，构建 AIOps 数据中台与智能分析框架，统一汇聚全链路可观测性数据并实现异常智能检测；第二，落地基于日志聚类、图谱与智能告警收敛的平台化治理，结合分级自愈动作消除告警风暴；第三，通过自动化编排与值班剧本联动，实现故障定位时间与 MTTR 的可验证下降。

在本项目实施中，我们正是通过数据中台与智能分析、聚类图谱与收敛自愈、MTTR 与定位时间下降，完成了智能运维（AIOps）体系的建设与落地，具体实践如下。

## 4. 正文部分

### 4.1 在“跨域数据资产检索与共享申请”中构建运维数据中台与智能分析框架，实现异常预测
在数据资产智能管理平台的复杂分布式架构中，“跨域数据资产检索与共享申请”是典型的长链路主流程，涉及前端门户、微服务容器（Kubernetes）、底层数据库（TiDB/图数据库）以及跨域网关等多个组件，其产生的可观测性数据格式各异、极度分散。当区域中心的数据管家反馈“资产共享申请页面加载缓慢”时，运维人员往往需要在多个监控系统（如 Prometheus、ELK、SkyWalking）之间来回切换，异常检测与根因分析无从谈起。
在架构设计上，我们首先建设了统一的运维数据中台，将全链路的 Metrics（指标）、Logs（日志）与 Traces（分布式追踪）进行清洗、对齐与融合存储。在此数据基座之上，我们引入了机器学习算法构建 AIOps 分析能力：针对“跨域检索 API 成功率”、“审批流平均耗时”等关键业务时序指标，摒弃了传统的静态阈值，采用动态基线算法（如基于孤立森林或 3-Sigma 的时序异常检测）识别隐蔽的性能劣化；在容量预测方面，利用时序预测模型分析区域网关和底层存储的历史趋势，提前 72 小时输出扩容建议。在落地实施时，我们坚持“数据质量与字段字典先行”，要求所有微服务的日志与 Trace 必须严格遵循统一的 TraceID 与 SpanID 规范注入。同时，将 AI 模型的输出与运维专家的经验规则相结合，避免纯算法导致的“黑盒结论”让运维人员不敢采纳。
这一框架的落地，使得平台监控从过去死板的“告警轰炸”转向了对业务异常的主动、可解释的捕获。平台在节假日检索洪峰到来前就能精准预判跨域网关的瓶颈，为后续的故障收敛与自愈提供了坚实的语义基础。

### 4.2 在“敏感数据脱敏策略与访问控制”中基于聚类与图谱进行智能告警收敛，消除告警风暴
“敏感数据脱敏策略与访问控制”是平台的核心安全红线。在试运行期间，曾发生过一次由于区域中心专线网络抖动导致的短网中断。在短短 5 分钟内，系统同时抛出了“脱敏规则拉取失败”、“跨域鉴权网关超时”、“脱敏服务连接池耗尽”等 200 余条相似且冗余的告警短信，导致安全与运维值班人员的手机被彻底“淹没”，根本无法第一时间判断是脱敏算法崩溃还是底层网络故障。
为解决这一安全领域的“告警风暴”痛点，我们在 AIOps 平台中引入了智能告警收敛与自愈机制。首先，采用自然语言处理（NLP）技术中的日志聚类算法，将海量的安全异常日志进行模板化提取，提炼出有限的几种故障模式；其次，构建了“数据资产运维知识图谱”，将微服务拓扑、脱敏组件、CMDB 资源与历史网络工单深度关联。当风暴发生时，智能引擎能够利用拓扑关联算法，将这 200 多条离散的安全与鉴权告警沿着调用链收敛为一个核心的“区域专线网络抖动”根因事件。此外，针对已知的高频异常，我们配置了自动化的故障自愈策略。在落地管控上，我们将自愈动作严格“分级”：对于无损的鉴权流量切换执行全自动编排，而对于涉及脱敏策略降级或底层网络重置的动作，则仅向值班主任推送“建议执行”的预案，并强制要求双人复核后方可点击执行。
通过这套机制，平台的告警可读性得到了质的飞跃。无意义的重复告警被压缩了 90% 以上，值班人员不再被噪声干扰，能够直击问题核心，极大地减少了重复排障劳动。

### 4.3 在“月末全量数据质量检测与血缘解析”中结合编排与剧本联动，显著缩短 MTTR
在数据资产平台执行“月末全量数据质量检测与血缘解析”跑批任务期间，一旦发生核心服务宕机或资源耗尽，排障与恢复的速度将直接决定全集团数据治理报表的产出效率。过去，面对因海量跑批导致跨组件的复合型故障，运维团队需要拉通网络、数据库、研发等多个部门联合排查，平均定位时间往往长达 45 分钟，整体恢复时间动辄数小时，严重影响了业务体验。
为了将 AIOps 的价值转化为实际的业务连续性保障，我们将智能根因分析与自动化恢复剧本（Playbook）进行了深度联动。当 AIOps 引擎定位到跑批导致的故障根因后，会自动匹配并推送标准的处理剧本，甚至通过 Webhook 直接触发 CI/CD 或自动化运维平台的恢复任务。例如，在一次凌晨发生的“血缘解析节点内存泄漏 OOM”故障中，AIOps 系统在监测到内存曲线异常突增后，自动触发了根因诊断，判断为图数据库写入连接未释放导致进程假死。系统随即自动调用 K8s API 执行了异常 Pod 的优雅驱逐与异地重建，并同步向钉钉值班群推送了包含完整 Trace 链路和诊断结论的事件报告。整个过程中，从故障发生到血缘解析跑批任务恢复正常，普通用户仅感知到了约 3 分钟的响应卡顿。
在效果度量上，平台稳定运行期间，月末跑批类复杂故障的平均定位时间由传统的约 45 分钟断崖式缩减至 5 分钟级别，准确率保持在 85% 以上；系统的平均恢复时间（MTTR）更是由约 2 小时大幅降至约 20 分钟。运维团队的工作模式成功从“被动救火”转变为“可度量、可复盘的工程化处置”，全面保障了平台 99.99% 的高可用性。

## 5. 总结
在国有航空集团数据资产智能管理平台建设中，我通过构建运维数据中台与智能分析框架、落地聚类图谱与告警收敛及分级自愈体系、实现 MTTR 与定位时间的可验证下降为核心，完成了智能运维（AIOps）体系的架构设计与落地。平台自2025年8月上线以来，已稳定运行10个月，成功支撑超10万项资产目录管理、5000余条质量规则执行以及年10亿次以上数据服务调用，核心热点查询响应时间稳定控制在300毫秒级，复杂故障 MTTR 降至 20 分钟内，系统整体可用性达到99.99%以上，取得了较好的建设效果。

项目复盘发现架构存在不足：一是针对极小概率的“黑天鹅”复合型故障，当前 AIOps 模型由于缺乏足够的历史正样本数据，根因分析的准确率偶尔会发生退化；二是自动化恢复剧本的覆盖率仍显不足，部分涉及跨多个外部业务系统（如 IAM、工单系统）的协同自愈动作依然依赖人工干预。后续将针对性优化：引入基于大语言模型（LLM）的运维 Agent 辅助诊断，结合海量的外部开源运维语料与专家知识库，提升模型在面临未知故障时的泛化与推理能力；同时进一步丰富自动化编排引擎的剧本库，推动标准化接口与跨域自愈协议的建设，持续提升架构的韧性与运维效率，助力该航空集团数字化高质量发展。
