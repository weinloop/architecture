# 某航空集团数据资产智能管理平台建设论文素材——大模型及 AI Coding 技术应用

## 1. 摘要

2024 年 3 月，我参与了某航空集团数据资产智能管理平台建设。该项目面向集团总部、5 大区域中心和 23 个分公司，服务数据治理委员会、数据管家及运维人员等角色，提供数据资产目录、数据质量、数据安全、血缘分析、数据共享和智能问答等核心功能。在项目中，我担任系统架构师，负责 platform 总体架构设计和落地。

本文围绕大模型及 AI Coding 技术应用展开论述：
- 基于私有化大模型与 RAG 知识工程实现数据治理问答闭环
- 通过 IDE 与 CI/CD 集成的 AI Coding 及人机协同门禁保障代码安全
- 结合领域知识图谱与分级采纳机制抑制模型幻觉并落实合规审查

系统于 2025 年 4 月正式上线，稳定运行至今，年支撑超 10 亿次数据服务调用，核心查询响应稳定在 300 毫秒量级，系统整体可用性保持在 99.99% 以上，有力保障集团数据治理业务平稳开展。

## 2. 项目背景

某航空集团业务涵盖航空客运、旅客服务、机务维修、地面服务等领域，总部数据中心、5 大区域中心及 23 家分公司长期积累了大量分散在数据库、数据仓库中的多模态数据资产。依据民航智慧民航数据管理相关政策标准，集团启动数据资产智能管理平台建设，亟需构建统一的数据资产目录、质量规则库、安全策略库、血缘图谱及智能知识库，实现资产盘点、共享申请、质量治理、安全管控、智能运营一体化管理。平台需适配集团统一治理、区域分级运营、分公司轻量接入架构，支撑 10 万+ 资产目录、5000 条质量规则、年 10 亿次以上数据服务调用，同时满足 5000 并发用户、页面查询 2 秒内响应、系统可用性 99.99% 以上等高并发、高一致性技术指标。

本人作为中标方系统架构师，负责平台总体技术路线设计。经分析，平台在智能问答与研发提效中既要释放大模型与 AI Coding 的效率价值，又要守住高敏数据不出域与安全关键代码可审计的底线。若沿用公网模型直连或插件各自为政的做法，将导致越权召回、幻觉误导与交付风险难以收敛。

在年十亿次量级的服务调用、多区域协同与强合规等总体约束下，若上述矛盾仅通过局部工具堆叠来应对，将难以保障智能化服务的合规性与研发交付的稳定性。相比通用大模型方案，私有化部署结合领域知识约束的架构更适合本项目对数据主权与代码安全的严苛要求。所以我们团队决定基于大模型及 AI Coding 技术建设该平台。平台 2025 年 4 月顺利上线，稳定支撑总部-区域多级协同访问，全面达成建设目标。

## 3. 问题 2 回应 + 过渡

大模型及 AI Coding 技术是重塑数据治理交互体验与研发效能的前沿生产力，涵盖私有化模型应用、工程化代码生成与领域可信输出等维度。

本项目涉及旅客、航班等高敏感数据资产，智能化应用必须在严密的合规边界内运行。为此，项目引入大模型及 AI Coding 技术体系作为核心智力底座：
- 依托私有化部署的 **RAG** 知识工程，结合提示工程与双侧授权机制，实现智能问答在内网环境下的闭环与安全审计。
- 基于 **IDE** 与 **CI/CD** 流水线集成的 **AI Coding**，配合人机协同门禁与双人复核，保障生成代码的可审、可回滚与可归因。
- 整合**领域知识图谱**与**规则引擎**，对模型输出实施约束解码与分级采纳，从架构层面抑制幻觉并落实安全合规审查。

在本项目实施中，我们正是通过私有化 RAG 问答、研发全链路 AI 赋能、领域可信治理，完成了智能化技术的建设与应用，具体实践如下。

## 4. 正文部分

### 4.1 基于私有化大模型与 RAG 知识工程，解决智能问答场景中的内网闭环难题

在“智能资产检索与共享辅助”业务场景中，平台存在的核心痛点为：跨域检索常关联高敏感业务上下文，用户期望通过自然语言快速理解治理规则，但传统模式下若将原始文本送入公网模型，极易导致敏感信息泄露、产生越权建议，且问答结果无法事后审计。

若持续采用“全文粘贴式”问答或通用知识库模式，极易引发数据出境风险、越权召回与幻觉误导等各类问题，进而制约智能化服务在强监管环境下的推广应用。

为破解上述难题，我们将对该场景进行私有化智能问答升级，同时在体系架构层面重点推进 RAG 知识脱敏、双侧授权访问与引用水印追溯能力建设落地。

在实际落地执行中，我们先聚焦知识治理与合规召回环节，将目录元数据分级脱敏后写入向量库，统一切片规范；再联动检索与生成双侧，接入数据域级鉴权机制，由内网推理服务完成生成，并在前端叠加引用锚点与水印，形成闭环审计证据链。

经此优化设计，自然语言查询实现秒级响应，问答全路径锁死在内网与策略边界内，试点周期内显著降低了越权解释风险，为跨域数据共享业务的高效、安全运行提供了智能化支撑。

### 4.2 通过 AI Coding 与流水线门禁策略，解决核心后台开发场景中的交付安全难题

在“跑批计算与血缘解析迭代”场景下，平台面临的核心矛盾是：此类模块逻辑复杂、迭代压力大，利用 AI 辅助编码可提升效能，但脱敏与权限等高敏感代码若未经审核直接合入，可能在海量任务中放大安全缺陷。

如果继续沿用“生成即合入”的粗放研发模式，就容易出现关键路径失守、主干代码难以回滚、责任界面模糊等问题，从而影响核心计算链路的稳定演进。

针对上述问题，我们将 AI Coding 与 CI/CD 流水线人机协同门禁引入该场景，并在架构上重点落实了分层授权生成、AI 辅助 Code Review 与安全关键清单机制。

在具体实施过程中，我们在开发侧限定 AI 产出类型，优先对单测与 Mock 服务进行模板化生成；再联动流水线，对安全关键文件清单内的变更强制触发双人复核与签核流程，并在合入时统一打标模型版本与操作者，推进研发全链路可追溯。

通过该机制的落地，试点模块研发效率提升约四成，主干代码变更保持高度可审计与可归因，未出现关键路径代码事故，验证了 AI Coding 工程化体系在处理复杂后台逻辑交付时的风险控制价值。

### 4.3 采用领域知识图谱与规则约束方案，解决敏感域应用场景中的模型幻觉难题

在“脱敏策略生成与动态授权”应用场景，平台当前亟待解决的核心问题是：大模型生成的 SQL 或安全策略草案若出现字段臆造或口径偏差，将直接冲击平台的脱敏与审批闸门，产生合规模糊地带。

若持续采取“模型直接给出结论、业务凭经验采纳”的轻治理模式，极易滋生策略建议与元数据不一致、误执行越权操作、审计证据链断裂等各类问题，直接影响平台满足合规红线的能力。

围绕上述痛点，我们将对该场景实施基于知识图谱约束与分级采纳的重塑，并在底层架构层面重点夯实规则裁决、约束解码与后校验等能力建设工作。

在具体落地推进过程中，我们先行布局术语对齐环节，将资产字段与审批动作接入领域知识图谱，从源头规避口径偏差；同步融合规则引擎强裁决，模型仅输出可被采纳的草案，不直接执行核心操作，并配合专家抽检与对抗回归测试，强化整体合规成效。

凭借该套优化设计方案，直接影响安全策略的错误采纳率压制至零事故目标，显著收敛了模型幻觉。该方案在敏感域内形成了“有限度、可负责”的 AI 使用路径，满足了监管审计对证据链完整性的预期。

## 5. 总结

在某航空集团数据资产智能管理平台建设中，我们以私有化 RAG 问答、AI Coding 流水线门禁、领域知识图谱约束为核心，完成大模型及 AI Coding 技术架构落地。平台 2025 年 4 月上线后持续稳定运行，智能化问答闭环锁内网，核心研发提效显著，年支撑超 10 亿次数据服务调用。依托大模型能力，平台交互体验与交付效能大幅提升：问答风险得到有效收敛，样板代码效率提升四成，安全策略建议实现可复核。

项目复盘发现架构存在不足：
- 一是平台架构扩展性不足，新增业务模块与第三方系统集成时，适配周期较长，难以快速响应业务迭代需求。
- 二是自动化运维能力有待完善，部分配置部署与版本更新依赖人工操作，存在人为操作风险。

后续将针对性优化：
- 推行插件化、低耦合的架构演进，标准化接口与扩展协议，缩短新功能集成落地周期。
- 同时完善全流程自动化运维体系，实现配置自动化、部署流水线化，降低人工干预成本与操作风险，持续探索提升架构灵活性与运维效率，助力该航空集团数字化高质量发展。
