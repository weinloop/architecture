# 论大模型及 AI Coding 技术在数据资产智能管理平台中的应用
2024年3月，我参与了某航空集团数据资产智能管理平台的建设。该平台面向集团总部、5大区域中心和23个分公司，服务数据治理委员会、数据管家及运维人员等角色，提供数据资产目录、数据质量、数据安全、血缘分析、数据共享和智能问答等全流程管理功能。在该项目中，我担任系统架构师，负责平台的总体架构设计和落地实施。本文围绕**大模型**及 **AI Coding** 技术应用展开论述，**基于私有化大模型与 RAG 知识工程**实现数据治理问答闭环与分域脱敏访问控制，**通过 IDE 与 CI/CD 集成的 AI Coding 及人机协同门禁**保障安全关键代码可审计交付，**结合领域知识图谱与分级采纳机制**抑制模型幻觉并落实安全合规审查；平台于 2025年4月正式上线，稳定运行至今，年支撑超 10 亿次数据服务调用，核心热点查询响应稳定在 300 毫秒量级，系统可用性保持在 99.99% 以上，获得客户高度认可。

---

## 项目背景
随着智慧民航建设战略的深入推进，航空集团数据治理相关政策标准逐步落地，集团总部数据中心、5大区域中心及23家分公司长期积累的多模态数据资产亟需规范化管理，同时对自然语言访问资产目录、质量规则与血缘影响研判，以及研发侧在高压迭代下保质交付提出了更高要求。在此背景下，某航空集团于2024年3月启动数据资产智能管理平台的建设，旨在构建覆盖集团全层级的数字化管理平台，实现资产盘点、共享申请、质量治理、安全管控、智能运营一体化管理，支撑10万+资产目录、5000条质量规则、年10亿次以上数据服务调用，同时满足5000并发用户、页面查询2秒内响应、系统可用性99.99%以上的高要求。

我司中标后，我作为系统架构师负责平台的总体架构设计和落地实施，因为平台在智能问答与研发提效中既要释放**大模型**与 **AI Coding** 的效率价值，又要守住高敏数据不出域与安全关键代码可审计的底线，若沿用公网模型直连、插件各自为政与“生成即合入”的做法，将导致越权召回、幻觉误导与交付风险难以收敛，工程边界不可回滚、不可签核，所以我们团队决定**基于大模型及 AI Coding 技术建设该平台**。平台于 2025年4月上线，成功应对了跨域共享高峰、月末质量跑批与血缘解析迭代等业务场景，为集团数据治理业务平稳开展提供了有力支撑，得到了用户和客户的一致好评。

## 问题2回应
在该平台的建设落地中，我们重点建设了**大模型及 AI Coding 技术体系**，主要包括**私有化大模型与 RAG 知识工程及分域脱敏访问控制**、**AI Coding 与流水线人机协同门禁**、**领域知识图谱与分级采纳可信输出** 3 个部分。基于私有化部署与可编排的 **RAG**，依托**提示工程**、租户与数据域级检索前后双侧授权及引用水印与追溯，将智能问答与申请辅助锁在内网并统一审计口径，降低越权与出境风险；基于 **IDE** 与 **CI/CD** 集成的 **AI Coding**，结合**静态检查**、**AI 辅助 Code Review** 与安全关键文件清单联动，对高风险模块强制双人复核与签核，保障生成物可审、可回滚、可归因；基于**领域知识图谱**与**规则引擎**对模型输出进行约束解码与后校验，实行**分级采纳**、极端样例回归与专家抽检，以“**规则裁决、模型建议**”的方式抑制幻觉并落实安全合规审查。

得益于大模型及 AI Coding 技术体系的建设，平台上线运行后，成功应对了智能问答内网闭环、核心后台研发提效与敏感域合规模糊地带收敛等业务与工程场景，下面我将重点论述该技术体系在该平台的建设实施过程。

## 正文
**基于私有化大模型与 RAG 知识工程及分域脱敏访问控制**，解决智能问答与共享申请场景中的**内网闭环与合规边界**问题，提升了自然语言访问数据资产的可解释性与可审计性。在数据资产智能管理平台的建设中，跨域数据资产检索与共享申请常与旅客、航班、机组及维修等高敏上下文相关联，业务侧希望通过自然语言理解目录口径、质量规则与血缘影响；若将原始文本无差别送入通用公网模型，或采用公共知识库与“全文粘贴式”问答，极易在召回与回答环节泄露可识别细节、产生越权建议，并带来数据出境、答案不可审计等问题，难以在强监管环境下推广智能服务。

于是我们引入**私有化大模型**与 **RAG** 知识工程能力，首先在知识治理阶段将制度、目录元数据等按敏感级别脱敏或抽象化后写入**向量库**，统一**向量化**与切片规范；其次在检索与生成双侧接入租户与数据域维度的统一认证与访问校验，由**内网推理服务**完成生成，阻断未授权域知识进入上下文；同时在前端与回答链路叠加**引用锚点**与**可解释水印**，支撑事后追溯与合规抽检。得益于该体系的建设，自然语言查询与共享前检查可达毫秒至秒级响应，问答路径闭环在集团内网与策略边界内，在试点周期内显著降低隐性协作与越权解释风险，为跨域数据共享业务的高效、安全运行提供了保障。

**通过 AI Coding 与流水线人机协同门禁**，解决月末全量数据质量检测与血缘解析等核心后台模块的**研发提效与关键代码可审计**问题，保障了核心计算链路的稳定演进。该类跑批与解析链路逻辑重、需求变更多，借助 AI 可压缩样板与用例耗时，但脱敏、权限与血缘计算属于高敏感区，任一生成物若绕过评审进入主干，都可能在长周期大批量任务中被放大；若停留在“全靠手写”与“生成物直接合入”的两极，则效率瓶颈与审核失守并存，主干难以验证与回滚。

经过调研，我们决定将 **AI Coding** 与 **CI/CD**、**代码评审**联动纳入工程规范：首先在开发侧集成通过安全评估的企业级辅助能力，限定仓库范围与产出类型，优先对**接口骨架**、**单测**、**Mock** 与说明文档类产出进行模板化生成；其次在流水线中串联静态扫描与 AI 辅助 Code Review，对**安全关键文件清单**内变更强制双人复核与签核，合入统一打标操作者与模型版本；同时在试点模块建立分层授权与合入可追溯机制，异常时可快速阻断与回滚。得益于该机制的落地，试点模块样板类效率较传统方式提升约四成，主干变更保持可审、可回滚、可归因，未出现关键路径失守类事故，验证了 **AI Coding** 工程化体系在复杂后台链路中的风险控制价值。

**采用领域知识图谱与规则约束、分级采纳及专家抽检**，解决敏感数据脱敏策略、动态授权与安全审计场景中的**模型幻觉与合规审查**问题，满足平台安全红线与监管预期。大模型可加速 **SQL** 与策略草案产出，但若出现字段臆造、口径偏差或越权路径，将直接冲击脱敏与审批两大合规闸门；若采取“模型直接给结论、业务凭经验采纳”的轻治理方式，易产生建议与元数据不一致、误执行脱敏建议、审计证据链断裂等问题。

针对以上痛点，我们落实“**图谱约束 + 规则裁决 + 分级采纳**”的一体化方案：首先将资产字段、域关系与审批动作接入**领域知识图谱**，在主数据与术语层面对齐约束；其次在生成侧采用**约束解码**与后校验，由**规则引擎**对脱敏、共享与导出动作强裁决，模型仅输出可被拒绝或采纳的草案；同时将专家抽检、对抗样例与回归集纳入发布前门禁，并对**提示版本**实行灰度与回滚。得益于该方案的建设，试点周期内将直接影响安全策略的错误采纳压降至零事故目标，显著收敛幻觉带来的合规模糊地带，在敏感域内形成“有限度、可负责”使用大模型的可复核路径，满足监管审计要求。

## 3. 总结
在本平台的建设落地中，通过构建完整的**大模型及 AI Coding 技术体系**，我们成功将数据资产智能管理平台打造成了在内网可治理、在研发链可签核、在敏感域可复核的智能化平台，平台于 2025年4月上线以来，成功应对了智能问答高峰、核心跑批与血缘解析迭代、敏感策略协同等业务与工程场景，保障了年 10 亿次以上数据服务调用的稳定运行，有效支撑了集团数据资产的全生命周期管理，受到了用户和客户的一致认可。

在项目的复盘会上，我们也发现了不足之处，比如平台架构扩展性不足，新增业务模块与第三方系统集成时适配周期较长，难以快速响应业务迭代需求；同时自动化运维能力有待完善，部分配置部署与版本更新依赖人工操作，存在人为操作风险。后续，我们拟推行插件化、低耦合的架构演进，标准化接口与扩展协议，缩短新功能集成落地周期；同时完善全流程自动化运维体系，实现配置自动化、部署流水线化，降低人工干预成本与操作风险，持续提升架构灵活性与运维效率，助力航空集团数字化高质量发展。
