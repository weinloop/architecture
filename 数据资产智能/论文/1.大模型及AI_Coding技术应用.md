# 论大模型及 AI Coding 技术在数据资产智能管理平台中的应用
2024年3月，我参与了某航空集团数据资产智能管理平台的建设。该平台面向集团总部、5大区域中心和23个分公司，服务数据治理委员会、数据管家及运维人员等角色，提供数据资产目录、数据质量、数据安全、血缘分析、数据共享和智能问答等全流程管理功能。在该项目中，我担任系统架构师，负责平台的总体架构设计和落地实施。本文围绕**大模型**及 **AI Coding** 技术应用展开论述，**基于私有化大模型与 RAG 知识工程**实现数据治理问答闭环与分域脱敏访问控制，**通过 IDE 与 CI/CD 集成的 AI Coding 及人机协同门禁**保障安全关键代码可审计交付，**结合领域知识图谱与分级采纳机制**抑制模型幻觉并落实安全合规审查；平台于 2025年4月正式上线，稳定运行至今，年支撑超 10 亿次数据服务调用，核心热点查询响应稳定在 300 毫秒量级，系统可用性保持在 99.99% 以上，获得客户高度认可。

---

## 项目背景
随着智慧民航建设战略的深入推进，航空集团数据治理相关政策标准逐步落地，集团总部数据中心、5大区域中心及23家分公司长期积累的多模态数据资产亟需规范化管理，同时对自然语言访问资产目录、质量规则与血缘影响研判，以及研发侧在高压迭代下保质交付提出了更高要求。在此背景下，某航空集团于2024年3月启动数据资产智能管理平台的建设，旨在构建覆盖集团全层级的数字化管理平台，实现资产盘点、共享申请、质量治理、安全管控、智能运营一体化管理，支撑10万+资产目录、5000条质量规则、年10亿次以上数据服务调用，同时满足5000并发用户、页面查询2秒内响应、系统可用性99.99%以上的高要求。

我司中标后，我作为系统架构师负责平台的总体架构设计和落地实施，因为平台在智能问答与研发提效中既要释放**大模型**与 **AI Coding** 的效率价值，又要守住高敏数据不出域与安全关键代码可审计的底线，若沿用公网模型直连、插件各自为政与“生成即合入”的做法，将导致越权召回、幻觉误导与交付风险难以收敛，工程边界不可回滚、不可签核，所以我们团队决定**基于大模型及 AI Coding 技术建设该平台**。平台于 2025年4月上线，成功应对了跨域共享高峰、月末质量跑批与血缘解析迭代等业务场景，为集团数据治理业务平稳开展提供了有力支撑，得到了用户和客户的一致好评。

得益于大模型及 AI Coding 技术体系的建设，平台上线运行后，成功应对了智能问答内网闭环、核心后台研发提效与敏感域合规模糊地带收敛等业务与工程场景，下面我将重点论述该技术体系在该平台的建设实施过程。

## 正文
**基于私有化大模型与RAG知识工程及分域脱敏访问控制，针对性解决智能问答与共享申请场景中的内网闭环与合规边界问题，提升自然语言访问数据资产的可解释性与可审计性。** 在数据资产智能管理平台建设过程中，跨域数据资产检索与共享申请场景常关联旅客、航班、机组及维修等高敏感上下文，业务侧需求通过自然语言快速理解目录口径、质量规则与血缘影响。传统模式下，若将原始文本无差别送入通用公网模型，或采用公共知识库与“全文粘贴式”问答方式，极易在知识召回与回答生成环节泄露敏感可识别细节、产生越权操作建议，同时存在数据出境、问答结果不可审计等风险，无法在强监管环境下推广智能服务。
针对上述痛点，引入私有化大模型与RAG知识工程能力，构建合规智能问答体系：
- 一是在知识治理阶段，将业务制度、目录元数据等按敏感级别完成脱敏或抽象化处理后写入向量库，统一规范向量化与文本切片标准，保障知识安全；
- 二是在检索与生成双侧接入租户与数据域维度的统一认证与访问校验机制，由内网推理服务完成回答生成，严格阻断未授权域知识进入上下文，确保访问合规；
- 三是在前端展示与回答生成链路中，叠加引用锚点与可解释水印，支撑事后追溯与合规抽检，完善审计证据链。

通过该体系的落地，自然语言查询与共享前检查可实现毫秒至秒级响应，问答全路径闭环在集团内网与策略边界内，试点周期内显著降低隐性协作与越权解释风险，为跨域数据共享业务的高效、安全运行提供有力支撑。

**采用AI Coding与流水线人机协同门禁，解决月末全量数据质量检测与血缘解析等核心后台模块的研发提效与关键代码可审计问题，保障核心计算链路的稳定演进。** 此类跑批与解析链路逻辑复杂、需求变更频繁，借助AI辅助编码可有效压缩样板代码与测试用例编写耗时，但脱敏处理、权限控制与血缘计算属于高敏感区域，任一生成代码若绕过评审直接进入主干分支，都可能在长周期、大批量任务中放大缺陷；传统模式若陷入“全靠手写”与“生成物直接合入”的两极，将同时存在研发效率瓶颈与审核失守风险，导致主干代码难以验证与回滚。
针对上述痛点，将AI Coding与CI/CD流水线、代码评审流程联动，纳入工程化规范：
- 一是在开发侧集成通过安全评估的企业级AI辅助编码能力，限定代码仓库范围与产出类型，优先对接口骨架、单元测试、Mock服务与说明文档等非核心敏感类产出进行模板化生成，降低风险；
- 二是在CI/CD流水线中串联静态代码扫描与AI辅助Code Review环节，对安全关键文件清单内的代码变更，强制执行双人复核与签核流程，合入代码时统一打标操作者与模型版本，确保可追溯；
- 三是在试点模块建立分层授权与合入可追溯机制，一旦发现异常代码，可快速阻断合入并完成版本回滚。

通过该机制的落地，试点模块样板类代码研发效率较传统方式提升约四成，主干代码变更保持可审计、可回滚、可归因，未出现关键路径失守类事故，验证了AI Coding工程化体系在复杂后台链路中的风险控制价值。

**采用领域知识图谱与规则约束、分级采纳及专家抽检，解决敏感数据脱敏策略、动态授权与安全审计场景中的模型幻觉与合规审查问题，满足平台安全红线与监管预期。** 大模型可快速加速SQL语句与安全策略草案的产出，但若出现字段臆造、口径偏差或越权路径建议，将直接冲击脱敏处理与审批流程两大合规闸门；传统“模型直接给出结论、业务凭经验采纳”的轻治理方式，易产生建议与元数据不一致、误执行脱敏建议、审计证据链断裂等问题，无法满足合规要求。
针对以上痛点，落实“图谱约束+规则裁决+分级采纳”一体化解决方案：
- 一是将资产字段、域关系与审批动作接入领域知识图谱，在主数据与术语层面实现统一对齐与约束，从源头规避口径偏差；
- 二是在生成侧采用约束解码与后校验机制，由规则引擎对脱敏处理、数据共享与导出等动作进行强裁决，模型仅输出可被拒绝或采纳的草案，不直接执行核心操作；
- 三是将专家抽检、对抗样例与回归测试集纳入发布前门禁，对提示词版本实行灰度发布与回滚机制，确保模型输出合规。

通过该方案的建设，试点周期内将直接影响安全策略的错误采纳率降至零事故目标，显著收敛模型幻觉带来的合规模糊地带，在敏感域内形成“有限度、可负责”使用大模型的可复核路径，满足监管审计要求。

## 3. 总结
在本平台的建设落地中，通过构建完整的**大模型及 AI Coding 技术体系**，我们成功将数据资产智能管理平台打造成了在内网可治理、在研发链可签核、在敏感域可复核的智能化平台，平台于 2025年4月上线以来，成功应对了智能问答高峰、核心跑批与血缘解析迭代、敏感策略协同等业务与工程场景，保障了年 10 亿次以上数据服务调用的稳定运行，有效支撑了集团数据资产的全生命周期管理，受到了用户和客户的一致认可。

在项目的复盘会上，我们也发现了不足之处，比如平台架构扩展性不足，新增业务模块与第三方系统集成时适配周期较长，难以快速响应业务迭代需求；同时自动化运维能力有待完善，部分配置部署与版本更新依赖人工操作，存在人为操作风险。后续，我们拟推行插件化、低耦合的架构演进，标准化接口与扩展协议，缩短新功能集成落地周期；同时完善全流程自动化运维体系，实现配置自动化、部署流水线化，降低人工干预成本与操作风险，持续提升架构灵活性与运维效率，助力航空集团数字化高质量发展。

## 正文归纳（论点—痛点现状—问题冲突—解决方案—落地实施—成果）

下文依据正文三节论述，按六项步骤抽取要点，便于答辩对照与经验复盘。

### 专题一：私有化大模型与 RAG 知识工程及分域脱敏访问控制

1. **论点**：以内网私有化推理与 **RAG** 知识工程为载体，把智能问答与共享前辅助收敛到可审计、可分域授权的策略边界内。
2. **痛点现状**：跨域检索与共享常与旅客、航班、机组及维修等高敏上下文相关，业务希望通过自然语言理解目录口径、质量规则与血缘影响。
3. **问题冲突**：公网通用模型或“全文粘贴式”问答易导致越权召回、可识别细节泄露、数据出境及答案溯源困难，难以满足强监管推广条件。
4. **解决方案**：制度与目录元数据分级脱敏或抽象后入向量库；检索与生成双侧统一认证与数据域校验；回答链路叠加引用锚点与可解释水印。
5. **落地实施**：统一向量化与切片规范；由内网推理服务完成生成并阻断未授权域知识进入上下文；前端展示可追溯字段支撑抽检。
6. **成果**：自然语言查询与共享前检查可达毫秒至秒级响应；问答闭环限于集团内网与策略边界；试点周期显著降低隐性协作与越权解释风险。

### 专题二：AI Coding 与流水线人机协同门禁

1. **论点**：将 **AI Coding** 纳入 **CI/CD** 与强制评审门禁，使样板效率提升与安全关键代码可签核、可回滚并行不悖。
2. **痛点现状**：月末质量跑批与血缘解析等后台模块逻辑重、变更频，纯手工交付瓶颈突出；生成物若直入主干在长周期大批量任务中易被放大。
3. **问题冲突**：“全靠手写”拖慢迭代；“生成即合入”又可能失守脱敏、权限与血缘等高危区，主干难以验证与回滚。
4. **解决方案**：限定仓库与产出类型；流水线串联静态扫描与 AI 辅助 Code Review；安全关键清单变更双人复核与签核。
5. **落地实施**：优先对接口骨架、单测、Mock 与文档类产出模板化生成；合入统一标注操作者与模型版本；试点模块分层授权与快速阻断回滚。
6. **成果**：试点模块样板类效率较传统方式提升约四成；主干变更保持可审、可回滚、可归因；关键路径未出现失守类事故。

### 专题三：领域知识图谱与规则约束、分级采纳及专家抽检

1. **论点**：以“图谱约束 + 规则裁决 + 分级采纳”抑制幻觉，使大模型在敏感域仅输出可被拒绝或采纳的草案。
2. **痛点现状**：模型可加速 SQL 与策略草案产出，但字段臆造、口径偏差会直接冲击脱敏与审批闸门。
3. **问题冲突**：若模型结论不经裁决直接执行，易出现建议与元数据不一致、误采纳脱敏策略、审计证据链断裂。
4. **解决方案**：字段域关系与审批动作接入领域知识图谱对齐；约束解码与规则引擎强裁决；专家抽检与对抗回归纳入发布门禁。
5. **落地实施**：生成侧后校验；提示版本灰度与回滚；抽检与极端样例回归闭环。
6. **成果**：试点周期直接影响安全策略的错误采纳压降至零事故目标；敏感域形成有限度、可负责、可复核的大模型使用路径，满足监管审计。
