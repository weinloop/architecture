# 大模型及 AI Coding 技术应用

## 1. 摘要

本文围绕大模型及 AI Coding 技术应用展开论述：
- 基于私有化大模型与 RAG 知识工程 实现数据治理问答闭环
- 通过 IDE 与 CI/CD 集成的 AI Coding 及人机协同门禁 保障代码安全
- 结合领域知识图谱与分级采纳机制 抑制模型幻觉并落实合规审查

## 2. 项目背景

经分析，平台在**智能问答与研发提效中**既要**释放大模型与 AI Coding 的效率价值**，又要守住**高敏数据不出域**与**安全关键代码可审计的底线**。若沿用**公网模型**直连或**插件各自为政**的做法，将导致**越权召回**、**幻觉误导与交付风险**难以收敛。

相比**通用大模型**方案，私有化部署结合**领域知识约束**的架构更适合本项目对**数据主权与代码安全**的严苛要求。所以我们团队决定基于**大模型及 AI Coding 技术**建设该平台。

## 3. 问题 2 回应 + 过渡

大模型及 AI Coding 技术是重塑数据治理交互体验与研发效能的前沿生产力，涵盖私有化模型应用、工程化代码生成与领域可信输出等维度。

本项目涉及旅客、航班等高敏感数据资产，智能化应用必须在严密的合规边界内运行。为此，项目引入大模型及 AI Coding 技术体系作为核心智力底座：
- 依托私有化部署的 **RAG** 知识工程，结合提示工程与双侧授权机制，实现智能问答在内网环境下的闭环与安全审计。
- 基于 **IDE** 与 **CI/CD** 流水线集成的 **AI Coding**，配合人机协同门禁与双人复核，保障生成代码的可审、可回滚与可归因。
- 整合**领域知识图谱**与**规则引擎**，对模型输出实施约束解码与分级采纳，从架构层面抑制幻觉并落实安全合规审查。

在本项目实施中，我们正是通过私有化 RAG 问答、研发全链路 AI 赋能、领域可信治理，完成了智能化技术的建设与应用，具体实践如下。

## 4. 正文部分

### 1. 私有化大模型与RAG知识工程（内网闭环）
- 场景：**智能资产检索**
- 问题：直连公网大模型→敏感数据泄露、越权建议、无审计留痕→不合规、无法规模化落地
- 方案：**升级私有化智能问答架构，私有化大模型+RAG知识工程**
- 落地：核心知识分级脱敏切片存入内网向量库；IAM鉴权、内网推理；前端锚点+水印全程追溯
- 效果：查询秒级响应、内网闭环、无合规事故，支撑敏感域智能化落地
---
### 2. AI Coding与流水线门禁（交付安全）
- 场景：**核心后台研发迭代**
- 问题：AI Coding生成即合入、无管控→敏感代码失守、无法回滚，难平衡效能与安全
- 方案：**AI Coding+人机协同门禁，分层授权+AI审核+安全清单管控**
- 落地：限定AI生成范围；CI/CD流水线安全清单、双人复核、合入打标追溯
- 效果：研发效率提升40%、主干代码可审计、无安全事故，支撑模块快速安全迭代
---
### 3. 领域知识图谱与规则约束（模型幻觉）
- 场景：**敏感域大模型生成操作建议**
- 问题：模型幻觉→策略偏差、越权风险，缺乏管控、不合规
- 方案：**领域知识图谱约束+分级采纳，规则裁决+约束解码+后校验**
- 落地：术语规范、知识图谱接入；规则引擎限定输出；专家抽检校验合规性
- 效果：策略错误采纳率为零、幻觉收敛、口径统一、审计可追溯，提升平台可靠性
