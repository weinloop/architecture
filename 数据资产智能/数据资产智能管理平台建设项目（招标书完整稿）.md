## 第二章 投标人须知

### 一、总则

#### 1. 项目概况

**1.1 项目名称**：国有航空集团有限公司数据资产智能管理平台建设项目

**1.2 项目背景**：

国有航空集团有限公司是国内三大航空运输集团之一，拥有员工8万余人，运营飞机超过800架，年旅客运输量超过1.2亿人次。集团下辖五大区域管理中心（华北、华东、华南、华西、华中）及23家分公司，业务涵盖航空客运、货运物流、航空食品、飞机维修、地面服务等多个领域。

随着集团数字化转型深入推进，数据已成为核心战略资产。在民航局《智慧民航建设数据管理政策标准体系》等行业政策指引下，明确航空数据治理、共享与智能化管理的建设要求。为全面提升数据资产管理能力，实现数据资产的"可见、可用、可管、可控"，集团决定建设覆盖全集团的数据资产智能管理平台，构建统一的数据资产管理体系，支撑业务创新和管理决策。

**1.3 建设目标**：

（1）**统一数据资产目录**：建立覆盖全集团的数据资产统一目录，实现数据资产的全面盘点和分类管理；

（2）**数据资产全生命周期管理**：实现数据资产从产生、采集、存储、加工、使用到销毁的全生命周期管理；

（3）**数据质量智能管控**：建立数据质量规则库，实现数据质量的自动检测、预警和改进；

（4）**数据安全分级保护**：建立数据安全分级分类体系，实现敏感数据的自动识别和保护；

（5）**数据价值评估与运营**：建立数据资产评估模型，支撑数据资产的价值评估和运营变现；

（6）**区域协同与共享**：实现五大区域中心、23个分公司的数据资产协同管理和安全共享；

（7）**智能能力深度赋能**：基于集团自建大模型，实现数据资产管理全流程智能化辅助；智能问答准确率≥90%，智能分析响应时间≤3秒，智能推荐匹配度≥85%；构建民航行业数据资产专属智能知识库，覆盖核心业务场景智能支撑。

## 第五章 技术规格书

### 一、项目总体要求

#### 1.1 建设背景

星空航空集团经过多年信息化建设，已积累海量业务数据，涵盖航班运行、旅客服务、机务维修、货运物流、财务人力等各个领域。但当前数据管理存在以下问题：

- **数据孤岛严重**：各业务系统独立建设，数据标准不统一，难以共享；
- **数据质量不高**：缺乏统一的数据质量管理机制，数据准确性、完整性、一致性难以保障；
- **数据资产不清**：数据资产家底不清，价值难以评估；
- **安全风险突出**：敏感数据识别和保护能力不足，合规风险较高；
- **区域协同困难**：五大区域中心、23个分公司数据管理各自为政，难以形成集团级数据能力。

#### 1.2 建设目标

通过本项目建设，实现以下目标：

**（1）数据资产全面盘点**
- 建立覆盖全集团的数据资产目录，盘点数据资产不少于**10万项**；
- 实现数据资产的自动发现和分类分级。

**（2）数据质量显著提升**
- 建立覆盖核心业务的数据质量规则库，规则不少于**5000条**；
- 数据质量合格率提升至**98%以上**。

**（3）数据安全有效保障**
- 实现敏感数据自动识别，识别准确率不低于**95%**（经智能增强后敏感数据智能识别准确率≥**98%**）；
- 建立数据安全分级分类体系，覆盖全部数据资产。

**（4）数据价值充分释放**
- 支撑数据共享交换，年数据服务调用量不低于**10亿次**；
- 支撑数据产品开发，年数据产品化收入不低于**5000万元**。

**（5）区域协同高效运转**
- 实现五大区域中心数据资产统一管理；
- 实现23个分公司数据资产按需接入。

**（6）智能能力深度赋能**
- 基于集团自建大模型，实现数据资产管理全流程智能化辅助；
- 智能问答准确率≥**90%**，智能分析响应时间≤**3秒**，智能推荐匹配度≥**85%**；
- 构建民航行业数据资产专属智能知识库，覆盖核心业务场景智能支撑。

```
│                    数据资产应用层                         │
│    （资产盘点、质量管控、安全保护、价值评估、运营监控）        │
├─────────────────────────────────────────────────────────┤
│                    数据资产能力层                         │
│    （元数据采集、血缘分析、影响分析、智能推荐、知识图谱）      │
├─────────────────────────────────────────────────────────┤
│                  大模型集成层（集团自建大模型）               │
│     （代理适配、推理调度、知识库、语义解析与场景编排）          │
├─────────────────────────────────────────────────────────┤
│                    数据中台基础层                         │
│    （数据集成、数据开发、数据治理、数据服务、数据存储）        │
├─────────────────────────────────────────────────────────┤
│                    基础设施层                             │
│    （计算、存储、网络、安全资源、容器编排、推理算力池）         │
└─────────────────────────────────────────────────────────┘
```

### 三、功能需求

#### 3.1 数据资产目录管理

**3.1.1 资产自动发现**
- 支持自动扫描数据库、数据仓库、大数据平台、文件系统等数据源；
- 支持自动识别表、字段、视图、存储过程等数据对象；
- 支持元数据自动采集，采集频率可配置（实时、小时、天）。

**3.1.2 资产分类分级**
- 支持按业务主题（航班、旅客、机务、货运等）分类；
- 支持按数据类型（结构化、半结构化、非结构化）分类；
- 支持按敏感级别（公开、内部、敏感、机密）分级；
- 支持自定义分类分级标准。

**3.1.3 资产目录服务**
- 提供统一的数据资产搜索门户，支持全文检索；
- 支持数据资产详情查看，包括元数据、业务属性、技术属性、管理属性；
- 支持数据资产申请、审批、授权流程；
- 支持数据资产订阅和变更通知。

#### 3.2 数据质量管理

**3.2.1 质量规则管理**
- 支持空值检查、重复检查、格式检查、范围检查、一致性检查等规则类型；
- 支持自定义SQL规则、正则表达式规则；
- 支持规则模板库和规则复用。

**3.2.2 质量检测执行**
- 支持实时检测和批量检测；
- 支持检测任务调度（定时、事件触发）；
- 支持检测结果的自动评分和评级。

**3.2.3 质量问题处理**
- 支持质量问题自动告警（短信、邮件、钉钉、企业微信）；
- 支持质量问题工单流转；
- 支持质量问题趋势分析和根因分析。

#### 3.3 数据安全管理

**3.3.1 敏感数据识别**
- 支持正则匹配、关键字匹配、机器学习等多种识别方式；
- 内置民航业敏感数据识别模型（旅客身份信息、航班动态信息、机组信息等）；
- 支持识别结果的人工确认和修正；
- 基于集团自建大模型增强模糊语义、上下文关联的敏感数据识别，并支持识别规则的智能生成与迭代优化。

**3.3.2 数据脱敏**
- 支持静态脱敏和动态脱敏；
- 支持替换、掩码、加密、泛化等多种脱敏算法；
- 支持脱敏策略的细粒度配置（按用户、按角色、按场景）。

**3.3.3 数据访问控制**
- 支持基于角色的访问控制（RBAC）；
- 支持基于属性的访问控制（ABAC）；
- 支持数据访问审计，记录完整访问日志。

#### 3.4 数据血缘与影响分析

**3.4.1 血缘自动采集**
- 支持SQL解析自动采集血缘关系；
- 支持ETL工具（DataWorks、Kettle、Informatica等）血缘采集；
- 支持手工维护血缘关系。

**3.4.2 血缘图谱展示**
- 支持全链路血缘图谱展示；
- 支持血缘关系的向上追溯和向下追踪；
- 支持血缘影响范围分析；
- 支持基于大模型的跨系统、跨层级血缘智能补全与校验，以及变更影响智能评估与风险提示。

#### 3.5 数据价值评估

**3.5.1 价值评估模型**
- 支持数据成本法、收益法、市场法等多种评估方法；
- 支持数据质量、数据应用、数据稀缺性等维度综合评估；
- 支持评估模型的自定义配置。

**3.5.2 价值运营看板**
- 提供数据资产价值总览看板；
- 支持按部门、按业务域、按数据类型的价值分析；
- 支持数据资产价值趋势分析。

#### 3.6 智能能力模块

##### 3.6.1 智能问答与助手
- 基于集团自建大模型，构建数据资产管理专属智能助手，支持自然语言交互；
- 支持数据资产查询、质量问题咨询、安全规则解读、血缘关系分析等场景的智能问答；
- 支持多轮对话、上下文理解，可生成结构化查询结果与可视化图表；
- 支持私有化部署，所有交互数据不对外泄露，符合民航数据安全规范。

##### 3.6.2 智能数据质量分析与预警
- 基于集团大模型的语义理解能力，自动识别数据质量问题的根因，生成整改建议；
- 支持异常数据趋势的智能预警，结合业务场景预判潜在质量风险；
- 自动生成数据质量分析报告，支持自定义报告模板与推送频率。

##### 3.6.3 智能数据资产推荐与运营
- 基于集团大模型的用户行为分析与业务语义理解，为不同角色（业务人员、数据分析师、管理人员）智能推荐高价值数据资产；
- 智能生成数据资产运营建议，包括数据产品化方向、共享策略优化、价值提升路径等；
- 支持基于大模型的非结构化数据资产（如机务维修文档、航班运行报告）的智能解析与结构化提取，纳入资产目录管理。

##### 3.6.4 智能敏感数据识别增强
- 基于集团自建大模型，优化敏感数据识别能力，对模糊语义、上下文关联的敏感数据（如隐式旅客隐私、机组涉密信息）实现精准识别；
- 支持敏感数据识别规则的智能生成与迭代优化，降低人工规则配置成本。

##### 3.6.5 智能血缘与影响分析增强
- 基于大模型的语义推理能力，实现跨系统、跨层级的血缘关系智能补全与校验；
- 智能分析数据变更的潜在影响范围，生成影响评估报告与风险提示。

### 六、技术方案

本章依据第五章《技术规格书》编制，面向国有航空集团“总部—五大区域中心—23个分公司”的组织与网络格局，给出可落地、可验收、可运维的一体化建设方案（含集团自建大模型与智能能力）。

---

#### 6.1 总体架构设计

**6.1.1 分层架构与职责边界**

平台采用**云原生微服务**总体架构，自上而下划分为多层，各层通过标准 API、事件总线与统一身份认证解耦，满足弹性伸缩与独立演进：

| 层级 | 核心职责 | 关键组件（示例） |
|------|-----------|------------------|
| 数据资产门户层 | 统一入口、目录检索、数据地图、服务目录与自助化申请 + 智能助手交互入口 | Web 门户、移动端、API 网关、BFF、智能问答前端组件 |
| 数据资产应用层 | 盘点、质量、安全、价值评估、运营监控与流程编排 + 智能能力应用封装 | 资产中心、质量中心、安全中心、价值中心、运营中心、智能助手服务 |
| 数据资产能力层 | 元数据采集、血缘/影响分析、智能推荐、知识图谱 + 大模型能力适配 | 元数据引擎、血缘服务、图谱服务、推荐引擎、大模型适配网关 |
| 大模型集成层 | 集团大模型调用、推理调度、知识库管理、语义解析 | 大模型代理服务、推理资源池、知识库引擎、语义理解组件 |
| 数据中台基础层 | 集成、开发、治理、服务化与存储 | 数据集成、批流计算、数据开发、数据服务、湖仓一体 |
| 数据与消息平面 | 跨域复制、主题域模型、数据产品封装 | Kafka/Pulsar、对象存储、数据湖、数仓 |
| 基础设施层 | 计算、存储、网络、安全资源与容器编排 + 大模型推理资源 | Kubernetes、服务网格、日志/监控、密钥管理、GPU/CPU推理节点池 |

**6.1.2 技术路线选型原则**

- **微服务**：Spring Cloud Alibaba（或同等能力）实现注册发现、配置中心、限流熔断、分布式事务协调；核心高吞吐链路可采用 Go 补充。
- **容器与编排**：Kubernetes 为唯一编排基座；支持鲲鹏/飞腾等国产 CPU 与麒麟/统信 OS 的镜像构建与调度亲和性配置。
- **数据存储**：关系型以 MySQL/PostgreSQL/OceanBase/TiDB 为主；检索以 Elasticsearch 为主；缓存 Redis；消息 RocketMQ/Kafka。
- **大数据**：Hadoop 生态（HDFS/YARN 或等价）、Spark（批）、Flink（流）、与现有数仓/湖仓体系对接。
- **前端**：Vue 3 或 React 18，组件化设计；移动端支持 iOS、Android、鸿蒙（UniApp/原生混合方案二选一，以集团统一移动规范为准）。
- **大模型集成**：基于集团自建大模型的标准化接口（RESTful/gRPC）开发适配层，支持模型推理请求的负载均衡、熔断降级；推理资源池采用 GPU/CPU 混合部署，核心场景（智能问答、敏感数据识别）优先调度 GPU 资源，非核心场景（批量报告生成）调度 CPU 资源。
- **知识库管理**：采用向量数据库（如 Milvus/Chroma）存储数据资产专属知识向量，与集团大模型联动实现语义检索与上下文增强。
- **隐私与安全**：大模型推理输入输出数据经国密算法加密，推理过程采用隔离容器运行，禁止数据落地，推理日志接入集团安全审计平台。

**6.1.3 多活与容灾策略**

- **同城双活 + 异地灾备**：总部数据中心为主生产；同城备中心实时同步关键元数据、配置与审计日志；异地冷/温备满足 RPO/RTO 指标并与等保三级审计要求一致。
- **服务级 SLA**：门户与目录检索、数据服务网关纳入最高可用域；批处理与血缘重算可降级为异步任务队列，保障核心在线能力。

**6.1.4 集成边界**

与现有 IAM（4A/统一认证）、工单系统、消息平台（短信/邮件/钉钉/企业微信）、日志平台、CMDB、DevOps 流水线对接；对外数据服务统一经 API 网关暴露，内网服务经零信任策略访问。

#### 6.2 数据资产管理平台详细设计

**6.2.1 数据资产目录管理**

- **自动发现**：部署采集 Agent/无代理扫描（按数据源类型），覆盖关系型库、数仓、Hadoop/Hive、对象存储、FTP/SFTP、消息主题等；识别表、视图、字段、分区、存储过程、作业定义等对象。
- **采集策略**：支持实时（CDC/日志）、小时、天级调度；支持全量与增量；对大规模库表采用分片并行采集，满足“≥1000 表/小时”性能目标并通过水平扩展线性提升。
- **分类分级**：预置航空业务主题域（航班、旅客、机务、货运、财务人力等）；支持结构化/半结构化/非结构化三元分类；敏感级别映射公开/内部/敏感/机密，并允许集团扩展标签体系（多标签、多继承）。
- **目录服务**：Elasticsearch 全文检索 + 图谱辅助检索；资产详情聚合元数据、业务属性、技术属性、管理属性（Owner、SLA、成本分摊）；内置申请—审批—授权—订阅—变更通知闭环，与统一流程引擎集成。

**6.2.2 数据质量管理**

- **规则库**：内置规则模板（空值、重复、格式、范围、跨表一致性、缓慢变化一致性等），支持 SQL 规则与正则；规则版本化、灰度发布与影响评估；目标规则条目≥5000 条并可按主题域分库管理。
- **执行引擎**：实时（流式质量探针）与批量（Spark/Flink/SQL 引擎）双模式；调度支持 Cron、事件触发、上游作业成功回调；结果自动评分、评级与趋势归档。
- **问题闭环**：告警多渠道；自动生成质量工单并关联数据 Owner 与系统 Owner；根因分析结合血缘下游影响、历史缺陷聚类与样本对比。

**6.2.3 数据安全管理**

- **敏感识别**：规则 + 关键字 + 民航领域机器学习模型（旅客身份、航班动态、机组等）；人机协同复核与反馈学习，目标识别准确率≥95%；经大模型增强后，敏感数据智能识别准确率≥98%。
- **脱敏**：静态脱敏（离线交付）与动态脱敏（查询/接口实时）；替换、掩码、加密、泛化等算法按字段级策略配置，支持按用户/角色/场景（开发/测试/分析/对外）差异化。
- **访问控制**：RBAC 为基础，ABAC 扩展（环境、数据分级、终端、地理围栏等属性）；全链路访问审计，日志留存≥6 个月并支持不可抵赖与时间戳签名。

**6.2.4 血缘与影响分析**

- **采集**：SQL 解析（方言适配 Oracle/MySQL/Hive/SparkSQL 等）、ETL 工具插件（DataWorks、Kettle、Informatica 等）、作业日志解析；允许手工补录与冲突仲裁策略。
- **展示与分析**：全链路有向图；上游追溯、下游追踪；变更影响范围评估（表/字段/作业/API 消费者），与质量告警、发布门禁联动。

**6.2.5 数据价值评估**

- **模型**：成本法、收益法、市场法可配置权重；综合维度含质量分、调用热度、稀缺性、合规成本、替代成本等。
- **看板**：集团总览 + 按部门/业务域/数据类型钻取；价值趋势与“高价值低质量”预警清单，支撑数据产品化与对外服务定价参考。

**6.2.6 非功能对齐**

- 页面响应：简单查询≤2s，复杂查询≤5s（通过缓存、预聚合、异步查询与分页优化达成）。
- 数据导入：批量通道≥10 万条/秒（分布式并行写入 + 批量提交）。
- 并发：门户与网关层水平扩展，会话与令牌集中管理，支撑≥5000 在线用户。
- 可用性：多副本、健康检查、自动故障转移，目标系统可用性≥99.99%。
- 存储：湖仓与对象存储规划≥5PB，冷热分层与生命周期策略降低成本。

**6.2.7 智能能力模块设计（基于集团自建大模型）**

##### 6.2.7.1 整体集成架构

采用“代理适配层 + 能力封装层 + 应用场景层”三级集成模式：

1. **代理适配层**：封装集团大模型的底层接口，统一请求/响应格式，实现模型版本兼容、推理资源调度、熔断降级与限流控制；
2. **能力封装层**：将大模型能力拆解为语义理解、智能问答、根因分析、智能推荐等原子能力，与数据资产平台现有功能模块解耦集成；
3. **应用场景层**：针对数据资产管理各场景（问答、质量、安全、运营），组合原子能力形成场景化智能服务。

##### 6.2.7.2 智能问答与助手

- **知识库构建**：采集数据资产管理相关文档（制度规范、操作手册、规则说明）、结构化数据（资产目录、质量规则、安全策略），经清洗、分词、向量化后存入向量数据库，构建民航数据资产专属知识库；
- **交互流程**：用户自然语言提问 → 前端组件解析 → 代理适配层调用集团大模型 + 向量数据库语义检索 → 大模型结合知识库生成答案 → 能力封装层格式化结果（文本/图表/链接） → 前端展示；
- **私有化保障**：所有提问内容、知识库数据、推理结果均存储于集团私有环境，代理适配层配置访问白名单，仅允许数据资产平台服务调用。

##### 6.2.7.3 智能数据质量分析与预警

- **根因分析**：将数据质量异常数据、血缘关系、历史整改记录输入集团大模型，结合业务语义理解生成根因分析结论与整改建议；
- **风险预警**：基于时序数据训练大模型子任务，对数据质量指标（如空值率、重复率）的趋势进行预判，当偏离基线阈值时自动触发预警，并推送至对应数据 Owner；
- **报告生成**：大模型自动整合质量检测结果、根因分析、整改进度，生成多维度质量分析报告，支持 PDF/Excel 格式导出与定时推送。

##### 6.2.7.4 智能数据资产推荐与运营

- **用户画像构建**：基于用户角色、访问频次、查询内容、业务场景等维度构建用户画像，存入向量数据库；
- **智能推荐**：将用户画像与数据资产属性（业务域、价值评分、使用频次）经集团大模型语义匹配，生成个性化资产推荐列表，推送给用户门户；
- **运营建议生成**：大模型分析数据资产的调用量、质量评分、共享范围等数据，结合民航行业数据产品化案例，生成资产运营优化建议（如新增共享场景、优化数据产品封装方式）。

##### 6.2.7.5 智能敏感数据识别增强

- **模型微调**：基于集团自建大模型，使用民航行业敏感数据样本（旅客信息、航班涉密数据、机组信息等）进行微调，提升行业专属敏感数据识别精度；
- **上下文识别**：大模型通过语义理解，识别隐式关联的敏感数据（如“某航班机组住宿信息”未直接标注敏感，但通过上下文判定为机密）；
- **规则智能生成**：大模型基于敏感数据识别结果，自动生成对应的安全规则（如脱敏策略、访问控制规则），经人工审核后批量生效。

##### 6.2.7.6 智能血缘与影响分析增强

- **血缘补全**：针对人工未维护、ETL 工具未采集的血缘关系，大模型通过解析 SQL 脚本、业务文档、数据流转日志，智能补全跨系统血缘关系；
- **影响推理**：当某数据资产发生变更时，大模型基于血缘关系与业务语义，推理受影响的下游资产、业务系统、数据产品，生成影响评估报告，并标注风险等级。

##### 6.2.7.7 性能与安全保障

- **性能优化**：采用推理结果缓存（Redis）、向量检索加速、大模型分片推理等方式，满足“智能问答响应≤3秒”的性能要求；推理资源池基于 Kubernetes 实现弹性伸缩，应对并发请求波动；
- **安全管控**：大模型调用需通过统一身份认证（IAM），推理请求携带访问令牌；推理数据传输采用 TLS 1.3 加密，向量数据库存储加密；所有大模型调用日志接入集团审计平台，留存≥6 个月。

#### 6.3 数据中台建设方案

**6.3.1 主题域与数据模型**

对齐航班运行、旅客服务、机务维修、货运物流、财务人力等核心业务，建立集团级**概念模型—逻辑模型—物理模型**三层体系；主题域模型与资产目录、血缘、质量规则绑定，实现“模型即治理”。

**6.3.2 数据集成与开发**

- **集成**：批量（全量/增量）、实时 CDC、API 拉取、文件交换统一在集成层编排；支持多云与混合云数据源安全出域。
- **开发**：可视化开发与 SQL/Notebook 双模式；代码版本与发布流水线对接；开发与生产环境隔离，敏感数据默认脱敏。

**6.3.3 数据服务化**

- 对内：标准 REST/gRPC、JDBC/ODBC 代理、GraphQL（可选）统一经服务网关，限流、鉴权、计量。
- 对外：数据产品封装为 API/数据集/订阅包，支撑年调用量≥10 亿次目标（通过集群扩展、缓存、异步化与 CDN 加速组合实现）。
- **计量计费**：调用次数、数据量、峰值并发多维计量，为价值评估与产品化收入（年≥5000 万元业务目标）提供数据支撑。

**6.3.4 湖仓一体与计算**

- 热数据入仓支撑高并发查询；冷数据入湖支撑 AI 与非结构化分析；统一元数据与权限策略跨湖仓生效。

#### 6.4 区域中心部署方案

**6.4.1 拓扑模式**

采用**“集团总部逻辑控制中心 + 五大区域中心数据平面 + 分公司轻量接入”**：

- **总部**：全局资产目录主索引、全局策略（分级分类、质量基线、安全基线）、跨域编排与审计汇聚；部署大模型代理适配层主节点、全局智能知识库、智能能力调度中心，负责全集团智能策略统一管理。
- **区域中心**（华北/华东/华南/华西/华中）：区域级元数据采集与缓存、区域质量执行集群、区域 API 网关、区域只读副本与就近计算；与总部目录双向同步（最终一致），关键策略总部强一致下发；部署大模型推理边缘节点、区域级知识库缓存，承接本区域的智能问答、质量分析等请求，降低总部算力压力；推理结果同步至总部审计。
- **23 个分公司**：部署轻量采集器与边缘网关，按需同步元数据摘要与订阅数据；带宽受限场景启用压缩、差分同步与离线介质摆渡预案；无大模型本地部署，通过轻量网关接入区域中心推理节点，实现智能能力就近调用。

**6.4.2 网络与安全**

- 专线/VPN 加密传输，TLS 1.3；跨域访问经统一身份与最小权限；区域间共享走受控数据交换区，默认禁止横向直连。

**6.4.3 发布与阶段验收**

- 与招标文件阶段划分一致：基础平台（总部）→ 五大区域中心分批上线 → 分公司接入与终验前稳定运行 3 个月；每阶段具备可测、可回滚、可审计的发布清单。

#### 6.5 数据治理实施方案

**6.5.1 组织与制度**

- 建立集团数据治理委员会、数据 Owner、数据管家（Steward）三级责任体系；明确认责矩阵与 RACI。
- 输出《数据标准管理办法》《数据质量管理办法》《数据安全分级分类细则》《数据共享开放细则》等制度包，与平台流程固化。

**6.5.2 数据标准**

- 命名规范、代码标准、主数据标准、指标口径标准；标准与模型、质量规则、目录属性联动，发布即生效、变更即评估影响。

**6.5.3 资产盘点路径**

- **三阶段盘点**：① 自动发现快速摸底；② 业务确认与补录（目标≥10 万项资产条目）；③ 分级分类与质量规则挂载。每阶段有量化里程碑与验收抽样方法。

**6.5.4 质量提升闭环**

- 以核心业务域优先（航班、旅客、机务等），制定质量基线与整改 SLA；平台侧跟踪整改率、复发率、Owner 响应时效。

**6.5.5 培训与变革管理**

- 分层培训：管理层（价值与合规）、技术层（平台与开发规范）、业务层（目录与申请）；配套考试与上岗认证，降低推广阻力。

#### 6.6 安全架构设计

**6.6.1 等保三级与民航合规**

- 安全物理/网络/主机/应用/数据/管理六大维度对照等保三级控制点逐项映射；满足《民用航空信息安全管理规定》及集团安全审计要求。

**6.6.2 密码与应用安全**

- 国密 SM2/SM3/SM4 用于签名、摘要、对称加密等场景；证书与密钥托管于 HSM/KMS；应用层防注入、防重放、防越权；接口签名与时间窗校验。

**6.6.3 数据安全全生命周期**

- 采集、传输、存储、使用、共享、销毁各环节策略一致；存储透明加密/字段加密按分级执行；销毁支持软删保留期与硬删审计。

**6.6.4 安全运营**

- SIEM 对接、态势感知大屏（可选）、漏洞扫描与渗透测试纳入版本发布门禁；红蓝对抗与应急预案年至少一次演练。

**6.6.5 第三方测评配合**

- 预留性能压测与安全测评接口与环境快照；配合第三方机构完成性能测试与安全测评，作为初验、阶段验收与终验依据之一。

**6.7.2 项目管理方法**

- 采用敏捷迭代 + 里程碑门禁；双周迭代演示，月度 steering committee。
- 范围、进度、成本、质量、风险、沟通、采购、干系人九大知识领域文档化；关键风险（数据源不配合、历史系统无文档、网络策略变更）预置应对预案。

**6.7.3 迁移与割接**

- 蓝绿/金丝雀发布；元数据与配置迁移脚本化；割接窗口与回滚演练；业务低峰期切换。

**6.7.4 验收配合**

- 初验：功能清单与性能抽测；阶段验收：区域部署清单与跨域策略验证；终验：全量指标、第三方报告、运维移交与知识库完整性。

#### 6.8 售后服务方案

**6.8.1 质保期**

- 终验合格后 **36 个月**质保；质保内设计/制造/实施缺陷免费修复。

**6.8.2 服务等级**

- **7×24 技术支持**；重大故障 **≤2 小时**响应，一般故障 **≤4 小时**响应（含远程接入与现场升级机制，按故障级别定义）。
- 提供专属服务经理、二线专家与原厂（若涉及商业产品）升级通道。

**6.8.3 运维与优化**

- 年度健康检查与容量规划；版本补丁与安全漏洞修复窗口；性能调优与索引/模型优化纳入常规服务。
- 定期对集团大模型适配层进行版本升级，同步大模型迭代能力；每季度更新数据资产智能知识库，补充新的业务规则、制度文档；基于用户反馈优化智能问答、推荐等能力的效果，持续提升准确率与匹配度；提供大模型推理资源的容量规划与性能调优服务，保障并发与响应指标达标。

**6.8.4 知识转移**

- 完整交付运维手册、应急预案、Runbook、API 文档、培训录像与管理员认证；质保末期满前完成最终知识转移验收。
