# 国有航空集团数据资产智能管理平台

# TOGAF 对齐四层企业架构

本文档依据 `TOGAF架构方法.md` 规范，对齐《数据资产智能管理平台建设项目（招标书完整稿）》建设背景、目标、技术规格与第六章技术方案要点，输出业务、应用、数据、技术四层架构，可直接用于方案编写与架构图制作。

---

## 一、业务架构

### 1. 业务愿景与目标层

| 战略方向 | 子目标（对齐招标书） |
|---------|----------------------|
| 数字化转型 | 建成云原生、微服务、容器化统一平台；支撑「总部—五大区域中心—23分公司」协同的数据资产管理体系。 |
| 服务质量与体验 | 统一门户与目录检索；数据服务年调用量≥10亿次；页面简单查询≤2秒、复杂查询≤5秒。 |
| 成本与效率 | 资产自动发现与规则复用降低人工盘点成本；湖仓冷热分层与生命周期策略优化存储与算力。 |
| 应急与韧性 | 同城双活+异地灾备；重大故障≤2小时响应；系统可用性≥99.99%。 |

**核心价值表述**：实现数据资产「可见、可用、可管、可控」，在等保三级与民航合规框架下释放数据价值并支撑数据产品化（年数据产品化收入目标≥5000万元）。

### 2. 核心业务流程层（端到端）

| 业务模块 | 核心执行动作 |
|---------|--------------|
| 资产盘点与编目 | 自动发现→元数据采集→业务确认与补录→分类分级→目录发布与订阅通知。 |
| 质量治理 | 规则配置/模板复用→实时/批量检测→评分评级→告警→工单流转→根因与整改闭环。 |
| 安全合规 | 敏感识别（规则/模型/大模型增强）→脱敏策略→RBAC/ABAC授权→访问审计与留存。 |
| 血缘与变更 | SQL/ETL/日志采集→图谱展示→影响分析→与质量告警、发布门禁联动。 |
| 价值与运营 | 价值模型计算→多维看板→共享计量→产品封装与运营建议（含智能推荐）。 |
| 智能服务 | 自然语言交互→知识检索与推理→结构化结果/报告输出→全链路审计。 |
| 区域协同 | 总部策略下发→区域采集与缓存/就近服务→分公司轻量接入与差分同步。 |

### 3. 组织架构与职责层

| 职能/角色 | 职责要点 |
|-----------|----------|
| 数据治理委员会 | 战略决策、重大标准与跨域共享政策审批。 |
| 数据 Owner | 主题域数据认责、质量基线签署、敏感分级确认。 |
| 数据管家（Steward） | 目录属性维护、规则协同、业务口径解释。 |
| 集团总部（信息中心/数据团队） | 全局目录主索引、全局策略、大模型代理与知识库主节点、审计汇聚。 |
| 区域中心 | 区域元数据采集与缓存、区域质量执行、区域网关与推理边缘节点、与总部目录同步。 |
| 分公司 | 轻量采集器、边缘网关、按需同步元数据摘要与订阅数据。 |
| 安全与审计 | 等保三级对照、渗透与测评配合、SIEM/日志留存。 |

### 4. 业务能力与价值链层

| 能力类型 | 能力构成 |
|---------|----------|
| 核心能力 | 统一资产目录、全生命周期管理、质量规则库与检测引擎、分级分类与敏感保护、血缘与影响分析。 |
| 支撑能力 | 数据中台集成与开发、湖仓一体、数据服务网关与计量、向量知识库与大模型适配。 |
| 管理能力 | 制度与标准包、RACI认责、流程引擎（申请—审批—授权—订阅）、运维与容量规划。 |
| 服务能力 | 自助检索与数据地图、API/数据集/订阅包、智能问答与报告、移动端与多渠道告警。 |

## 二、应用架构

### 1. 支撑平台服务层

| 模块 | 核心能力 |
|------|----------|
| 集成与连接 | 批量/CDC/API/文件交换编排；多云与混合云安全出域；分公司压缩与差分同步。 |
| 检索与目录索引 | 全文检索（如 Elasticsearch）；目录主索引与图谱辅助检索。 |
| 消息与事件 | Kafka/RocketMQ 等主题与事件驱动质量检测、变更通知、异步血缘重算。 |
| 采集与代理 | 数据库/数仓/湖/Hadoop/对象存储/FTP/消息主题采集 Agent 或无代理扫描。 |
| 边缘与区域 | 区域缓存、区域 API 网关、区域推理节点与知识库缓存（降总部算力压力）。 |

### 2. 智慧集成服务层

| 模块 | 核心能力 |
|------|----------|
| 统一门户与 BFF | Web/移动统一入口；资产目录、数据地图、搜索、服务目录、智能助手前端组件。 |
| API 网关 | 统一鉴权、限流熔断、计量与对外服务暴露边界。 |
| 可视化与报表 | 价值运营看板、质量趋势、调用热度与多维钻取。 |
| 调度与流程 | 检测任务调度（定时/事件）、工单与审批流、发布门禁。 |
| 移动与消息 | iOS/Android/鸿蒙适配；短信、邮件、钉钉、企业微信告警推送。 |

### 3. 核心业务服务层

| 服务域 | 核心能力 |
|--------|----------|
| 资产中心 | 自动发现、分类分级、资产详情聚合、申请/审批/授权/订阅/变更通知。 |
| 质量中心 | 规则库（≥5000条目标）、实时/批量执行、评分评级、工单与根因分析。 |
| 安全中心 | 敏感识别、动静态脱敏、RBAC+ABAC、访问审计。 |
| 血缘中心 | SQL/ETL/日志血缘、图谱、影响评估、与大模型补全/校验联动。 |
| 价值中心 | 成本法/收益法/市场法可配置模型、价值总览与钻取。 |
| 运营中心 | 调用计量、数据产品封装、运营监控与智能运营建议。 |
| 智能助手服务 | 问答、质量报告、推荐、敏感规则生成、血缘补全与影响推理的场景化封装。 |

### 4. 应用集成与数据流层

| 集成方式 | 用途 |
|---------|------|
| REST/gRPC | 数据服务、大模型代理、内部微服务调用。 |
| JDBC/ODBC 代理、可选 GraphQL | 分析类消费与统一数据出口。 |
| 消息队列 | 异步检测、血缘重算、跨域同步与解耦峰值。 |
| 数据同步 | 湖仓复制、区域—总部目录双向同步（最终一致+关键策略强一致）。 |
| 统一认证 | 对接 4A/IAM、零信任策略；大模型调用携带令牌。 |
| DevOps 与配置 | 与 CMDB、流水线对接；配置中心与密钥管理（KMS/HSM）。 |

## 三、数据架构

### 1. 数据源层

| 来源类型 | 覆盖范围（对齐招标书） |
|---------|------------------------|
| 业务系统 | 航班运行、旅客服务、机务维修、货运物流、财务人力等。 |
| 数据平台 | 关系库、数据仓库、Hadoop/Hive、大数据平台、对象存储。 |
| 文件与交换 | FTP/SFTP、运行报告、机务文档等非结构化来源。 |
| 消息与日志 | 消息主题、作业日志、用于血缘与审计。 |
| 外部与第三方 | 受控数据交换区接入；默认禁止横向直连。 |
| 终端与边缘 | 分公司轻量采集器、区域网关汇聚的摘要与订阅数据。 |

### 2. 数据采集与传输层

| 机制 | 核心能力 |
|------|----------|
| 多协议与多模式 | 实时 CDC/日志、小时/天级调度、全量与增量。 |
| 性能与扩展 | 分片并行采集；元数据采集≥1000表/小时并通过水平扩展线性提升。 |
| 传输安全 | 专线/VPN、TLS 1.3；跨域最小权限。 |
| 校验与缓冲 | 完整性校验、背压与缓存、失败重试与可观测性指标。 |

### 3. 数据存储层

| 存储形态 | 用途 |
|---------|------|
| 关系型/分布式关系型 | MySQL/PostgreSQL/TiDB/OceanBase 等元数据、配置、流程与事务数据。 |
| 检索与时序需求 | Elasticsearch 等支撑目录检索与日志检索场景。 |
| 缓存 | Redis（会话、令牌、推理结果缓存等）。 |
| 湖仓一体 | 热数据入仓高并发查询；冷数据入湖支撑 AI 与非结构化分析；规划≥5PB 与冷热分层。 |
| 向量库 | Milvus/Chroma 等承载资产专属知识向量。 |
| 对象存储 | 大文件、模型产物、报告附件等。 |

### 4. 数据处理与治理层

| 模块 | 核心能力 |
|------|----------|
| 清洗与转换 | 集成管道中的标准化、码表对齐、主题域模型落地。 |
| 质量管控 | Spark/Flink/SQL 引擎支撑批量；流式质量探针支撑实时。 |
| 安全与加密 | 国密 SM2/SM3/SM4；存储透明加密/字段级加密按分级执行。 |
| 元数据与血缘 | 技术/业务/管理属性统一模型；血缘有向图与版本管理。 |
| 生命周期 | 软删保留期、硬删审计、归档与销毁策略。 |

### 5. 数据服务层

| 服务形态 | 核心能力 |
|---------|----------|
| 查询与搜索 | 目录全文检索、资产详情、图谱辅助检索。 |
| API 与订阅 | REST/gRPC、数据集与订阅包；变更通知。 |
| 分析与可视化 | 价值与质量多维分析、异步复杂查询与预聚合。 |
| 导出与报告 | PDF/Excel 质量报告、影响评估报告等。 |
| 计量 | 调用次数、数据量、峰值并发多维计量，支撑价值评估与产品化收入核算。 |

## 四、技术架构

### 1. 硬件基础设施层

| 组件 | 要点 |
|------|------|
| 计算 | x86/国产 CPU（鲲鹏/飞腾）；总部生产与同城备、异地灾备；GPU/CPU 混合推理池。 |
| 存储 | 高性能块存储、分布式文件与对象存储、备份介质。 |
| 网络 | 总部与五大区域中心专线/VPN；受控数据交换区。 |
| 边缘 | 区域推理节点；分公司轻量网关与采集器。 |

### 2. 软件基础设施层

| 组件 | 要点 |
|------|------|
| 操作系统 | 麒麟/统信等国产化适配。 |
| 中间件 | Redis、RocketMQ/Kafka、注册配置中心。 |
| 容器与微服务 | Kubernetes 唯一编排基座；Spring Cloud Alibaba（或同等）；可选服务网格。 |
| 大数据 | Hadoop 生态、Spark（批）、Flink（流）。 |
| DevOps | 流水线、镜像仓库、灰度/蓝绿/金丝雀发布。 |

### 3. 网络架构层

| 维度 | 要点 |
|------|------|
| 拓扑 | 总部逻辑控制+区域数据平面+分公司轻接入；区域间共享经交换区。 |
| 隔离 | 生产/测试/开发环境隔离；推理隔离容器。 |
| 优化 | 就近计算与区域只读副本；带宽受限差分同步与离线摆渡预案。 |

### 4. 安全架构层

| 维度 | 要点 |
|------|------|
| 合规 | 等保三级；《民用航空信息安全管理规定》。 |
| 身份与权限 | 统一 IAM；RBAC+ABAC；接口签名与时间窗。 |
| 数据安全 | 分级分类、脱敏、加密传输（TLS 1.3）、推理全程加密与禁止不当外发。 |
| 监控审计 | 操作审计、推理日志全量留存≥6个月、SIEM 对接。 |

### 5. 云计算与边缘计算层

| 维度 | 要点 |
|------|------|
| 部署模式 | 私有云/混合云/多云适配（阿里云、腾讯云、华为云等）。 |
| 弹性 | K8s HPA/VPA 与推理资源池联动；大模型推理≥200并发请求/秒目标。 |
| 云边协同 | 区域边缘推理与知识库缓存；结果同步总部审计。 |
| 私有化大模型 | 仅私有化调用，禁止公共大模型外联；代理适配、版本兼容、熔断降级。 |

### 6. 运维管理层

| 领域 | 要点 |
|------|------|
| 监控告警 | 全栈可观测；门户与网关最高可用域；异步任务可降级。 |
| 日志 | 集中采集、关联追踪、留存策略与合规。 |
| 配置与密钥 | 配置中心、KMS/HSM 托管。 |
| 备份恢复 | 元数据、配置、审计日志同城实时同步与异地灾备策略。 |
| 性能与容量 | 年度健康检查、索引与模型优化、容量规划（含推理算力）。 |
| 应急 | Runbook、演练、7×24 支持及响应 SLA（重大≤2小时、一般≤4小时）；质保 36 个月。 |

## 五、四层闭环联动（TOGAF 对齐说明）

- **业务驱动应用**：端到端流程映射为资产中心、质量中心、安全中心、血缘中心、价值中心、运营中心与智能助手服务边界。
- **应用牵引数据**：目录、质量、血缘、计量与知识库均绑定主题域概念—逻辑—物理模型，实现「模型即治理」。
- **技术承载非功能**：云原生、多活容灾、等保与国密、性能指标（并发、响应、采集吞吐、存储规模）与智能指标（问答准确率≥90%、分析≤3秒、推荐匹配度≥85%、敏感智能识别≥98%）在技术分层中逐项落实。
- **数据回流业务**：调用计量与价值看板反哺数据产品化与共享策略，完成从战略到运营的数据闭环。
