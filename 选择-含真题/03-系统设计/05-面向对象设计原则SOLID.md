# 面向对象设计原则（SOLID 等）

**知识点编号**：3.4.2  
**权重**：重点 ★★★★★

## 核心要点

| 原则 | 要点 |
|------|------|
| **单一职责 S** | 一类一事 |
| **开闭 O** | 对扩展开放、对修改关闭；常靠抽象层 |
| **里氏替换 L** | 子类可替换基类；变化侧用抽象 |
| **接口隔离 I** | 接口小而专；客户端只依赖所需方法 |
| **依赖倒置 D** | **抽象不依赖细节，细节依赖抽象**；面向接口编程 |
| **组合/聚合复用** | 优先组合聚合而非继承 |
| **最少知识（迪米特）** | 尽量少与其他实体耦合 |

**口诀扩展**：开闭为基；**依赖倒置** 常是实现开闭的手段。

## 系统架构设计师 · 真题汇编

### （2015年11月·第26题）
「抽象不应依赖细节，细节应依赖抽象，针对接口编程」对应（）。  
**答案**：D **依赖倒置原则**
