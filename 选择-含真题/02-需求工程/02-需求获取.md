# 需求获取

**知识点编号**：2.1  
**权重**：次重点 ★★★☆☆

## 核心要点

- **含义**：确定并理解各干系人的需求与约束。
- **常用方法**（熟悉即可）：

| 方法 | 要点 |
|------|------|
| 用户访谈 | 结构化 / 非结构化；准备—过程—后续 |
| 问卷调查 | 高效、低成本；细节与返还率受限 |
| 采样 | 样本大小 ≈ α×(可信度系数/可接受错误)²，α 常取 0.25 |
| 情节串联板 | 图片讲故事；被动/主动/交互；利于界面早期评审，耗时 |
| 联合需求计划 JRP | 会议式与客户深度合作；组织难度大 |
| 收集资料 / 获取文档 / 参加业务实践 | 文献、既有文档、实地体验 |

**口诀**：访问踩窗帘，合收十文 — 访、问、踩（采样）、窗（串联）帘；合、收、十（实践）、文（文档）。

## 考点提示

方法名与适用场景、优缺点匹配；采样公式选择题可记形态。
