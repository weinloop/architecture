# 需求验证 · 评审、检查、走查

**知识点编号**：2.7  
**权重**：非重点 ☆☆☆☆☆（概念辨析可考）

## 核心要点

- **需求验证（确认）**：确保 SRS **正确、无歧义、可验证** 等，降低后期返工成本。  
- **需求测试**：早期无执行代码，多为 **基于文本的概念级** 测试思路。  
- **评审类型**：

| 类型 | 特点 |
|------|------|
| **评审** | **正式**会议，向干系人展示工作产品，征求意见与批准 |
| **检查** | **正式且严格**，非作者小组 **逐条核查** 错误与标准符合性 |
| **走查** | **非正式**，作者带领团队讲解，同事提意见 |

## 考点提示

三者 **正式程度与目的** 对比；SRS 评审发现二义性需求。
