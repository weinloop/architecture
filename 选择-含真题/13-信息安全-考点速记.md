## 二、系统安全分类（★★★☆☆）

| 类型 | 要点 |
|------|------|
| **实体安全** | 环境、设备、媒体；防损失与破坏 |
| **运行安全** | 风险管理、审计跟踪、备份与恢复、应急 |
| **信息安全** | OS 安全、数据库安全、网络安全、病毒防护、访问控制、加密、认证 |
| **人员安全** | 意识、法律、技能 |

**案例（航空运营智能管理平台）**：

- **实体**：机房、云平台可用区、介质与密钥硬件（若有）。
- **运行**：**99.99% 可用**、**备份与演练**、**漏洞扫描与应急响应**、**异地登录与暴力破解告警**。
- **信息**：**数据库加密、WAF、防 SQL 注入/XSS/DDoS**、**访问控制与审计**。
- **人员**：运维与客服 **权限审批**、保密培训、操作规范。

## 三、等级保护（★★★☆☆）

| 等级 | 适用对象 |
|------|----------|
| 第一级 用户自主保护 | 普通内联网用户 |
| 第二级 系统审计保护 | 需保密的非重要单位，内联网/国际网商务 |
| 第三级 安全标记保护 | 地方国家机关、金融机构等重要单位 |
| 第四级 结构化保护 | 中央级国家机关、广电等关键部门 |
| 第五级 访问验证保护 | 国防关键部门、依法特殊隔离单位 |

**案例（航空运营智能管理平台）**：航空信息中心承载 **重要行业信息系统与大量公民出行数据**，工程上常按 **第三级及以上** 要求建设（定级以主管部门测评为准）；答题时 **“重要单位/行业关键信息基础设施”** 对应 **第三级** 最常被选中。

## 四、数据加密（★★★★★）

- **对称**：AES（128/192/256）、DES（56）、3DES（168/有效 112）、IDEA（128）、RC5；国产 SM1/SM4/SM7。**非对称**：RSA、ECC；国产 SM2/SM9（SM2 椭圆曲线、SM9 基于标识）。
- **杂凑**：SM3、MD5、SHA 系列；用于完整性、摘要。

**案例（航空运营智能管理平台）**：**旅客身份证、银行卡号** 等 **静态敏感字段** 宜用 **AES/SM4** 等 **对称加密** 存库；**密钥协商、API 互信** 可用 **RSA/ECC/SM2**；**完整性校验** 用 **SM3/SHA-256**；国密合规场景优先 **SM 系列**。

---

## 五、认证技术：数字签名与数字证书（★★★★★）

- **Hash 特性**：易压缩、单向、高灵敏、抗碰撞；SHA-256、SM3 等。
- **数字签名**：对**摘要**用私钥加密；验证时用公钥解密摘要，与对原文 Hash 结果比对。**只签摘要不签全文**。
- **数字证书**：CA 用私钥对“持有者信息+公钥”等签名，形成证书；对方用 CA 公钥验证证书，获得可信公钥。解决**公钥真实性**问题。
- **场景**：信源验证（私钥签名、公钥验签）；密钥交换（公钥加密会话密钥，私钥解密）。

**案例（航空运营智能管理平台）**：

- **数字签名**：与外部机构 **对账文件、回调通知** 可对 **摘要** 做 **私钥签名**，对方 **验签** 防篡改。
- **证书**：对外 **HTTPS** 使用 **CA 签发 X.509**；内部服务 **mTLS** 用 **企业 CA** 绑定服务身份。
- **Hash**：日志指纹、**安装包/镜像校验** 防供应链篡改。
- **合规**：**对接民航局旅客身份核验** 解决 **真人真证**（与 TLS/证书 **信道安全** 互补，属业务与合规层）。

## 六、密钥管理体制（★★★☆☆）

| 体制 | 适用 | 特点 |
|------|------|------|
| **KMI** | 封闭网 | KDC 分发，静态/动态，秘密信道 |
| **PKI** | 开放网 | CA、证书，标准平台 |
| **SPK** | 规模化专用网 | LPK/CPK 等 |

**案例（航空运营智能管理平台）**：互联网开放访问以 **PKI/CA 证书体系** 为主（**HTTPS**）；内网若封闭可配合 **KMS/KDC** 管理 **数据加密密钥**；专网大规模场景可了解 **SPK** 考点即可。

---

## 七、安全协议（★★★★★）

| 协议 | 层次/类型 | 要点 |
|------|-----------|------|
| **SSL** | 传输层 | 握手/记录/警报；身份认证、加密、完整性；HTTPS 即 HTTP+SSL |
| **HTTPS** | 应用层+SSL | 安全通道、防窃听、X.509 |
| **IPSec** | 网络层，IPv4/6 | IP 层加密与认证，端到端 |
| **PGP** | 应用，邮件 | 基于 RSA，邮件加密与签名 |

**案例（航空运营智能管理平台）**：

- **TLS 1.2+**（素材 **SSL/TLS 全程加密**）：浏览器/App 与网关之间 **握手协商会话密钥**，防窃听与中间人。
- **HTTPS**：**HTTP over TLS**，保护 **购票与个人信息**。
- **IPSec**（若采用）：**专线/站点 VPN** 加密 **机房—云** 或 **多 AZ** 东西向（视架构）。
- **PGP**：考点为主；实际企业邮件可换 **S/MIME** 等，论文不必展开。

## 八、访问控制（★★★★★）

| 类型 | 思想 | 适用 |
|------|------|------|
| **DAC** | 主体自主授权，依身份与组 | 多用户 |
| **MAC** | 主体与客体安全属性，强制规则 | 高安全 |
| **RBAC** | 用户—角色—权限 | 大型系统如 DBMS |
| **TBAC** | 权限随任务上下文，动态 | 工作流 |
| **OBAC** | 对象 ACL，可继承、派生 | 对象多、变化频繁 |

### 访问控制模型（★★★☆☆）

- **BLP**：保密性；**上读 no read up**、**下写 no write down**。
- **Biba**：完整性；**下读 no read down**、**上写 no write up**。

**案例（航空运营智能管理平台）**：

- **RBAC**：素材 **基于 RBAC 的细粒度权限、角色自定义、权限申请审批**；角色如 **旅客服务、调度、财务只读、安全审计**。
- **DAC/MAC**：桌面共享文件为 **DAC**；若划分 **秘密/内部** 标签的运维系统可用 **MAC**（考点辨析）。
- **TBAC**：**检修工单、退票审批** 等流程中 **按任务环节** 动态授权。
- **BLP**：防 **低权限用户读高敏旅客明细**（保密性）；**Biba**：防 **低可信数据污染** 高可信账务（完整性）。

## 九、单点登录（★★★☆☆）

- **Kerberos**：信任第三方，票据认证。**外壳脚本**：一次认证激活多系统，口令同步等需配合。

**案例（航空运营智能管理平台）**：信息中心可建 **统一身份认证**，一线 **客服/运维** 登录 **票务、看板、检修** 多子系统时 **单点登录**，减少口令泄露面；与 **异地登录检测、异常操作告警** 联动。

## 十、速记清单（考前扫一眼）

1. **安全四类**：实体、信息、运行、人员。
2. **等级**：一普二商三地方四中央五国防。
3. **对称**：AES/DES/3DES；**非对称**：RSA/ECC；**国产**：SM2/SM9 非对称，SM3 杂凑，SM4 分组。
4. **数字签名**：Hash 摘要+私钥加密；验证=公钥解密+比摘要。**数字证书**：CA 签公钥，验签得可信公钥。
5. **协议**：SSL 传输层、HTTPS=HTTP+SSL、IPSec 网络层、PGP 邮件。
6. **访问控制**：DAC 自主、MAC 强制、RBAC 角色；BLP 上读下写、Biba 下读上写。
