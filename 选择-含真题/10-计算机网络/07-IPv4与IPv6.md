10.7.IPv4 VS IPv6（次重点★★★☆☆)

IPv6  通过更大的地址空间、简化报头、 内置安全和自动配置等特性 ，解决了 IPv4  地址耗尽和扩展性不足的问题。其他维度的对比如下表所示 ，其中标记颜色的重点记忆。


| 对比维度 | IPv4 | IPv6 |
| --- | --- | --- |
| 地址空间 | 32  位（ 约 43  亿个地址） | 128  位（ 约  3.4 × 1038   个地址） |
| 地址表示 | 点分十进制（ 如： 192.168.1.1 ） | 冒    号   分    隔    十    六    进    制   （     如   ： 2001:0db8:85a3::8a2e:0370:7334） |


| 地址分配 | 手动配置 / DHCP（ 需公有 /  私有地址规划 ） | 无状态地址自动配置（SLAAC）+ DHCPv6  可选 |
| --- | --- | --- |
| 报头格式 | 可变长报头（ 20-60  字节 ），包含校验和、分片等字段 | 固定 40  字节报头 ，简化字段（ 无校验和 ，分片由源端处理） |
| NAT  依赖 | 依赖 NAT  解决地址短缺问题（如家庭网络共享公网 IP ） | 无需 NAT（地址充足 ，支持端到端通信） |
| 安全性 | 无内置加密 ，需额外配置 IPsec | 强制支持 IPsec（ AH/ESP  协议） |
| 路由效率 | 路由表庞大 ，处理复杂 | 路由表精简（聚合性更好） ，转发速度更快 |
| 应用场景 | 传统网络、低地址需求场景 | 物联网、大规模网络、云服务 、未来互联网基础设施 |
| 过渡成本 | 已成熟 ，但面临地址枯竭问题 | 部署成本高（ 需升级设备 /  软件） ，但长期扩展性更好 |

## 系统架构设计师 · 真题汇编

> 来源：[计算机网络.pdf](../../选择真题-分类/计算机网络.pdf)。**完整题干、选项、正确答案与解析**。

### （2016年11月·第11题）

在IPv6无状态自动配置过程中，主机将其（）附加在地址前缀1111 1110 10之后，产生一个链路本
地地址。
问题（1）
（A）IPv4地址
（B）MAC地址
（C）主机名
（D）随机产生的字符串

**正确答案：** B

**解析**

本题考察的是IPv6 的无状态地址自动配置机制中的链路本地地址生成方法。
 IPv6 中，链路本地地址（Link-Local Address）用于本地链路上的通信，不可路由。其地址前缀
固定为 1111 1110 10（即 FE80::/10），后面跟一系列 0，然后是接口标识符。
A选项 IPv4地址：IPv6 与 IPv4 是两个独立协议族，IPv4 地址不会用于构造 IPv6 地址，除非在
某些过渡技术中，题干描述与其无关。
B选项 MAC地址：正确，IPv6 无状态自动配置中，主机会使用其网络接口的 MAC 地址，通过 EUI-
64 转换机制生成接口标识符，然后附加在前缀 FE80::/10 后面，构成链路本地地址。
C选项 主机名：主机名是高层信息，与底层链路地址无关，不能参与链路本地地址的生成。
D选项 随机产生的字符串：虽然有些隐私扩展机制可能引入随机值用于地址生成，但题干特指标准
的链路本地地址生成，不使用随机字符串。
 因此，选项 B 正确。

### （2015年11月·第11题）

以下关于 IPv6 的论述中，正确的是（）。
问题（1）
（A）IPv6 数据包的首部比 IPv4 复杂
（B）IPv6 的地址分为单播、广播和任意播3种
（C）IPv6 的地址长度为 128 比特
（D）每个主机拥有唯一的 IPv6 地址

**正确答案：** C

**解析**

本题考察的是IPv6 的基本特性与寻址方式。
A选项：IPv6 的首部比 IPv4 更简单，设计上减少了首部字段数量，并且去掉了不必要的选项字段
，以提高路由转发效率，因此该说法错误。
B选项：IPv6 的寻址方式是单播（Unicast）、组播（Multicast）、任播（Anycast），没有广播
（Broadcast）概念，因此该说法错误。
C选项：IPv6 地址长度为 128 位（bit），这是 IPv4 地址 32 位的 4 倍，能够提供极为庞大的地
址空间，该说法正确。
D选项：IPv6 主机通常会拥有多个 IPv6 地址（如链路本地地址、全局单播地址、回环地址、组播
地址等），并不是唯一一个，因此该说法错误。
 因此，正确答案是 C。
