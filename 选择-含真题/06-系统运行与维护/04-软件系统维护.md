# 软件系统维护

**知识点编号**：6.4  
**权重**：次重点 ★★★☆☆

## 核心要点

交付后至淘汰期间，为**改正错误**或**满足新需求**而修改软件；前提是软件**可维护**。

| 类型 | 动因 | 说明 |
|------|------|------|
| **改正性** | 错误、缺陷、误用 | 诊断并修复（修 Bug） |
| **适应性** | **外部环境/数据环境变化**（OS、硬件、规则、格式等） | 为适应变化而修改 |
| **完善性** | 用户提出**新功能/性能/体验** | 扩充、增强、调优、重构（非单纯修故障、非仅环境变） |
| **预防性** | 预先提高可维护性、可靠性 | 再工程思想：「用今天方法整理昨天系统，服务明天」 |

**辨析**：**适应性** = 外部/环境变了要跟上；**完善性** = 用户要更多更好。

### 提高可维护性（论文可用）

| 阶段 | 要点 |
|------|------|
| 需求 | 预见变更与可移植 |
| 设计 | **高内聚低耦合**、可扩展、可移植、可复用构件 |
| 编码 | 规范、注释、复用 |
| 测试 | 充分测试与测试文档，减轻后期维护 |
| 维护 | **配置管理**、文档同步、培训 |

### （2014年11月·第21题）

在软件的使用过程中，用户往往会对软件提出新的功能与性能要求。为了满足这些要求，需要修改
或再开发软件。在这种情况下进行的维护活动称为（）。

问题（1）
（A）改正性维护
（B）适应性维护
（C）完善性维护
（D）预防性维护

**正确答案：** C

**解析**

本题考察的是软件维护类型的分类与适用场景。
 软件维护分为四种类型：改正性维护、适应性维护、完善性维护和预防性维护。
A选项改正性维护：主要针对消除软件中已发现的缺陷和错误，是修复性的维护，与提升功能和性能
无关，错误。
B选项适应性维护：是为了适应运行环境（硬件、软件平台、外部接口等）或业务需求变化而做的修
改，不专指新增功能或性能优化，错误。
C选项完善性维护：旨在根据用户提出的新功能和性能需求，对软件进行修改或再开发，以提高其质
量和性能，这是题干描述的情况，正确。
D选项预防性维护：在软件运行正常时，为了防止潜在问题而进行的修改，以提高系统可靠性和稳定
性，与题干无关，错误。
 因此，选项 C 正确。

### （2024年11月·第27题）

系统上线运行后，用户提出新的需求，需要进行改造代码，提升系统性能，提高运行效率，扩展更
多功能，这属于（）。
问题（1）
（A）改正性维护
（B）适应性维护
（C）完善性维护
（D）预防性维护

**正确答案：** C

**解析**

本题考察的是软件系统维护的分类。
软件维护是指软件在投入使用后，为了修复缺陷、适应环境变化或满足用户新需求所进行的修改与
演进过程。根据维护目的的不同，通常分为以下四类：
A选项改正性维护：用于修复系统中的缺陷或错误，比如程序崩溃、输出错误等，是“补救性质”的
维护方式。
B选项适应性维护：由于外部环境变化（如操作系统升级、数据库平台更换、法律法规调整）而引起
的软件修改，以适应新的环境。
C选项完善性维护：指系统上线后，用户基于业务发展提出的新需求，如功能扩展、性能优化、效率
提升等，这种维护是对系统的改进和增强，题干中正是描述了这一情况，因此此选项正确。
D选项预防性维护：是为提高系统将来的可维护性或稳定性所做的结构优化或技术重构，不是基于当
前问题或需求提出的修改。
 因此，选项 C 正确。

### （2021年11月·第23题）

在软件开发和维护过程中,一个软件会有多个版本,（）工具用来存储、更新、恢复和管理一个软件
的多个版本。
问题（1）
（A）软件测试
（B）版本控制
（C）UML建模

（D）逆向工程

**正确答案：** B

**解析**

本题考察的是软件维护中版本控制的概念。
 在软件开发与维护过程中，随着功能的增加、缺陷的修复以及性能的优化，软件会不断演化，形成
多个版本。为了保证开发过程有序进行，需要使用版本控制工具来管理不同版本的代码和资源。
A选项软件测试：用于验证和确认软件质量，与版本管理无关，错误。
B选项版本控制：通过工具（如 Git、SVN 等）存储、更新、恢复和管理软件的多个版本，支持多人
协作开发，防止版本冲突，正确。
C选项UML建模：用于软件设计阶段的建模描述，不负责版本管理，错误。
D选项逆向工程：从现有软件系统推导出设计文档或规格说明，不是版本管理工具，错误。
 因此，选项 B 正确。

### （2013年11月·第19题）

以下叙述中，（）不属于可行性分析的范畴。
问题（1）
（A）对系统开发的各种候选方案进行成本/效益分析
（B）分析现有系统存在的运行问题
（C）评价该项目实施后可能取得的无形收益
（D）评估现有技术能力和信息技术是否足以支持系统目标的实现

**正确答案：** B

**解析**

本题考察的是项目可行性分析的范围。
A选项对系统开发的各种候选方案进行成本/效益分析：这是经济可行性分析的重要内容，用于评估
投入与产出的比例，属于可行性分析范畴，正确。
B选项分析现有系统存在的运行问题：这是现状调查与问题分析的内容，属于系统调查阶段，不是可
行性分析的直接内容，错误，正确答案应选此项。
C选项评价该项目实施后可能取得的无形收益：这是经济可行性分析的组成部分，通常包括提高工作
效率、改善企业形象等，属于可行性分析范畴，正确。
D选项评估现有技术能力和信息技术是否足以支持系统目标的实现：这是技术可行性分析的主要内容
，判断系统开发是否有技术支撑，属于可行性分析范畴，正确。
 因此，正确答案是 B。
