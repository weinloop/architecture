# 系统运行管理

**知识点编号**：6.2  
**权重**：非重点（了解）

## 核心要点

对信息系统**运行管控、状态记录、检查评价**，支撑管理与决策。

- **日常运行管理**：用户、网络、软件资源等。
- **用户管理**：账号与权限、外部用户、**统一用户**（方便、安全、减负）、认证方式（口令、IC 卡、动态口令、USB Key 等）、**安全审计**。
- **网络资源管理**：**性能、故障、配置、计费、安全**（五大功能）。
- **软件资源**：**构件库**、**分发**（部署/补丁/远程）、**文档**管理。

### （2022年11月·第35题）

软件复杂性度量中，（）可以反映原代码结构的复杂度。
问题（1）
（A）模块数
（B）环路数
（C）用户数
（D）对象数

**正确答案：** B

**解析**

本题考察的是软件复杂性度量的概念，特别是衡量代码结构复杂度的指标。
A选项 模块数：模块数反映的是程序中模块的数量，虽然模块数量较多可能增加系统的复杂度，但
它并不直接反映代码的结构复杂性。
B选项 环路数：环路数（Cyclomatic Complexity）是衡量程序复杂度的常用指标，它通过计算程序
中的独立路径数量来反映代码的复杂度。环路数越多，表示代码逻辑越复杂，决策结构越多，因此
它能够直接反映原代码结构的复杂度。
C选项 用户数：用户数是指使用系统的用户数量，与代码复杂度没有直接关系。
D选项 对象数：对象数指的是系统中包含的对象数量，虽然对象的数量可能影响系统的设计复杂度
，但它与代码的执行逻辑复杂度没有直接关系。
 因此，B选项 环路数是正确答案。

### （2020年11月·第14题）

进行系统监视通常有三种方式：一是通过（），如 UNIX/Linux 系统中的 ps、last 等；二是通过
系统记录文件查阅系统在特定时间内的运行状态；三是集成命令、文件记录和可视化技术的监控工
具，如（） 。
问题（1）
（A）系统命令
（B）系统调用
（C）系统接口
（D）系统功能
问题（2）
（A）Windows 的 netstat
（B）Linux 的 iptables
（C）Windows 的 Perfmon
（D）Linux 的 top

**正确答案：** A,C

**解析**

本题考察的是系统监视的常用方法与工具。系统监视是系统管理员和运维人员保障系统稳
定运行的重要手段，常见方式包括直接使用命令、查阅日志文件、使用综合监控工具等。
问题1：
A选项 系统命令：例如 UNIX/Linux 系统中的 ps（查看进程状态）、last（查看登录记录）等，属
于通过命令直接获取系统运行状态的方法，符合题意，正确。
B选项 系统调用：这是应用程序与操作系统内核交互的接口，用于实现功能调用，不是直接用于系
统监视的主要方式，错误。
C选项 系统接口：指系统对外提供的调用接口，不特指监视功能，错误。
D选项 系统功能：过于笼统，没有明确对应到命令监视方式，错误。
 因此，选择 A。
问题2：

A选项 Windows 的 netstat：用于查看网络连接和端口状态，属于命令工具，不是集成可视化的综
合监控工具，错误。
B选项 Linux 的 iptables：是防火墙规则配置工具，不是监控工具，错误。
C选项 Windows 的 Perfmon：性能监视器（Performance Monitor），集成了命令、日志、可视化图
表等功能，可监控 CPU、内存、磁盘、网络等多方面的性能，是典型的综合监控工具，正确。
D选项 Linux 的 top：用于动态查看进程和资源占用情况，但主要是命令行工具，不是集成可视化
监控的工具，错误。
 因此，选择 C。

### （2019年11月·第14题）

通常用户采用评价程序来评价系统的性能，评测准确度最高的评价程序是（）。在计算机性能评估
中，通常将评价程序中用得最多、最频繁的（）作为评价计算机性能的标准程序，称其为基准测试
程序。
问题（1）
（A）真实程序
（B）核心程序
（C）小型基准程序
（D）核心基准程序
问题（2）
（A）真实程序
（B）核心程序
（C）小型基准程序
（D）核心基准程序

**正确答案：** A,B

**解析**

本题考察的是计算机系统性能评价方法的分类与应用。
 性能评估的方法包括使用真实程序、核心程序、小型基准程序和合成基准程序等，不同方法的精确
度与适用性不同。
问题1：
A选项真实程序：直接选取实际用户使用的应用程序作为测试对象，测试结果能最真实反映系统在真
实环境下的性能表现，评测准确度最高，正确。
B选项核心程序：从真实程序中提取最常用的部分，虽然代表性较高，但不如完整的真实程序全面
，错误。
C选项小型基准程序：为特定功能或算法设计的小规模程序，便于比较不同系统，但不能全面反映实
际性能，错误。
D选项核心基准程序：并非性能评估的标准分类，错误。
 所以选择 A。
问题2：
A选项真实程序：并非基准测试的标准形式，而是整体应用程序，错误。
B选项核心程序：指从真实程序中提取出最频繁、最重要的计算部分，用作标准测试程序，是目前公
认较好的系统性能评估方法，正确。
C选项小型基准程序：适合快速对比，但覆盖面较窄，错误。
D选项核心基准程序：非标准分类，错误。
 所以选择 B。

### （2016年11月·第13题）

把应用程序中应用最频繁的那部分核心程序作为评价计算机性能的标准程序，称为（）程序。
（）不是对 Web 服务器进行性能评估的主要指标。

问题（1）
（A）仿真测试
（B）核心测试
（C）基准测试
（D）标准测试
问题（2）
（A）丢包率
（B）最大并发连接数
（C）响应延迟
（D）吞吐量

**正确答案：** C,A

**解析**

本题考察的是基准测试程序与 Web 服务器性能评估指标。
问题1：
A选项 仿真测试：是通过模拟应用环境来评估性能，不特指核心程序测试，错误。
B选项 核心测试：虽然名称接近，但行业通用称谓是“基准测试”，错误。
C选项 基准测试：选取应用程序中最频繁使用的核心部分作为标准程序来评估系统性能，是公认的
测试方法，正确。
D选项 标准测试：泛指按照统一标准进行的测试，不特指此类性能测试，错误。
 所以选择 C。
问题2：
A选项 丢包率：属于网络质量指标，不是 Web 服务器性能的主要评价指标，正确。
B选项 最大并发连接数：衡量服务器同时处理连接的能力，是重要性能指标，错误。
C选项 响应延迟：反映处理请求所需时间，是重要性能指标，错误。
D选项 吞吐量：衡量单位时间内处理请求的数量，是重要性能指标，错误。
 所以选择 A。

### （2015年11月·第13题）

为了测试新系统的性能，用户必须依靠评价程序来评价机器的性能，以下4种评价程序，（）评测的
准确程度最低。
问题（1）
（A）小型基准程序
（B）真实程序
（C）核心程序
（D）合成基准程序

**正确答案：** D

**解析**

本题考察的是系统性能评价方法中各种基准程序的特点和准确度差异。
 系统性能测试常用的程序类型包括真实程序、核心程序、小型基准程序和合成基准程序。它们的评
测准确度依次递减：真实程序 > 核心程序 > 小型基准程序 > 合成基准程序。真实程序最能反映实
际工作负载，而合成基准程序由于高度抽象和模拟性强，偏离真实环境较多，因此准确度最低。
A选项小型基准程序：此类程序规模较小，通常选取部分典型功能或操作进行测试，比合成基准程序
更接近实际，但仍无法完整反映实际应用场景，准确度中等偏低。不是四者中最低的。
B选项真实程序：直接使用真实的业务负载或实际应用程序进行测试，能最真实地反映系统在实际环
境中的性能，因此准确度最高。
C选项核心程序：从真实程序中抽取出使用频率最高、最能体现系统特点的部分形成测试程序，其准
确度仅次于真实程序，不是最低。
D选项合成基准程序：为测试系统性能而人为构造的程序，它通常模拟系统的典型行为，但不可避免
与真实场景存在差距，因此评测结果的可信度最低。此项符合题意。

### （2014年11月·第11题）

在实际应用中，用户通常依靠评价程序来测试系统的性能。以下评价程序中，（）的评测准确程度
最低。事务处理性能委员会（Transaction Processing Performance Council, TPC）是制定商务应
用基准程序（benchmark）标准规范、性能和价格度量，并管理测试结果发布的非营利组织，其发布
的TPC-C是（）的基准程序。
问题（1）
（A）核心程序
（B）真实程序
（C）合成基准程序
（D）小型基准程序
问题（2）
（A）决策支持
（B）在线事务处理
（C）企业信息服务
（D）联机分析处理

**正确答案：** C,B

**解析**

本题考察的是系统性能测试的评价程序类型及 TPC 基准程序应用领域。
问题1：
 不同类型的性能评价程序，其评测准确性排序为：真实程序 > 核心程序 > 小型基准程序 > 合成
基准程序。
A选项 核心程序：抽取应用程序中最关键的部分进行评测，准确性较高。
B选项 真实程序：直接使用实际应用程序进行测试，准确性最高。
C选项 合成基准程序：通过构造模拟典型负载特征的程序进行测试，与实际环境差距较大，准确性
最低，正确。
D选项 小型基准程序：是提炼的小规模测试程序，准确性居中。
 选择选项 C。
问题2：
A选项 决策支持：TPC-H/TPC-DS 属于决策支持型基准，不是 TPC-C 的范畴。
B选项 在线事务处理：TPC-C 是 OLTP（Online Transaction Processing）系统的基准测试，模拟
订单处理、支付、发货等业务场景，正确。
C选项 企业信息服务：非 TPC-C 测试重点。
D选项 联机分析处理：对应 OLAP，通常使用 TPC-H/TPC-DS，而非 TPC-C。
 选择选项 B。

### （2013年11月·第12题）

把应用程序中应用最频繁的那部分核心程序作为评价计算机性能的标准程序，称为（）程序。
（）不是对 Web 服务器进行性能评估的主要指标。
问题（1）
（A）仿真测试
（B）核心测试
（C）基准测试
（D）标准测试
问题（2）
（A）丢包率
（B）最大并发连接数
（C）响应延迟

（D）吞吐量

**正确答案：** C,A

**解析**

本题考察的是计算机性能评估方法与 Web 服务器性能指标。
问题1：
A选项仿真测试：利用模拟环境来评估性能，但不特指频繁使用的核心程序，错误。
B选项核心测试：并不是标准的计算机性能评价术语，错误。
C选项基准测试：指从应用程序中抽取最常用、最具代表性的核心程序，作为标准程序测试计算机性
能，是目前通用的评测方法，正确。
D选项标准测试：是泛称，不是具体的性能评价方法，错误。
 所以选择 C。
问题2：
A选项丢包率：属于网络传输性能指标，而非 Web 服务器本身的性能评估指标，错误，正确答案应
选此项。
B选项最大并发连接数：衡量服务器同时支持的连接能力，是 Web 服务器的重要指标，正确。
C选项响应延迟：反映用户请求到服务器响应的时长，是性能评估的重要指标，正确。
D选项吞吐量：反映单位时间内服务器处理请求的能力，是常见性能指标，正确。
 所以选择 A。
