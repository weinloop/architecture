# CMMI · 理论基础与五级成熟度

**知识点编号**：1.4.2  
**权重**：次重点 ★★★☆☆

## 核心要点

- **CMMI**：CMM 的 **整合升级**；强调 **系统工程 + 软件工程**，适合 **系统集成** 类企业。  
- **四大知识域**（口诀 **细软开工**）：**系统工程（SE）**、**软件工程（SW）**、**集成产品与过程开发（IPPD）**、**供应商采购（SS）**。

### 五个成熟度级别（CMMI）

| 级别 | 要点 |
|------|------|
| **初始级** | 过程 **随意、混乱** |
| **已管理级** | **项目级** 过程：策划、文档化、执行、监督、控制 |
| **已定义级** | **组织级** 标准过程 **制度化** |
| **量化管理级** | 产品、服务、过程性能 **定量目标** |
| **优化级** | 持续改进，**动态调整** 过程 |

**口诀**：**储罐定粮油** — 储（初）罐（管）定（定）粮（量）油（优）。*储罐里放粮油。*

## 考点提示

CMMI 与 CMM **级别命名不完全相同**（如 CMMI 第二级为 **已管理级**）；勿混淆两套口诀与顺序细节，以 **教材表格** 为准背诵。

### （2022年11月·第20题）
