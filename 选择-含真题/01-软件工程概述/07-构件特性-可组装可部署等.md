# 构件的定义与特性（萨默维尔）

**知识点编号**：1.2.4.1  
**权重**：重点 ★★★★★

## 核心要点

| 特性 | 要点 |
|------|------|
| **可组装性** | 外部交互经 **公开接口**；对外可访问自身信息（方法、属性等） |
| **可部署性** | **自包含**；常以 **二进制** 形式在构件平台运行，部署前 **无须编译**（服务可由提供者部署） |
| **文档化** | 完全文档化；接口 **语法乃至语义** 可界定 |
| **独立性** | 无特殊依赖即可组装部署；需外部服务时在 **请求接口** 显式声明 |
| **标准化** | 符合 **标准构件模型**（接口、元数据、文档、组成、部署等） |
| **可见性** | **无外部可见状态**（特例：可由容器管理状态对外呈现；不影响功能的属性可对外可见） |

**口诀**：竹鼠挡毒标 — 竹（组装）鼠（部署）档（文档）毒（独立）镖（标准化）。

## 考点提示

选择题常考特性描述与 **可见状态** 的表述；口诀辅助记忆六项。

## 系统架构设计师 · 真题汇编

### （2022年11月·第31题）
有关构件特性，不正确的是（）。  
**答案**：D（“作为部署单元是可拆分的”）  
**要点**：部署单元应完整、不可随意拆分破坏封装。

### （2022年11月·第32题）
（）是一个已命名的一组操作的集合。  
**答案**：A 接口  
**要点**：构件对外交互通过接口。

### （2022年11月·第34题）
有关构件演化，不正确的是（）。  
**答案**：D（安装新版本必须终止所有已有版本—过于绝对）  
**要点**：实际可支持多版本共存、热部署等策略。

### （2021年11月·第27题）
关于软件构件，错误的是（）。  
**答案**：C（同一进程中可有多个同一构件拷贝—与“至多一个拷贝”类表述冲突）  
**要点**：以教材/真题解析为准，记“拷贝数量”易错。

### （2017年11月·第26题）
构件与对象不同，构件（）。  
**答案**：B（可利用容器管理对外可见状态）  
**要点**：构件原则上无外部可见状态，但可由容器托管呈现。

### （2023年11月·第48题）
关于构件的说法，错误的是（）。  
**答案**：B（构件具有外部可见状态）  
**要点**：规范表述为无外部可见状态（或经容器例外）。

### （2023年11月·第53题）
（1）外部交互必经公开接口且可提供自身信息访问；（2）自包含、二进制、部署前无须编译。  
**答案**：（1）A 可组装性；（2）B 可部署性  
**要点**：与卡片表格一一对应。

### （2024年5月·第22题）
关于构件，不正确的是（）。  
**答案**：C（构件不能配置以适应系统）  
**要点**：构件应支持配置以适应不同应用。

### （2017年11月·第27题）
接口标准化是对（）的标准化。  
**答案**：B（接口中消息模式、格式和协议）  
**要点**：不仅是命名或单一数据格式。

### （2016年11月·第25题）
面向构件编程 COP 所需基本支持包括（）。  
**答案**：C（多态性、模块封装性、后期绑定和装载、安全性）  
**要点**：COP 与 OO 相关但强调组装与后期绑定。

### （2021年11月·第28题）
构件交互复杂性中，（）易产生数据竞争和死锁。  
**答案**：A 多线程  
**要点**：并发与同步问题。
