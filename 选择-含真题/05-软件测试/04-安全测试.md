# 安全测试

**知识点编号**：5.4  
**权重**：次重点 ★★★☆☆

## 核心要点

从**应用安全**与**数据安全**等维度发现隐患、验证防护能力。

| 类型 | 内容要点 |
|------|-----------|
| 数据加密 | 存储/传输加密，算法是否合规 |
| 身份验证 | 非法登录、密码策略、重试限制、越权访问 |
| 日志 | 事务、错误、IP、用户等可追溯 |
| Session | 超时、回退、会话数据保护 |
| 备份与恢复 | 全量/增量/差量；**RTO/RPO** |
| 访问控制 | 权限隔离、策略与实现一致 |
| 漏洞 | **XSS、SQL/命令注入**等；Burp 等工具 |
| 网络 | 端口最小暴露（如 80/443） |
| 服务端脚本 | 上传、目录遍历、文件包含等 |
| 防火墙 | 规则、流量、抗 DDoS 等 |

### （2025年5月·第58题）

在软件测试过程中，回归测试的主要目的是哪一项（）。
问题（1）
（A）确认系统满足所有非功能性需求
（B）验证新增功能是否按设计工作
（C）找出由于修改缺陷或代码变更引入的新问题
（D）提高测试覆盖率，确保所有路径都被测试过

**正确答案：** C

**解析**

本题考察的是软件测试中“回归测试”的目的。
A项错误，非功能性需求测试通常属于性能测试、安全测试等，不是回归测试的主要目标。
B项错误，验证新增功能通常是功能测试或确认测试的内容。
C项正确，回归测试的核心目的是在已有系统上进行修改（如修复缺陷、添加新功能）后，确认这些
变更没有引入新的缺陷或破坏已有功能。
D项错误，提高测试覆盖率是测试设计策略的目标，而不是回归测试的直接目的。
 因此，选择选项C。
