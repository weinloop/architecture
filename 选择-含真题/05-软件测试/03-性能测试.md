# 性能测试

**知识点编号**：5.3  
**权重**：次重点 ★★★☆☆

## 核心要点

| 类别 | 要点 |
|------|------|
| **负载测试** | 性能指标**已有一项或多项达瓶颈**时，让系统**继续运行一段时间**，观测运行中各项指标；即在**已达最大负载（压力）**时的测试 |
| **压力测试** | **持续加压（加负载）**，测各项性能指标，直到 CPU 等某指标**达瓶颈**为止（强调「加压过程」） |
| **并发测试** | 负载+压力结合思路：递增加载找瓶颈，综合交易与资源指标评估并发能力 |
| **容量测试** | 针对某指标**极限值**（最大并发、库表规模等）验证在极限下是否仍可用 |
| **可靠性测试** | 提高/评估可靠性；发现影响可靠性的缺陷 |
| **强度测试** | **异常/恶劣/超负荷**下系统是否扛得住、性能下降是否可接受（含最差环境） |
| **配置测试** | 寻**最优资源分配**；Web 客户端环境多变，选典型配置组合 |
| **速度测试** | 网络连接速度 + 关键业务处理速度 |

**辨析**：**强度**偏「造极限/恶劣条件」；**压力**偏「持续加压过程」。

### （2019年11月·第28题）

软件性能测试有多种不同类型的测试方法，其中，（）用于测试在限定的系统下考查软件系统极限
运行的情况，（）可用于测试系统同时处理的在线最大用户数量。
问题（1）
（A）强度测试
（B）负载测试
（C）压力测试
（D）容量测试
问题（2）
（A）强度测试
（B）负载测试
（C）压力测试
（D）容量测试

**正确答案：** A,D

**解析**

此题考察软件测试的相关概念。
 问题 1：用于测试在限定的系统资源条件下考查系统极限运行的情况。
本题考察的是软件性能测试中“强度测试”的概念。
A选项强度测试：模拟极端恶劣的环境（如网络不稳定、低内存等）来测试系统在资源受限条件下的
稳定性，符合题意。
B选项负载测试：关注系统在逐步增加负载下的性能变化，不专注于资源极端条件。
C选项压力测试：测试系统在超负荷条件下的反应，主要关注系统的崩溃点，与题意描述的“限定资
源”不同。
D选项容量测试：是测试系统能处理的最大并发用户数量，不涉及极限资源条件。
 因此，正确答案为 A. 强度测试。
问题 2：用于测试系统可同时处理的在线最大用户数量。
本题考察的是“容量测试”的定义，也被称为并发测试。
A选项强度测试：关注资源匮乏下的运行，不涉及并发用户数。
B选项负载测试：测试系统在不同负载条件下的响应时间和稳定性，但不专注于并发用户容量。
C选项压力测试：测试系统的崩溃点，着重于极限压力下的稳定性，与并发用户数无关。
D选项容量测试：测试系统能同时处理的最大用户数，符合题干描述。
 因此，正确答案为 D. 容量测试。

### （2017年11月·第31题）

软件确认测试也称为有效性测试，主要验证（）。确认测试计划通常是在需求分析阶段完成的。根
据用户的参与程度不同，软件确认测试通常包括（）。
问题（1）
（A）系统中各个单元模块之间的协作性
（B）软件与硬件在实际运行环境中能否有效集成
（C）软件功能、性能及其他特性是否与用户需求一致
（D）程序模块能否正确实现详细设计说明中的功能、性能和设计约束等要求
问题（2）
（A）黑盒测试和白盒测试
（B）一次性组装测试和增量式组装测试
（C）内部测试、Alpha、Beta 和验收测试
（D）功能测试、性能测试、用户界面测试和安全性测试

**正确答案：** C,C

**解析**

本题考察的是软件确认测试的概念与分类。
 确认测试（Validation Testing）也称为有效性测试，旨在确保软件产品与用户需求完全一致，而
不仅仅是符合设计文档。确认测试的计划通常在需求分析阶段完成，并且在测试过程中会有用户或
用户代表的参与。
问题1：
A选项 系统中各个单元模块之间的协作性：这是集成测试（Integration Testing）的主要目标，错
误。
B选项 软件与硬件在实际运行环境中能否有效集成：这是系统测试或验收测试中的一部分，但不是
确认测试的定义重点，错误。
C选项 软件功能、性能及其他特性是否与用户需求一致：这是确认测试的核心目的，验证是否满足
用户需求，正确。
D选项 程序模块能否正确实现详细设计说明中的功能、性能和设计约束等要求：这是验证测试
（Verification Testing）的目标，错误。
 所以选择 C。
问题2：
A选项 黑盒测试和白盒测试：这是测试方法的分类，与确认测试的用户参与程度无关，错误。
B选项 一次性组装测试和增量式组装测试：这是集成测试的策略分类，错误。
C选项 内部测试、Alpha、Beta 和验收测试：这是按照用户参与程度划分的确认测试类型，正确。
内部测试由开发团队完成，Alpha 测试在受控环境中进行，Beta 测试在用户环境中进行，验收测试
由用户最终确认软件可交付。
D选项 功能测试、性能测试、用户界面测试和安全性测试：这是按测试内容划分的类型，不是按用
户参与程度分类，错误。
 所以选择 C。

### （2023年11月·第6题）

下列属于 Web 新型测试方法的是（）。
问题（1）
（A）A/B测试
（B）α测试
（C）β测试
（D）压力测试

**正确答案：** A

**解析**

本题考察的是Web应用测试方法的相关概念。
A选项A/B测试：A/B测试是一种常见的Web新型测试方法，广泛用于Web和App应用中。其基本思想是
将用户随机分为两组或多组，分别接受不同版本的页面或功能，通过对比各组的表现来评估哪一种
方案更能满足目标需求。A/B测试可以帮助产品团队做出数据驱动的决策，常用于用户体验、界面设
计等方面的优化。
B选项α测试：α测试是软件开发过程中的早期测试，通常由开发团队内部的成员进行，目的是在软
件发布前发现和修复错误。它并不是Web应用中新型的测试方法，主要适用于传统软件的测试。
C选项β测试：β测试是产品发布前的第二阶段测试，通常由外部用户进行。测试目的是通过用户反
馈发现产品的问题。它也不是Web新型测试方法，主要适用于软件发布前的最终用户验证。
D选项压力测试：压力测试用于测试系统在超负荷情况下的表现，评估其在高负载条件下的稳定性和
性能。它是性能测试的一种类型，不是Web新型测试方法，而是传统测试方法。
 因此，选项 A 正确。

### （2025年5月·第17题）

以下哪一项不属于 Web 服务器的性能评测方法（）。
问题（1）
（A）基准性能测试
（B）压力测试
（C）可靠性测试
（D）UI 测试

**正确答案：** D

**解析**

本题考察的是Web 服务器性能评测的常用测试方法。
A. 基准性能测试（Benchmarking） 正确，属于性能测试的一种。用于评估服务器在正常负载下的
响应时间、吞吐量等性能指标。
B. 压力测试（Stress Testing） 正确，用于测试服务器在超出正常负载情况下的性能表现，如系
统崩溃点、恢复能力等。
C. 可靠性测试（Reliability Testing） 正确，在广义上用于测试系统长期运行下的稳定性，通常
也包含在服务器性能综合评估中。
D. UI 测试（User Interface Testing） 错误，UI 测试是针对前端界面交互性、可用性、界面逻
辑的测试，与 Web 服务器性能无直接关系。

### （2025年11月·第16题）

测试精确度从高到低排列（）。
问题（1）
（A）真实程序、核心程序、小型基准测试、合成基准测试
（B）真实程序、小型基准测试、合成基准测试、核心程序
（C）核心程序、小型基准测试、合成基准测试、真实程序
（D）真实程序、合成基准测试、小型基准测试、核心程序

**正确答案：** A

**解析**

本题考察的是系统性能测试中不同基准程序的精确度层级。
 基准测试程序可以分为真实程序、核心程序、小型基准程序和合成基准程序，它们的精确度依次递
减：真实程序最贴近实际，核心程序提取关键部分，小型基准程序规模更小但代表性有限，而合成
基准程序模拟性最强、偏差最大。掌握其排序有助于根据测试场景选择合适的评测工具。选择选项
A。

### （2025年11月·第57题）

（）是通过确定一个系统的瓶颈或者不能接受的性能点，来获得系统能提供的最大服务级别的测试
。
问题（1）
（A）压力测试
（B）基准测试
（C）负载测试
（D）稳定性测试

**正确答案：** A

**解析**

本题考察的是性能测试类型中“压力测试（Stress Testing）”的概念与作用。
 在性能测试中，不同的测试类型有不同的目的：
负载测试（Load Testing）：验证系统在正常或预期负载下的性能表现；
压力测试（Stress Testing）：验证系统在极限负载条件下的表现，确定性能瓶颈和最大承载能
力；
稳定性测试（Stability/Endurance Testing）：验证系统长时间运行的稳定性；
基准测试（Benchmark Testing）：建立性能基准，用于对比和回归测试。

A选项压力测试：通过不断增加系统负载，超出系统设计的正常工作范围，以确定系统的瓶颈、性能
极限和不可接受的性能点。题干所述“确定系统瓶颈或不能接受的性能点”正是压力测试的核心目
标，因此该选项正确。
B选项基准测试：是为了获取系统在特定环境下的性能基线，用于对比不同系统或同一系统不同版本
间的性能，不用于发现瓶颈，因此错误。
C选项负载测试：是模拟系统在正常或预期负载下的运行情况，目的是评估系统的性能指标是否达标
，而不是找出系统的最大承载极限，因此错误。
D选项稳定性测试：又称持久性测试，用于评估系统在长时间运行下的性能稳定性，例如是否存在内
存泄漏或性能衰减等，不涉及性能极限的确定，因此错误。
因此，正确答案为 A。
