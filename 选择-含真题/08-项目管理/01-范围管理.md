# 范围管理

**知识点编号**：8.1  
**权重**：次重点 ★★★☆☆

## 核心要点

确定项目**边界**：哪些工作属于本项目，保证干系人对**产品**与**过程**有共同理解。

软考教材常见 **5 过程**（与 PMBOK 六过程差在是否单列「收集需求」，以教材为准）：

| 过程 | 作用 |
|------|------|
| **编制范围管理计划** | 如何定义、确认、控制范围 |
| **定义范围** | 形成**项目范围说明书** |
| **创建 WBS** | 自上而下拆到**工作包**，总和 = 项目全部工作 |
| **确认范围** | **正式验收**已完成的可交付成果 |
| **范围控制** | 监督状态、管理**范围基准**变更 |

### （2017年11月·第16题）

项目范围管理中，范围定义的输入包括（）。
问题（1）
（A）项目章程、项目范围管理计划、产品范围说明书和变更申请
（B）项目范围描述、产品范围说明书、生产项目计划和组织过程资产
（C）项目章程、项目范围管理计划、组织过程资产和批准的变更申请
（D）生产项目计划、项目可交付物说明、信息系统要求说明和项目质量标准

**正确答案：** C

**解析**

本题考察的是项目范围管理中范围定义过程的输入项。
 根据高项考试的知识框架，范围定义的输入主要包括：
项目章程：提供项目的高层描述与总体目标；
项目范围管理计划：说明如何定义和管理范围；
组织过程资产：包括组织的标准、历史档案和经验教训等；
批准的变更申请：如果项目范围有调整，必须纳入已批准的变更内容。
A选项：包含了“产品范围说明书”和“变更申请”，但缺少“组织过程资产”，且“变更申请”未
限定为“批准的”，错误。
B选项：包含“项目范围描述”和“产品范围说明书”，不是PMBOK所列标准输入项，错误。
C选项：完全符合考试参考答案及教材中的旧版定义，正确。
D选项：属于其他项目管理过程的输出或质量管理相关文档，不是范围定义输入，错误。
 选择选项 C。

### （2015年11月·第18题）

关于项目范围管理描述，正确的是（）。
问题（1）
（A）项目范围是指信息系统产品或者服务所应包含的功能
（B）项目范围描述是产品范围说明书的重要组成部分
（C）项目范围定义是信息系统要求的度量
（D）项目范围定义是生产项目计划的基础

**正确答案：** D

**解析**

本题考察的是项目范围管理的基本概念和范围定义的作用。
A选项项目范围是指信息系统产品或者服务所应包含的功能：错误。该描述是对产品范围的定义，而
不是项目范围。项目范围包括为了交付这些产品或服务所必须完成的所有工作，因此项目范围远不
止产品功能。
B选项项目范围描述是产品范围说明书的重要组成部分：错误。实际上是产品范围说明书为制定项目
范围说明书**提供基础和依据，二者之间的逻辑顺序反了。
C选项项目范围定义是信息系统要求的度量：错误。项目范围定义关注的是“做什么”与“不做什么
”，是项目边界的定义，不是对系统需求进行量化的度量方式。
D选项项目范围定义是生产项目计划的基础：正确。明确的项目范围是制定项目计划（包括时间表、
成本预算、人力资源配置等）的前提，因此项目范围定义是项目计划编制的基础。
 因此，选项 D 正确。

### （2013年11月·第17题）

详细的项目范围说明书是项目成功的关键，（）不属于项目范围定义的输入。
问题（1）
（A）项目章程
（B）项目范围管理计划
（C）批准的变更申请
（D）项目文档管理方法

**正确答案：** D

**解析**

本题考察的是项目范围管理中“定义范围”过程的输入内容。
A选项项目章程：是范围定义的重要输入之一，它提供了高层次的项目描述、目标、主要干系人和总
体边界，为后续的范围界定提供了基本依据，因此属于有效输入。
B选项项目范围管理计划：是项目管理计划中的组成部分，定义了如何制定、监控和控制项目范围
，是指导范围定义工作的主要依据，也属于该过程的输入。
C选项批准的变更申请：虽然不是PMBOK第六版以后定义范围过程的标准输入，但在一些老版本教材
或项目调整背景中，作为变更引发重新定义范围的触发条件，也有可能被提及，因此并不完全错误
。
D选项项目文档管理方法：是项目整体管理中的一部分，涉及文档的创建、控制与存储方式，与具体
的范围定义过程无直接关系，不属于定义范围的输入。
 因此，正确答案是 D。
