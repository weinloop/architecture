# 可靠性的定量描述

**知识点编号**：7.2  
**权重**：重点 ★★★★★

## 核心要点

| 指标 | 英文提示 | 含义 |
|------|-----------|------|
| **MTBF** | Between Failures | 两次故障间平均时间；教材口径常含检测与修复时段 |
| **MTTR** | Repair/Recover | **修复**平均时间 |
| **MTTA** | **Acknowledge** | 告警到**确认**平均时间 |
| **MTTF** | **Failure** | 运行到**失效**（不含修）平均时间 |
| **MTTD** | **Detect** | 故障发生到**被检出**平均时间 |

- **MTBF ≈ MTTF**（当 **MTTR、MTTD** 很小时，真题常考）。  
- **失效概率 F(t)**：随时间增大，t→∞ 趋近 1。  
- **可靠度 R(t)**：规定时间内**不失效**的概率，**R(t)=1−F(t)**，R(0)=1。  
- **规定时间**：常用 **CPU 执行指令时间总和**度量运行时间（教材表述）。

**可用性**：与可靠度、维修等相关；区别于纯**性能**指标（如数据延迟）。

**可靠性测试**：**广义**侧重获取数据、**评价**可靠性；**狭义**在给定**操作/运行剖面**下测可靠性（与「以找缺陷为主」的泛化测试区分）。

### （2022年11月·第15题）

系统（）是指在规定的时间内和规定条件下能有效地实现规定功能的能力。它不仅取决于规定的使
用条件等因素，还与设计技术有关。常用的度量指标主要有故障率（或失效率）、平均失效等待时
间、平均失效间隔时间和可靠度等。其中，（）是系统在规定工作时间内无故障的概率。
问题（1）
（A）可靠性
（B）可用性
（C）可理解性
（D）可测试性
问题（2）
（A）失效率
（B）平均失效等待时间
（C）平均失效间隔时间
（D）可靠度

**正确答案：** A,D

**解析**

本题考察的是系统可靠性及可靠性度量指标的基本概念。
 可靠性强调在规定时间与条件下无故障完成规定功能；可用性强调“能用”的时间占比，它受可靠
性与可维护性共同影响；常见度量中，可靠度R(t)刻画“在规定时间内无故障”的概率，是可靠性
的核心概率度量。
问题 1：
A选项可靠性：指系统在规定时间、规定条件下完成规定功能的能力，符合题干定义，正确。
B选项可用性：指系统处于可运行状态的时间比例，通常与可靠性、可维修性（如MTTR）共同决定
，不等同于题干所给定义。
C选项可理解性：指理解系统结构与代码/文档的难易程度，属于可维护性子特性之一，与题干无关
。
D选项可测试性：指验证系统是否满足需求的难易程度，也属于可维护性子特性，与题干无关。
 选择选项 A。
问题 2：
A选项失效率：单位时间内发生故障的速率（常记为λ），描述故障发生强度，不是“在规定时间内
无故障的概率”。
B选项平均失效等待时间：从失效到开始维修的平均等待时长，偏向可维护性指标，不是无故障概率
。
C选项平均失效间隔时间：两次相邻失效之间的平均时间（常与MTBF/MTTF相关），是时间指标而非
概率。
D选项可靠度：系统在规定工作时间内无故障的概率，通常记为R(t)，与题干后半句完全一致，正确
。
 选择选项 D。

### （2022年11月·第50题）

平均失效等待时间（mean time to failure，MTTF）和平均失效间隔时间（mcan time between
failure，MTBF）是进行系统可靠性分析时的要指标，在失效率为常数和修复时间很短的情况下
，（）。
问题（1）
（A）MTTF 远远小于 MTBF
（B）MTTF 和 MTBF 无法计算
（C）MTTF 远远大于 MTBF

（D）MTTF 和 MTBF 几乎相等

**正确答案：** D

**解析**

本题考察的是系统可靠性分析中的四个核心指标，包括 MTTF、MTTR、MTBF 与系统可用性
公式。
 平均失效等待时间（MTTF）= 1/λ，λ 为失效率。
 平均故障修复时间 (MTTR) = 1/μ，μ 为修复率。
 平均失效间隔时间又称平均无故障时间(MTBF) = MTTR + MTTF。
 系统可用性 = MTTF/(MTTR+MTTF)×100%

A选项 MTTF 远远小于 MTBF：在 MTTR 很小的情况下，MTBF≈MTTF，不存在“远远小于”的关系
，错误。
B选项 MTTF 和 MTBF 无法计算：在失效率 λ 和修复率 μ 已知时，这两个指标是可以直接计算的
，错误。
C选项 MTTF 远远大于 MTBF：与实际公式 MTBF = MTTF + MTTR 不符，错误。
D选项 MTTF 和 MTBF 几乎相等：公式 MTBF = MTTF + MTTR，当 MTTR 很小时，MTBF 与 MTTF 的差
值可以忽略，因此几乎相等，正确。
 选择选项 D。

### （2024年11月·第13题）

下列不属于可用性的选项是（）。
问题（1）
（A）可用时间
（B）可用时间间隔
（C）数据延迟时间

（D）故障间隔

**正确答案：** C

**解析**

本题考察的是系统可用性（Availability） 的相关概念。
A选项可用时间：指系统在一定时间内能够提供正常服务的总时间，是衡量系统可用性的直接指标。
时间越长，系统可用性越高，因此属于可用性指标。
B选项可用时间间隔：指系统能够持续正常运行的时间长度，实际上与故障间隔（MTBF）密切相关
，也用于评估系统的稳定性和可用性。
C选项数据延迟时间：指数据在处理或传输过程中所经历的延迟时间，反映的是系统的性能，尤其是
响应时间或吞吐率方面的指标，而非系统是否“可用”，所以它不属于可用性范畴。
D选项故障间隔：即平均无故障时间（MTBF），是可用性的常用度量指标之一。故障间隔越长，说明
系统更稳定、更可用。
 因此，选项 C 数据延迟时间不属于系统可用性的指标，正确答案为 C。

### （2025年11月·第19题）

系统运行过程中，其失效率函数 f(t) 用于描述系统在时间 t 发生故障的概率特性。关于失效率函
数 f(t) 的下列说法中，哪一项是错误的？（）
问题（1）
（A）f(0)=0，表示系统在刚开始运行时不会发生故障
（B）所有系统最终都会出现故障
（C）f(t) 在 (0，+∞) 上单调递减
（D）f(t) 的取值范围在 0 到 1 之间

**正确答案：** C

**解析**

本题考察的是可靠性工程中失效率函数（hazard rate）f(t） 的定义及性质。
A选项 f(0)=0，表示系统刚运行时不会发生故障：这是典型的“早期失效忽略”假设，可以认为正
确。
B选项 所有系统最终都会出现故障：可靠性工程假设系统在无限时间下故障概率趋近于 1，即系统
不会永远正常工作，因此该说法通常视为正确。
C选项 f(t) 在 (0，+∞) 上单调递减：单调递减肯定是错误，因为一开始是0，最终趋近于 1。因

此此选项错误。
D选项 f(t) 在 0 到 1 的范围内：失效率是概率肯定是在 0-1 之间。
 选择选项 C。

### （2025年11月·第20题）

下列关于可靠性测试的说法中，哪一项是错误的（）。
问题（1）
（A）定义系统剖面是广义可靠性测试的重要手段
（B）可以通过可靠性测试发现系统薄弱点
（C）广义可靠性测试是指通过测试发现软件缺陷并提高软件质量
（D）狭义可靠性测试是指在给定操作剖面的条件下，通过获取可靠性数据来评价软件可靠性

**正确答案：** C

**解析**

本题考察的是可靠性测试的分类及其目的。
广义可靠性测试：强调对软件可靠性的整体评价过程，包括定义运行剖面、设计测试用例、执行测
试、收集可靠性数据、分析失效并排错等。其核心目标是获取可靠性数据并评价软件可靠性，而非
以发现缺陷为主要目的。
狭义可靠性测试：为了获取可靠性数据，在给定操作剖面（Operational Profile）下执行预先设计
的测试用例，模拟用户实际使用过程，用测试数据评价软件的可靠性。具有“面向缺陷”和“模拟
实际使用场景”的特点。
A选项 定义系统剖面是广义可靠性测试的重要手段：广义可靠性测试的关键步骤之一就是定义运行
剖面（操作剖面），用于模拟系统真实环境中的使用情况，为后续测试用例设计和可靠性评价提供
依据。因此，正确。
B选项 可以通过可靠性测试发现系统薄弱点：此说法正确。可靠性测试不仅能够收集可靠性数据
，还可以暴露系统中的薄弱点，例如模块易失效区域、设计不完善之处，以及在实际工作负载下可
能发生的稳定性问题。这也是可靠性测试的主要价值之一。
C选项 广义可靠性测试是指通过测试发现软件缺陷并提高软件质量：这是错误的。广义可靠性测试
的核心不在“发现缺陷”，而在“评价可靠性”。
D选项 狭义可靠性测试是指在给定操作剖面的条件下，通过获取可靠性数据来评价软件可靠性。正
确。
 选择选项 C。

### （2025年11月·第22题）

在单片机系统性能与可靠性评价指标中，下列组合中哪一组指标最符合单片机系统常用的考量要素
（）。
问题（1）
（A）吞吐量、MTBF、功耗
（B）吞吐量、MTBF、响应时间
（C）吞吐量、功耗、响应时间
（D）MTBF、功耗、响应时间

**正确答案：** D

**解析**

本题考察的是单片机（嵌入式系统）性能指标与可靠性指标的核心特征。
 单片机系统不同于通用计算机系统，其关注点主要集中在：低功耗、可靠运行能力（如 MTBF）、
实时响应能力（响应时间）。反之，吞吐量是服务器、网络系统、数据库系统中常见的性能指标
，并非嵌入式系统的核心指标。
A选项吞吐量、MTBF、功耗：包含吞吐量，而吞吐量不属于单片机常用评价指标，因此该组合不正确
。
B选项吞吐量、MTBF、响应时间：包含吞吐量，因此也不是单片机主要评价指标组合。

C选项吞吐量、功耗、响应时间：同样包含吞吐量，不符合嵌入式系统评价特性。
D选项MTBF、功耗、响应时间：均为单片机系统最主要的关注指标，因此该选项正确。
