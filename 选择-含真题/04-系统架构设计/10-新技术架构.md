# 新技术架构（云 · 中间件 · 集成等）

**知识点编号**：教材散见 / 真题补充  
**权重**：重点（选择题频）

## 核心要点

- **云计算三层**：**SaaS**（应用/软件即服务）→ **PaaS**（平台）→ **IaaS**（基础设施）；自顶向下对应 **应用层 / 平台层 / 基础设施层**。
- **消息中间件**：异步、松耦合；点对点 / 发布-订阅；**非**「一组对象跨平台」——后者更像**分布式对象**中间件（CORBA/RMI 等）。
- **分布式中间件**：常见分类强调 **交互支持** + **公共服务**（安全、事务等可纳入公共服务）。
- **物联网**：常见 **感知 / 网络 / 应用** 三层 → 多归类为**层次型**架构风格。
- **EAI**：原则强调应用**独立性**；服务层次自底向上通常含通信、数据/消息转换、应用连接等，**最上层常为流程控制**（以真题教材表述为准）。
- **数字化/企业架构等**：抓住「战略—业务—技术」对齐与**治理能力**等关键词，选项排除法。

### （2022年11月·第1题）

云计算服务体系结构如下图所示，图中①、②、③分别与 SaaS PaaS Iaas相对应，图中①、②、
③应为（）
问题（1）
（A）应用层、基础设施层、平台层
（B）应用层、平台层、基础设施层
（C）平合层、应用层、基础设施层
（D）平台层、基础设施层、应用层

**正确答案：** B

**解析**

本题考察的是云计算服务模型与体系结构的基本概念。
①应用层（对应 SaaS）：软件即服务（SaaS）是基于互联网的软件交付模式，用户通过浏览器等方
式直接使用应用程序，无需关心底层平台与硬件环境，如在线办公、在线CRM系统等。
②平台层（对应 PaaS）：平台即服务（PaaS）为开发者提供运行、开发、测试和部署应用程序的平
台环境，包括操作系统、中间件、数据库等，用户专注于应用开发而无需管理底层硬件和系统软件
。
③基础设施层（对应 IaaS）：基础设施即服务（IaaS）通过网络提供虚拟化的计算资源、存储、网
络等基础设施，用户可按需租用计算机硬件资源，如云服务器、云存储等。
 因此，①②③依次为应用层、平台层、基础设施层，选项 B 正确。

### （2022年11月·第12题）

通常，嵌入式中间件没有统一的架构风格，根据应用对象的不同可存在多种类型，比较常见的是消
息中间件和分布式对象中间件。以下有关消息中间件的描述中，不正确的是（）。
问题（1）
（A）消息中间件是消息传输过程中保存消息的一种容器
（B）消息中间件具有两个基本特点：采用异步处理模式、应用程序和应用程序 调用关系为松耦合
关系
（C）消息中间件主要由一组对象来提供系统服务，对象间能够跨平台通信
（D）消息中间件的消息传递服务模型有点对点模型和发布-订阅模型之分

**正确答案：** C

**解析**

本题考察的是消息中间件的概念、特点与传输模式。
 消息中间件主要解决应用系统之间的异步通信和解耦问题，常用于分布式系统的数据传输与集成。
A选项消息中间件是消息传输过程中保存消息的一种容器：正确。消息中间件在消息传递过程中起到
缓冲与路由作用，确保消息可靠送达。
B选项消息中间件具有异步处理模式与松耦合特点：正确。发送方与接收方不需同时在线，且无需直

接调用对方，减少系统耦合度。
C选项主要由一组对象提供系统服务且可跨平台通信：错误。这是分布式对象中间件（如CORBA、
Java RMI）的特点，不是消息中间件的特征。
D选项传输模型有点对点和发布-订阅两种：正确。点对点用于一对一传递，发布-订阅用于一对多广
播式传递。
 因此，选项C错误，正确答案为C。

### （2022年11月·第24题）

在分布式系统中，中间件通常提供两种不同类型的支持，即（）
问题（1）
（A）数据支持和交互支持
（B）交互支持和提供公共服务
（C）数据支持和提供公共服务
（D）安全支持和提供公共服务

**正确答案：** B

**解析**

本题考察的是分布式系统中间件的功能类型。
 中间件作为连接分布式系统中不同组件或应用的桥梁，主要解决异构环境下组件协作与资源共享的
问题。
A选项 数据支持和交互支持：数据支持不是中间件功能的主要分类之一，错误。
B选项 交互支持和提供公共服务：交互支持用于协调不同组件之间的通信与协作；公共服务是可复
用的服务实现（如安全、事务、日志等），可供不同组件调用，完全符合分布式中间件的功能分类
，正确。
C选项 数据支持和提供公共服务：缺少交互支持这一核心功能，错误。
D选项 安全支持和提供公共服务：安全支持属于公共服务的一部分，不是与其并列的中间件功能类
型，错误。
 因此选择 B。

### （2021年11月·第33题）

软件架构风格是描述某一特定应用领域中系统组织方式的惯用模式，按照软件架构风格，物联网系
统属于 （） 软件架构风格。
问题（1）
（A）层次型
（B）事件系统
（C）数据线
（D）C2

**正确答案：** A

**解析**

本题考察的是物联网系统的架构风格。
A选项层次型：正确。物联网系统通常采用三层架构：感知层、网络层和应用层。每一层负责不同的
功能，彼此之间通过接口进行通信，符合层次型架构风格的基本特征。因此，物联网系统属于层次
型架构风格。
B选项事件系统：错误。事件驱动架构适用于系统中大量异步事件交互的场景，如图形界面或部分中
间件，不是物联网整体架构的主要风格。
C选项数据线：错误。数据线（即流水线）风格适用于数据处理过程明确、顺序固定的系统，如编译
器，不适用于物联网系统的整体架构。
D选项C2：错误。C2架构是一种以组件和连接件为核心的架构风格，强调解耦与消息驱动，适合可插
拔系统，但不是描述物联网系统的主流架构风格。
 因此，选项A正确。

### （2019年11月·第16题）

系统应用集成构建统一标准的基础平台，在各个应用系统的接口之间共享数据和功能，基本原则是
保证应用程序的（）。系统应用集成提供了4个不同层次的服务， 最上层服务是（）服务 。
问题（1）
（A）独立性
（B）相关性
（C）互操作性
（D）排他性
问题（2）
（A）通信
（B）信息传递与转化
（C）应用连接
（D）流程控制

**正确答案：** A,D

**解析**

本题考察的是系统应用集成（EAI） 的相关概念与服务层次。
 EAI 通过构建统一平台来集成不同的应用系统，确保数据和功能的共享。其核心原则和提供的服务
层次对于系统的设计与实施至关重要。
问题1：
A选项独立性：EAI的核心原则之一是确保各应用程序的独立性，即各个系统可以独立运行，同时通
过统一平台进行集成。这种独立性有助于系统的灵活性和可扩展性，正确。
B选项相关性：相关性并不是EAI的核心原则，错误。
C选项互操作性：虽然互操作性在EAI中非常重要，但其重点是系统之间能够正常交互，而非应用程
序的独立性，错误。
D选项排他性：排他性并不是EAI的基本原则，它更强调系统之间的协同工作，而非限制性的特性
，错误。
 所以选择 A。
问题2：
通讯服务、信息传递与转化服务、应用连接服务、流程控制服务：这是标准EAI体系的从下至上的四
层结构。
A选项通信：通信服务是EAI提供的最基础的服务，但它属于较低层次的服务，错误。
B选项信息传递与转化：信息传递与转化服务是EAI中的第二层服务，负责确保信息在不同系统之间
的转换和传递，错误。
C选项应用连接：应用连接服务是EAI的第三层服务，用于实现不同应用系统的连接，错误。
D选项流程控制：流程控制服务是EAI的最上层服务，负责业务流程的管理和控制，确保整个系统的
协调性和高效运行，正确。
 所以选择 D。

### （2023年11月·第31题）

CDN 和反向代理的基本原理都是（）。
问题（1）
（A）缓存
（B）负载均衡
（C）路由转发
（D）NAT转换

**正确答案：** A

**解析**

本题考察的是CDN（内容分发网络）和反向代理的基本原理。

A选项 缓存：CDN和反向代理的基本原理确实是缓存。CDN通过在全球各地的分发节点缓存用户常访
问的内容，使得用户在访问网站时能从距离自己最近的服务器获取数据，从而提高访问速度，减轻
源站的负载。反向代理则是将请求代理到后端的不同服务器上，如果请求的资源在反向代理服务器
的缓存中，代理服务器直接返回缓存数据给用户，而不需要再次请求后端服务器。缓存是这两种技
术的共同点，因此A选项是正确的。
B选项 负载均衡：负载均衡虽然在CDN和反向代理中也有应用，但它并不是它们的基本原理。负载均
衡主要是用于分发请求到多个服务器，确保流量均匀分配以避免某个服务器过载。虽然CDN和反向代
理都可以通过负载均衡提升性能，但缓存是其最基本的工作原理。
C选项 路由转发：路由转发主要是指在网络中根据路由表决定数据包的转发路径。虽然CDN和反向代
理在某种程度上涉及到路由和数据转发，但它们的核心机制还是基于缓存，而非单纯的路由转发。
D选项 NAT转换：NAT（网络地址转换）通常用于解决IP地址不足的问题，它与CDN和反向代理的基本
原理无关。因此，D选项不适用。
 综上所述，A选项是正确的，CDN和反向代理的基本原理都是缓存。

### （2023年11月·第47题）

关于区块链不包含（）。
问题（1）
（A）私有链
（B）公有链
（C）行业链
（D）联盟链

**正确答案：** C

**解析**

本题考察的是区块链的基本概念。
A选项私有链：私有链是指由一个组织或实体控制的区块链网络，访问权限是受限的，只有授权用户
可以加入。它是区块链的一个常见类型，属于区块链技术的范畴。
B选项公有链：公有链是公开的区块链，任何人都可以加入并参与网络的操作。比特币和以太坊是公
有链的典型例子，是区块链技术的核心形式之一。
C选项行业链：行业链并不是区块链的一个标准分类。区块链技术分为私有链、公有链和联盟链，并
没有专门称为行业链的类型。虽然某些区块链应用可以针对特定行业（如金融、物流等），但“行
业链”并不是一个正式的区块链类别。
D选项联盟链：联盟链是由多个组织共同管理和控制的区块链，它介于私有链和公有链之间，常用于
需要多个组织共同维护的场景，比如企业之间的合作。
 因此，选项 C 正确。

在数字孪生生态系统中，（）包括描述、诊断、预测、决策四个方面。

（A）数据互动层
（B）模型构建层
（C）仿真分析层

本题考察的是数字孪生生态系统的结构与分层功能，知识来源于《数字孪生应用白皮书

A选项数据互动层：主要负责数据采集、传输与处理，是实现数字孪生动态性的关键支撑，错误。

型支撑，提供数字孪生在各类场景中的通用应用能力，正确。

### （2024年5月·第6题）

物联网是指通过信息传感设备，按约定的协议，将任何物体与网络相连接，物体通过信息传播媒介
进行信息交换和通信，以实现智能化识别、定位、跟踪、监管等功能。物联网应用通常分为三层
，分别是（）。
问题（1）
（A）感知层、网络传输层和操作系统层
（B）应用层、中间件和操作系统层
（C）感知层、协议层和应用层
（D）感知层、网络传输层和应用层

**正确答案：** D

**解析**

本题考察的是物联网体系结构的三层模型。
A选项操作系统层、B选项中间件和操作系统层：操作系统和中间件属于支持平台范畴，不是物联网
分层结构的标准组成，错误。
C选项协议层：协议是网络通信的一部分，但“协议层”不是物联网三层结构的标准术语，错误。
D选项感知层、网络传输层和应用层：这是物联网系统最常见的三层结构，也被称为“物联网三层架
构”：
感知层：负责数据采集和识别，利用传感器、RFID、摄像头等设备。
网络传输层：负责将感知层获取的数据通过各种通信方式（如Wi-Fi、5G、以太网等）传输至应用
层。
应用层：面向用户，处理数据并提供具体服务，如智能家居、智慧城市、工业自动化等。
因此，正确答案为：D. 感知层、网络传输层和应用层。

### （2024年5月·第19题）

以下关于REST的描述中，（）是不正确的。
问题（1）
（A）REST的状态转移是借助HTTP方法来实现
（B）URI和资源是多对多关系
（C）REST是一种设计风格而不是一个架构
（D）REST是以资源为中心构建的

**正确答案：** B

**解析**

本题考察的是REST（Representational State Transfer）架构风格的基本概念和资源建模
原则。
A选项REST的状态转移是借助HTTP方法来实现：REST通过使用HTTP的GET、POST、PUT、DELETE等方法
来表示对资源的操作，是状态转移（State Transfer）机制的核心，正确。
B选项URI和资源是多对多关系：这是错误的。REST中的URI（统一资源标识符）用于唯一标识某一资
源，虽然一个资源可能有多个URI（例如为了兼容历史系统），但一个URI只能对应一个资源。因此
，URI与资源的关系应为“多对一”或“多（URI）对一（资源）”，而不是多对多，选项描述错误
，正确答案。
C选项REST是一种设计风格而不是一个架构：REST不是正式的架构，而是一种Web系统的设计风格
，由Roy Fielding在其博士论文中提出，正确。
D选项REST是以资源为中心构建的：REST中的一切都是资源，设计核心是资源的表示、标识和操作方
式，正确。

 因此，正确答案为：B. URI和资源是多对多关系。

### （2024年5月·第45题）

UDDI是一种用于（）Web Service的技术，它是Web Service协议栈的一个重要部分。
问题（1）
（A）描述、发现、集成
（B）描述、发现、开发
（C）描述、利用、开发
（D）描述、连接、集成

**正确答案：** A

**解析**

本题考察的是SOA 中 Web Service 的核心协议组成，重点涉及 UDDI 在服务描述和发现中
的作用。
 UDDI（Universal Description, Discovery and Integration）是Web服务的一个核心组成部分
，它提供了服务注册与查找的机制，使得服务可以被广泛发现与使用。
A选项描述、发现、集成：这是UDDI的核心功能，UDDI用于描述服务元数据、发现服务提供者、集成
现有服务，是最准确的表述。
B选项描述、发现、开发：UDDI 并不直接用于开发服务，它关注的是服务的注册与查找，因此“开
发”并不准确。
C选项描述、利用、开发：“利用”这个说法不够专业，且“开发”仍然不是 UDDI 的直接用途，因
此此选项不正确。
D选项描述、连接、集成：“连接”不是UDDI的功能，连接是由SOAP或其他协议实现的，而UDDI更多
是服务注册与查找，因此也不准确。
 因此，选项 A 正确。

### （2024年5月·第52题）

下面不属于云计算之虚拟化技术的是（）。
问题（1）
（A）KVM
（B）Xen
（C）Hyper-V
（D）LVS

**正确答案：** D

**解析**

本题考察的是云计算中虚拟化技术的相关概念。
A选项KVM：Kernel-based Virtual Machine，是基于Linux内核的虚拟化技术，支持完全虚拟化，属
于主流虚拟化方案之一。
B选项Xen：开源的虚拟机监控器，支持半虚拟化和完全虚拟化，是早期云计算平台常用的虚拟化技
术。

### （2025年11月·第58题）

（）是以谓词形式来表示动作的主体、客体，是一种叙述性知识表示方法。
问题（1）
（A）逻辑表示法
（B）产生式表示法
（C）框架
（D）面向对象

**正确答案：** A

**解析**

本题考察的是人工智能知识表示方法。
 知识表示是人工智能系统中用于描述知识、推理规则以及实体之间关系的核心内容，常见的知识表
示方法包括逻辑表示法、产生式表示法、框架表示法和面向对象表示法等。
逻辑表示法：以谓词逻辑为基础，适合表达事实性、关系性知识（叙述性表示）。
产生式表示法：以规则（IF…THEN…）形式表示过程性知识。
框架表示法：以结构化形式表达对象及其属性。
面向对象表示法：以类与对象的方式封装数据与操作，适合复杂系统建模。
A选项逻辑表示法：逻辑表示法是以谓词逻辑（Predicate Logic） 为基础的知识表示方式，采用
“谓词 + 参数”的形式描述事实和关系。例如：Loves(John, Mary) 表示“约翰爱玛丽”，其中
“Loves”是谓词，“John”和“Mary”分别是动作的主体和客体。这种方式能够精确地表示对象间
的逻辑关系，是一种叙述性知识表示方法，用于表达知识的语义和推理关系。因此该选项正确。
B选项产生式表示法：产生式表示法是一种过程性知识表示方法，通常以“如果……那么
……”（IF…THEN…）的规则形式表示知识。例如：IF 天气下雨 THEN 带伞。这种方法强调条件与
动作之间的因果关系，不属于叙述性知识表示，而属于规则型知识表示。
C选项框架：框架（Frame）是一种结构化的知识表示方式，类似于面向对象的“类”，通过“槽
（Slot）”和“值（Value）”来表示实体及其属性。框架适合表示静态知识和层次结构关系，不以
谓词逻辑为核心。

D选项面向对象：面向对象是一种将数据和操作封装在一起的知识组织方式，用于软件建模和实现。
虽然也能表达知识，但主要用于程序设计和对象建模，不属于叙述性知识表示。
 因此，正确答案是 A. 逻辑表示法。
