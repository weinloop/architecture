# ABSD · 基于架构的软件开发

**知识点编号**：4.4  
**权重**：次重点～重点（选择题常考）

## 核心要点

- **ABSD（Architecture-Based Software Design）**：**架构驱动** = **商业目标 + 质量目标 + 功能需求** 的组合驱动（非「产品、设计活动」等干扰项）。
- **三个基础**：（1）**功能分解**（沿用内聚/耦合）；（2）用**架构风格**满足质量与商业目标；（3）**软件模板**复用结构。
- **递归、自顶向下**：顶层→概念子系统 + 模板；再细化至**构件与类**（真题常见空）。
- **用例**：功能需求；**质量场景**：变更/性能/可靠性/交互等，含**预期与非预期**场景。
- **六子过程（口诀：许设温敷实验）**：**需求** → **设计** → **文档化**（规格说明 + 测试/质量设计说明；须**分发并保持最新**）→ **复审**（外部用户/领域专家；可搭最小可运行系统）→ **实现**（组装与测试）→ **演化**（需求变化归类→计划→增删改构件→更新交互→组装测试→评审）。
- **视角**：静态看功能组织；动态看并发与行为；教材中「逻辑/进程/实现/配置」作 ABSD 视图特例记忆。
