# DSSA · 特定领域软件体系结构

**知识点编号**：4.7  
**权重**：次重点 ★★★★☆

## 核心要点

- **DSSA**：面向**特定领域**共性需求与变化点的可复用领域架构。
- **垂直域**：某一**系统族**可行解决方案的通用架构；适合**成熟稳定**领域。
- **水平域**：多系统/系统族间**功能区域共性**；不局限于单领域成熟度。
- **三类基本活动**：**领域分析**（领域模型/共同需求）→ **领域设计**（得到 DSSA）→ **领域实现**（可重用信息组织与开发）。
- **四类角色**：领域专家；**领域分析**人员（偏知识工程）；**领域设计**；**领域实现**。
- **五阶段（建立过程）**：定义范围 → 定义领域元素（词典）→ 定义设计与实现**约束** → 定义领域模型与体系结构 → **产生/搜集可重用产品单元**。

### （2023年11月·第29题）

关于 DSSA 说法正确的是 （）
问题（1）
（A）DSSA 适用于所有领域
（B）不同领域的 DSSA 使用方法完全相同
（C）由于领域不同，DSSA 使用存在一定差异
（D）DSSA 主要用于制造业领域

**正确答案：** C

**解析**

本题考察的是DSSA（Domain-Specific Software Architecture，领域特定软件架构） 的
基本概念。
 DSSA 是根据特定领域的需求来设计和实现的架构，它考虑到不同领域的特性和需求，因此在不同
领域中使用时可能存在差异。
A选项 DSSA 适用于所有领域：
 这项说法不完全正确。虽然DSSA可以应用于多种领域，但它并不是“适用于所有领域”，因为不同
领域的需求和特性会导致DSSA在创建和使用时有差异。因此，这个选项不准确。
B选项 不同领域的 DSSA 使用方法完全相同：
 这项说法也是错误的。由于不同领域有不同的需求和开发方法，不同领域的DSSA在应用时会有一定
的差异。DSSA的创建和使用往往需要根据具体领域的特点进行调整。因此，这个选项不成立。
C选项 由于领域不同，DSSA 使用存在一定差异：
 这个选项是正确的。DSSA的应用会根据所处领域的不同有所不同。例如，制造业的DSSA可能更侧重
于生产过程的自动化，而金融领域的DSSA则可能注重数据安全和交易的高效性。因此，DSSA的创建
和使用方法会根据行业需求和开发者的习惯有所变化。
D选项 DSSA 主要用于制造业领域：
 这个选项是不正确的。尽管DSSA可以应用于制造业，但它并不限于此。DSSA可以广泛应用于不同的
行业领域，如金融、医疗、电子商务等。因此，将其仅限于制造业是不准确的。
 综上，C选项是正确的。

A选项领域专用语言：领域专用语言（DSL）是用于特定领域的编程语言或符号集，它可以在设计过
程中使用，但不是领域设计阶段的主要成果。
B选项参考架构：参考架构是指导系统设计的框架和模型，但它更多的是在架构设计中产生，而不是
领域设计阶段的主要成果。
C选项特定领域软件架构：在DSSA的领域设计阶段，主要成果是特定领域软件架构（Domain-
Specific Software Architecture）。它定义了一个特定领域的架构模型，帮助开发人员构建符合
领域需求的软件系统。
D选项构件规格说明：构件规格说明描述的是构件的具体要求，属于实现阶段的内容，和领域设计阶
段不太相关。
 因此，选项 C 正确。

### （2024年5月·第26题）

在特定应用领域软件体系结构的设计中，（）阶段的主要目标是获得领域模型。
问题（1）
（A）领域实现
（B）领域设计
（C）领域建模
（D）领域分析

**正确答案：** D

**解析**

本题考察的是特定领域软件体系结构（DSSA）中的领域分析阶段。
A选项领域实现：错误，领域实现是后期将领域设计具体化为代码和系统的阶段，不负责建立领域模
型。
B选领域设计：错误，领域设计是在领域分析之后，根据领域模型进行通用架构设计的阶段，其目标
是设计而非分析。
C选项领域建模：错误，这虽然听起来像是建立领域模型的阶段，但它通常是领域分析的一个具体活
动，而不是完整的独立阶段。

D选项领域分析：正确。领域分析是 DSSA 的第一阶段，其核心目标就是获得领域模型，识别领域系
统的共性需求、变异点以及构建领域的抽象表示，指导后续架构设计。
 因此，正确答案为 D。

### （2024年11月·第66题）

下列关于建立DSSA的过程的说法，哪个是正确的（）。
问题（1）
（A）顺序的、递归的、反复的
（B）顺序的、循环的、一次性的
（C）并发的、递归的、反复的
（D）并发的、循环的、反复的

**正确答案：** C

**解析**

本题考察的是DSSA（Domain-Specific Software Architecture，特定领域软件架构）建立
过程的特性。
 建立 DSSA 的过程通常分为 5 个阶段，每个阶段都包含输入、输出、所需解决的问题以及验证标
准。整个过程的特点如下：
并发性（Concurrent）：各阶段之间可以同时进行，而不是严格线性顺序。
递归性（Recursive）：一些步骤在更精细的层次上会再次重复，用于逐层深入细化。
反复性（Iterative）：每个阶段可能需要多次迭代，每次完善更多细节，类似螺旋模型。
选项 A 和 B 错误在于“顺序”和“一次性”不符合 DSSA 的迭代本质。
选项 D 的“循环”表达不如“递归”准确，容易混淆控制结构的概念。
 因此，正确答案是：C. 并发的、递归的、反复的。

### （2025年11月·第8题）

在特定领域架构中，领域分析的结果通常得到是（）。
问题（1）
（A）领域模型
（B）领域知识
（C）特定领域
（D）领域样本

**正确答案：** A

**解析**

本题考察的是领域分析（Domain Analysis）在领域架构与领域工程中的作用和产出物。
 领域分析的核心任务是：在特定应用领域内识别共性需求、抽象概念、对象、关系和约束，从而形
成可复用的领域知识结构。其最终输出通常是领域模型，用于规范描述该领域的核心概念与关系。
A选项 领域模型：领域分析的主要成果就是领域模型，它描述了领域中的关键概念、属性、关系与
规则，是后续领域架构与系统设计的基础。因此该选项正确。
B选项 领域知识：领域知识在分析过程中会被收集，但“领域知识”不是领域分析的标准产物，更
像是过程中的输入或中间资料，而不是最终模型化的结果，因此该选项不准确。
C选项 特定领域：特定领域是领域分析的对象，而不是分析后的产出，因此该选项不正确。
D选项 领域样本：领域样本（如典型案例、示例系统）可能用于分析，但它也不是正式的输出成果
，因此此选项不正确。
 因此，本题正确答案为 A。
