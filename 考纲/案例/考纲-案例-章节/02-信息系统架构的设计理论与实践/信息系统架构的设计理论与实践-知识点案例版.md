## 2.1 信息系统架构概述

### 考纲要点

- 信息系统架构的定义和发展
- 信息系统架构的分类（集中式和分布式）
- 信息系统常用的四种架构模型
- 单机应用模式
- 客户机/服务器（C/S）模式（两层 C/S、三层 C/S 或 B/S、多层 C/S、MVC）
- 面向服务架构（SOA）模式
- 企业数据交换总线
- 企业信息系统的总体框架（战略系统、业务系统、应用系统、信息基础设施）

### 考试详细内容

- **定义**：对信息系统的**结构要素、关系与原则**的高层描述，承载业务与质量目标。
- **发展**：从主机—C/S—B/S—**SOA**—**微服务/云原生**—**数据平台/智能** 持续演进。
- **集中式 vs 分布式**：集中易治理；分布式利于**伸缩与高可用**，但带来一致性与运维复杂度。
- **四种模型**（教材常见归类，名称以教材为准）：如**文件/单机**、**C/S**、**B/S**、**分布式/互联网** 等变体。
- **C/S**：胖客户端能力强；**B/S**：免安装易部署；**多层**：表现—应用—数据分离；**MVC**：表现与逻辑分离。
- **SOA**：**服务契约**、松耦合、可组合；常与 **ESB** 做异构集成。
- **企业总体框架分层**：
  - **战略系统**：目标与指标体系。
  - **业务系统**：流程与领域能力。
  - **应用系统**：软件模块与服务。
  - **信息基础设施**：网络、平台、数据与安全基座。

### 航空运营智能管理平台案例

- **形态**：典型**分布式、云原生**大型信息系统；旅客侧 **B/S 与移动 App**（多终端），中心侧 **微服务集群**。
- **服务化**：七大模块以**可复用服务**组织，素材明确**云原生微服务 + 分布式事务**，可视为 **SOA 思想在运行时的现代实现**（REST/gRPC、事件、API 网关）。
- **集成**：**数据同步服务**承担与遗留系统/外部系统的**交换与对齐**；对接**民航局身份核验**属跨组织集成。
- **分层对应**：
  - **战略**：智慧民航、效率提升 30%+、99.99% 可用。
  - **业务**：航线/航班/票务/旅客/检修全流程。
  - **应用**：各模块子服务。
  - **基础设施**：弹性计算、分布式存储、消息与流处理、安全与运维。

## 2.2 信息系统架构设计方法（ADM）

### 考纲要点

- 开放式企业架构框架标准（TOGAF）
- ADM 架构开发方法（定义、ADM 方法的开发阶段划分、详细活动说明）

### 考试详细内容

- **TOGAF**：企业架构框架，提供**方法（ADM）**、**内容元模型**、**技术参考模型**等组件（概念层）。
- **ADM（Architecture Development Method）**：迭代循环，常见阶段包括（**以教材版本为准**）预备、架构愿景、业务架构、信息系统架构（数据/应用）、技术架构、机会与解决方案、迁移规划、实施治理、架构变更管理等。
- **要点**：**干系人共识**、**基线架构与目标架构**、**差距分析**、**迁移路线图**。

### 案例

- **架构愿景**：统一运营平台，消除孤岛，支撑高并发与智能分析。
- **业务架构**：运控、营销、客服、机务等能力与流程在平台上的映射。
- **应用/数据架构**：七大模块边界、主数据（航线/机型/旅客）、**批流一体**数据平台。
- **技术架构**：容器、微服务、缓存、负载均衡、TLS、RBAC。
- **迁移**：分阶段上线核心链路与数据治理（素材未展开细节，答题写“分阶段迁移与对账”）。

## 2.3 信息化总体架构方法

### 考纲要点

- 信息化的一般概念（定义、生产力、建设和特征）
- 信息化工程建设方法
- 信息化架构模式（数据导向架构和流程导向架构）
- 信息化建设生命周期
- 信息化工程总体规划的方法论（关键成功因素法、战略目标集转化法和企业系统规划法）
- 信息化资源管理
- 国际和国内有关信息化的标准、法律和规定

### 考试详细内容

- **信息化**：广泛利用信息技术提升生产与管理效能的过程；**生产力**体现在效率、质量、创新。
- **特征**：渗透性、全局性、长期性。
- **建设方法**：总体规划分步实施、标准先行、平台化、数据治理与安全保障。
- **架构模式**：
  - **数据导向**：主数据、数据仓库/中台、以数据共享驱动应用。
  - **流程导向**：BPM、端到端流程编排、以流程 KPI 驱动系统。
  - 实际项目常**混合**：流程在线化 + 数据资产化。
- **生命周期**：规划—建设—运维—优化—退役。
- **BSP（企业系统规划法）**：自上而下识别过程与数据类，建立 **CU 矩阵**。
- **CSF（关键成功因素）**：少数关键目标驱动信息化优先级。
- **SST（战略目标集转化）**：组织战略 → 信息系统目标。
- **资源管理**：人力、预算、数据资产、软硬件与云资源、知识。
- **法规标准**：网络安全、数据安全、个人信息保护、行业监管（答题写框架）。

### 案例

- **数据导向**：素材强调 **10PB+ 历史**、**主题分析**、**跨系统联动**，**数据服务模块**是典型数据驱动载体。
- **流程导向**：第 4 节**多模块联动**（购票→同步→校验→行程→通知→检修阈值→分析）是端到端流程数字化。
- **规划方法**：可用 **CSF** 归纳“高并发票务稳定、数据驱动决策、合规安全”为关键成功因素；用 **BSP 思想**识别“航班/客票/旅客/设备”等数据类与共享需求。
- **法规**：旅客信息保护、身份核验、脱敏与日志留存（素材第 7 节）。

## 2.4 案例分析

### 考纲要点

- 价值驱动的体系结构
- Web 服务在 HL7 上的应用
- 以服务为中心的企业整合

### 考试详细内容

- **价值驱动体系结构（VDSA 类思想）**：架构决策追溯到**业务价值与可度量结果**（收入、成本、风险、体验），避免为技术而技术。
- **HL7 + Web 服务**：HL7 是**医疗健康信息交换**标准体系；**FHIR** 等常基于 **REST/HTTP** 资源化 API；说明**行业标准 + Web 服务**实现互操作。
- **以服务为中心的企业整合**：通过**标准服务契约**整合异构系统（ESB/API 网关/事件总线），实现流程协同与数据一致。

### 案例

- **价值驱动**：将 **99.99% 可用、≤1s 购票、效率提升 30%+** 等映射为架构投资决策与优先级（缓存、弹性、分布式事务、可观测）。
- **HL7 类比普通**：航空领域存在**行业消息与数据交换标准**（考试若要求 HL7，则按教材答医疗场景；本项目可写“**类比**：本平台通过 **API/同步服务**对接**民航局身份核验**与内部遗留系统，同属**行业集成**”）。
- **服务为中心整合**：**票务、旅客、航信、数据服务**通过 API 与事件整合；**企业整合**体现为统一门户、统一身份、统一数据服务层。
