## 8.1 安全架构概述

### 考纲要点

- 信息安全面临的威胁
- 安全架构的定义和范围（产品安全架构、安全技术体系架构和审计架构）
- 信息安全相关的国内外标准及组织
- 主要安全模型
- 状态机模型（BLP）模型
- Bell-LaPadula 模型
- Biba 模型
- Clark-Wilson（CWM）模型
- Chinese Wall 模型

### 考试详细内容

#### 威胁

- 人为/非人为；窃听、篡改、重放、否认、拒绝服务、恶意代码、越权、社会工程等。

#### 安全架构范围

- **产品安全架构**：软件/系统内置安全机制。
- **安全技术体系架构**：纵深防御、分层控制（网络、主机、应用、数据）。
- **审计架构**：日志、留痕、合规证明、追责。

#### 标准与组织（答题框架）

- **ISO/IEC 27001**、**等级保护**体系、**TC260** 等；**网络安全法、数据安全法、个人信息保护法**。

#### 经典安全模型（抓性质）

| 模型 | 核心性质（简记） |
|------|------------------|
| **BLP / Bell-LaPadula** | **保密性**（不上读不下写 *- 以教材规则为准） |
| **Biba** | **完整性**（不上写下读） |
| **Clark-Wilson** | 商业完整性：**职责分离（SoD）**、TP/CDI |
| **Chinese Wall** | 利益冲突隔离，动态调整访问 |

### 航空运营智能管理平台案例

- **威胁对齐素材**：**SQL 注入、XSS、DDoS**、撞库与暴力破解、越权、内部滥用。
- **纵深防御**：**TLS 传输** + **字段加密存储** + **RBAC** + **WAF/审计** + **备份**。
- **模型落地（论述可用）**：
  - **Biba/完整性**：票务与库存数据防篡改，强一致事务。
  - **Clark-Wilson**：**权限申请审批**、关键操作双人复核思想；票务与财务对账。
  - **Chinese Wall**：不同业务条线数据访问隔离（扩展表述，素材未细写）。
- **审计架构**：关键操作日志 **≥1 年**、可追溯。

## 8.2 系统安全体系架构规划框架

### 考纲要点

- 安全技术体系架构
- 信息系统安全体系规划（技术体系、组织机构体系和管理体系）
- 信息系统安全规划框架

### 考试详细内容

- **技术体系**：身份与访问、加密、边界、监测响应、备份恢复、开发安全。
- **组织体系**：角色、职责、应急组织。
- **管理体系**：制度、培训、供应商、变更与配置管理。
- **规划框架**：现状评估 → 目标与差距 → 路线图 → 度量与改进。

### 案例

由**航空信息中心**统筹；**RBAC + 审批**属管理流程；**漏洞扫描、渗透测试**属技术+管理闭环；**备份保留 ≥3 年**属持续合规。

## 8.3 信息安全整体架构设计（WPDRRC 模型）

### 考纲要点

- WPDRRC 体系架构模型概述（6 个环节和 3 大要素）
- 信息安全体系架构设计方法
- 安全保障体系（确定安全策略、统一配置和管理防病毒系统、网络安全管理）
- 信息安全体系架构（物理安全、系统安全、网络安全、应用安全和安全管理）
- 面向企业安全控制系统的安全架构设计

### 考试详细内容

- **WPDRRC**（常见表述）：**预警、保护、检测、响应、恢复、反击**（不同教材用词略异，**以教材为准**）；**人、技术、管理**三要素。
- **五层安全**：物理 / 网络 / 系统 / 应用 / 管理（与考纲“物理、系统、网络、应用、管理”一致）。

### 案例

| 环节 | 素材落点 |
|------|-----------|
| 预警 | 异地登录、异常操作、异常流量监测 |
| 保护 | TLS、加密、RBAC、脱敏 |
| 检测 | 漏洞扫描、入侵特征、应用层攻击检测 |
| 响应 | 告警、封禁、限流、应急 |
| 恢复 | 备份与归档、业务连续性 |
| 反击 | 配合监管溯源（合规语境，谨慎表述） |

## 8.4 网络安全体系架构设计

### 考纲要点

- OSI 的安全体系架构概述
- OSI 安全架构（7 层协议）
- 鉴别框架
- 访问控制框架
- 机密性框架
- 完整性框架
- 抗抵赖框架

### 考试详细内容

- **OSI 安全**：安全机制映射到层（物理到应用）；**五大安全服务**：鉴别、访问控制、机密性、完整性、抗抵赖（教材分法为准）。
- **框架**：各服务可实现的机制组合（加密、数字签名、哈希、证书等）。

### 案例

- **鉴别**：登录、会话令牌、对接**民航局身份核验**。
- **访问控制**：**RBAC**、API 授权。
- **机密性/完整性**：**HTTPS**、数据库加密、消息完整性校验。
- **抗抵赖**：票务与权限变更**审计日志**、时间戳与不可抵赖操作记录。

## 8.5 数据库系统的安全设计

### 考纲要点

- 数据库安全设计的评估标准
- 数据库的完整性设计方法（设计原则、作用和示例）

### 考试详细内容

- **评估**：访问控制、审计、加密、备份、注入防护、最小权限。
- **完整性**：实体/参照/用户定义完整性；事务 ACID；触发器与约束。

### 案例

- **敏感列加密**（身份证、银行卡）；**预编译参数**防 **SQL 注入**。
- **票务库存**约束防超卖；**主外键**保证订单—航班一致。
- **审计表**或数据库级审计跟踪特权操作。

## 8.6 系统架构的脆弱性分析

### 考纲要点

- 软件脆弱性的定义、特点和分类
- 软件脆弱性的生命周期
- 软件脆弱性的分析方法
- 典型的软件架构脆弱性分析（分层、C/S、B/S、事件驱动、MVC、微内核和微服务）

### 考试详细内容

- **脆弱性**：可被威胁利用的弱点（缺陷、配置错误、弱口令）。
- **生命周期**：引入 → 发现 → 利用/修复 → 残留风险。
- **分析**：威胁建模、攻击面分析、渗透测试、代码审计、依赖扫描。
- **架构相关弱点**：
  - **分层**：跨层绕过、过度信任 DAL。
  - **C/S/B/S**：会话劫持、XSS、注入。
  - **事件驱动**：消息伪造、重放、顺序错乱。
  - **微服务**：横向移动、服务间信任过大、供应链。

### 案例

- **微服务**：素材防范 **DDoS**、需 **服务间 mTLS/鉴权**（概念）；**分布式事务**复杂度高增加配置脆弱性。
- **B/S**：**XSS** 与 **CSRF** 面；高并发下**限流**不当导致可用性风险。
- **事件驱动**：订单事件需**签名/幂等**防重放。

## 8.7 案例分析

### 考纲要点

- 电子商务系统的安全性设计
- 基于混合云的工业安全架构设计

### 考试详细内容

- **电商安全**：交易完整性、支付安全、账户安全、风控、日志与取证。
- **混合云工业安全**：OT/IT 边界、分区隔离、零信任、数据不出域、工控协议安全。

### 航空运营智能管理平台案例（对照）

| 考纲案例 | 与本项目的对应 |
|----------|----------------|
| **电子商务安全** | **在线票务**同属高价值交易：**强一致、防刷、防 DDoS、TLS、敏感信息保护、审计**（素材第 7 节 + 5.2 性能） |
| **混合云工业安全** | **设备运行数据 + 检修 + 大数据** 可类比“**生产型数据入云**”：**专网/逻辑隔离、脱敏、最小权限、审计**；本平台以信息中心与**云资源**为主，混合云属扩展论述 |

**统一收束**：素材已覆盖 **权限、数据、操作、合规** 四类安全；架构上配合 **云原生弹性** 与 **全链路加密审计**，形成**纵深防御**。
