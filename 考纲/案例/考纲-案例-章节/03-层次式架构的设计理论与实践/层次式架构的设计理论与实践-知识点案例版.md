## 3.1 层次式架构概述

### 考纲要点

- 层次式架构的定义和特性
- 层次式架构的一般组成（表现层、中间层、数据访问层和数据层）

### 考试详细内容

- **定义**：将系统按**职责**垂直划分为若干层，上层依赖下层服务，**单向依赖**以降低耦合。
- **特性**：职责清晰、可替换性（如更换 UI 技术）、利于分工与测试；注意避免**跨层穿透**与**过度分层**。
- **经典四层**：
  - **表现层（Presentation）**：用户交互、输入校验、展示与路由。
  - **中间层（Business / Application）**：领域规则、用例编排、事务边界（有时再细分领域层与服务层）。
  - **数据访问层（DAL / Persistence）**：ORM、DAO、Repository，屏蔽存储细节。
  - **数据层（Data）**：关系库、缓存、消息、对象存储、数仓等。

### 航空运营智能管理平台案例

- **表现层**：旅客 Web/App、运控与客服**数据可视化**（素材：多终端、≤3 秒更新）。
- **中间层**：票务规则、改签退票、运力校验、检修计划与 AI 分析编排；对应七大模块中的**业务服务**。
- **数据访问层**：封装对业务库、缓存、消息、大数据引擎的访问；支撑 **5000 TPS** 与 **PB 查询** 的分路径实现（OLTP vs OLAP）。
- **数据层**：分布式存储、实时流、离线批处理、备份归档（**99.999% 可靠性**）。

**与云原生关系**：物理上多为**多服务部署**，逻辑上每个服务内部仍常采用**分层或六边形端口适配**，二者不矛盾。

## 3.2 表现层框架设计

### 考纲要点

- 设计模式（MVC、MVP 和 MVVM）
- XML 技术
- UIP 设计思想
- 动态生成设计思想

### 考试详细内容

- **MVC**：Model-View-Controller，**控制器**协调模型与视图；Web 常见。
- **MVP**：Presenter 承担逻辑，View 被动；便于测试。
- **MVVM**：ViewModel + 数据绑定；前端框架常用。
- **XML**：配置、界面描述、数据交换；**Schema** 约束结构。
- **UIP（User Interface Process）**：将**用户界面流程**从业务中抽出，统一导航与状态（微软模式，按教材）。
- **动态生成**：根据配置/权限/角色**动态渲染**页面或表单，降低变更成本。

### 案例

- **旅客端**：航班查询、下单、改签流程适合 **MVVM/MVC**；多终端复用 **API**。
- **运营大屏**：组件化可视化，**动态配置**指标卡片与告警阈值展示。
- **权限与角色**：RBAC 下菜单与按钮级**动态显示**（素材：细粒度权限）。
- **XML/JSON**：接口契约与前后端分离中的**交换格式**（现代项目 JSON 更常见，考试答“结构化数据与校验”即可）。

## 3.3 中间层架构设计

### 考纲要点

- 业务逻辑层组件设计
- 业务逻辑层工作流设计
- 业务逻辑层实体设计
- 业务逻辑层框架

### 考试详细内容

- **组件**：按领域拆分服务模块（票务、旅客、航信等），高内聚。
- **工作流**：长流程用 BPM/状态机/编排（购票、支付、出票、通知链路）；与 **Saga/补偿** 结合处理分布式事务。
- **实体**：领域对象（航班、客票、旅客、检修工单）与**充血/贫血模型**选型。
- **框架**：Spring、领域事件、规则引擎、工作流引擎等（概念层）。

### 案例

- **实体**：`Flight`、`TicketOrder`、`PassengerTrip`、`MaintenanceAlert`。
- **工作流**：第 4 节联动链路——下单→同步→校验→行程→通知→检修阈值→分析，属于**跨服务编排**。
- **组件**：票价计算、库存扣减、身份核验适配、流式告警分别封装。
- **框架**：云原生下中间层常体现为**微服务 + 消息**，工作流由**编排器**或**事件驱动**实现。

## 3.4 数据访问层设计

### 考纲要点

- 访问模式
- ORM、Hibernate 与 CMP 2.0 设计思想
- 灵活运用 XML Schema
- 事务处理设计
- 连接对象管理设计

### 考试详细内容

- **访问模式**：在线事务（CRUD）、批量、CQRS（读写分离）、缓存旁路（Cache-Aside）。
- **ORM**：对象—关系映射；**Hibernate** 为 Java 生态代表；**CMP 2.0**（EJB 容器管理持久化）为历史考点，理解“容器托管持久化”即可。
- **XML Schema**：报文、配置、集成接口的结构校验。
- **事务**：本地 ACID、分布式事务（2PC/TCC/Saga）；票务需**强一致**。
- **连接管理**：连接池、超时、熔断、读写分离路由。

### 案例

- **票务**：ORM + **悲观/乐观锁**防超卖；**分布式事务**保障跨服务一致性（素材）。
- **大数据**：分析查询走**专用引擎/列存**，与 OLTP **读写分离**。
- **集成**：对接民航局核验报文可用 **Schema 校验**保证字段合规。
- **连接池**：高并发下必须调优，配合 **≤1s** 响应目标。

## 3.5 数据层架构规划与设计

### 考纲要点

- 数据库设计与类的设计融合的方法
- 数据库设计与 XML 设计融合的方法

### 考试详细内容

- **DB ↔ 类**：表—实体映射、聚合根与表边界、值对象嵌入、继承映射策略。
- **DB ↔ XML**：导出/交换格式、配置驱动批处理、主数据同步文件；注意**规范化 vs 交换宽表**。

### 案例

- **OLTP 库**：票务、旅客、航班表设计支撑一致性与索引策略。
- **分析侧**：主题宽表、汇总表服务于 **PB 级查询 ≤5s/10s**。
- **交换**：与外部系统用**标准字段定义**（类 XML/JSON 契约）；**数据同步服务**承载落地。

## 3.6 物联网的层次架构设计

### 考纲要点

- 物联网的一般层次架构（三层体系结构）

### 考试详细内容

- **常见三层**：**感知层**（传感、终端）—**网络层**（接入、传输）—**应用层**（平台与业务）。
- 扩展：**平台层/边缘层**细分。

### 案例

- **感知层**：航班动态、设备运行参数、旅客终端操作、机场传感数据接入（素材：秒级采集）。
- **网络层**：移动网络、机场专网、云上 VPC。
- **应用层**：本平台业务与数据服务、可视化与告警处置。
- 与 **边缘计算** 结合：航站区预处理再入云，降低延迟与带宽压力（与第 10 章未来技术呼应）。

## 3.7 案例分析

### 考纲要点

- 电子商务网站
- 基于物联网架构的电子小票服务系统

### 考试详细内容

- **电子商务网站**：典型分层 + 购物车/支付/库存 + 高并发 + 安全。
- **物联网电子小票**：感知（终端出票/扫码）—网络—云端对账与营销；强调**端到端可追溯**。

### 航空运营智能管理平台案例（对照论述）

| 考纲案例 | 与本项目的类比 |
|----------|----------------|
| 电子商务网站 | **在线票务**同属高并发交易：查询—下单—支付—库存—履约—售后（改签退票）；指标上对齐 **TPS、响应时间、一致性** |
| 物联网电子小票 | **电子行程/电子客票/通知推送**构成凭证链；运行与设备数据类似**端侧采集 + 云端分析**；强调**日志留痕与可追溯**（素材第 7 节） |

答题可写：**层次式架构在交易系统中统一表现为“前端接入—业务编排—数据访问—多类型存储”，并通过物联网三层理解端侧数据进入平台的路径**。
