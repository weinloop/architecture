## 4.1 云原生架构概述

### 考纲要点

- 云原生的背景、定义和特征
- 云原生架构的设计原则
- 架构模式（服务化架构模式、Mesh 化架构模式、Serverless 模式、存储计算分离模式、分布式事务模式、可观测架构和事件驱动架构）

### 考试详细内容

#### 背景、定义与特征

- **背景**：业务快速变化、海量并发、弹性需求、DevOps 与持续交付；传统单体与固定容量难以为继。
- **定义（常见表述）**：面向云特性（弹性、自动化、分布式）设计的应用构建与运行方式；**CNCF** 强调容器化、微服务、**声明式 API**、自动化运维。
- **特征**：弹性伸缩、故障隔离、可观测、声明式配置、不可变基础设施、DevSecOps。

#### 设计原则（概括）

- **服务拆分与自治**：按业务域划分，独立部署。
- **弹性与韧性**：无单点、限流熔断、重试与降级。
- **声明式与 GitOps**：期望状态驱动集群。
- **安全左移**：供应链、镜像扫描、最小权限。
- **可观测性三位一体**：**指标 Metrics、日志 Logs、链路 Traces**。

#### 考纲列出的架构模式

| 模式 | 要点 |
|------|------|
| **服务化** | 微服务/SOA 现代形态，API/事件契约，独立生命周期 |
| **Mesh 化** | 数据平面代理接管流量、安全、可观测；控制平面统一下发策略 |
| **Serverless** | 事件触发、按使用计费、免运维节点；冷启动与供应商锁定需权衡 |
| **存储计算分离** | 计算（容器/任务）与状态（DB/对象存储/消息）解耦，独立扩展 |
| **分布式事务** | 2PC、TCC、Saga、可靠消息；与一致性级别权衡 |
| **可观测架构** | 统一采集、SLI/SLO、告警与复盘 |
| **事件驱动** | 异步解耦、削峰、最终一致性 |

### 航空运营智能管理平台案例

素材明确 **云原生微服务弹性扩展架构** + **分布式事务** + **分布式缓存** + **负载均衡** + **动态扩容**，与高并发（**≥10 万并发**、**≥5000 TPS**、**≤1s**）及 **99.99% 可用** 一致。

- **服务化**：七大业务域微服务 + 基础支撑平台。
- **存储计算分离**：业务交易与 **PB 级分析**、**流式计算** 分层；状态外置到分布式存储与消息。
- **分布式事务**：保障票务 **强一致性**。
- **事件驱动**：订单/航变等事件驱动通知、同步与分析（与第 4 节链路一致）。
- **可观测**：日志保留、审计、指标支撑容量与故障定位（素材第 7 节与运维需求）。
- **Mesh/Serverless**：考纲要求掌握概念；本项目可写“**可选增强**：Mesh 统一 mTLS 与流量策略；**模型推理/批任务**可 Serverless 化降峰谷成本”（不虚构已上线细节）。

## 4.2 云原生架构相关技术

### 考纲要点

- 容器技术
- 云原生微服务技术
- 无服务器技术
- 服务网格

### 考试详细内容

#### 容器技术

- **OCI 镜像**、**容器运行时**、**Kubernetes** 编排：Pod、Deployment、Service、HPA（水平扩缩容）。
- **价值**：环境一致、快速发布、资源隔离。

#### 云原生微服务技术

- **注册发现**、**配置中心**、**API 网关**、**熔断限流**、**客户端负载均衡**。
- **12-Factor**（概念）：配置外置、无状态进程等。

#### 无服务器技术（Serverless）

- **FaaS**、**托管事件总线**、**Serverless 容器**；适合突发、间歇任务。

#### 服务网格（Service Mesh）

- **数据面**（如 sidecar）+ **控制面**；**mTLS**、**灰度发布**、**可观测**。
- 与 **微服务** 关系：把横切能力从业务代码下沉。

### 案例

- **容器/K8s**：实现素材中的 **弹性扩展**、**滚动发布**、**多 AZ**（概念）部署。
- **微服务**：票务、旅客、数据服务等独立扩缩；网关统一鉴权与路由。
- **Serverless**：**T+1 批处理**、**模型训练任务** 可按需调度；大促临时函数扩容（论述可用）。
- **Service Mesh**：在多团队、多语言服务共存时统一流量与安全策略；与 **TLS、零信任** 方向一致。

### 航空运营智能管理平台案例（与考纲五类案例对照）

| 考纲案例类型 | 与本项目的对应写法 |
|--------------|----------------------|
| **旅行公司云原生改造** | 直接同类：航班/旅客/票务/行程，**旺季峰值**与**航变突发**驱动弹性与容灾 |
| **电商业务改造** | **在线票务**与电商同源：购物车式查询、下单、库存、支付一致性、促销（动态票价） |
| **快递公司核心系统改造** | 强调**全程可视化**、**时效与异常处理**；本平台对应航班动态、行李与保障链路（素材侧重运营与票务，可弱写） |

**统一叙述模板（答题可用）**：

本项目以 **云原生微服务** 实现业务解耦与弹性，以 **分布式事务与缓存** 保障票务强一致与性能，以 **数据服务** 承载批流一体与 AI 模型，对应行业中台 + 数字化运营实践；指标上对齐 **TPS、延迟、可用性、数据可靠性**。
