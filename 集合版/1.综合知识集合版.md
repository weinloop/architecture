# 综合知识背诵版

## 1\. 软件工程概述 ★★★★★

### 1\.1 软件工程相关定义 ★★★☆☆

【考频：低频】

- **软件工程**：将系统化、规范化、可度量的方法应用于软件的开发、运行和维护

- **软件危机**：落后的软件生产方式无法满足迅速增长的计算机软件需求

- **软件生命周期**：可行性分析与项目开发计划 → 需求分析 → 概要设计 → 详细设计 → 编码 → 测试 → 维护

### 1\.2 软件开发方法 ★★★★★

【考频：高频】

|方法类型|核心思想|适用场景|
|---|---|---|
|**结构化方法**|自顶向下、逐层分解、模块化|需求明确的中小型系统|
|**面向对象方法**|封装、继承、多态|复杂系统、需求易变|
|**原型化方法**|快速构建原型、迭代改进|需求不明确、探索性项目|
|**敏捷方法**|个体交互、可工作软件、客户协作、响应变化|快速迭代、需求频繁变更|

**记忆口诀：结原敏面（结缘敏面）**

### 1\.3 软件过程 / 生命周期模型 ★★★★★

【考频：高频】

|模型名称|特点|适用场景|
|---|---|---|
|**瀑布模型**|线性顺序、阶段明确、文档驱动|需求明确、稳定的项目|
|**增量模型**|分批增量交付、逐步完善|需求逐步明确、资金分阶段投入|
|**迭代模型**|多次迭代、每次迭代完成部分功能|大型复杂项目、高风险项目|
|**螺旋模型**|瀑布 \+ 增量 \+ 风险分析|大型复杂高风险项目|
|**喷泉模型**|面向对象、迭代重叠、无明显阶段|面向对象开发|
|**V 模型**|测试与开发对应、左开发右测试|需求明确、对质量要求高|
|**RUP**|用例驱动、架构为中心、迭代增量|大型面向对象项目|

**记忆口诀：瀑增迭螺喷 V RUP（不增迭螺喷 V 入）**

#### V 模型对应关系【高频必考】

|开发阶段|对应测试阶段|测试依据|
|---|---|---|
|编码|单元测试|详细设计说明书|
|详细设计|集成测试|概要设计文档|
|概要设计|系统测试|需求分析文档|
|需求分析|验收测试|用户需求|

#### RUP 九大核心工作流【高频】

业务建模、需求、分析与设计、实现、测试、部署、配置与变更管理、项目管理、环境

**记忆口诀：建需分食测，建仓前需先分好粮食（建需分实测）**

### 3\.4 面向对象设计 ★★★★★

【考频：高频】

#### SOLID 设计原则【高频必考】

|原则|全称|核心思想|
|---|---|---|
|S|单一职责原则|一个类只负责一个功能领域中的相应职责|
|O|开闭原则|对扩展开放，对修改关闭|
|L|里氏替换原则|子类可以替换父类出现在父类能够出现的任何地方|
|I|接口隔离原则|使用多个专门的接口，而不使用单一的总接口|
|D|依赖倒置原则|依赖于抽象，不依赖于具体实现|

**记忆口诀：SOLID（固体）**

### 3\.5 结构化设计 ★★★★★

【考频：高频】

#### 模块设计原则【高频必考】

- **高内聚**：模块内部各元素联系紧密

- **低耦合**：模块之间联系松散

**内聚强度排序（从弱到强）**：
偶然内聚 → 逻辑内聚 → 时间内聚 → 过程内聚 → 通信内聚 → 顺序内聚 → 功能内聚

**耦合强度排序（从强到弱）**：
内容耦合 → 公共耦合 → 外部耦合 → 控制耦合 → 标记耦合 → 数据耦合 → 非直接耦合

**记忆口诀：偶逻时过通顺功（偶路过通顺成功）**

### 7\.2 软件可靠性的定量描述 ★★★★★

【考频：高频必考】

|指标|全称|定义|
|---|---|---|
|MTTF|Mean Time to Failure|平均失效时间，从开始到故障|
|MTBF|Mean Time Between Failures|平均故障间隔时间，MTTF\+MTTR|
|MTTR|Mean Time to Repair|平均修复时间，故障到恢复|
|MTTD|Mean Time to Detect|平均故障检测时间|
|MTTA|Mean Time to Acknowledge|平均确认时间|

**核心公式**：

- 可用性 = MTTF / \(MTTF \+ MTTR\) × 100%

- MTTF = 1/λ（λ 为失效率）

- MTTR = 1/μ（μ 为修复率）

## 15\. 计算公式汇总

### 15\.1 可靠性相关

- **可用性** = MTTF / \(MTTF \+ MTTR\) × 100%

- **MTTF** = 1/λ （λ 为失效率）

- **MTTR** = 1/μ （μ 为修复率）

- **串联系统可靠性**：R = R₁ × R₂ × \.\.\. × Rₙ

- **并联系统可靠性**：R = 1 \- \(1\-R₁\)\(1\-R₂\)\.\.\.\(1\-Rₙ\)

### 15\.2 McCabe 环形复杂度

- V \(G\) = 区域数

- V \(G\) = E \- N \+ 2 （E = 边数，N = 结点数）

- V \(G\) = P \+ 1 （P = 判定结点数）

### 15\.3 流水线

- **流水线执行时间** = 第一条指令时间 \+ \(n\-1\) × 流水线周期

- **吞吐率** = n / 流水线执行时间

- **加速比** = 不使用流水线时间 / 使用流水线时间

### 15\.4 阿姆达尔定律

- 加速比 = 1 / \[\(1\-f\) \+ f/k\]

- f = 可改进部分比例，k = 改进加速倍数

### 15\.5 项目管理

- **CV 成本偏差** = EV \- AC

- **SV 进度偏差** = EV \- PV

- **CPI 成本绩效指数** = EV / AC

- **SPI 进度绩效指数** = EV / PV
