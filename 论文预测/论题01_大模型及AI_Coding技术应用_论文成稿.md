## 1. 摘要（约300字）

2024年3月，我参与某航空公司运营智能管理平台建设，项目面向航空运营机构、近百个运营基地机场，覆盖数千万常旅客会员，年均服务旅客超3000万人次，提供航空信息管理、旅客全流程服务、票务交易、航空检修预警、数据智能分析等核心业务功能。项目中，我担任系统架构师，全面负责平台架构设计与核心技术落地。本文围绕**大模型及 AI Coding 技术在航空运营场景中的落地**展开论述，通过**私有化大模型与 RAG 航务知识工程实现敏感数据内网闭环与脱敏访问控制**，基于**IDE 与 CI/CD 集成的 AI Coding 与人机协同门禁保障安全关键代码可审计交付**，结合**领域知识图谱与分级采纳机制抑制模型幻觉并落实航安合规审查**。系统于2025年8月正式上线，截至2026年5月已稳定运行10个月，各项功能及性能指标均达到预设标准，获得客户高度认可。

## 2. 项目背景（约500字）

某航空公司需管理覆盖全部航线网络的近百个运营基地与机场，服务数千万常旅客会员、年服务旅客超3000万人次，为其提供票务、值机、行程查询、航班变动通知、航空检修协同等全场景服务；原有多系统分散、烟囱式建设，故障影响面大、协同效率低，无法满足7×24小时稳定可用与节假日高并发下的**智能化研发与安全合规协同**要求。随着国家智慧民航建设战略深入推进，航空运输行业数字化、智能化转型迫在眉睫，《"十四五"民用航空发展规划》《智慧民航建设路线图》等政策明确要求推动航空运营全流程数字化、智能化升级，提升运输效率与安全水平。在此背景下，该航空公司于2024年3月启动航空运营智能管理平台建设，旨在构建覆盖全部航线网络、近百个运营基地及数千万常旅客会员的数字化管理平台，实现航线、航班、票务等核心业务全流程智能管控，提供全场景便捷服务，提升运营效率与服务体验。

我司中标后，我以系统架构师身份负责平台整体架构设计与核心技术落地。航班动态、机组排班、旅客服务强依赖机型手册、航路规则、气象等**高敏感航务知识**，若依赖公网大模型裸用将面临数据出境与合规风险；同时燃油、载重平衡、行李分拣等逻辑迭代频繁，纯手写难以跟上业务节奏，但**安全关键代码**绝不能「生成即上线」。平台需在提效与航安之间建立可审计、可回滚的工程化边界。

为此，我们团队决定基于**大模型及 AI Coding 技术**，**采用私有化部署 LLM、RAG 构建航务知识库与提示模板库，配套数据分级审批与入向量库前脱敏**；在研发侧**集成 AI Coding 插件与流水线中的 AI 辅助审查**，对核心航务模块实行清单化门禁与双人复核。平台于2025年8月正式上线，成功应对多轮节假日高并发压力，高效完成年度航班调度、设备检修预警及旅客全流程服务，为旅客提供7×24小时信息支持，上线10个月稳定运行，各项指标达标，获得客户与用户一致认可。

## 3. 问题2回应 + 过渡

由于本项目**航务知识高度专业且敏感，公网模型不可用；研发提效需求迫切但安全关键路径必须可签核、可追溯**，所以选用**大模型及 AI Coding 技术的应用**，其核心包括：第一，**私有化 LLM 与 RAG、提示工程及脱敏访问控制**，将智能问答与延误预测辅助等能力限制在内网与合规边界内；第二，**AI Coding 与 CI/CD、代码评审联动**，突出数值与安全敏感点变更的人机协同；第三，**领域知识图谱与规则约束、分级采纳与专家抽检**，抑制幻觉并明确「效率不能以牺牲航安复核为代价」。

在本项目的实施中，我们通过**私有化知识工程保障数据治理基线、AI Coding 门禁保障关键代码可审计、分级治理保障模型输出可信**，完成了**大模型及 AI Coding 技术**的建设与落地，具体实践如下。

## 4. 正文部分

**论述链（每段正文均按此顺序展开）：** **场景** → **架构决策** → **关键技术** → **具体实现落地** → **效果**。

### 一、大模型技术选型与私有化知识工程（约500–510字）

春运、暑运与雷暴叠加的大面积延误中，签派与呼叫中心话务陡增，席位须核对机型性能包线、MEL/CDL，并协调改航、备降与旅客签转；若依赖分散网盘与关键字检索，不仅响应慢，也易漏最新 NOTAM。我们认定航务智能必须与数据主权同构，敏感运行数据不出境。架构上不直连公网API，在内网部署经授权的**私有化LLM**推理集群；以**RAG**将机型手册、航路通告、气象译码、运行政策按「章节—条款—例外」切片并向量化，延误辅助、话术草稿与风险提示仅绑定评审入库的提示模板与Few-shot样本，禁止私存咒语。关键技术：切片粒度与重叠按条款交叉引用设置；**混合检索**（关键词加向量）降低术语歧义；旅客身份、机组排班、舱单相关字段入向量库前脱敏哈希，并与数据分级审批联动，未批域不参与召回，向量库按业务域隔离防误召。落地实施：知识入库经业务部门会签与版本冻结，提示模板双人审批，ETL记录血缘；推理侧设API网关统一鉴权与限流，并对疑似提示注入拦截留痕；界面展示引用编号与仅供参考水印。相关度不足时提示补充关键信息。效果：航务问答与辅助输出在**内网闭环**完成，**合规边界清晰**，为后续AI生成代码复用同一数据治理基线。

### 二、AI Coding 工具链与研发流程集成（约500–510字）

调度、行李分拣、载重平衡等模块业务规则复杂、版本迭代快，样板代码、异常分支与Mock若全靠手写，迭代窗口常被业务压缩；燃油计划、载重平衡、舱单计算等路径属于安全关键领域，任何「生成即合并」都会把不可解释的偏差带入生产。架构上将AI明确定位为**副驾驶**而非自动驾驶员：**IDE**集成经评审的企业**AI Coding**插件，用于生成函数骨架、入参校验、单元测试初稿与接口Mock；**CI/CD**流水线叠加AI辅助代码审查规则，对魔法数、边界条件、异常分支、并发与资源释放进行高亮与风险提示。**核心航务算法清单**内变更必须通过人工复核，关键模块执行双人策略由开发与签派或资深架构交叉确认，清单按季度滚动评审。落地方面，合并请求模板强制填写变更动机与是否涉及载重燃油舱单，AI审查报告作为必看附件但不替代电子签核；提交元数据标记是否经AI辅助及所用模型与插件版本，满足事后审计。还将静态扫描、单测覆盖率与关键路径回归纳入同一质量门禁，避免出现「AI写了但团队无人懂」的灰色代码。效果上形成**谁生成、谁修改、谁复核可追溯**的交付链路，提效体现在重复劳动减少与测试左移，而安全关键把关强度不降级。

### 三、幻觉治理、知识产权与航安合规的折中实践（约500–510字）

试运行阶段曾出现航路限制类建议前后不一致、与NOTAM冲突等现象；团队同时担忧代码片段与运行数据经外部SaaS外泄，以及训练数据知识产权归属不清。治理策略坚持「模型可以聪明，但边界必须硬」：建设领域**知识图谱**并与**规则引擎**联动，对禁航区、机型跑道长度、起飞重量上限等硬性约束一律以规则判定为准，大模型只承担解释、排序与可读说明生成，杜绝听起来合理但事实越界。代码研发侧优先启用内网代码语义检索与企业版或私有化工具链，默认关闭向公网上传仓库与训练数据；确需使用云端能力时须经法务评估并纳入白名单与数据出境评估。落地时对模型输出与研发产物实行可自动采纳、建议采纳、必须人工确认三级策略，调度、载重、舱单等一律落在第三级；每轮发布前组织对抗样例集覆盖极端天气、临时限载与备降叠加，并安排专家抽检签字，纳入航安合规审查清单。对疑似幻觉输出提高审计抽样比例，并按月与签派专家对照复盘。度量上以故事点吞吐与生产缺陷密度综合评估，调度等相关模块经多轮迭代开发效率约提升百分之四十，同时用制度红线重申效率不能以牺牲航安复核为代价。效果上形成智能提效与合规审计并行的治理范式，为后续扩展更多AI场景预留制度接口与工具链扩展点。

## 5. 总结

本平台响应智慧民航建设政策，以**私有化航务知识工程、AI Coding 人机协同门禁、幻觉与合规分级治理**为核心，构建航空运营全流程一体化管理体系，2025年8月上线后稳定运行10个月，超额达成预期目标。上线以来，系统日均处理票务交易超12万笔，核心业务响应时间≤800毫秒，运营效率提升35%，旅客投诉率下降40%，设备故障预警准确率92%，系统可用性达99.993%，峰值处理能力突破5500 TPS，成功应对节假日高并发压力，获行业与旅客广泛认可。项目复盘发现架构存在不足：一是高并发叠加场景下，微服务间同步通信偶有延迟，跨模块数据同步耗时增加；二是各模块资源占用不均，辅助服务资源利用率偏低、核心模块高峰资源紧张。后续将针对性优化：引入异步通信与消息队列技术，重构通信链路；搭建智能资源调度平台，通过AI算法实现容器化资源动态分配，提升资源利用率与系统抗突发能力，持续深化技术融合，助力智慧民航高质量发展。
