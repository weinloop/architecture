## 一、试卷形态与答题结构（稳定规律）

1. **四选一**：每次考试提供 **4 道** 论题，考生选 **1 道** 撰写（摘要正文约 **2200～2500 字**，以当年答题纸说明为准）。  
2. **三段式要求高度一致**：近年题目普遍固定为三个子问题，对应：  
   - **（1）项目背景与个人职责**（真实项目、角色清晰、规模适中）；  
   - **（2）理论与概念**（术语准确、分点作答、最好与教材章节对齐）；  
   - **（3）项目实践**（问题—方案—效果，有数据或对比更佳）。  
3. **主题覆盖**：一套试卷通常同时覆盖 **软件工程/维护**、**架构风格与集成**、**数据与中间件**、**分布式/云与质量保障** 中的若干方向，便于不同背景考生选题。

## 二、培训机构归纳：「双轨制」命题框架（与真题互证）

培训机构从长期真题中归纳出 **两条稳定规律**，与本仓库 **2024-11～2025-11** 三套试卷 **高度吻合**：一套卷中往往 **既有「能扣教材」的论题**，也有 **追热点、追工程落地** 的论题。

### 规律 A：必有一道「教材向 / 方法向」基础题

命题常锚定 **架构设计方法、建模与质量保障** 一类可背书、可分类的「基本功」，与 **第7～10章** 联系紧密，且 **每隔数年轮换重现**。

| 常见母题（培训列举） | 教材大致落点（第2版） |
|----------------------|------------------------|
| **DSSA**（领域特定软件架构） | **第7章** 及相关领域工程表述 |
| **架构建模**、**分层架构风格** | **§1.1.2**、**§7.3** 架构风格与分层 |
| **ABSD**（基于架构的软件设计） | **第7章** 架构设计过程 |
| **MDA / 模型驱动** | 教材与历年卷曾出现；**建模方法**分散于软件工程章节 |
| **面向对象分析（OOA）** | **§5.3** 及面向对象方法 |
| **架构评估与质量保证**（ATAM/SAAM、效用树等） | **第8章** 质量属性与架构评估 |
| **架构演化** | **第9章** 可靠性与演化相关论述 |
| **软件测试**（含性能测试、测试管理） | **§5.x** 软件测试 |
| **可靠性体系**（设计、评价、测试） | **第9章**、**§5** 测试与质量 |

> **与最近三次的对照**：2024-11 **软件维护**、**SOA**；2025-05 **软件测试（AI 方向）**；2025-11 **性能测试** —— 均属于 **易与教材/考纲条文对齐** 的一类；**OOA、MDA、ABSD、DSSA、纯「架构评估」** 在 **最近三次 PDF 中未单独成题**，按轮换逻辑 **仍可作为 2026 的「基础槽位」候选**。

### 规律 B：始终紧跟「新技术、新概念、业务落地」

卷中必有题目指向 **产业热点**，要求结合 **真实业务场景** 写清 **如何选型、如何落地、效果如何**。

| 热点簇（培训列举） | 最近三次真题中的体现 |
|--------------------|------------------------|
| **云原生** | 2025-11 **云原生数据库** |
| **微服务**（可与 SOA/集成呼应） | 分布式事务、秒杀等题中间接体现 |
| **Serverless** | 2025-11 **Serverless 架构模式** |
| **分布式事务** | 2024-11 **论分布式事务及其解决方案** |
| **数据湖 / 湖仓一体** | 尚未在最近三次中单独命题，可作 **递补热点** |
| **多源异构集成** | 2024-11 同主题 |
| **多模型数据管理** | 2025-05 同主题 |
| **云原生数据库** | 2025-11 同主题 |
| **边云协同 / 边缘计算** | 最近三次 PDF 未单列，**案例卷高频**，论文 **中概率递补** |

**写作要点（培训与通用评分一致）**：第二问写清 **概念与分类**；第三问必须落到 **业务流程 + 架构决策 + 关键技术步骤 + 效果**，避免只堆概念。

## 三、历年主题脉络（简表）

根据 **`历年论文题目.png`** 与公开汇总，长期可观察到：

| 方向 | 典型论题形态 |
|------|----------------|
| **架构风格与演进** | SOA、微服务、EDA、Serverless、ABSD、MDA/MDE、架构评估 |
| **数据与平台** | 持久层、NoSQL/多模型、数据湖、分片、多源集成、大数据架构（如 Lambda）、云原生数据库 |
| **质量与过程** | 可靠性、测试（含性能测试）、软件维护、DevOps/协同 |
| **云与新型计算** | 企业上云选型、边缘计算、云原生、Serverless |
| **安全与并发** | 系统安全策略、Web 负载均衡、秒杀/高并发（近年案例向） |

**结论**：论文不是「猜一题背一篇」，而是 **在熟悉 2～3 个主项目素材的前提下，按模块准备「可迁移段落」**（背景一段、技术栈一段、难点与效果若干）。  
**与双轨制结合**：同一项目素材应能 **既讲清一次「教材型」小标题（如测试类型、质量属性）**，又能 **讲清一次「热点型」落地（如云原生组件、消息与一致性）**。

| 题号 | 论题 | 第二问（理论要点） | 双轨标签 |
|------|------|---------------------|----------|

## 五、最近三次的「出题规律」归纳（融合版）

1. **双轨并行**：见 **第二节**；命题在 **第7～8章（方法/质量）** 与 **云原生/数据/分布式热点** 之间交替占位。  
2. **架构类与「当年热点」强相关**  
   - 2025-05：**EDA**、**负载均衡** 与案例/下午卷中的高并发、消息化一致。  
   - 2025-11：**云原生数据库**、**Serverless**、**秒杀** 与云、弹性、大促场景一致。  
   - **AI + 测试**（2025-05）体现命题对 **大模型辅助工程** 的关注。  
3. **数据类题目「由通到专」**  
   - 2024-11：**多源异构集成**（偏集成与管线）。  
   - 2025-05：**多模型数据管理**（偏同一平台多模型）。  
   - 2025-11：**云原生数据库**（偏云上形态与架构选型）。  
   → 数据线在 **集成 → 多模型 → 云原生数据** 上递进，2026 可能继续向 **湖仓、实时化、治理与成本** 延伸（与培训「数据湖/湖仓」热点一致）。  
4. **质量保障类「单独成题」频率高**  
   - 维护（2024-11）、**软件测试+AI**（2025-05）、**性能测试**（2025-11）—— **测试与运维期质量** 是稳定槽位。  
5. **分布式与一致性**  
   - 2024-11 **分布式事务** 与 **第14章** 中 **XA/BASE/TCC/Saga** 等可直接对应；2026 若再考，可能换成 **最终一致、Saga 编排、Outbox** 等表述。

## 七、2026 年 5 月命题方向预测（融合：轮换 + 双轨）

> 说明：预测基于 **双轨制**、**轮换规律、技术热度与教材可写性**，**不保证与真题一致**；正式考试以卷面为准。

### 1. 「教材向」高概率递补（培训轮换逻辑）

下列主题在 **2024-11～2025-11 三套 PDF 中未以独立大题出现**，若 2026-05 **加强基础槽位**，出现概率相对上升：

| 候选 | 备考动作 |
|------|----------|
| **ABSD / 架构设计过程** | 背 **第7章** 活动与产出物关键词，准备「需求→架构→评审」项目叙事 |
| **MDA / 模型驱动** | 与 2024-05 历年表曾出现 MDA 呼应；准备 **UML/元模型/代码生成** 一条线 |
| **架构评估（ATAM/SAAM）** | 背 **第8章** 效用树、风险/敏感点/权衡点；准备一次评审经历 |
| **DSSA / 领域建模** | 准备 **领域参考架构 + 可变点** 的工厂/政务/金融片段 |
| **OOA** | **2023-11** 曾考；可复用 **用例/分析类** 项目段落 |
| **可靠性设计/评价** | **第9章** + 测试章节；与「高可用、故障注入」项目结合 |

### 2. 「热点向」高概率（与培训热点表 + 本仓库原预测合并）

| 预测方向 | 理由 |
|----------|------|
| **微服务架构与治理** | 案例常年出现；与 **§1.1.2**、**第14章** 衔接 |
| **湖仓一体 / 实时数据平台 / 数据治理** | 培训单列 **数据湖/湖仓**；承接多源、多模型、云原生库 |
| **可观测性与 SRE** | Serverless、秒杀之后的自然延伸 |
| **边云协同** | 培训热点；案例卷已多，论文 **中概率** |
| **安全架构（零信任、API 安全）** | 大纲呼应；与微服务边界结合 |
| **企业集成 / API 经济 / 中台** | SOA 的「现代版」叙述 |

### 3. 中概率与低概率（保持原判）

- **中**：单元测试与 CI、大模型/RAG 应用架构（与 AI 测试题区分）、低代码与架构管控。  
- **低**：单独 DSSA/评估大题之外的 **极冷门** 组合；仍建议 **第8章效用树** 随时能写一段。

## 八、备考动作清单（针对 2026 年 5 月）

1. **固化 1 个主项目叙事**：业务背景、技术栈、你的职责、量化结果。  
2. **按双轨各备一条「可迁移段落」**：  
   - **教材向**：质量属性 / 测试分类 / 架构评估 / ABSD 或 OOA（择一背熟）；  
   - **热点向**：云原生组件、数据管线、一致性、限流降级（择一与项目绑定）。  
3. **第二问专门背「分类+定义」**：与最近三次一致（负载均衡三类、分布式事务四类、性能测试指标、Serverless 组成、秒杀手段等）。  
4. **第三问用 STAR 变体**：**场景—架构决策—关键技术—具体实现落地—效果**。  
5. **培训补充自检**：能否用同一项目分别回答 **「纯教材小题」** 与 **「热点落地小题」** 各一题口播 5 分钟。
