# 航空运营智能管理平台：从需求到运维的完整知识闭环

下文将以“航空运营智能管理平台”（面向航司运营机构与近百个运营基地机场，覆盖数千万常旅客、年服务旅客超 3000 万人次量级）为载体，将系分架构的 11 大知识模块与项目全生命周期深度绑定，形成“需求 - 设计 - 实现 - 运维 - 基础支撑”的完整知识闭环，核心逻辑如下：

# 1. 整体概述

项目全生命周期与知识模块的对应关系：

1. **启动阶段（为什么做）**：对应“系统规划与分析”—— 通过项目背景分析明确立项价值（破解烟囱式系统分散、协同效率低、难以支撑 7×24 高可用与节假日高并发等问题），按 6 步规划流程制定建设方案，通过四维度可行性分析验证项目落地性，输出《系统设计任务书》与《需求规格说明书》，同时为论文“项目背景”提供素材。

2. **规划阶段（怎么做）**：对应“项目管理”与“软件工程”—— 项目管理通过范围、进度、成本管理控制项目边界与落地节奏（如按业务域拆分为票务、航班、旅客服务、检修预警、数据分析等子系统，锚定关键里程碑与上线节点）；软件工程确定“增量迭代模型”，标准化开发流程（需求评审 → 代码开发 → 测试验收），保障多方协作下项目有序推进（如 100+ 人规模的跨团队工程）。

3. **设计阶段（做什么）**：对应“软件需求工程”“系统设计”“软件架构设计”—— 需求工程通过 5 步流程细化需求（如用户需“航班变动多渠道通知”“异地值机与行程查询”“航空检修协同与预警”），系统设计完成“结构化 + 面向对象”双维度方案（如 DFD 图梳理数据流，UML 图定义订单、库存、旅客等对象关系），架构设计选择“微服务 + 分布式”架构（如核心业务数据由关系型数据库持久化，Kafka 承担日均海量消息与异步解耦，Redis 支撑热点查询与分布式状态），满足高可用、可扩展与合规类质量属性。

4. **实现与测试阶段（做出什么）**：对应“软件实现与测试”—— 技术选型适配场景（旅客与管理端 Web 常用 Vue + Spring Cloud 体系，移动终端可采用跨端方案），通过“黑盒 + 白盒 + 自动化测试”验证系统质量（如性能与压力测试验证高峰时段万级并发、核心链路响应时延等指标），最终通过部署流程（容器化部署 + 冒烟测试）实现系统上线。

5. **运维阶段（如何长期用）**：对应“系统运行与维护”—— 制定 7×24 小时运维制度（监控指标如可用性、核心接口时延、资源与中间件健康度），通过“并行切换”完成新旧系统过渡，基于业务变化演化架构（如春运或航班大面积延误时的弹性扩容、多航司策略灰度），确保系统持续服务。

6. **基础支撑（底层靠什么）**：对应“数据库”与“计算机基础”—— 数据库设计“三级模式”（外模式适配角色与权限视图，内模式优化索引与分区以支撑海量订单与审计查询），通过并发控制、备份与审计机制保障数据安全与合规；计算机基础为平台提供硬件（多核 CPU、高速存储与网络）、网络（分层拓扑与安全域隔离）、操作系统（可按信创要求选型）支撑，数学与工程方法（如图模型、预测与优化）服务于航线需求、排班与运维决策类场景。
