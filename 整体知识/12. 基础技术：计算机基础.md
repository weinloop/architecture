# 12. 基础技术：计算机基础 —— 支撑平台运行的底层技术

## 12.1. 计算机系统结构

### 1. 计算机系统结构组成与平台适配

- **硬件系统**：“航空运营智能管理平台”服务端典型配置为“**多核 CPU + 大容量内存 + NVMe + 大容量数据盘**”—— CPU 满足**高并发 JVM/容器**与 TLS 卸载场景；内存支撑**连接池、堆内缓存、PageCache**；NVMe 承载**数据库与日志热数据**，HDD/冷存可选承载**历史日志、冷归档、备份**，平衡性能与成本；（信创场景可选用**国产 ×86/ARM 平台+兼容认证机型**；）

- **软件系统**：分层 —— 底层**服务器操作系统**（如麒麟 Kylin OS V10 等国产系统以满足合规）；中间层**数据库、缓存、消息、注册配置**；上层**订座/运行/旅客/分析等业务应用**；

- **系统性能优化**：对**高峰 API、报文接入、SSL 握手**等路径，可调整**进程/线程优先级与 CPU 亲和性**、适当增大**套接字与连接队列**；利用 **PageCache** 支撑顺序写日志与批量 ingest；维护 **SSD 健康与 TRIM 策略**降低写放大。

### 2. 存储器系统与数据存储策略

- **存储器层次结构**：**CPU Cache → 内存 → NVMe → 机械/温存储 → 异地备份**；Cache/内存承载**热路径与在线事务**；SSD 承载**随机读写敏感的数据库与索引**；廉价层承载**可延迟访问的历史附件与磁带网关导出**；

- **存储策略应用**：对**高吞日志或运行事件流**可采用**内存缓冲区批量落盘/批量入列**，降低小包随机写；对**订单与支付流水**采用**双副本/同步复制 + 异地备份**满足 RPO/RTO。

### 3. 输入输出系统与设备交互

- **输入侧**：**自助值机、二维登机牌扫描枪、CUSS/自助托运终端**等通过串口/USB/以太网上送；**地服 App** 使用摄像头（登机牌/行李牌）、定位与**离港状态回填**；接入层将设备报文规范为 **JSON/内部 DTO** 进入服务；

- **输出侧**：**运控中心室内大屏**经视频控制器输出 **HDMI/DP**，刷新**航班态势、告警与 KPI**；柜台**热敏登机牌/行李条打印机**经 **CUPS/厂商驱动** 管理打印队列与重试。

## 12.2. 计算机网络

### 1. 计算机网络基础与平台网络架构

- **协议栈**：**MQTT/Kafka（TCP）** 或 **AMQP** 承载运行与异步事件；旅客与员工侧 **HTTPS（TLS 1.2+）** 保护凭证与行程；内部微服务可用 **gRPC/HTTP2** 降低时延与头开销；对极低开销探测可用 **UDP**（以丢包可容忍为前提）；

- **IP 规划（示意）**：**DMZ/渠道区、应用区、数据区、运维管理区**分属不同网段；核心服务器与网络设备**静态地址**，访客或临时终端可走受控 **DHCP**；

- **子网与路由**：按**职能 VLAN** 隔离东西向流量，仅放行**最小端口矩阵**；静态/动态路由将南北向与跨区流量导向**下一代防火墙与负载均衡**。

### 2. 网络体系结构与协议应用（OSI 与 TCP/IP 映射）

- **物理/链路层**：机房光纤、双绞线、PoE AP、到航站楼的**专线或 MPLS**；

- **网络层**：**IP 寻址与 ECMP**，跨区访问经**边界设备**；

- **传输层**：**TCP** 用于交易与文件；**UDP** 可用于特定实时监控（需应用层补偿）；

- **应用层**：**HTTPS、gRPC、消息协议**；加固含 **mTLS 服务网格、证书轮转、MQTT over TLS**；运维**远程走 VPN + 堡垒机**。

### 3. 网络工程与平台网络部署

- **拓扑**：**核心—汇聚—接入**分层；生产与测试**逻辑/物理隔离**；核心区**高带宽低时延**互联数据库与中间件；

- **设备配置**：**VLAN** 分隔渠道、应用、数据、带外管理；**LACP** 提升上行可靠性与带宽；**防火墙/WAF** 策略默认拒绝，仅暴露 API 网关与健康检查；

- **监控维护**：**Zabbix/Prometheus+SNMP** 观测端口、错包、温度与 power；**iperf、ping、trace** 定期基线；**漏扫与固件**按计划升级。

## 12.3. 操作系统

### 1. 进程管理与平台进程调度

- **进程/线程**：报文接入与 HTTP **工作线程池**分区防阻塞；重型**报表/换季批处理**可**多进程**隔离故障域，进程间可用**队列/共享段**传递大块只读数据；

- **调度与资源限额**：关键链路与批处理区分**优先级与 cgroup CPU 份额**；为**接入容器**设置**内存上限**防 OOM 拖垮节点，结合 **JVM/堆外**调优。

### 2. 存储器管理与内存优化

- **分页与回收**：热数据尽量驻留；批处理大对象用后即释；

- **虚拟内存**：规模按内核建议配置 swap（若以 SSD 为 swap，需权衡寿命）；优先通过**水平扩容与限流**缓解内存尖峰；

- **监控**：`top`/`pidstat`/`smem` 追踪**泄漏与缓存命中率**，结合 **Redis/DB buffer pool** 调参。

### 3. 文件管理与平台文件存储

- **文件系统**：系统分区 **ext4**；数据库与大附件可选 **XFS**；**logrotate** 管理应用与接入 trace，控制保留天数；

- **权限**：二进制 **755**、密钥与账单样例 **640**、公共配置 **644**，避免全局可写；

- **备份**：`rsync`/`rdiff` 同步配置与静态资源；定期从冷备**抽检恢复**。

### 4. 国产操作系统适配

- 在**麒麟**等环境验证 **MySQL/Redis/JDK/容器运行时**兼容矩阵，补齐**缺失包与调优库**；

- **内核参数**：放大 `somaxconn`、`file-max`、`nf_conntrack`、`txqueuelen` 等以支撑高峰连接与短事务风暴；

- **终端安全**：EDR/杀毒、基线核查与**最小化安装**，满足等保要求。

## 12.4. 数学与工程

### 1. 图论应用与平台业务优化

- **图建模**：可将**航站楼内保障节点**（登机口、行李转盘、除冰坪）或**航路网络中的可选航段**抽象为**图** —— 资源为**顶点**，可通行/可飞航段为**边**，边上权重可为**时间、成本、流量限制**；

- **图算法示例**：

    - **Dijkstra**：不正常航班场景下求**改签联程或机组/航材调配**的**时间最短路径**（权重随限制动态更新）；

    - **Kruskal**：新开航站/卫星厅时，**廊桥下穿、行李系统的最小连通建设成本**（示意）；

    - **最大流**：在给定安检口、摆渡车或除冰位**容量上界**下，求**高峰登机构型的可调度流量分配**（示意）；

- **与运行孪生结合**：图权重可随**实时延误、关闭跑道**刷新，**遍历**快速找出瓶颈子图辅助排班。

### 2. 预测与决策在平台运营中的应用

- **时间序列**：对**航线需求、客座、行李量**做 **ARIMA/Prophet** 等预测，支撑**运力与地服人力排班**；

- **回归分析**：分析**延误时长与限流、天气、前序航班叠绳**等因素，指导**缓冲时间与波次编排**；

- **决策树**：对**不正常航班工单**按**影响人数、国际/国内、衔接风险**等特征自动**分级路由**；

- **AHP**：在**多个架构升级备选方案**（成本、工期、风险、可运维性）之间**加权评分**选型。
